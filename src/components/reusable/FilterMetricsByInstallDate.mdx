Paywall, trial, and purchase metrics can be grouped by two different dates:

- **The event date** — when the paywall was viewed, the trial started, or the purchase happened.
- **The install date** — when the user first opened the app.

The two views can show very different numbers for the same date range. The **Filter metrics by install date** checkbox controls which one the dashboard uses:

- **Unchecked (default)**: Metrics are grouped by event date.
- **Checked**: Metrics are grouped by install date.

**Example.** You set the date range to April 1–30 and look at trials.

- **Unchecked**: Shows trials that *started* in April, no matter when those users installed.
- **Checked**: Shows trials from users who *installed* in April, no matter when their trials started.

Use the install-date view to measure user-acquisition performance for a specific cohort. Use the event-date view to measure paywall or onboarding activity in a specific period.
