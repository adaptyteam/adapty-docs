Apple Ads (AA) attribution arrives asynchronously after `Adapty.activate()`. If you call `getPaywall` early, attribution often hasn't landed yet and Adapty resolves the placement against the default audience — bypassing your ASA-segmented paywalls. `AdaptyProfile.appliedAttributionSources` lets the app detect when ASA attribution has been applied to the profile, so the paywall request can wait until ASA segmentation will resolve correctly.

## Before you start

You need:
- Adapty iOS SDK **3.17.1** or later.
- Apple Ads configured for the app in Adapty. See [Apple Ads](apple-search-ads).

## How it works

After `Adapty.activate()`, the SDK requests Apple Ads attribution from Apple in the background and forwards the result to Adapty's backend. When ASA becomes the active attribution source for the profile, the SDK delivers an updated `AdaptyProfile` whose `appliedAttributionSources` array contains `.appleSearchAds`.

An empty array can mean any of:

- Apple Ads attribution hasn't been processed yet for this profile.
- No attribution has arrived at all.

Even with an empty array, `getPaywall` is still safe to call — Adapty resolves the request against whichever audience matches the current profile state, typically the default audience.

## Implementation

1. **Activate the SDK.** See [Install & configure the iOS SDK](sdk-installation-ios).
2. **Subscribe to profile updates** by conforming to `AdaptyDelegate` and implementing `didLoadLatestProfile`. If you haven't set up the delegate yet, see [Listen to subscription updates](ios-check-subscription-status#listen-to-subscription-updates) for the setup steps.
3. **Check `appliedAttributionSources` on every profile update.** When the array contains `.appleSearchAds`, request the paywall — Adapty will return the ASA-segmented variant.

```swift
extension <YourAdaptyDelegateImpl>: AdaptyDelegate {
    nonisolated func didLoadLatestProfile(_ profile: AdaptyProfile) {
        if profile.appliedAttributionSources.contains(where: { $0 == .appleSearchAds }) {
            // load paywall
        }
    }
}
```

4. **Apply a timeout** of 3–5 seconds. If `.appleSearchAds` never appears, call `getPaywall(placementId:)` anyway when the timer fires. Configure a [fallback paywall](fallback-paywalls) for the placement so the user is never stuck on the splash screen.
