The CLI requires authentication to call the Adapty API. Two authentication methods are available:

- [Device flow](#log-in-with-device-flow): Interactive browser login. Recommended for local use.
- [Manual API token](#use-a-manual-api-token): Static token for CI/CD pipelines and scripts.

## Log in with device flow

The device flow lets you authenticate interactively without managing tokens manually. The CLI opens the Adapty Dashboard in your browser, where you confirm a short-lived code and authorize access. Your credentials are saved to `~/.adapty/credentials.json` for future commands.

To log in:

1. In your terminal, run:
   ```bash
   adapty auth login
   ```
2. The CLI prints a verification code in `XXXX-XXXX` format and opens the Adapty Dashboard in your browser.
3. On the authorization page, confirm the code matches your terminal output.
4. Click **Authorize**. The browser shows "CLI authorized! You can close this tab."
5. Back in the terminal, the CLI confirms you are authenticated.

:::note
Screenshot needed: browser auth page at /cli-auth showing the code, Authorize button, and Deny button.
:::

If the code expires before you authorize, or if you click **Deny**, run the following command again to restart the flow:

```bash
adapty auth login
```

## Use a manual API token

A manual token is a static credential you create in the Adapty Dashboard. Use it in CI/CD pipelines and scripts where a browser is not available.

To create a token, in **Settings → Developer API**, click **Generate token**. Copy the token — it won't be shown again.

Pass the token to the CLI in one of two ways:

- **`--token` flag**: Applies to a single command.
  ```bash
  adapty auth status --token your_token_here
  ```
- **`ADAPTY_TOKEN` environment variable**: Applies to all commands in the session.
  ```bash
  export ADAPTY_TOKEN=your_token_here
  adapty auth status
  ```

## Token precedence

When multiple credential sources are present, the CLI uses this order:

| Priority | Source |
|---|---|
| 1 (highest) | `--token` flag |
| 2 | `ADAPTY_TOKEN` environment variable |
| 3 | `~/.adapty/credentials.json` (set by `adapty auth login`) |

The first matching source is used; the others are ignored.

## Manage authentication

### Check authentication status

To see your current authentication state, run:

```bash
adapty auth status
```

:::note
Screenshot needed: example terminal output of `adapty auth status` when authenticated and when unauthenticated.
:::

### Log out

To remove your stored credentials, run:

```bash
adapty auth logout
```

This deletes `~/.adapty/credentials.json`. Tokens passed via `--token` or `ADAPTY_TOKEN` are not affected.

## Token errors

If a token is revoked or becomes invalid, CLI commands return a 401 error. To re-authenticate, run:

```bash
adapty auth login
```
