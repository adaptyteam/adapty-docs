Localizing your flows makes them available in multiple languages. In the Flow Builder, you localize content screen by screen and track translation progress as you go.

:::tip
Finish setting up your flow in the default locale before adding other languages.
:::

## Add and set up localization

1. In the left panel, click <Inline id="localizations.svg" alt="Localizations" />, click **Add locale**, and select the languages to add.

2. To translate the new locales from your default language right away, turn on **Translate content from default locale using AI**. You can review and edit the translations afterward.

<ZoomImage id="new-locale-flow.webp" width="500px" alt="Add locales dialog with AI translation option" />

3. Click **Add locales**. Each added locale appears as a column in the localization table, pre-filled with the default language values.

4. To work on one screen at a time, select it from the **Screens** dropdown in the left panel. By default, the table shows all screens.

5. To focus only on what's missing, turn on the **Missing only** toggle in the left panel. The table filters to show only untranslated rows.

<ZoomImage id="locales-table.webp" width="700px" alt="Localization table" />

## Translate with AI

You can also translate locales with AI after adding them — for example, to fill in locales you added without auto-translation, or to retranslate updated content.

1. In the top-right corner, click **AI Translate**.

2. Choose an option:
   - **Translate missing only**: Translates only lines that aren't translated yet. Best for incremental updates.
   - **Translate all**: Retranslates every line, including ones already translated.

:::tip
To translate a single row, open the row's context menu (**⋮**) and select **AI Translate row**. To translate a specific set of rows, select the checkboxes next to them and click **AI Translate**.
:::

<ZoomImage id="ai-translate-flow.webp" width="700px" alt="AI Translate menu in the localization table" />

## Export and import for external translation

You can export the localization file to share with translators and import the translated results.

In the top toolbar, click **Import / Export**.

<ZoomImage id="import-export.webp" width="700px" />

### Export file format

Exporting produces a `.tsv` (tab-separated) file with one row per translatable element. The columns are:

| Column | Description |
|--------|-------------|
| `Screen` | The screen the element belongs to (e.g., `Welcome`, `Quiz`) |
| `Element` | Auto-generated element identifier within that screen. You can change it in **Interactions** > **Element ID**. |
| `Property` | The property type (e.g., `content`) |
| `[default_locale]` | The default language code (e.g., `en`) |
| `[locale]` | One column per added locale (e.g., `fr`, `es`) |

Example:

```
Screen	Element	Property	en	fr	es
Welcome	title	content	Turn words into art	Transformez les mots en art
Welcome	subtitle	content	Create stunning images in seconds with AI	Créez des images en quelques secondes
Quiz	quiz-title	content	What will you create?
```

:::note
Leave locale columns blank for untranslated lines — Adapty will treat them as missing.
:::

### Import file requirements

- **Format**: `.tsv` (tab-separated values)
- **Headers**: must include `Screen`, `Element`, `Property`, and at least one locale column
- **Locale column names**: must match the locale codes of locales already added to the flow. Importing a file with locale codes not present in the flow results in an error.
- **Partial import**: you can include only a subset of rows; rows not in the file keep their current values

## Translate manually

You can also type translations directly into any cell in the localization table.

To manage a specific row, open its context menu (**⋮**):

- **AI Translate missing only**: Translates only the untranslated cells in that row.
- **AI Translate row**: Translates every cell in that row, including ones already translated.
- **Reset to default**: Reverts the row's translation to the default language values.
