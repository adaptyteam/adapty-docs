Before Adapty can show retention messages, Apple needs a way to reach it when a subscriber cancels. You set up that connection here — in sandbox first to test it, then in production.

## Before you begin

- Add an [App Store Connect API key](app-store-connection-configuration) to Adapty. Adapty uses it to register the realtime URL and communicate with Apple.
- Request access to [Apple's Retention Messaging program](https://developer.apple.com/documentation/retentionmessaging) and wait for Apple's approval. Not every app is eligible, so ask early — without approval, Apple rejects the connection.

## Connect Adapty in sandbox

You set this up in Apple's sandbox environment first, so you can verify the connection before any real subscriber sees a message.

In the Adapty Dashboard, go to **Retention Messaging** and click **Configure Sandbox URL**. A setup card opens and guides you through the connection.

{/* screenshot here */}

On the card, click **Configure URL**. Adapty registers its endpoint with Apple through the App Store Connect API. If App Store Connect already has a different URL set, the card shows **Replace URL** instead — click it to point Apple at Adapty.

This URL is dedicated to Retention Messaging. It's separate from the App Store Server Notifications you set when connecting your App Store Connect account.

When it succeeds, Adapty confirms with "Sandbox URL has been configured."

{/* screenshot here */}

## Test the connection

Before you can go to production, Apple requires a sandbox test (it calls this a **performance test**). It confirms that Adapty answers Apple's request fast enough — within about 700 ms.

The test runs against a sandbox transaction, so first make a purchase with a Sandbox Apple Account, the same way you test purchases.

On the same card, click **Start Test**. The test takes a few minutes. If it fails, check the connection, then click **Retry Test**.

{/* screenshot here */}

## Switch to production

You can only connect production after the sandbox test passes — until then, the production step stays disabled. Once it passes, click **Configure URL** under the production setup. Adapty confirms with "Production URL has been configured."
