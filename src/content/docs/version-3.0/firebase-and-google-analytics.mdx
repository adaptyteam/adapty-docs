If you want to include Adapty data in Google Analytics, the data needs to go through Firebase — Google's mobile app analytics platform. GA4 cannot receive mobile data any other way. Even if you only use GA4, you'll need to set up Firebase and link it to your GA4 property.

Once connected, your `purchase` and `refund` events appear in GA4 with revenue, currency, and product details. From there, you can use them as Google Ads conversions, build GA4 audiences from subscription state, and analyze conversion funnels alongside the in-app behavior you already track.

## What you can do with this integration

- See which actions suggest a future conversion: detect the correlation between Adapty's subscription events and GA4 events. Adapty Analytics contains funnel data, GA4 tracks in-app behavior, but the two need to be combined for a complete picture.
- Register `purchase` and `refund` events in Google Analytics. After this, Google Ads can classify these events as **conversions** to optimize your click bidding.
- Build GA4 audiences with `subscription_state` to target Google Ads campaigns at non-paying users. Adapty [segments](segments) cover the same logic for in-app targeting but don't reach Google's ad inventory.
- Feed GA4's predictive audience models. GA4 reads your purchase events to predict future revenue, top spenders, and likely churners. The resulting audiences are shared with Google Ads automatically.
- Track the same user across devices and improve Google Ads conversion matching. When you identify users with `Adapty.identify()`, Adapty forwards their `customer_user_id` to GA4 as `user_id` — GA4 and Google Ads use this to link sessions and conversions back to the same person.

## How does this integration work

1. When a user first opens your app, the Firebase SDK creates a unique identifier for the installation — the **Firebase App Instance ID**. GA4 requires this ID on every mobile event — web users are tracked separately.
2. Your app's code passes the Firebase ID to Adapty's SDK. From that moment, to fulfill the GA4 requirements, Adapty tags that installation's events with the Firebase ID.
3. When the user makes a purchase, Adapty's servers forward the purchase event to GA4 with the Firebase ID attached. This data is exchanged outside of the app, server-to-server.
4. GA4 uses the Firebase ID to identify which installation is responsible for the purchase. Now you can see what the user did in the app before buying.

### Limitations

- **This integration only carries mobile-app purchases.** No mobile app means no Firebase ID. Users who pay on the web without installing the app don't reach GA4 this way. To track web users, take advantage of [FunnelFox's Firebase integration](https://funnelfox.com/docs/integrations/subscription-management/adapty) or GA4's web data stream.
- **Paddle purchases aren't included.** This integration doesn't currently support Paddle. Paddle purchases stay in Adapty Analytics and don't reach GA4 via this path.
- **Stripe purchases inherit Adapty's Stripe integration limitations.** See [Stripe integration limitations](stripe#current-limitations).


## Setup instructions

### Configure Firebase

1. Open [Firebase Console](https://console.firebase.google.com/) and open or create a project. To keep production analytics clean from sandbox events, use a separate Firebase project for development builds.
2. Link the project to a GA4 property. Firebase prompts you during project creation, or you can add the link at any time via **Project settings** > **Integrations** > **Google Analytics**.

    <ZoomImage id="firebase-analytics-integration-settings.webp" width="700px" alt="Adapty Dashboard Firebase integration settings page" />

3. In **Project settings** > **General** > **Your apps**, add an entry for each platform you ship on (iOS / Android / Web). For Stripe, add a Web app entry — there's no native Stripe app type. Each entry generates a unique **Firebase App ID** and a corresponding data stream in Google Analytics. You'll paste the ID into Adapty's Firebase integration settings during the setup.

    <ZoomImage id="firebase-app-id.webp" width="700px" alt="Firebase Console showing the App ID for an iOS app entry" />

### Configure Adapty

1. Open [**Integrations** > **Firebase**](https://app.adapty.io/integrations/firebase) in the Adapty Dashboard.

2. Enable the **Firebase integration** toggle.

3. Enter credentials for each platform you ship on. Adapty needs both a **Firebase App ID** and a **Google Analytics API secret** for each platform:

   | Field | Where to find it |
   | --- | --- |
   | **Firebase App ID** (iOS / Android / Stripe) | Firebase Console > **Project settings** > **General** > **Your apps**. Each entry has its own App ID. |
   | **Google Analytics API secret** (iOS / Android / Stripe) | Google Analytics > **Admin** > **Data streams** (app platform) > **Measurement Protocol API secrets** > **Create**. The secret is platform-specific. |

   <ZoomImage id="ga4-firebase-secret.webp" width="700px" alt="Google Analytics Measurement Protocol API secret creation screen" />

4. Configure how Adapty forwards revenue and user data. The four controls share one row in the dashboard:

   - **Revenue definition** dropdown: Gross revenue, Proceeds after store commission, or [Proceeds after store commission and taxes](controls-filters-grouping-compare-proceeds#display-gross-or-net-revenue).
   - **Send user properties** toggle: When on, events include `subscription_state` and `subscription_product_id`. To use them in GA4 reports or audiences, see [Use subscription data in GA4](#use-subscription-data-in-ga4-reports-and-audiences).
   - **Report user's currency** toggle: When on, Adapty converts every transaction's local currency to your account's reporting currency before forwarding to GA4.
   - **Send trial price** toggle: Trial starts are valuable — most paying users start with a trial. But Google Ads' bid optimization only treats events with revenue as worth chasing. Turn this on to assign a placeholder price to each trial so Google sees them as conversions and optimizes ad spend toward acquiring trial starters. When on, the **Trial price multiplier** field appears — for example, `0.5` reports half the subscription price during trial.

   <ZoomImage id="revenue-data-options-firebase.webp" width="700px" alt="Firebase integration revenue data row with the four controls" />

5. Map Adapty events to GA4 event names. Adapty exposes a separate maps for **iOS** and **Android** so you can use different names per platform. **Stripe purchases use the iOS event map** — there's no separate Stripe map.

    Google Analytics enforces strict Measurement Protocol limits — 40-character event names, 24-character user-property names, and 36-character values. GA4 silently drops custom events that exceed these limits.

    :::warning
    Some events map to GA4's reserved ecommerce vocabulary — `purchase` and `refund`. Google Ads conversion import, GA4 revenue reports, and predictive audiences depend on these exact strings. Override the defaults only if you don't need those features.
    :::

   <ZoomImage id="ios-event-map-firebase.webp" width="700px" alt="Adapty Dashboard Firebase event names configuration" />

6. Click **Save**. Adapty starts forwarding events to GA4 within a few minutes.

### Configure your app code

:::tip
Make sure your app includes the <InlineTooltip tooltip="Firebase SDK">[iOS](https://firebase.google.com/docs/ios/setup), [Android](https://firebase.google.com/docs/android/setup), [Flutter](https://firebase.google.com/docs/flutter/setup), [Unity](https://firebase.google.com/docs/unity/setup), [React Native](https://rnfirebase.io/), and [Capacitor](https://github.com/capawesome-team/capacitor-firebase)</InlineTooltip>.
:::

Adapty needs to include the **Firebase App Instance ID** with each event — otherwise nothing reaches GA4 (`MISSING_INTEGRATION_ID`).

After `FirebaseApp.configure()` and `Adapty.activate()`, ask the Firebase SDK for the App Instance ID. Pass it to Adapty via `setIntegrationIdentifier`. Run this once per app launch, before any purchase flow.

### Verify the integration

The fastest way to confirm events are flowing is Firebase DebugView:

1. On a test device, run your app with [Firebase debug mode enabled](https://firebase.google.com/docs/analytics/debugview#enable_debug_mode).
2. Trigger a sandbox purchase or any event you've enabled in the Adapty Dashboard.
3. Open Firebase Console > **Analytics** > **DebugView**. Events appear within seconds, with all parameters.

Standard GA4 reports — Realtime, the Reports overview, and audiences — populate within minutes to 24 hours, depending on the report. DebugView is the only place to confirm in real time that the integration is working.

## Use subscription data in GA4 reports and audiences

This step is optional. By default, GA4 hides user properties. To use `subscription_state` and `subscription_product_id` in reports, Explorations, or audiences, register each as a custom dimension. This is useful for filtering by subscription state, building Google Ads audiences of paying users, or feeding predictive models.

When you complete the setup, Adapty will populate these dimensions for upcoming GA4 events. Existing events won't be updated.

1. In Google Analytics, open **Admin** > **Custom definitions**.
    <ZoomImage id="firebase-ga4-custom-definitions.webp" width="700px" alt="Google Analytics Custom Definitions" />
2. Click **Create custom dimensions**.
3. For each property, set:
   - **Dimension name**: Any human-readable name, for example "Subscription state".
   - **Scope**: **User**.
   - **User property**: `subscription_state` or `subscription_product_id`. The name must match exactly — GA4 is case-sensitive.

    <ZoomImage id="ga4-new-custom-dimension.webp" width="500px" alt="Google Analytics New Custom Definition" />

## Troubleshooting

### Events don't appear in GA4

- Confirm the Firebase App Instance ID is set on the profile before the first purchase. Events without it are dropped with `MISSING_INTEGRATION_ID`.
- Confirm the GA4 property linked in the Firebase Console matches the data stream whose API secret you used in Adapty.
- Confirm the credentials in Adapty match the event's platform — iOS App ID and iOS API secret on iOS, Android on Android, Stripe on Stripe.

### Sandbox events pollute production data

Adapty forwards sandbox and production transactions to the same Firebase project. See [Set up Firebase and link it to GA4](#set-up-firebase-and-link-it-to-ga4) — using a separate Firebase project for development builds avoids this entirely.

### Adapty Analytics and GA4 numbers disagree

- **SDK adoption**: GA4 only sees events from users whose app version forwards a Firebase App Instance ID. Older app versions without that call are invisible to GA4 but counted in Adapty.
- **Sandbox events**: Sandbox transactions reach GA4 the same way as production ones unless you isolate them in a separate Firebase project.
- **Sampling**: GA4 Explorations sample high-volume tables. Cross-check against the Realtime view or standard reports for unsampled counts.

### Custom event names are rejected by GA4

GA4 limits event names to 40 characters, alphanumeric and underscores only, starting with a letter. Rename any custom Adapty events in the dashboard that violate these limits.
