A virtual currency is an in-app currency — such as coins, gems, or credits — that users earn or buy and then spend inside your app. In Adapty, you define each currency once and link it to your products. Purchases then grant credits automatically, and Adapty tracks a separate balance for every user. Your app keeps its own spending logic: it reads balances and grants or spends credits through the Server API.

## How it works

1. **Create a currency**: Define a currency in the dashboard with a code, a name, and an optional description.
2. **Link it to products**: Map the currency to consumable or subscription products and set how many credits each purchase grants.
3. **Credits are granted automatically**: When a user buys a linked consumable or renews a linked subscription, Adapty adds the configured credits to their balance.
4. **Read and spend the balance**: Your app reads each user's balance and grants or spends credits through the Server API.
5. **Track every change**: Adapty records each balance change as a transaction and shows it in the user's profile.

## Use cases

- **Subscription allowances**: Grant a set number of credits on each renewal, and expire unused credits at the end of the billing cycle.
- **Consumable packs**: Sell coin or gem packs as consumable products that grant non-expiring credits.
- **Rewards and loyalty**: Grant credits automatically when a user makes a purchase.
- **In-app economies**: Run several currencies at once — for example, gold, silver, and gems — and convert between them.
- **Manual grants**: Top up a user's balance from their profile to compensate a user, run a promo, or test a flow.
