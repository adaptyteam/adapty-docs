The builder includes three media element types under the **Media** category: Image, Video, and Icon.

:::tip
To make an image or video cover the entire screen — including the areas behind the notch and the home indicator — position it as fixed with all offsets at 0 and select **Ignore safe area**. See [Layout and positioning](manage-paywall-ui-elements#ignore-safe-area).
:::

## Image

Upload a `.JPG`, `.PNG`, or `.GIF` file up to 20 MB.

- **Aspect** — controls how the image fits its container:
  - **Fit** — scales the image to fit inside the container without cropping.
  - **Fill** — stretches the image to fill the container.
  - **Cover** — scales the image to cover the container, cropping if needed. Default.
- **Use custom media ID** — see [Custom media ID](#custom-media-id) below.

## Video

Upload an `.MP4` or `.WEBM` file up to 50 MB and no longer than 30 seconds.

- **Aspect** — Fit, Fill, or Cover. Defaults to Fill.
- **Loop** — replay the video continuously. On by default.
- **Use custom media ID** — see [Custom media ID](#custom-media-id) below.

Videos do not play in the builder preview — the canvas shows a still frame. On the device at runtime, the video plays muted by default. With Loop on, it repeats indefinitely.

### Trigger an action when the video ends

:::link
Main article: [Actions](onboarding-actions)
:::

The Video element supports an **On playback finished** trigger that fires when the video reaches the end. Set it up in the **Interactions** panel to navigate to another screen, reveal a CTA, or run any other action.

## Icon

<ZoomImage id="builder-icon-selection.webp" width="300px" float="right" alt="Icon picker with the Tabler Icons library" />

Pick from the bundled [Tabler Icons](https://tabler.io/icons) library, with thousands of icons in two visual styles:

- **Stroke** — outline only.
- **Filled** — solid fill.

Search the picker by keyword to find an icon. Set the icon's color in the **Color** picker — pick a saved [color style](builder-styling) or set a custom color.

## Custom media ID

:::important
You can also set a custom media ID for an image and video [background](paywall-head-picture).
:::

Tag an image or video element with a custom media ID to swap it at runtime from your app code. Use this for [personalized visuals](get-pb-paywalls#customize-assets) — for example, showing the user's selected avatar.

The media you upload in the builder serves as the fallback. If your code doesn't provide media for the ID at runtime, the fallback renders instead.

To enable a custom media ID on an Image or Video element:

1. Select the **Use custom media ID** checkbox under the upload area.
2. Enter a media ID.
3. Upload a fallback image or video.

In your app code, fetch media by its ID — see [Customize assets](get-pb-paywalls#customize-assets) for the SDK API.
