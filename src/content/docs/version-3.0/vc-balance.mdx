*Intro: each user profile shows that user's virtual currency balances; from the same place you can manually top up a balance and review every balance change in the event history. Lives on the profile page in Profiles/CRM. No heading; stands alone.*

## View balances in a user profile

*Outline: where the Virtual currency card appears on the profile and what it shows — each currency listed by code with its current balance (e.g., COINS 12,450; GEMS 3,280). Note that multiple currencies are listed; mention held / available if shown.*

## Top up a balance manually

*Outline: on the Virtual currency card, click the credit (+) control to open the Credit virtual currency dialog. Steps:*

- *Select the currency (the dialog shows its current balance).*
- *Enter the amount to credit.*
- *Confirm; a "Balance updated" confirmation appears and the balance updates immediately.*

*Note: a manual credit is logged as a `dashboard_transaction` in the user's history. Crediting cannot be undone — issue a debit to reverse. Useful for compensation, manual promos, and testing.*

## Top-up events in the event history

*Outline: VC balance changes appear in the profile's event timeline. Manual top-ups show as `dashboard_transaction` (e.g., "Credited 250 COINS"). Other VC event types that also appear: `store_purchase`, `subscription_renewal`, `trial_start`, `api_transaction`, `expiry`, and `refund`. Each entry expands to show details (type, amount, related product, etc.).*
