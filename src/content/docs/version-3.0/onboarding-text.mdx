Add titles, paragraphs, or lists with a single click, style them to match your brand, and use [dynamic variables](onboarding-variables) to personalize content for each user.

## Set up text styles

The **Styles** panel includes pre-configured text styles: H1, H2, H3, Button Label, Body, Caption, and Small Label. Click any style to edit its font family, weight, size, alignment, decoration, and other properties.

When you [add a text element](#add-text), you can choose from one of the styles you set up here.

:::important
Changes to a style apply to all text elements using it, across all screens.
:::

To create a new style:

1. Click <Inline id="Styles.webp" alt="Palette"/> to open the **Styles** panel.
2. On the **Text** tab, click **Create style**.

<ZoomImage id="new-text-style.webp" width="500px" alt="New text style" />

3. Enter a name and configure the typography:
- **Font family**: Choose a typeface. For details on working with custom fonts, see the [guide](using-custom-fonts-in-paywall-builder).
- **Weight** and **size**: Set the font weight and size.
- **Color**: Set the text color.
- **Line height**: Set the spacing between lines, or leave it on **Auto**.
- **Alignment**: Set horizontal (left, center, right) and vertical (top, middle, bottom) alignment.
- **Decoration**: Apply underline or strikethrough.
- **Truncate**: Limits the number of lines displayed. Text beyond that limit is truncated. Useful when content length varies due to dynamic variables or localization.
4. Click **Create**.

:::note
SF Pro isn't available for cross-platform use. We recommend Inter as a close alternative.
:::

<ZoomImage id="text-style-details.webp" width="500px" alt="Text style configuration options" />

## Add text

To add a text element:
1. Click **+** at the top left. Select **Text**. Choose from one of the [text styles](#set-up-text-styles) that were pre-configured or set by you.

<ZoomImage id="new-text.webp" width="500px" alt="New text" />

2. Click the new element and edit its content in the **Content** section in the **Design** panel on the right.
3. If needed, adjust the typography in the **Design** panel. Or, select the text in the preview to open a tooltip for quick style customization.
4. Optionally, in the **Design** and **Interaction** panels, you can also apply any other configurations available for components in the flow. For more details, see [Styling elements](builder-styling).

:::tip
If you need to use the same text element on multiple screens, copy and paste it: select the element, press Ctrl+C (or ⌘+C on Mac), navigate to another screen, and press Ctrl+V (or ⌘+V on Mac) to paste.
:::

<ZoomImage id="text-settings.webp" width="500px" alt="Text element settings" />

### Change styling for parts of text

To only change the style for some parts of a text element:

1. Select a part of a text element in the **Content** section.
2. In the tooltip that appears, change the text color, apply bold, underline, italic, or strikethrough formatting, or add a URL.

The preview updates immediately.

<ZoomImage id="edit-text-part.webp" width="500px" alt="Edit text part" />

## Add links

Flows support two ways to turn text into clickable links. Pick based on what role the text plays:

- **Inline link** — for a URL inside flowing copy, like a "Learn more" reference embedded in a paragraph. Always opens in the in-app browser.
- **an Open URL action** — for standalone tap targets, like a Terms of Service button. Can open in either the in-app or external browser.

### Inline link

To turn part of a text element into a link:

1. Select the text in the **Content** section.
2. In the formatting tooltip, click the link icon.
3. Paste the destination URL into the pop up.

<ZoomImage id="inline-link-demo.webp" width="400px" alt="Formatting tooltip with the link button highlighted" />

### Open URL action

:::link
Main article: [Actions](onboarding-actions#open-url)
:::

To turn an entire button into a link:

1. Add a [Button](builder-elements#buttons) — or use the **Links** preset, a ready-made row with Restore / Terms of Service / Privacy Policy buttons.

    <ZoomImage id="links-button-template.webp" width="500px" alt="The Links preset on canvas with Terms of use, Restore, and Privacy policy buttons" />

2. Select the button in the **Layers** panel and open the **Interactions** tab in the right panel.

    <ZoomImage id="button-selected.webp" width="500px" alt="Terms of Use button selected in the Layers panel, with the nested text element visible but unselected" />

3. Set the destination for the [**Open URL**](onboarding-actions#open-url) action.

    :::important
    An empty Open URL action [blocks previewing and publishing](builder-save-publish#troubleshooting).
    :::

    <ZoomImage id="open-url.webp" width="500px" alt="Open URL action configuration in the Interactions panel" />

## Add lists

:::note
List elements are containers made up of individual item components. For simple bullet or numbered lists within flowing text, use a text element and apply the desired formatting via the **Design** panel.
:::

1. Click **+** at the top left. Select **List** and choose one of the list templates.

2. Go to the **Design** panel on the right to edit list items or upload an image as an item marker.
