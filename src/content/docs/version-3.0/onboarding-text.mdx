Add titles, paragraphs, or lists with a single click, style them to match your brand, and use [dynamic variables](onboarding-variables) to personalize content for each user.

:::tip
Prompt the <Inline id="sparkle.svg" alt="AI Editor" /> [AI Editor](builder-ui#ai-editor) to rewrite your copy or change the styling of a text element.
:::

## Set up text styles

The **Styles** panel includes pre-configured text styles: H1, H2, H3, Button Label, Body, Caption, and Small Label. Click any style to edit its font family, weight, size, alignment, decoration, and other properties.

When you [add a text element](#add-text), you can choose from one of the styles you set up here.

:::important
Changes to a style apply to all text elements using it, across all screens.
:::

To create a new style:

1. Click <Inline id="Styles.webp" alt="Palette"/> to open the **Styles** panel.
2. On the **Text** tab, click **Create style**.

<ZoomImage id="new-text-style.webp" width="500px" alt="New text style" />

3. Enter a name and configure the typography — font family, weight, alignment, and [other properties](#typography-properties).

    :::warning
   [Custom fonts](using-custom-fonts-in-flow-builder) behave differently from built-in fonts. Read the guide — some controls don't apply, and each font variant needs its own file.    
   :::

   <ZoomImage id="text-style-details.webp" width="500px" alt="Text style configuration options" />


4. Click **Create**.


## Add text

To add a text element:
1. Click **+** at the top left. Select **Text**. Choose from one of the [text styles](#set-up-text-styles) that were pre-configured or set by you.

<ZoomImage id="new-text.webp" width="500px" alt="New text" />

2. Click the new element and edit its content in the **Content** section in the **Design** panel on the right.
3. If needed, adjust the [typography properties](#typography-properties) in the **Design** panel. Or, select the text in the preview to open a tooltip for quick style customization.
4. Optionally, in the **Design** and **Interaction** panels, you can also apply any other configurations available for components in the flow. For more details, see [Styles and appearance](builder-styling).

:::tip
If you need to use the same text element on multiple screens, copy and paste it: select the element, press Ctrl+C (or ⌘+C on Mac), navigate to another screen, and press Ctrl+V (or ⌘+V on Mac) to paste.
:::

<ZoomImage id="text-settings.webp" width="500px" alt="Text element settings" />

### Change styling for parts of text

:::warning
**Bold** and **Italic** formatting has no effect on text using [custom fonts](using-custom-fonts-in-flow-builder). To apply variants of a custom font, upload each variant as a separate font file and select it from the **Font family** dropdown.
:::

To only change the style for some parts of a text element:

1. Select a part of a text element in the **Content** section.
2. In the tooltip that appears, change the text color, apply bold, underline, italic, or strikethrough formatting, or add a URL.

The preview updates immediately.

<ZoomImage id="edit-text-part.webp" width="500px" alt="Edit text part" />

## Add an old price

The Old Price element shows an inflated "before" price next to a product's real price, so the discount is visible at a glance. It displays the product's price multiplied by a factor you choose and crosses it out automatically.

You don't edit the element's content. The value is calculated from the product's store price at runtime, so the currency always matches the real price.

To add an old price:

1. Click **+** at the top left. Select **Text**, then select **Old Price**.

   <ZoomImage id="old-price-menu.webp" width="500px" alt="Element menu with the Text category expanded, showing the Old Price item" />

2. In the **Layers** panel, drag the element inside a product card. The element also works inside product cards saved as reusable components.

   :::important
   The Old Price element must sit inside a product card — the price is taken from that card's product. Outside a product card, the builder shows an "Old price" placeholder instead of a price.
   :::

3. In the **Design** panel, in the **Price** section, set the **Multiplier**. Adapty multiplies the product's price by this value. For example, with the default multiplier of 2, a $12.99 product shows a crossed-out $25.98.

   <ZoomImage id="old-price-multiplier.webp" width="500px" alt="Design panel with the Price section and the Multiplier field for an Old Price element" />

4. If needed, adjust the [typography properties](#typography-properties). The element supports the same styling as regular text.

   <ZoomImage id="old-price-result.webp" width="400px" alt="Product card on the canvas showing a crossed-out old price next to the real price" />

:::tip
To cross out another product's real price — for example, the monthly plan's price on a yearly card — build the price row from text elements and variables instead. See [Show a strikethrough price with a discount badge](strikethrough-price).
:::

## Typography properties

Every text element and text style shares the same set of typography controls:

- **Font family**: Choose a typeface — a built-in font or a [custom font](using-custom-fonts-in-flow-builder).

    :::warning
    Different devices include different fonts, or may render the same font differently. If the font is not present on the device, the system will use a default font (SF Pro / Roboto). To consistently display the same font across devices, upload a [custom font](using-custom-fonts-in-flow-builder). 
    :::

- **Weight**: Set the font weight.

    :::warning
    The **Weight**, **Bold**, and **Italic** controls do not apply to [custom fonts](using-custom-fonts-in-flow-builder), whether set in the **Styles** panel, the **Design** panel typography section, the inline toolbar, or the text-part formatting tooltip. To apply variants of a custom font, upload each variant as a separate font file and select the right one from the **Font** dropdown.
    :::

- **Size**: Set the font size in pixels.
- **Color**: Set the text color.
- **Line height**: Set the spacing between lines, or leave it on **Auto**.
- **Alignment**: Set horizontal (left, center, right) and vertical (top, middle, bottom) alignment.
- **Decoration**: Apply underline or strikethrough.
- **Truncate**: Limit the number of lines displayed. Text beyond that limit is truncated. Useful when content length varies due to dynamic variables or localization.

## Add links

Flows support two ways to turn text into clickable links. Pick based on what role the text plays:

- **Inline link** — for a URL inside flowing copy, like a "Learn more" reference embedded in a paragraph. Always opens in the in-app browser.
- **an Open URL action** — for standalone tap targets, like a Terms of Service button. Can open in either the in-app or external browser.

### Inline link

To turn part of a text element into a link:

1. Select the text in the **Content** section.
2. In the formatting tooltip, click the link icon.
3. Paste the destination URL into the pop up.

<ZoomImage id="inline-link-demo.webp" width="400px" alt="Formatting tooltip with the link button highlighted" />

### Open URL action

:::link
Main article: [Actions](onboarding-actions#open-url)
:::

To turn an entire button into a link:

1. Add a [Button](builder-elements#buttons) — or use the **Links** preset, a ready-made row with Restore / Terms of Service / Privacy Policy buttons.

    <ZoomImage id="links-button-template.webp" width="500px" alt="The Links preset on canvas with Terms of use, Restore, and Privacy policy buttons" />

2. Select the button in the **Layers** panel and open the **Interactions** tab in the right panel.

    <ZoomImage id="button-selected.webp" width="500px" alt="Terms of Use button selected in the Layers panel, with the nested text element visible but unselected" />

3. Set the destination for the [**Open URL**](onboarding-actions#open-url) action.

    :::important
    An empty Open URL action [blocks previewing and publishing](builder-save-publish#troubleshooting).
    :::

    <ZoomImage id="open-url.webp" width="500px" alt="Open URL action configuration in the Interactions panel" />
## Add conditional text

Conditional text changes what a text element displays based on a condition. For example, a heading can show one message when a user selects the annual plan and a different message for the monthly plan. Conditional text works like [conditional visibility](onboarding-element-visibility), but it swaps the content instead of showing or hiding the element.

To set up conditional text:

1. Select a text element on the canvas.
2. In the **Design** panel, in the **Content** section, select **Conditional**.

<ZoomImage id="flow-conditional-text.webp" width="500px" alt="Content set to Conditional in the Design panel" />

3. Build the **if** condition. Choose a property from the **Custom**, **Products**, or **Elements** tab, set the operator, and enter the value to match. For details on the property types, see [Conditional visibility](onboarding-element-visibility).
4. Under **then**, enter the text to display when the condition is true. Rich text formatting works the same way as for [regular text](#change-styling-for-parts-of-text). To insert a [variable](onboarding-variables), click **&#123; &#125; Add variable**.
5. Under **else**, enter the fallback text to display when no condition matches.
6. (Optional) Click **+ Add else/if** to add more conditions, each with its own text.

:::tip
To edit conditional text for another language, switch the active locale at the bottom of the editor. Add languages in the **Localizations** panel first — see [Add locale in Flow Builder](add-paywall-locale-in-adapty-paywall-builder).
:::

<ZoomImage id="flow-conditional-text-1.gif" width="500px" alt="Setting up conditional text with if, then, and else branches" />

## Add lists

:::note
List elements are containers made up of individual item components. For simple bullet or numbered lists within flowing text, use a text element and apply the desired formatting via the **Design** panel.
:::

1. Click **+** at the top left. Select **List** and choose one of the list templates.

2. Go to the **Design** panel on the right to edit list items or upload an image as an item marker.
