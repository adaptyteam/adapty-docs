Each email in a sequence has a purpose — the role it plays in guiding the user toward a purchase. The AI assigns purposes automatically, choosing a combination designed to maximize engagement for your app's category.

## Available purposes

There are two format types: standard (with hero images) and non-standard (text-first, no hero image).

| Purpose | Format | Description |
|---|---|---|
| Welcome | Standard | Creates a positive first impression |
| Product Discovery | Standard | Highlights key features with cards |
| Offer | Standard | Promotes a deal or incentive |
| Educational | Standard | Provides valuable domain knowledge (3–5 body paragraphs) |
| Reminder | Standard | References an incomplete user action |
| Social Proof | Standard | Includes testimonials and statistics |
| Final Chance | Standard | Creates urgency with a deadline |
| Story Testimonial | Standard | Features a realistic success story with an AI-invented name and results |
| Re-engagement | Standard | Targets inactive users warmly |
| Upsell | Standard | Compares feature tiers |
| Personal Coach Letter | Non-standard | First-person letter from a named persona; no hero image |
| Fake Transactional | Non-standard | Mimics a system notification; typically produces higher open rates |
| System Notification | Non-standard | Account-alert style with minimal formatting |

## Diversity rules

To prevent repetitive sequences, the AI enforces these constraints:

- A 7-email sequence must use at least **4 different purposes**.
- **Consecutive emails cannot share the same purpose.**
- Every sequence must include **at least 1 non-standard format** (Personal Coach Letter, Fake Transactional, or System Notification) — these feel like personal messages or system alerts, breaking the pattern of promotional emails.
- Purpose selection is influenced by your **app category** — for example, Fake Transactional works well for subscription apps but not for weather or utility apps.

## Purpose-specific behaviors

**Non-standard formats** (Personal Coach Letter, Fake Transactional, System Notification) omit hero images entirely and always use Layout C (text-first). See [Email layouts](mail-email-layouts).

**Educational and Story Testimonial** allow extended body copy (3–5 paragraphs) instead of the standard 1–2.

:::warning
**Story Testimonial** content is AI-invented. The AI generates a realistic first name and specific results (for example, "Maria lost 12 lbs in 6 weeks"). This content is entirely fictional. Ensure this aligns with your legal and compliance requirements before using Story Testimonial emails.
:::

The AI assigns purposes automatically. You cannot manually set the purpose for an individual email — use **Regenerate** to request a new version. The AI may assign a different purpose based on the updated content and position in the sequence.
