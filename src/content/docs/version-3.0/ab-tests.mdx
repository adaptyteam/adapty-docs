Boost your app revenue by running A/B tests in Adapty. Compare different flows, paywalls and onboardings to find what converts best — no code changes needed. For example, you can test:

- Subscription prices
- Paywall design, copy, and layout
- Trial periods and subscription durations
- Onboarding designs

## Prerequisites

Before setting up an A/B test, you must have:

- **Placements**: One or more [placements](placements) where a flow, paywall or onboarding is displayed.
- **For flows**: At least two [flows](adapty-flow-builder).
- **For paywalls**: At least two [paywalls](paywalls).
- **For onboardings**: At least two [onboardings](onboardings).

:::warning
If you are not using the [Adapty Flow builder](adapty-flow-builder) or [Adapty Paywall builder](adapty-paywall-builder), [send paywall views to Adapty](present-remote-config-paywalls#track-paywall-view-events) with `.logShowFlow()` (iOS SDK v4+) / `.logShowPaywall()`. Without this method, Adapty cannot calculate paywall views in the test, and conversion stats will be inaccurate.
:::

## A/B test types

Adapty supports two main A/B test types:

- **Regular**: Runs on a single flow/paywall/onboarding placement.
- **Crossplacement**: Runs across multiple paywall placements, showing the same variant to a user everywhere. Currently available only for paywalls.

For a full comparison of types, use cases, and priority rules, see [A/B test types](ab-test-types).
