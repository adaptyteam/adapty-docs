To sell virtual currency credits, define a virtual currency once, then link it to your products.

You do this in two steps — both described in this article. Once you link a product, each purchase or renewal grants credits automatically.

## Create a virtual currency

In the Adapty Dashboard, go to **Virtual currency**.

{/* screenshot here */}

To create a virtual currency:

1. Click **New virtual currency**. The drawer opens on the **General** step.
2. Fill in the currency details:
    - **Code**: The unique, **permanent** identifier for the currency within your app, such as `COINS`. Identifies the currency in the server-side API and is used to track the users' balance.
    
        When choosing the name, use uppercase Latin letters, digits, and underscores only, up to 32 characters. Lowercase letters are automatically transformed into uppercase letters, and spaces are replaced with underscores.
    - **Name**: The display name, such as `Gold`. You can change it later.
    - **Description**: An optional note about the currency.
3. Click **Continue** to move to the **Link Products** step.

:::important
You can't change the currency's **Code** after you create it. You can still edit the **Name** after the fact.
:::

When you introduce a new currency, the starting balance for every user profile is 0.

## Link products

Link a product to a virtual currency so that a purchase or renewal grants credits in that currency. You can link one product to several currencies, and one currency to several products. You can link products now, during the **Link Products** step, or add them later when you edit the currency.

During the **Link Products** step, the **Associated products** section lists the products that grant this currency. To add one, click **Add associated products**, then choose a product.

The credit settings depend on the product type:

- **Subscription products**: Set **Credit per cycle**, the number of credits granted on each renewal. Optionally, set **Credit on trial start** to grant credits when a free trial begins.
- **One-time products**: Set **Credit amount**, the number of credits granted per purchase. One-time credits never expire.

Click **Save** to create the currency with its product links.

{/* screenshot here */}

When you remove a link, future grants stop, but credits already granted stay. Changes to product links apply from the next store event, not retroactively — existing user balances stay as they are.

### Credit expiration

For subscription products, you can decide whether credits carry over between cycles or reset at the end of each cycle. 

The **Credits expire at the end of each billing cycle** toggle controls this behavior::

- **On**: Credits from one cycle expire when Adapty grants the next cycle's credits. At renewal, Adapty resets any unused credits from the previous cycle to 0.
- **Off**: Credits accumulate across cycles and never expire.

One-time products have no expiry toggle — their credits never expire.

When expiry is on, credits follow the subscription lifecycle:

- **On renewal**: Adapty resets unused credits from the previous cycle to 0, then grants the new cycle's credits.
- **During billing retry or grace period**: Adapty grants no new credits while the store retries payment.
- **On cancellation**: Credits stay until the end of the paid period.
- **When the subscription expires**: Adapty resets any remaining credits to 0 and records an `expiry` transaction.
- **On upgrade or downgrade**: The new product's credit mapping applies from the next cycle.

## Next steps

After you create a currency and link products, purchases grant credits automatically. From here:

- Track each user's balance and top it up manually on the [Virtual currency balance](vc-balance) page.
- If you haven't set up the products that should grant credits, [create them first](create-product).
- Grant, spend, and read balances at runtime through the [server-side API](server-side-api-specs).
