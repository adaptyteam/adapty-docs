*Intro: what this covers — creating a virtual currency and linking it to products in a single flow, so that purchases automatically grant credits. Creation happens in a two-step side drawer: General, then Link Products. No heading; stands alone.*

## Create a virtual currency

*Outline: open Virtual currencies, click Create currency; the drawer opens on the General step. Fields:*

- *Code — required, unique per app; latin letters, digits, and underscore only; max 32 characters; cannot be changed after creation (e.g., `COINS`).*
- *Name — required; display name (e.g., Gold Coins).*
- *Description — optional.*

*Note: code is immutable after creation; mention duplicate-code and invalid-code errors. New currencies start at a 0 balance for every profile.*

## Link products

*Outline: step 2 of the drawer (Link Products). Add one or more product mappings so that purchasing a product grants credits in this currency. A product can be linked to several currencies, and a currency can be linked to several products.*

*The settings differ by product type:*

- *Subscriptions: Credit per cycle (granted on each renewal), Credit on trial start (optional), and the expiry setting (see below).*
- *Consumables: Credit amount (granted per purchase). Consumable credits are always non-expiring.*

*Note: changes apply from the next store event, not retroactively — existing balances stay. Removing a mapping stops future grants but keeps existing balances. A product linked to a currency can't be deleted until it's unlinked.*

### Set whether credits expire

*Outline: this is the "duration" of granted credits. For a subscription mapping, toggle "Credits expire at the end of each billing cycle":*

- *On — credits granted at renewal expire at the next renewal date; unused expiring credits are zeroed when the next cycle is credited.*
- *Off — credits are non-expiring.*

*Consumable credits are always non-expiring, so there's no toggle for them.*
