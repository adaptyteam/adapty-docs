Navigation and branching lets you guide users through every step of your flow: use static routes to send everyone to core screens, and dynamic navigation to adapt the flow based on user choices.

:::link
The Flow builder treats navigation as an action type. Learn more about actions [here](onboarding-actions).
:::

## Navigate between screens

You can configure static and dynamic navigation, using different flow elements.

### Static navigation

Static navigation directs all users to the same target screen. To set it up:
1. Add and select any element users can tap — button, quiz with options, or toggle.
2. Open the **Interactions** panel on the right. There, click **Add trigger**.

   If tapping a quiz option must navigate users to a different screen right away without them tapping a button, you need to select one quiz option at this stage.

   <ZoomImage id="add-trigger.webp" width="500px" alt="Add trigger button" />

3. Set up the **On tap** trigger:
   - **Action**: Select **Navigate to screen**.
   - **Destination**: Choose the destination screen.

  <ZoomImage id="static-nav.webp" width="500px" alt="Static navigation" />

### Dynamic navigation

Dynamic navigation routes users based on their quiz answers, toggle state, and custom attributes.
Any [selectable element](onboarding-actions#set-up-selectable-elements) can be a condition for dynamic navigation.

To set it up:
1. Add and select an element that will navigate users.
2. Open the **Interactions** panel on the right. There, click **Add trigger**.

   If tapping a quiz option must navigate users to a different screen right away without them tapping a button, you need to select one quiz option at this stage.

   <ZoomImage id="add-trigger.webp" width="500px" alt="Add trigger button" />

3. Set up the **On tap** trigger:
   - **Action**: Select **Conditional**.
   - **Conditions**: Set the conditional navigation actions. Learn more [here](onboarding-actions#conditional-actions).

<ZoomImage id="conditional-action.webp" width="500px" alt="Conditional actions" />

## Close flow

If your user journey calls for closing the flow, you can set it up using buttons or single-answer quizzes:

1. Add and select an element that must close the flow on tap..
2. Open the **Interactions** panel on the right. There, click **Add trigger**.
   <ZoomImage id="add-trigger.webp" width="500px" alt="Add trigger button" />

3. Set up the **On tap** trigger:
   - **Action**: Select **Close flow**.
