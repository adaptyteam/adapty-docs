You can create a flow from a template or from scratch.

## Create flow

1. Open the **Flows** page.
2. Click **Create flow**. This opens the Template Library.
3. Pick a template or start from scratch.

### Use a template

The Template library contains several templates that act as a starting point for your flow. Each is a complete flow with multiple screens, interactive elements, and working navigation. You can edit any element to customize it.

To apply a template:

1. In the Template library, browse the template cards. Each card shows preview screenshots from a flow.
2. Click **Use as template** on the card you want.

The template loads in the builder. From here, you can change any element, screen, or property.

### Start from scratch

{/* TODO: SCREENSHOT */}

Click **Start from Scratch** in the Template library to open a flow with basic, reusable screens — a welcome screen, quiz screens, and a paywall. Copy what you need into your design, or delete them for a completely clean start.

## Change template

You can switch templates from inside the builder. Open the Screens panel and click the **Templates** <Inline id="template-library.svg" alt="Templates" /> button to reopen the Template library, then apply a new template the same way as during initial creation.

:::danger
Applying a new template discards your current flow draft. There's no confirmation prompt, and the action can't be undone. The published flow is unaffected, but can't be restored for further editing.
:::

## Custom fonts in templates

:::link
Main article: [Custom fonts in the Flow Builder](using-custom-fonts-in-flow-builder.md)
:::

<ZoomImage id="custom-font-warning.webp" width="300px" alt="Custom font chip on a template card with the warning tooltip" float="right" />

Templates marked with a **Custom font** chip use custom fonts. These fonts don't ship with the mobile SDK. Click the chip to reveal the list of used fonts and the download link.

To see the intended typography on device, add the font files to your app bundle.

:::warning
If you change a published flow to use a new custom font, users on app versions that don't bundle the font see the system fallback.
:::

Fonts live in the app bundle, not in Adapty. To avoid the fallback, duplicate the flow, switch the font in the copy, and show the new flow only to users on [app versions that include the font](segments).
