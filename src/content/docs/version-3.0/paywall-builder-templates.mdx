## Use a template

The Template library contains several templates that act as a starting point for your flow. Each is a complete flow with multiple screens, interactive elements, and working navigation. You can edit any element in a template to completely customize it.

### Apply a template

:::danger
When you apply a template, the current flow draft is discarded. There is no confirmation prompt, and the action cannot be undone. The published flow is unaffected, but cannot be restored for further editing.
:::

1. In the Template library, browse the template cards. Each card shows preview screenshots from a flow.
2. Click **Use as template** on the card you want.

The template loads in the builder. From here, you can change any element, screen, or property.

### Custom fonts in templates

:::link
Main article: [Custom fonts in the Flow Builder](using-custom-fonts-in-flow-builder.md)
:::

<ZoomImage id="custom-font-warning.webp" width="300px" alt="Custom font chip on a template card with the warning tooltip" float="right" />

Templates marked with a **Custom font** chip use custom fonts. These fonts don't ship with the mobile SDK. Click the to reveal the list of used fonts and the download link.

Some templates use custom fonts that do not ship with the mobile SDK. 


To see the intended typography on device, add the font files to your app bundle yourself.



The chip is informational only. Clicking **Use as template** still applies the template immediately — there is no warning at apply time.

:::warning
Without the font in your app bundle, mobile users see the system fallback font instead of the template's intended font.
:::

### Open the templates library from inside the builder

You can open the template library from inside the builder.

Open the Screens panel and click the **Templates** <Inline id="template-library.svg" alt="Templates" />  button.
