Apple's Retention Messaging API shows a custom message on the iOS Cancel Subscription screen, before a subscriber confirms the cancellation. Adapty allows you to author the message in the dashboard and deliver it to Apple automatically. A subscriber who taps Cancel sees your message the moment they're about to leave.

## How it works

Adapty sits between your dashboard and Apple:

1. You author a message in Adapty and submit it for Apple's review.
2. Adapty uploads the approved message to Apple's servers.
3. When a subscriber taps Cancel in iOS Settings, Apple calls Adapty in real time to retrieve the message for that product and locale.
4. Apple displays the message received from Adapty on in the cancel screen.

## Requirements

- Retention Messaging is an **iOS exclusive** feature that uses Apple's Retention Messaging API.
- Your app must have access to [Apple's Retention Messaging program](https://developer.apple.com/documentation/retentionmessaging).
- You need an App Store Connect API key in Adapty. Adapty uses it to register your realtime URL and upload messages to Apple.

## Get started

- **[Set up the connection](retention-messaging-setup)**: Connect Apple's Retention Messaging realtime URL to Adapty, test in sandbox, then run in production.
- **[Author a default message](retention-messaging-create)**: Write a title and body, and optionally add an image. Adapty shows it when a subscriber cancels, and translates it into your other locales.
