The Flow Builder creates responsive layouts. You do not drag elements to exact coordinates — instead, you nest them inside **containers** that automatically arrange their children. 

The container decides the elements' direction (vertical or horizontal), alignment, and spacing. Individual elements can then fine-tune their size and margins, or — when needed — break out of the flow with absolute or fixed positioning.

:::link
For visual properties like fill, borders, and effects, see [Element styling](builder-styling).
:::

## Layout

<ZoomImage id="flow-builder/layout.webp" width="300px" float="right" alt="Layout settings" />

Layout is the primary tool for arranging elements on screen. Each container automatically distributes its children according to a set of rules — direction, alignment, and spacing. The layout elements available in the builder are:

* **Vertical Stack**: Arranges children from top to bottom
* **Horizontal Stack**: Arranges children from left to right
* **Divider**: A visual separator between elements
* **[Carousel](paywall-carousel)**: A horizontally scrollable set of slides
* **[Bottom Sheet](paywall-carousel)**: A sliding overlay panel that reveals additional content when the user taps a button

Stacks are the primary building blocks of a screen. You can nest them inside each other to build complex layouts. Each container has a **Layout** section in the right panel that controls how its children are distributed. 

To group elements into a new container, use the **Wrap** [layer action](paywall-layout-and-products#layer-actions). To remove a container and promote its children, use **Unwrap**.

<ZoomImage id="flow-builder/alignment-distribution-demo.webp" width="600px" alt="Alignment and distribution options" />

:::link
For details on the screen and layer hierarchy, see [Screens and Layers](paywall-layout-and-products).
:::

### Direction

* <Inline id="free-layout.svg" alt="Free" /> **Free**: No automatic layout. Children are positioned independently (useful when children use absolute positioning)
* <Inline id="vertical-layout.svg" alt="Vertical" /> **Vertical**: Children stack from top to bottom, like rows in a column
* <Inline id="horizontal-layout.svg" alt="Horizontal" /> **Horizontal**: Children arrange from left to right, like items in a row

### Element order

Children render in the order they appear in the **Layers** panel. In a vertical stack, the top item in the list appears at the top of the screen. In a horizontal stack, the top item appears on the left. Drag elements in the Layers panel to reorder them, or use **Move Up** and **Move Down** [layer actions](paywall-layout-and-products#layer-actions).

### Alignment

<ZoomImage id="flow-builder/alignment.webp" width="100px" float="right" alt="Layout settings" />

The alignment grid controls where children sit along the container's cross axis. In a vertical stack, alignment controls the children's horizontal placement (left, center, or right). In a horizontal stack, it controls their vertical placement (top, middle, or bottom).

### Distribution

Distribution determines how space is divided among children along the main axis:

* **Gap** <Inline id="gap.svg" alt="Gap" /> (default): A fixed pixel value between adjacent children
* **Space Between**: Children spread to the edges; equal gaps appear between them
* **Space Around**: Equal space surrounds each child, with half-size gaps at the edges
* **Space Evenly**: Equal space before, between, and after all children


### Clip content

Visually clips content that extends beyond the container's boundaries. Disable this to allow overflow (for example, a badge that intentionally extends past the card edge).

## Position

<ZoomImage id="flow-builder/position-relative.webp" width="300px" float="right" alt="Position toggle" />

By default, every element's position is automatically determined by its container's layout. The **Position** toggle allows you to break it out of the normal flow and manually position it.

<ZoomImage id="flow-builder/positioning-demo.webp" width="700px" alt="Absolute and fixed positioning in action" />

### Relative (default)

The element stays in the normal layout flow. Its position is determined automatically by the parent container's layout rules — you cannot drag it freely. Use **Margin** to adjust the space around a relative element.

Use relative positioning for the vast majority of content: text blocks, images, cards, buttons, and list items.

### Absolute

<ZoomImage id="flow-builder/position-absolute.webp" width="300px" float="right" alt="Absolute position controls" />

The element lifts out of the normal flow and overlays other content. It no longer affects the layout of neighboring elements.

When you select **Absolute**, additional controls appear:

* **Offset fields** (T, L, R, B): Set the distance in pixels from the element to each edge of its parent container
* **Anchor grid**: Click a point on the 3×3 grid to choose which corner, edge, or center of the parent the element anchors to
* **Horizontal anchor** <Inline id="horizontal-positioning.svg" alt="Horizontal positioning" /> (Left / Center / Right) and **Vertical anchor** <Inline id="vertical-positioning.svg" alt="Vertical positioning" /> (Top / Center / Bottom): Dropdowns that control the same anchor point as the grid
* **Z-index**: A numeric field that controls the element's [stacking order](#stacking-order) relative to siblings. Elements with higher values appear on top

Use absolute positioning for decorative overlays, badges, close buttons, and icons placed on top of images.

:::tip
To stretch an absolute element across the full width of its parent, set the horizontal anchor to **Left**, then add a **Right** offset of 0. The element will pin to both edges.
:::

### Fixed

<ZoomImage id="flow-builder/position-fixed.webp" width="300px" float="right" alt="Fixed position controls" />

The element ignores its parent container entirely and pins to the device viewport. It stays visible while the user scrolls — the page content moves underneath it.

Fixed positioning uses the same controls as Absolute (offsets, anchor grid, Z-index). All offsets are relative to the screen edges instead of the parent.

Use fixed positioning for sticky buttons at the bottom, floating navigation bars, and persistent headers.

## Sizing

<ZoomImage id="flow-builder/sizing.webp" width="300px" float="right" alt="Sizing controls" />

Each element has **Width** and **Height** controls. Click the dropdown to choose a sizing mode:

* **Fill**: The element stretches to occupy all available space in its parent. The pixel value shown is the computed result.
* **Hug**: The element shrinks to fit its content. The pixel value shown is the computed result.
* **Fixed**: The element uses the exact pixel value you specify, regardless of parent or content size. The only available mode for elements with absolute or fixed positioning.

<ZoomImage id="flow-builder/sizing-spacing-demo.webp" width="600px" alt="Fill, Hug, and Fixed sizing modes" />

## Spacing

<ZoomImage id="flow-builder/spacing.webp" width="300px" float="right" alt="Spacing box model" />

Set spacing values independently for each side of the element.

* **Margin**: The space between the element and its neighbors. Does not extend beyond the boundaries of the parent container, regardless of its value.
* **Padding**: The space between the element's boundary and its content.

Text elements only have margin. Screens only have padding. Both are available for containers and other elements with child content.

## Stacking order

<ZoomImage id="flow-builder/flow-builder-layer-context-menu.webp" width="300px" float="right" alt="Layer context menu" />

Elements further down in the **Layers** panel render on top of earlier ones. 

**Absolute** and **Fixed** elements can overlap other elements regardless of their order in the Layers panel. Use the **Z-index** field to control which one appears on top — higher values win.

**Bring to Front** and **Send to Back** [layer actions](paywall-layout-and-products#layer-actions) set the Z-index to the highest or lowest value among siblings.
