A retention message is what a subscriber sees on the iOS Cancel Subscription screen. You create a default message — a title and body, optionally with an image. Adapty shows it per product and locale when a subscriber cancels.

## Before you begin

- Create at least one [product](create-product) for the message to cover.
- [Set up Retention Messaging](retention-messaging-setup) and test it in sandbox environment.

## Create a default message

To start, in the Adapty Dashboard go to **Retention Messaging**, then click **Create Default**.

{/* screenshot here */}

Choose where the message runs:

- **Create a sandbox message**: For testing. Sandbox messages never reach real users.
- **Create a production message**: For real users. Available once you've set your production URL and passed the sandbox performance test.

The message form opens with three steps: **General**, **Content**, and **Localization**.

### General

- **Name**: An internal name for the message, up to 60 characters.
- **Products**: The products this message covers. Add up to 20.

Click **Next**.

{/* screenshot here */}

### Content

Write the message in English. You can localize it in the next step.

- **Title**: Up to 66 characters.
- **Body**: Up to 144 characters.
- **Image**: Optional. A PNG with the maximum width of 3840px and 160–2160px height, no transparency. Add **Alt text** to describe it.

Click **Next**.

{/* screenshot here */}

### Localization

Adapty translates the message into your other locales.

1. Click **Add Locale**, then choose the locales to add.
2. Click **AI Translate All Cells** to fill the translations.
3. Click any cell to edit a translation manually.

{/* screenshot here */}

## Submit for review

When the message is ready, click **Submit To Review**. Apple reviews it, and its state changes to **In review**, then **Live** or **Rejected**. Production stays locked until Apple approves at least one locale.
