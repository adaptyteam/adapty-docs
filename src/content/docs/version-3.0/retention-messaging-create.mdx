A retention message shows subscribers a reason to stay, right when they try to cancel. It appears on the iOS Cancel Subscription screen. Apple shows the message that matches the subscriber's product and locale.

There are two kinds of retention message:

- **Default**: The fallback message Apple shows when no promo applies. It's required, so create it first.
- **Promo**: A message tied to a promotional offer or a plan switch. Use it to offer specific subscribers a better deal before they leave.

## Before you begin

- Create at least one [product](create-product) for the message to cover.
- [Test Retention Messaging](retention-messaging-setup) in sandbox mode.

## Create a message

Open the Retention Messaging page in the Adapty Dashboard. Click **Create Default** for a default message, or **Create Message** for a promo message.

<ZoomImage id="retention-select-default-or-promo.webp" width="700px" alt="The Create Default and Create Message buttons in the Retention Messaging page header" />

Select the environment — a sandbox message for testing, or a production message for real users.

<ZoomImage id="retention-new-message.webp" width="700px" alt="New message start screen with Create a sandbox message and Create a production message options" />

### General

Name the message and add the products that will trigger the message when cancelled.

<ZoomImage id="retention-message-step-1-general.webp" width="700px" alt="Default message General step with the Name field and Products section" />

For a **promo** message, configure the incentive that will persuade the subscriber to stay:

- **Promo offer**: A promotional offer you [previously set up for the product](app-store-offers).

    <ZoomImage id="retention-messaging-promo.webp" width="500px" alt="Promo message General step with an associated offer selected for a product" />

- **Plan switch**: A different product from your catalog.

    <ZoomImage id="retention-messaging-switch-plan.webp" width="500px" alt="Promo message General step with an associated plan selected for a product" />

### Content

Write the message in English. A preview on the right shows the message on the Cancel Subscription screen as you type.

The **Default** message may include an image.

<ZoomImage id="retention-message-step-2-content.webp" width="700px" alt="Content step with title, body, and image fields, and a phone preview of the Cancel Subscription screen" />

### Localization

English is the source language. Add the other languages your subscribers use, then translate the content.

1. Click **Add Locale** and choose which locales to support.
2. Click **AI Translate All Cells** to automatically translate the [content](#content).
3. Click any cell to edit a translation.

Hover over a locale to preview it on the Cancel Subscription screen.

<ZoomImage id="retention-message-step-3-localization.webp" width="700px" alt="Localization step with the locale table and the AI Translate All Cells button" />

## Submit for review

When the message is ready, click **Submit To Review**. Apple reviews it, and the status changes to **In review**, then **Live** or **Rejected**.

The message goes live for production users once Apple approves at least one locale.

:::note
Apple approves each locale separately. Subscribers see your message only in approved locales — there's no automatic fallback to English.
:::
