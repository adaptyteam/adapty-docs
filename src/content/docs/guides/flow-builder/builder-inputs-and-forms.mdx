The Flow Builder provides two types of interactive elements for processing user input:

- **[Inputs](#inputs):** Typed fields for text, email, number, phone number, date, and time. Use them to request specific data like a name, birthday, or contact details.
- **[Quizzes](#quizzes):** Option lists, grids, and rating scales where users select from predefined choices. Use them to learn about user preferences, segment your audience, or ask for a quick rating.

Once collected, you can put the responses to work in a few ways:

- **Conditional navigation:** Route users to different screens based on their answers — for example, show a tailored flow for beginners vs. advanced users.
- **Variables:** Store responses and reference them elsewhere in the flow, such as addressing the user by name on a later screen.
- **App code:** Handle the data outside the flow to update user profiles, trigger analytics events, or save preferences to your backend.

## Inputs

Use inputs to collect typed data from users — such as a name, email address, or date of birth.

### Add an input

<ZoomImage id="add-input.webp" width="400px" float="right" alt="Add input" />

1. Click **+** at the top left.
2. Select **Input**.
3. Choose the input type:
   - **Text:** Any short text entry.
   - **Email:** Email addresses, with optional format validation.
   - **Password:** Secure text entry, with configurable requirements.
   - **Number:** Numeric values, with configurable format.
   - **Phone number:** Telephone numbers.
   - **Date:** Opens a date picker.
   - **Time:** Opens a time picker.
   - **Date and time:** Opens a combined picker.

### Configure an input

:::link
For more details about visual settings — layout, style, and visibility — see [Element styling](builder-styling).
:::

<ZoomImage id="input-settings.webp" width="400px" float="left" alt="Add input" />

For all input types, you can configure the following in the **Design** tab:

- **State:** Define how the input looks in different situations. Switch between **Default**, **Active**, **Invalid**, and **Disabled** and apply different visuals to each.
- **Placeholder:** Hint text displayed inside the empty field.
- **Leading and trailing icons:** Add icons inside the input field.
- **Typography:** Text styling for the value displayed in the field.

<ZoomImage id="input-settings-password.webp" width="400px" float="left" alt="Add input" />

Some settings are specific to certain input types:

| Setting                    | Input types               |
|----------------------------|---------------------------|
| Clear button               | Text, Email               |
| Validate email format      | Email                     |
| Show password icon         | Password                  |
| Edit password requirements | Password                  |
| Number format              | Number                    |
| Date/time format           | Date, Time, Date and time |
| Min and max date           | Date, Date and time       |

### Use input values

When a user submits an input, you can save the value as a variable. To do this, use the **On Submit** action. Once saved, the variable is available across the flow — to personalize copy, populate another field, or drive conditional navigation.

To save an input value as a variable:

1. Select the input element.
2. In the **Design** tab, click **Add trigger** and add an **On Submit** trigger.
3. Set the action to **Set variable** and give the variable a name.

## Quizzes

Use quizzes to present users with predefined choices. Unlike inputs, quizzes don't have typed fields — users select from the options you define. Use them to collect preferences, segment users, or branch the flow based on their answers.

### Add a quiz

<ZoomImage id="quiz.webp" width="400px" float="left" alt="Add input" />

1. Click **+** at the top left.
2. Select **Quiz**.
3. Choose the quiz type:
   - **Icon/image/emoji options:** A vertical list of selectable options, each with an icon, image, or emoji alongside a text label.
   - **Icon/image/emoji grid:** A grid of selectable options, each with an icon, image, or emoji.
   - **Rating:** A scale for users to express a rating — numeric or star-based.

### Configure quiz actions

Each quiz option is a selectable element. In the **Interactions** panel, each option has its own **Element ID** and a **Group** shared across all options in the quiz. You can add actions that fire when a user taps an option.

To add an action to a quiz option:

1. Select the option.
2. Open the **Interactions** panel on the right.
3. Under **Selectable Triggers**, click **+ Add Action** next to the **On Tap** trigger.
4. Set the action — for example, set a variable or show a navigation button.

Repeat for each option.

<ZoomImage id="quiz-on-tap.webp" width="500px" alt="Add input" />

#### Set up conditional navigation

To route users differently based on their selection, set a conditional action on the **navigation button**, not the quiz option:

1. Select the navigation button.
2. In the **Interactions** panel, add an **On Tap** trigger with a **Conditional** action.
3. In the condition editor, open the **Elements** tab.
4. Set the condition — for example, *if [some option] is selected* — and assign a destination screen for each case.
