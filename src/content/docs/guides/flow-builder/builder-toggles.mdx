:::warning
Apple may reject apps that use a pre-selected trial toggle. A toggle set to "on" by default can be flagged as a manipulative dark pattern under App Store Review Guidelines — it implies user consent to a free trial without an explicit choice.

To avoid rejection, set the toggle to **off** by default and let users opt in to the trial themselves.
:::

A trial toggle is a binary switch that lets users choose between standard and trial-based products on a paywall. When the user changes its state, it triggers actions — such as swapping product groups, updating variables, or showing and hiding elements — instantly.

<ZoomImage id="trial-toggle.webp" width="500px" alt="Trial toggle element in the Flow Builder" />

To add a trial toggle:

1. On the canvas, click **+** on the target screen and select **Trial toggle**.
2. Open the **Interactions** tab in the right panel and click **Add trigger**.
3. Select **On Changed** and click **Add action**.
4. From the **Action** dropdown, select the action to perform — for example, **Show** or **Hide** to swap product groups based on the toggle state.
5. Select the element to show or hide.
