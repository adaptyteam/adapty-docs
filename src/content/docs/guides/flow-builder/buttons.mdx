The primary interactive element used to trigger user actions. Buttons are essential for executing purchases, navigating between screens, closing modals, submitting forms, and other key interactions within the paywall.

The Button is the fundamental interactive building block of the builder. It serves as a universal container designed to handle user taps and manage visual state transitions.

What it's for: This single component is the engine behind all clickable elements on the paywall:
CTA (Call to Action): Prominent purchase and subscription buttons.
Navigation: "Close", "Back", and "Skip" buttons.
Utility Links: Text links (Restore, Terms), which are technically instances of the Button component in "Ghost" mode.
Key Feature: The component acts as a polymorphic element. It supports 3 architectural styles (Fill, Secondary, Ghost) as shown in the system reference. Any preset dragged onto the canvas is simply a pre-configured instance of this base component. You can transform a "Close" button into a "Buy Now" button entirely through property edits.

## Presets

### Base

Description: A standard button configured as a Primary CTA. Comes with basic visual styles: solid theme color fill, 12px corner radius, and center alignment.

Interactions: ❌ None. The action list is empty. After adding this button, user must manually assign an action (e.g., Purchase or Next Screen) 

### Right icon

Description: A variation of the base button that includes a visual cue (icon). Typically used for navigation to indicate direction or progress ("Next", "Continue").

Reference: The second button in the preset list.

Structure: Unlike the base button, this preset creates a container with two nested elements:

📂 Button

T Text (Label)
★ Icon (Arrow / Chevron right) Absolute position

Interactions: ❌ None. The action list is empty. Just like the Base preset, you must manually assign an action (e.g., Navigate to Screen) after dropping the button onto the canvas.

### With subtitle

A button featuring a two-level text hierarchy. Used when a single label isn't enough and extra context is required (e.g., subscription terms or discount details).

Structure: This preset automatically generates a nested layout to align text vertically:

📂 Button (H-Stack, Align: Center) — Parent container.
📂 V-Stack (Gap: 2-4px, Align: Center) — Inner container for text wrapping.
T Text (Title, Bold, 16px)
T Text (Subtitle, Regular, 12px, Opacity 80%)

Interactions: ❌ None. The action list is empty. You must manually assign an action (typically Purchase, as this layout is most common for offer buttons).

### Pulse animation

A variant of the base button with a pre-configured "heartbeat" effect. Used to boost Click-Through Rate (CTR), as the dynamic motion automatically draws the user's eye to the Call to Action.

Structure:

📂 Button (Standard Base structure)
T Text

Visual Settings: The Animation section in the property panel comes pre-set with optimal values:

Type: Pulse.
Scale: ~10-20% (subtle expansion).
Behavior: The animation is set to loop automatically upon appearance.

Interactions: ❌ None. The action list is empty. Assign logic manually.

### Purchase

A specialized button designed to initiate payments. Typically features a Primary style (accent fill) with a strong Call to Action, such as "Subscribe", "Start Trial", or "Proceed to payment".

Interactions: ✅ Pre-configured. The Purchase action is already assigned to the On Tap trigger. No logic setup is required; upon adding to the screen, this button is immediately ready to trigger a transaction via App Stores (StoreKit / Google Billing) or Stripe (for Web).

### Secondary outline

A button with background and a visible stroke. Used for secondary actions to establish visual hierarchy, ensuring it doesn't compete with the Primary CTA.

Structure:

📂 Button (Standard Base structure)
T Text

Interactions: ❌ None. The action list is empty. Assign logic manually.

### Back

Description: A minimalist button consisting solely of an icon. Typically placed in the top corner of the screen. Used for navigation (going back) or dismissing the paywall.

Interactions: 
If used on the main paywall screen (modal): Assign the Close Flow action.
If used on a secondary screen (e.g., inside a funnel): Assign the Navigate Back action.

A pre-built footer navigation row. Includes the mandatory legal links required by App Store and Google Play guidelines (Terms of Use, Privacy Policy) along with the Restore Purchases option.

This section describes the new Paywall Builder, which works with:
- iOS, Android, and React Native SDKs version 3.0 or higher
- Flutter and Unity SDKs version 3.3.0 or higher

### Purchase buttons
Purchase buttons:
- Connect to selected products in your paywall
- Start the purchase when tapped

When you add a purchase button to your paywall, it automatically processes purchases your users make. So, you don't need to handle purchases in the app code.

:::note
You can attract more attention to purchase buttons by animating them. The Paywall builder currently supports **Arrow** and **Pulse** animation types. Note, that, to add the **Arrow** animation, first, you need to configure the **Arrow icon** in the **Content** section.

Each animation lets you choose an easing option (Linear, Ease In, Ease Out, Ease In Out) to control how it speeds up or slows down.

Animations are available in the Adapty iOS, Android, React Native, and Flutter SDKs starting from version 3.10.0. Follow the [migration guide](migration-to-android-310.md) for Android.
:::

### Custom buttons
You need custom buttons to:
- Close the paywall (`close`)
- Open a URL (`openUrl`)
- Restore purchases (`restore`)
- Sign in (`login`)
- Trigger custom actions (e.g., open another paywall)

To make most buttons work, you need to **handle their action IDs in your code**:
- [iOS](handle-paywall-actions.md)
- [Android](android-handle-paywall-actions.md)
- [React Native](react-native-handle-paywall-actions.md)
- [Flutter](flutter-handle-paywall-actions.md)
- [Unity](unity-handle-paywall-actions.md)

For example, a close button needs the `close` action handler.

:::important
`close` is handled automatically in the iOS, Android, and React Native SDKs. `openUrl` is handled automatically in the iOS and Android SDKs. However, if needed, you can override the default behavior.

`restore` is always handled automatically.
:::

When handling custom actions in your code, you can implement scenarios like:

- Opening another paywall
- Running multiple actions in sequence (like close and open)

Note that you would need to build these scenarios using the action handling system - they're not built-in features.

## Add buttons

Use a Pulse Button to draw attention and boost click-through rates. Or add a Countdown Button on trial expiration slides to create urgency and increase upgrades.

To add a button:
1. Click **Add** at the top left.
2. Select **Buttons** and choose one:
   - **Button**
   - **Pulse Button**
   - **Glossy Button**
   - **Pulse Glossy Button**
   - **Countdown Button**
3. Choose the [button action](onboarding-actions.md) from the **On Press** dropdown on the right:
   - **Navigate**: Moves the user to a specified onboarding screen.
   - **Show/Hide element**: Shows or hides a target element.
   - **Open paywall**: Opens the paywall screen for purchases. Learn how to handle opening paywall on [iOS](ios-handling-onboarding-events.md#opening-a-paywall), [Android](android-handle-onboarding-events.md#opening-a-paywall), [React Native](react-native-handling-onboarding-events.md#opening-a-paywall), and [Flutter](flutter-handling-onboarding-events.md#opening-a-paywall).
   - **Scroll to**: Scrolls the page to a specific element.
   - **Custom**: Runs your custom event logic. For example, ut can be used for opening a login window or requesting the app permissions. Learn how to handle custom action on [iOS](ios-handling-onboarding-events.md#custom-actions), [Android](android-handle-onboarding-events.md#custom-actions), [React Native](react-native-handling-onboarding-events.md#handle-custom-actions), and [Flutter](flutter-handling-onboarding-events.md#handle-custom-actions).
   - **Close onboarding**: Closes the onboarding flow. Learn how to handle onboarding closure on [iOS](ios-handling-onboarding-events.md#closing-onboarding), [Android](android-handle-onboarding-events.md#closing-onboarding),[React Native](react-native-handling-onboarding-events.md#closing-onboarding), and [Flutter](flutter-handling-onboarding-events.md#closing-onboarding).

To edit button text, click the button preview and make your changes in WYSIWYG mode.

## Button customization

Beyond the basic [element layout](onboarding-layout.md#element-layout), you can customize button appearance:

1. Select the button element on the left.
2. Go to **Styles** in the right menu.
3. Based on the button type, you can adjust these options:
   - **All buttons**: Width, padding, background, roundness, border, border color, shadows, next arrow and arrow size, right offset, text or countdown color, font, and line height.
   - **Pulse Button**: Animation duration and easing, shadow color and size, button grow.
   - **Glossy Button**: Glossy line color, width, angle, and animation duration.
   - **Pulse Glossy Button**: Animation duration and easing, shadow color and size, button grow, glossy line color, width, angle, and animation duration.
