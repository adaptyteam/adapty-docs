The **Flow Builder** is a visual no-code editor for paywalls and interactive screen sequences. Build a single-screen paywall to accept a purchase, or a multi-step onboarding that ends in a paywall. The Adapty SDK renders the flow natively in your app. After you deploy a flow, you can update its copy, design, and logic at any time — without an app release.

## What you can build

The Flow Builder supports two kinds of flows:

- **Paywalls** — screens that allow the user to purchase a [product](create-product). Adapty handles the transaction.
    :::tip
    For a step-by-step example, see the [Basic paywall screen guide](basic-paywall-screen).
    :::
    <ZoomImage id="sample-paywall.webp" width="300px" alt="Sample paywall" />

- **Onboarding flows** — multi-screen sequences with text input, quizzes, branching, and personalized content. End the flow in a paywall, or close it into your app.

    <ZoomImage id="sample-onboarding-flow.webp" width="600px" alt="Sample onboarding flow" />


## Build your flow

:::link
Main article: [Interface overview](builder-ui)
:::

Edit your flow in a WYSIWYG workspace with a device preview. Pick an element from the left panel and edit its properties on the right. Preview the result across devices, orientations, themes, and locales.


<ZoomImage id="flow-builder-overview.webp" width="600px" alt="Flow Builder interface with a paywall preview" />


### Screens and layers

:::link
Main article: [Screens and Layers](paywall-layout-and-products)
:::

Build a flow as a sequence of screens, each composed of nested layers. Reorder layers to change the visual order. Drag them between containers to restructure, or duplicate a whole screen to create a variant.

### Elements

:::link
Main article: [Elements](builder-elements)
:::

The Flow Builder includes a wide variety of elements:


<ZoomImage id="element-selection.webp" width="600px" alt="Element library in the Flow Builder" float="left" />


- **Containers** — [stacks](#automatic-layouts) with automatic item distribution, [tabs](builder-tabs), [carousels](paywall-carousel), etc.
- [Text elements](onboarding-text) with presets based on your saved styles.
- [Media](custom-media) — vector icons, images, and videos.
- [Buttons](paywall-buttons) of different shapes and sizes. Some include built-in actions — to purchase products, close the paywall, or restore earlier purchases.
- [Interactive quizzes](onboarding-quizzes) with different visual styles.
- [Paywall elements](#products-and-purchases) — product cards, [trial toggles](builder-toggles), [countdown timers](paywall-timer).
- [Inputs](builder-inputs-and-forms) — text, email, phone, date, number, and password.
- [Progress indicators](builder-loaders-and-progress-bars) — bars and loaders.


### Automatic layouts

:::link
Main article: [Element positioning](manage-paywall-ui-elements)
:::

The flow builder was designed for automatic layout management. Nest elements inside vertical or horizontal containers. Each container will distribute its children with consistent spacing, alignment, and distribution. Enable absolute or fixed positioning to overlay elements over one another.

<ZoomImage id="alignment-distribution-demo.webp" width="600px" alt="Alignment and distribution inside an automatic container" />

### Styles

:::link
Main article: [Element styling](builder-styling)
:::

Save reusable color and text styles, then apply them across every screen. Give each color style a light and a dark value to support dark mode automatically.

### Products and purchases

:::link
Main article: [Products and purchases](paywall-product-block)
:::

Add a [product card](create-product) to connect a product from your App Stores to the flow.

The localized title, price, and offer details fill in automatically. Use the built-in Purchase action to start a transaction.

## Add behavior

<ZoomImage id="add-action.gif" width="400px" alt="Adding an action in the Interactions tab" />

You can make any element interactive. Open the **Interactions** tab and specify an [action](onboarding-actions) to perform. Navigate to a different screen, open a URL, [show or hide a layer](onboarding-element-visibility).

### Variables

:::link
Main article: [Variables](onboarding-variables)
:::

Drop variables into any text field to address the user by name, show a product's price, or process user input in other ways.

Variables come from three sources: custom values your app code sets, product data from the stores, and element state (quiz selections, input values, toggle state).


### Quizzes, inputs, and forms

:::link
Main article: [Inputs and forms](builder-inputs-and-forms)
:::

Collect user data with [quizzes](onboarding-quizzes) and inputs. Reuse any answer as a variable — anywhere in the flow, or in your app code.

### Navigation and branching

:::link
Main article: [Navigation and interaction](builder-navigation-actions)
:::

Use **static navigation** to move users between screens in a fixed sequence. Use [conditional actions](onboarding-navigation-branching) to split the flow into branches for different users.

## Ready for production

### Dark mode

:::link
Main article: [Dark mode](paywall-dark-mode)
:::

Define a light and a dark value on every color style. The flow flips themes when the device does. Toggle modes from the bottom toolbar to verify the result.

<ZoomImage id="light-and-dark-mode.webp" width="600px" alt="Light and dark mode preview in the Flow Builder" />


### Multi-locale support

:::link
Main article: [Localization](add-paywall-locale-in-adapty-paywall-builder)
:::

Add locales in the Localization panel. Translate manually, import a `.tsv` from a translator, or use the built-in AI translation. Change the active preview locale from the bottom toolbar.

### Save and Publish

:::link
Main article: [Save and publish flows](builder-save-publish)
:::

- Save a draft to develop a flow without affecting your app's users.
-  To display a live flow, **publish it** and attach it to a [placement](create-placement).
- Use the `getFlow()` method to retrieve the flow in your app.The users will see the flow when they reach that placement.

:::tip
A placement can hold different flows for different [audiences](add-audience-paywall-ab-test).
:::

## Migrating from the old Paywall Builder

You can use the `getFlow()` with both legacy paywalls and new flows. When you're ready to migrate, simply attach a flow to your placement.
