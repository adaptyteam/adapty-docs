The **Flow Builder** is a visual no-code editor for screen sequences in your app.

- **Flexible customizable flows** — Build any kind of flow — single-screen paywall, multi-step sequences, surveys, anything in between.
- **Native rendering** — The Adapty SDK doesn't use web views to ensure a seamless user experience.
- **Update without redeploying** — Change copy, design, or logic any time. Updates reach your users without an app release.

<ZoomImage id="flow-builder-overview.webp" width="700px" alt="Flow Builder workspace" />


## What you can do

Build flows that:

- **Enable a purchase** — single-screen flows that handle an in-app sale.

    <ZoomImage id="sample-paywall.webp" width="300px" alt="Sample single-screen flow" />

- **Walk the user through a multi-step experience** — screen sequences with text, quizzes, branching, and personalized content.

    <ZoomImage id="sample-onboarding-flow.webp" width="600px" alt="Sample multi-screen flow" />

- **Collect user input** — use form elements to save and process user responses.

## Create a flow

Open the **Flows** page and click **Create flow**. Use the following tools to customize your flow:

- **[Element library](builder-elements)** — pre-built blocks for text, media, buttons, forms, and other content.

    <ZoomImage id="element-selection.webp" width="400px" alt="Element library in the Flow Builder" />

- **[Actions](onboarding-actions)** — navigate between screens, open URLs, run purchases, and more.

<ZoomImage id="add-action.gif" width="500px" alt="Adding an action in the Interactions tab" />

- **[Variables](onboarding-variables)** — use variable values in text or to enable trigger-based flow logic.
- **[Conditional navigation](onboarding-navigation-branching)** — branch the flow based on user input.
- **[Quizzes and inputs](builder-inputs-and-forms)** — collect and process user input.
- **[Dark mode](paywall-dark-mode)** — style elements to match the device theme.

    <ZoomImage id="light-and-dark-mode.webp" width="600px" alt="Light and dark mode preview in the Flow Builder" />

- **[Localization](add-paywall-locale-in-adapty-paywall-builder)** — manual or AI-enabled.
- **[Save and publish](builder-save-publish)** — safely work on drafts; publish and attach to a [placement](create-placement) when ready.

:::tip
A placement can hold different flows for different [audiences](add-audience-paywall-ab-test).
:::
