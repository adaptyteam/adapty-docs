## Stacking order (Z-index)

Bring to Front: Sets the node's Z-index to the highest value in the current context (moves to the end of the visual stack).
Send to Back: Sets the node's Z-index to the lowest value (moves to the beginning of the visual stack).

## Set position manually

This section defines how an element is positioned relative to its parent or the entire screen.Location: This section is situated within the Right Sidebar (Property Panel / Inspector) under the Position heading.Target Object: Applied to any visual element (Frame, Text Node, Group) that needs explicit positioning control relative to its parent or the viewport.

### Relative position

In Flow — Default Standard behavior for native apps. The element acts like a brick in a wall: it takes up space and pushes neighboring elements down or sideways.

How it works: The position is automatically determined by the parent container (Stack). You cannot move the element freely, but you can adjust Margins.

When to use: For 95% of content: text blocks, images, lists, cards.

### Absolute position

Absolute (Overlay) — Toggle ON Switches the element to overlay mode. The element "lifts" out of the flow and can be placed on top of other layers without affecting the layout of surrounding items.

How it works: The element is positioned relative to the corners or center of its parent.

When to use: For decorative shapes, icons over images, "Sale" badges, "Close" buttons.

### Fixed position

The element ignores the parent container and pins to the device screen edges (Viewport).

Behavior: The element "sticks to the glass" of the phone. The page content scrolls underneath, while the element stays fixed in place.

Example: A "Buy Now" button stuck to the bottom, floating navigation bar, header

## Auto-layout and wrapping

Wrap: Wraps the selected node(s) in a new parent container.
Wrap in Horizontal Stack: Creates a parent with flex-direction: row.
Wrap in Vertical Stack: Creates a parent with flex-direction: column.
Wrap in Frame: Creates a basic container (absolute/fixed positioning context).
Ungroup / Unwrap: Removes the selected container node and promotes its children to the grandparent level, preserving their relative order.

Move Up / Left: Decrements the node's index in the parent's children array (visually moves it up or left).
Move Down / Right: Increments the node's index in the parent's children array (visually moves it down or right).

### Props

#### Direction

Vertical (VStack): Elements are arranged from top to bottom.

Horizontal (HStack): Elements are arranged from left to right.

#### Gap

Sets a uniform spacing between all adjacent elements inside the stack.

#### Margins

Defines the space around the element (separating it from neighboring elements).

#### Padding

Defines the space inside the element, between its content and its boundary.

#### Clip content

Trigger: Click on the checkbox.
Logic: Immediate update (Immediate Update). If enabled, content exceeding the element's bounds is visually clipped in the canvas.

#### Wrap

Logic: The Wrap property is only relevant and active when the element is configured as a HStack.If Wrap is enabled, and the container size is fixed, elements will wrap. If the container size is dynamic (Auto), the container size will grow along the cross-axis to accommodate wrapped content.

##### Alignment

Controls how elements are positioned along the container’s cross-axis (for example, in a vertical stack this is horizontal alignment: left, center, right).

#####  Distribution

* Space between
* Space around
* Space evenly
* Mixed (off)
