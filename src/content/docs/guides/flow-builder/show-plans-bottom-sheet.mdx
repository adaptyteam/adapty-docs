This template surfaces a single highlighted offer first, with a discreet link to the full plan list. Tapping **Show all plans** slides up a bottom sheet that contains the other products, a purchase button, and the footer links.

Use it when one plan converts disproportionately well — the bottom sheet keeps the alternatives one tap away without crowding the main screen.

## Before you start

- [Create products](create-product) in the Adapty Dashboard.
- [Connect Adapty to the App Store and Google Play](integrate-payments).

## 1. Set up the screen layout

Configure the screen as a vertical container with the hero image as the background and the call-to-action elements pinned to the bottom.

For the full list of screen properties, see [Screens and layers — Screen settings](paywall-layout-and-products#screen-settings).

To configure the screen:

1. Click an empty area of the canvas to select the screen.
2. Under **System UI**, disable **Safe area** so the hero image extends to the screen edges.
3. Under **Layout**, set the direction to **Vertical** <Inline id="vertical-layout.svg" alt="Vertical" /> and the distribution to **Space between** — the CTA heading anchors to the top, and the buttons anchor to the bottom.
4. Under **Fill**, choose **Image** <Inline id="fill-image.svg" alt="Image" /> and upload your hero image.

<ZoomImage id="plans-bottom-sheet-screen-settings.webp" width="300px" alt="Screen settings with Image fill and Space between distribution" />

## 2. Add the CTA heading

The heading sits at the top of the screen and communicates the offer in one line.

1. Click **+** > **Text** > **H1**.
2. With the H1 selected, open the **Design** tab and edit the text in the **Content** field.

<ZoomImage id="plans-bottom-sheet-heading.webp" width="500px" alt="CTA heading added to the top of the screen" />

## 3. Add the bottom sheet

The bottom sheet is a layout container that slides up from the bottom of the screen. Add it now — hidden by default — so the products inside become available as screen variables in the next steps.

For more on bottom sheets and other layout containers, see [Elements — Layout](builder-elements#layout).

To add the bottom sheet:

1. Click **+** > **Layout** > **Bottom Sheet**.
2. With the bottom sheet selected, open the **Design** tab.
3. Under **Visibility**, set the state to **Hide** <Inline id="visibility-hide.svg" alt="Hide" />. The bottom sheet stays in the layer tree but doesn't appear until a button shows it.

<ZoomImage id="plans-bottom-sheet-add-sheet.webp" width="500px" alt="Bottom sheet added to the layer tree with Visibility set to Hide" />

## 4. Add the product list inside the bottom sheet

Place the three products inside the bottom sheet. One of them — the monthly subscription — also drives the price shown on the main CTA button.

For more on managing products, see [Set up purchases](paywall-product-block).

To add and configure products:

1. In the **Layers** panel, select the bottom sheet layer so new elements nest inside it.
2. Click **+** > **Products** and choose a layout preset. Vertical List works well for three plans.
3. Select each product card on the canvas and pick a product from the dropdown in the **Design** tab.
4. Select the monthly subscription card and enable **Set as default product** in the **Design** tab.

<ZoomImage id="plans-bottom-sheet-products.gif" width="700px" alt="Adding the Products element inside the bottom sheet and assigning products" />

## 5. Add the purchase button inside the bottom sheet

The bottom sheet needs its own purchase button to buy whichever plan the user selects from the list.

1. With the bottom sheet still selected in the **Layers** panel, click **+** > **Buttons** and choose a button preset.
2. With the button selected, open the **Interactions** tab in the right panel.
3. Click **Add trigger** > **On tap**, then click **Add action**.
4. Set **Action** to **Purchase** and **Product** to `products.selectedProduct`.

<ZoomImage id="plans-bottom-sheet-purchase-button.webp" width="500px" alt="Purchase action configured on the bottom sheet's purchase button" />

## 6. Add the footer links inside the bottom sheet

Terms of use, privacy policy, and restore purchases sit at the bottom of the sheet — the main screen stays uncluttered.

1. With the bottom sheet still selected in the **Layers** panel, click **+** > **Buttons** > **Links**. This adds a row with Restore Purchases, Terms of Use, and Privacy Policy.
2. Select the **Terms of Use** link, open the **Interactions** tab, and paste your terms URL into the **Open URL** field.
3. Repeat for the **Privacy Policy** link with your privacy URL.
4. Leave the **Restore Purchases** link as is. Its action is preconfigured.

<ZoomImage id="plans-bottom-sheet-footer-links.webp" width="500px" alt="Footer links inside the bottom sheet" />

## 7. Add the main subscribe button

The main button on the screen subscribes the user to the monthly plan in one tap. Its label uses the monthly product's price variable so the button stays in sync with the product.

1. In the **Layers** panel, click the screen so new elements add to the root, not inside the bottom sheet.
2. Click **+** > **Buttons** and choose a button preset.
3. With the button selected, open the **Design** tab and place the cursor in the **Content** field. Click the variable icon <Inline id="variable.webp" alt="Variable icon" /> and pick the price variable for the monthly product. Surround it with the rest of the label — for example, `Subscribe for {price}/month`.
4. Switch to the **Interactions** tab and click **Add trigger** > **On tap** > **Add action**.
5. Set **Action** to **Purchase** and **Product** to the monthly subscription product. Unlike the bottom sheet button, this one targets a specific product instead of `products.selectedProduct`.

<ZoomImage id="plans-bottom-sheet-main-button.webp" width="500px" alt="Main subscribe button with the price variable bound to the monthly product" />

## 8. Add the 'Show all plans' link

Below the main button, a borderless text link gives users a way to see the full plan list.

1. With the screen still selected in the **Layers** panel, click **+** > **Buttons** and choose a button preset. The Base preset works — you'll style it as a borderless link in the next step.
2. With the button selected, open the **Design** tab and edit the **Content** field to read `Show all plans`.
3. Style the button to look like plain text: under **Fill**, set it to a transparent color or remove the fill; under **Border**, choose **None** <Inline id="border-none.svg" alt="None" />.

For details on styling, see [Element styling](builder-styling).

<ZoomImage id="plans-bottom-sheet-show-link.webp" width="500px" alt="Show all plans button styled as a borderless text link" />

## 9. Open the bottom sheet on tap

Wire the **Show all plans** button to reveal the hidden bottom sheet.

1. Select the **Show all plans** button.
2. Open the **Interactions** tab and click **Add trigger** > **On tap** > **Add action**.
3. Set **Action** to **Show** and pick the bottom sheet element from the dropdown.

For more on the Show/Hide action, see [Actions — Show/hide elements](onboarding-actions#showhide-elements).

<ZoomImage id="plans-bottom-sheet-show-action.webp" width="500px" alt="Show action targeting the bottom sheet" />
