## Left panel: Style view

A centralized hub for managing the project's styles. This panel allows you to create and edit global styles (Colors and Typography) to ensure visual consistency and seamless Dark Mode support across the entire app.

Text: A list of all typography presets (Headers, Body, Captions).
Colors: A list of color styles (Brand colors, Backgrounds, Feedback states).

Core Principle: Updating a style in this panel immediately propagates changes to all elements across all screens where this style is applied.

## Left panel: Product view 

A control panel for mapping subscription products to paywall screens. It allows assigning, grouping, and configuring products for each specific screen in the flow without leaving the builder context.This panel solves the task of Mapping. We do not create products here (that happens via the Edit button); we only "place" existing products onto the "shelves" (screens) of our store.


## Left panel: Variables view

The Variables Panel is the central data hub of the builder. It allows users to define dynamic content and logic without writing code. Variables serve as placeholders that the SDK fills with real data at runtime. They are used for:

Text Binding: Replacing static text with dynamic values (e.g., "Hello, ```{{user.name}}```").
Logic & Visibility: Controlling element visibility (e.g., "Show Upgrade Button if isPremium is false").
Conditional Styles: Adapting UI based on themes or states.
