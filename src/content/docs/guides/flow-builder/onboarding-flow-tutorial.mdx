A multi-screen flow in the Flow Builder is a sequence of screens connected by navigation actions. The flow can stay linear, or branch based on user input collected on an earlier screen. This tutorial walks through the end-to-end process — creating screens, building their content, wiring navigation, and adding conditional branching — using a four-screen onboarding flow as the worked example.

The example uses:

- A **name input** that exposes the user's name as a variable for personalization.
- A **single-choice quiz** whose answer drives which screen the user sees next.
- **Two branching paths** with tailored copy for each audience segment.
- A **paywall** as the final screen.

The same pattern applies to any flow that personalizes content based on user input.

## Before you start

- [Create products](create-product) in the Adapty Dashboard. The example flow uses two — an Annual and a Monthly subscription.
- [Connect Adapty to the App Store and Google Play](integrate-payments).

## 1. Set up reusable styles

Reusable styles let you apply consistent typography and colors across every screen with a single click. Color styles hold a Light and a Dark variant, so the flow automatically supports both themes.

For full instructions, see [Element styling — Reusable styles](builder-styling#reusable-styles).

To set up the styles:

1. In the left panel, open the **Styles** <Inline id="styles.svg" alt="Styles" /> panel.
2. On the **Colors** tab, click **<Inline id="plus.svg" alt="Plus" /> Create style** and add the colors you'll reuse. For each color, pick a Light value, switch to the Dark tab, and pick a Dark value.

<ZoomImage id="flow-onboarding-color-styles.webp" width="500px" alt="Colors tab in the Styles panel with Light and Dark variants" />

3. On the **Text** tab, click an existing style to edit its font, weight, and size, or click **<Inline id="plus.svg" alt="Plus" /> Create style** to add custom presets.

<ZoomImage id="flow-onboarding-text-styles.webp" width="500px" alt="Text tab in the Styles panel with custom presets" />

## 2. Create the screens

A flow is a sequence of screens. Configure the first screen with the shared base — layout, background, and safe area — then duplicate it for the rest. This way, every screen shares the same foundation and you only set it up once.

For more on managing screens, see [Screens and Layers — Manage screens](paywall-layout-and-products#manage-screens).

To set up the screens:

1. Click an empty area of the canvas on the first screen to open the screen settings.
2. Under **System UI**, disable **Safe area** so backgrounds and edge-aligned elements can extend to the screen edges.
3. Under **Fill**, choose a background type and configure it — for example, an **Image** <Inline id="fill-image.svg" alt="Image" /> that sits behind every screen of the flow.
4. Under **Layout**, set the direction to **Vertical** <Inline id="vertical-layout.svg" alt="Vertical" /> and pick a distribution that fits your design.

<ZoomImage id="flow-onboarding-screen-settings.webp" width="300px" alt="Screen settings with background image, safe area off, and vertical layout" />

5. In the **Screens** section of the left panel, click the three-dot menu <Inline id="context.svg" alt="Context menu" /> on the first screen and choose **Duplicate**. Repeat until you have four screens in total — the second branching path will be added later by duplicating the first one.
6. Rename each screen to match its role — in our example: `Welcome`, `Quiz`, `Rock path`, and `Paywall`.

<ZoomImage id="flow-onboarding-screens.webp" width="300px" alt="Four named screens sharing the same base settings" />

## 3. Build the introduction screen

The first screen usually sets the tone — a headline, a feature list, and a call to action that opens the rest of the flow. In our example, this is the Welcome screen.

Click the **Welcome** screen in the **Screens** panel, then add the elements:

1. Add the main image. Click **+** > **Media** > **Image**, upload your image, and adjust margins if needed.

<ZoomImage id="flow-onboarding-welcome-image.webp" width="700px" alt="Top of the Welcome screen with the main image and headline" />

2. Add a headline: click **+** > **Text**, pick a heading style from your saved text styles, and edit the **Content** field.

<ZoomImage id="flow-onboarding-welcome-top.webp" width="700px" alt="Top of the Welcome screen with the main image and headline" />

3. Add the feature list. Click **+** > **List** > **Icon Cards**, then edit the icon and label in each card.

<ZoomImage id="flow-onboarding-welcome-features.webp" width="700px" alt="Icon Cards feature list below the headline" />

4. Add a primary navigation button at the bottom. It'll get an action wired up in the navigation step.

<ZoomImage id="flow-onboarding-welcome.webp" width="700px" alt="Final Welcome screen" />

## 4. Build the input and quiz screen

The second screen collects input from the user. In our example, it asks for a name and a single-choice answer that drives which path the user sees next.

For more on inputs and quizzes, see [Inputs and forms](builder-inputs-and-forms) and [Surveys and quizzes](onboarding-quizzes).

Click the **Quiz** screen in the **Screens** panel, then add the elements. Each group on the screen — intro, question + input, question + quiz — sits in its own Vertical Stack so related elements stay together visually.

1. Add the intro headline and body. Click **+** > **Text** > **H1** for the headline and **+** > **Text** > **Body** for the supporting copy.
2. Group the intro. Click **+** > **Layout** > **Vertical Stack**, drag the new stack to the top of the layer tree, then drag the H1 and body inside it.

<ZoomImage id="flow-onboarding-quiz-intro.gif" width="700px" alt="Intro headline and body grouped in a Vertical Stack at the top of the layer tree" />

3. Add the first question and input. Click **+** > **Text** for the question caption, then click **+** > **Inputs** > **Text** for the field.
4. Set the input's **Element ID** in the **Design** tab — in our example, `name`. This exposes the value as a variable other screens can reference.

<ZoomImage id="flow-onboarding-quiz-input-element-id.webp" width="700px" alt="Name input with Element ID name, grouped with its caption in a Vertical Stack" />

5. Group the caption and input in a Vertical Stack the same way as the intro.
6. Add the second question and quiz. Click **+** > **Text** for the caption, then click **+** > **Quiz** and pick a layout preset like Icon Options. Configure the options — in our example, `Rock` and `Hip hop`.
7. Group the caption and quiz in a Vertical Stack the same way.
8. Set the option IDs. Select each quiz option, open the **Interactions** tab, and set its **Element ID**. These IDs are referenced in the conditional navigation later.

<ZoomImage id="flow-onboarding-quiz-option-id.webp" width="700px" alt="Quiz group set to Single choice in Selectable Groups" />

9. Switch the quiz to single choice: click an empty area of the canvas to open **Screen settings**, scroll down to **Selectable Groups**, click the quiz group's name, and set the type to **Single choice**.

<ZoomImage id="flow-onboarding-quiz-single-choice.webp" width="700px" alt="Quiz group set to Single choice in Selectable Groups" />

10. Add a primary button at the bottom — this is the Next button that triggers the branching.

<ZoomImage id="flow-onboarding-quiz.webp" width="300px" alt="Final quiz screen with intro, name input, and single-choice quiz" />

## 5. Build the first branching path

Each path screen tailors content for one audience segment. In our example, the Rock path features rock-focused content — playlists, artists, and recommendations.

For more on variables, see [Variables](onboarding-variables).

To build the screen:

1. In the **Screens** panel, click the **Rock path** screen.
2. Add a headline. Place the cursor in the **Content** field where the personalization should appear, click the variable icon <Inline id="variable.webp" alt="Variable icon" />, and open the **Elements** tab. Pick the screen where the input lives — in our example, **Quiz** — then pick the input's value variable. The picker resolves it as `<elementId>.value` — in our example, `name.value`. At runtime, the headline updates with whatever the user typed.

<ZoomImage id="flow-onboarding-rock-variable.webp" width="700px" alt="Variable picker on the Elements tab showing the Quiz screen and name.value variable" />

3. Add the body copy as additional text elements, tuned to the audience segment for this path.
4. Add a primary button at the bottom.

<ZoomImage id="flow-onboarding-rock.webp" width="300px" alt="Final first path screen with personalized headline" />

## 6. Build the second branching path

Path screens usually share a layout — only the copy changes. Duplicate the first path screen and update the content.

To duplicate and update:

1. In the **Screens** panel, select the first path screen and press ⌘D / Ctrl+D to duplicate it. The copy appears at the end of the screen list.
2. Rename the copy — in our example, `Hip hop path` — and drag it to the correct place in the screen list, so it sits next to the screen it was duplicated from.

<ZoomImage id="flow-onboarding-duplicate.gif" width="700px" alt="Duplicated path screen renamed and updated for the other audience" />

3. Update the body copy for the other audience segment. The personalized headline keeps working — the variable carries over.

<ZoomImage id="flow-onboarding-hiphop.webp" width="300px" alt="Duplicated path screen renamed and updated for the other audience" />

## 7. Build the paywall

The final screen is the paywall — where the user can subscribe. For a fuller walkthrough of paywall mechanics, see [Create a basic paywall screen](basic-paywall-screen). The version below condenses that walkthrough.

Click the **Paywall** screen in the **Screens** panel, then add the elements:

1. Add a **Horizontal Stack** at the top and add a **Close** button inside it. The Close preset comes preconfigured.
2. Add the main image, the headline (with the same personalization variable as on the path screens), and a subheading as supporting copy.

<ZoomImage id="flow-onboarding-paywall-top.webp" width="300px" alt="Top of the paywall with close button, main image, and headline" />

3. Add the products: click **+** > **Products** and pick **Vertical List**. Assign each card a product from the dropdown in the **Design** tab.
4. Click the default product's card and enable **Set as default product** so it's pre-selected when the screen loads.

<ZoomImage id="flow-onboarding-paywall-products.gif" width="700px" alt="Product list with one card marked as default" />

5. Add the purchase button. Click **+** > **Buttons** and choose a preset. In the **Interactions** tab, click **Add trigger** > **On tap** > **Add action** and set **Action** to **Purchase** with **Product** as `products.selectedProduct`.

<ZoomImage id="flow-onboarding-paywall-purchase.webp" width="700px" alt="Top of the paywall with close button, main image, and headline" />

6. Add the footer links. Click **+** > **Buttons** > **Links** — the row contains Restore Purchases, Terms of Use, and Privacy Policy. The Restore link is preconfigured. For Terms and Privacy, select each link's text in the **Content** field, click the link icon in the formatting toolbar, and paste the URL.

<ZoomImage id="flow-onboarding-paywall.webp" width="700px" alt="Final paywall with products, purchase button, and links" />

## 8. Wire navigation between the screens

Screens don't connect to each other automatically. Use **On tap** triggers and **Navigate to** actions to connect each screen's primary button to its next screen. A screen that branches based on user input uses a **Conditional action** instead.

For more on navigation and conditional actions, see [Navigation and interaction](onboarding-navigation-branching) and [Actions — Conditional actions](onboarding-actions#conditional-actions).

To wire the navigation for the example flow:

1. **Static navigation from the introduction screen.** Open the Welcome screen, select the primary button, and switch to the **Interactions** tab. Click **Add trigger** > **On tap** > **Add action**, set **Action** to **Navigate to**, and pick the next screen — in our example, the Quiz screen.

<ZoomImage id="flow-onboarding-nav-static.gif" width="700px" alt="Static Navigate to action on the Welcome screen's primary button" />

2. **Conditional navigation from the quiz.** Open the Quiz screen, select the Next button, and add an **On tap** trigger with a **Conditional action**. Set up the IF/ELSE rule:
   - In the variable picker, open the **Elements** tab, pick the **Quiz** screen, and select `quiz.selectedOptionId`.
   - Use the **Equals** operator and compare against the ID of one of the options — in our example, the Rock option.
   - **IF** the comparison matches, fire **Navigate to** and pick the first path screen.
   - **ELSE**, fire **Navigate to** and pick the second path screen.

<ZoomImage id="flow-onboarding-nav-conditional.gif" width="700px" alt="Conditional action on the Quiz Next button with IF/ELSE branches" />

3. **Static navigation from each branching path to the paywall.** Repeat the pattern from step 1 on each path screen, with the paywall as the destination.

<ZoomImage id="flow-onboarding-nav-to-paywall.webp" width="700px" alt="Both paths point to the same paywall" />
