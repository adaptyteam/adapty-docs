Каждое письмо, которое отправляет Adapty Mail, содержит уникальную ссылку на чекаут для конкретного получателя. Нажав на неё, пользователь попадает в воронку веб-чекаута, которая определяет его по профилю, показывает предложение и обрабатывает платёж. Воронки чекаута хранятся в разделе **Web Paywalls** в Adapty Mail и редактируются встроенным **web paywall builder**.

## Требования \{#requirements\}

- Аккаунт веб-платёжного провайдера (Stripe, Paddle или PayPal) с настроенными продуктами подписки — его нужно будет подключить в web paywall builder.

Отдельный аккаунт для web paywall builder не нужен. Он входит в состав Adapty Mail: рабочее пространство создаётся автоматически при первом входе, а в редактор вы входите с помощью учётных данных Adapty. Это не зависит от пейволов, настроенных на странице пейволов основного дашборда Adapty — веб-пейволы Adapty Mail являются отдельными сущностями и управляются исключительно из Adapty Mail.

## Настройка воронки чекаута \{#set-up-your-checkout-funnel\}

1. В Adapty Mail перейдите в **Web Paywalls → Create**.
2. Выберите **Generate with AI**, чтобы создать воронку, или вставьте URL существующего веб-пейвола.
3. Если используете генерацию: нажмите **Log in to the paywall builder**. Web paywall builder откроется в новой вкладке. Если вы ещё не вошли, войдите с помощью учётных данных Adapty.
4. В builder'е подключите платёжного провайдера (Stripe, Paddle или PayPal) в настройках оплаты. Подробнее см. в разделе [Настройка веб-пейвола](web-paywall-configuration).
5. Проверьте сгенерированный пейвол и нажмите **Publish** в builder'е.

:::important
Пейвол должен быть опубликован, прежде чем он сможет обрабатывать трафик чекаута. Неопубликованные пейволы возвращают ошибку, когда пользователи переходят по ссылкам на чекаут из писем.
:::

## Как выглядит чекаут \{#what-the-checkout-looks-like\}

Когда пользователь нажимает на ссылку чекаута, он проходит четырёхэкранную воронку:

**1. Подтверждение email**
Приветствует пользователя по имени или email и подтверждает его личность перед продолжением.

**2. Главная страница конверсии**
Полноэкранная продающая страница. ИИ генерирует текст и изображения для каждого блока:

| Блок | Что генерирует ИИ |
|---|---|
| Заголовок | Яркий заголовок, ориентированный на преимущества |
| Подзаголовок | Поддерживающее ценностное предложение |
| Бейдж предложения | Бейдж срочности (без придуманных цен — использует расплывчатый промо-язык) |
| Кнопка CTA | Текст, ориентированный на действие, 2–5 слов |
| Преимущества | 3–6 карточек преимуществ с эмодзи и текстом |
| Функции | 3–8 описаний функций с заголовком и подзаголовком |
| Тарифы | Заголовок выбора тарифа и текст таймера предложения |
| Социальное доказательство | Текст о сообществе и 3–5 реалистичных отзывов пользователей |
| FAQ | 3–6 распространённых вопросов и ответов |
| Гарантия | Текст гарантии возврата денег или удовлетворённости |

**3. Успешная оплата**
Поздравительное сообщение с дальнейшими шагами и сгенерированным ИИ изображением.

**4. Ошибка оплаты**
Дружелюбное сообщение с предложением попробовать ещё раз. Состояние чекаута сохраняется.

## Как работает персонализация \{#how-personalization-works\}

Каждое письмо содержит уникальный URL чекаута с `customer_user_id` получателя и его email-адресом в виде параметров:

```
https://your-funnel.com/?cid={{customer_user_id}}&email={{email}}
```

Adapty генерирует эти URL автоматически при отправке каждого письма — никакой настройки в web paywall builder не требуется. Когда пользователь переходит по ссылке, builder считывает параметры для его идентификации. После завершения покупки Adapty связывает доход с конкретным письмом, которое привело к конверсии. Эти данные отображаются в [аналитике кампаний](mail-analytics).

## Устранение проблем \{#troubleshooting\}

| Проблема | Решение |
|---|---|
| Ссылка чекаута не открывается | Убедитесь, что пейвол опубликован в web paywall builder |
| Пользователь не идентифицирован в чекауте | Проверьте, что `Adapty.identify()` был вызван с правильным ID пользователя до отправки письма |
| Покупка не атрибутирована к письму | Проверьте, что параметр `cid` присутствует в URL чекаута — если параметры отсутствуют, обратитесь в службу поддержки |
