**Flow Builder** — это визуальный редактор без кода для создания последовательностей экранов в вашем приложении.

- **Любой тип флоу** — одноэкранные пейволы, многошаговые онбординги с пейволом и всё, что между ними.
- **Гибкий дизайн** — воссоздавайте сложные экраны визуально: без единой строчки кода.
- **Нативный рендеринг** — Adapty SDK не использует веб-вью, чтобы обеспечить бесшовный пользовательский опыт.
- **Обновления без релиза** — меняйте тексты, дизайн или логику в любой момент. Изменения доходят до пользователей без обновления приложения.

## Создание флоу \{#create-a-flow\}

Откройте страницу **Flows** и нажмите **Create flow**. Для настройки флоу используйте следующие инструменты:
- **[Библиотека элементов](builder-elements)** — готовые блоки для текста, медиа, кнопок, форм и другого контента.
- **[Действия](onboarding-actions)** — переход между экранами, открытие URL, запуск покупок и другое.
- **[Переменные](onboarding-variables)** — использование значений переменных в тексте или для логики флоу на основе триггеров.
- **[Условная навигация](onboarding-navigation-branching)** — ветвление флоу в зависимости от действий пользователя.
- **[Квизы и поля ввода](builder-inputs-and-forms)** — сбор и обработка пользовательского ввода.
- **[Тёмная тема](paywall-dark-mode)** — оформление элементов под тему устройства.
- **[Локализация](add-paywall-locale-in-adapty-paywall-builder)** — ручная или с помощью ИИ.
[Сохраните и опубликуйте](builder-save-publish) черновик, затем привяжите его к [плейсменту](create-placement). В одном плейсменте можно настроить разные флоу для разных [аудиторий](add-audience-paywall-ab-test).
