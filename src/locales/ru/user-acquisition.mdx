Adapty User Acquisition помогает связать расходы на рекламу с доходом от подписок в кампаниях web-to-app, давая вам полное представление об экономике приложения в одном месте.

Чтобы данные о доходах отображались в Adapty User Acquisition, сначала нужно включить интеграцию в дашборде Adapty. Никаких API-ключей, токенов или идентификаторов передавать не нужно — достаточно обновить и настроить Adapty SDK.

:::important
User Acquisition доступен при использовании:
- iOS, Android и Flutter SDK версии 3.9.0 или выше.
- React Native и Capacitor SDK версии 3.10.0 или выше.
- Unity SDK версии 3.12.0 или выше.
- Kotlin Multiplatform SDK версии 3.15.0 или выше.
:::

## Перед началом работы \{#before-you-start\}

Чтобы связать данные о доходах с показателями кампаний, нужно дать Adapty возможность отслеживать ваши покупки:

- Если вы **уже реализовали встроенные покупки с помощью Adapty** — на этом этапе ничего дополнительно делать не нужно.
- Если вы **ещё не реализовали встроенные покупки и хотите использовать Adapty** — выполните шаги из [гайда по быстрому старту](quickstart), чтобы делегировать обработку покупок Adapty.
- Если вы **уже реализовали встроенные покупки без Adapty** и не планируете переходить на Adapty, [установите Adapty SDK для вашей платформы в режиме наблюдателя](implement-observer-mode). На этом этапе достаточно добавить SDK в проект, активировать его с включённым режимом наблюдателя и передавать транзакции.

Эта настройка обеспечивает атрибуцию web-to-app:
- Когда пользователи устанавливают приложение, Adapty SDK получает данные об установке из параметров ссылки, и Adapty UA может получить детали кампании.
- Adapty SDK знает обо всех событиях, связанных с доходами внутри приложения, и может атрибутировать их к веб-кампаниям.

## Шаг 1. Включите интеграцию User Acquisition \{#step-1-enable-the-user-acquisition-integration\}

Чтобы начать отправку событий о доходах в Adapty UA:
1. Перейдите в раздел [Integrations > Adapty](https://app.adapty.io/integrations/user-acquisition) в дашборде Adapty.
2. Включите переключатель.

Как только события начнут поступать, вы увидите следующие данные для каждого события:
- Название события
- Статус
- Среда
- Дата и время

## Шаг 2. Подключите рекламную платформу и добавьте трекинговые ссылки \{#step-2-connect-your-ad-platform-and-add-tracking-links\}

Adapty использует трекинговые ссылки для связи установок приложения с данными кампании.
Трекинговую ссылку нужно использовать в качестве целевого URL в каждой рекламной кампании, которую вы хотите отслеживать в Adapty UA.

Если вы запускаете рекламу на нескольких платформах, настройте трекинговые ссылки для каждой платформы отдельно.

Adapty работает с рекламными платформами двумя способами:

- **Нативные интеграции (Meta Ads, TikTok Ads).** Adapty подключается напрямую к рекламной платформе. Трекинговые ссылки генерируются автоматически, а параметры кампании подставляются динамически в зависимости от места использования ссылки. Одну и ту же ссылку можно использовать в разных кампаниях, группах объявлений или креативах — Adapty автоматически получит корректные данные о кампании и расходах на рекламу.

- **Только трекинговые ссылки (все остальные рекламные платформы).** Adapty не подключается к рекламной платформе. Трекинговые ссылки создаются вручную, и все параметры кампании нужно задавать явно при создании ссылки. Данные о расходах на рекламу для таких платформ недоступны.

Чтобы создать трекинговую ссылку для Meta Ads:
1. Перейдите в раздел [Integrations > Meta](https://app.adapty.io/ua/integrations/facebook/accounts) в дашборде Adapty UA и нажмите **Continue with Facebook**.

<ZoomImage id="ua-connect-meta.webp" width="700px" />

<ZoomImage id="ua-fb-sign.webp" width="500px" />

3. Ознакомьтесь с запрашиваемыми разрешениями и нажмите **Save**.
4. Перейдите на вкладку **Web campaigns** и нажмите **Create campaign**. Выберите приложение и нажмите **Save**.

<ZoomImage id="ua-new-campaign.webp" width="500px" />

5. На вкладке **General** разверните раздел **iOS** и/или **Android** и вставьте URL приложения из App Store и/или Google Play. Затем нажмите **Save**.

<ZoomImage id="ua-url.gif" width="900px" />

6. Скопируйте значение поля **Click link** для **единой ссылки** или для ссылки на конкретную платформу. Затем в Meta Ads Manager откройте ваше объявление и вставьте эту ссылку в качестве целевого URL.

:::important
В поле **Website URL** вставьте `https://api-ua.adapty.io/api/v1/attribution/click`. Остальную часть ссылки вставьте в поле **URL parameters** в разделе **Tracking**. Это поможет получить одобрение вашего объявления в Meta. Подробнее — в [рекомендациях по настройке объявлений в Meta Ads Manager](meta-create-campaign).
:::

7. После запуска объявления в Meta Ads его данные станут доступны для анализа в дашборде Adapty UA.

Чтобы создать трекинговую ссылку для других рекламных платформ:

1. В дашборде Adapty UA перейдите в раздел **Tracking links** в боковом меню и нажмите **Create link**.

    <ZoomImage id="new-tracking-link.webp" width="700px" />

2. Выберите приложение из списка и нажмите **Next**.

    <ZoomImage id="ua-choose-app.webp" width="700px" />

3. Заполните параметры ссылки, чтобы привязать её к нужной кампании и объявлению.
4. По умолчанию создаётся One Link — он автоматически определяет платформу пользователя и после отслеживания клика перенаправляет его в App Store или Google Play.

    Если вы хотите использовать разные URL перенаправления для каждой платформы, снимите флажок **One Link** и укажите ссылки на сторы для каждой платформы вручную.

    - **iOS link** или **Android link** — опциональные версии для конкретных платформ, если вы хотите использовать отдельные ссылки для каждого стора.
7. Перейдите на вашу рекламную платформу и вставьте ссылку в объявление в качестве целевого URL.

## Шаг 3. Запустите кампанию web-to-app и смотрите результаты \{#step-3-launch-your-web-to-app-campaign-and-view-results\}

Как только кампания запущена и пользователи начинают устанавливать приложение, Adapty начинает атрибутировать установки и доходы к вашим кампаниям.

В [дашборде аналитики Adapty UA](ua-analytics) вы увидите метрики на уровне кампании:
- Установки и конверсии
- Доход от подписок и покупок
- Разбивка показателей по рекламной платформе, кампании, группе объявлений и креативу

Метрики появляются сразу после получения событий об установках и доходах из вашего приложения. Данные о расходах на рекламу доступны для платформ с нативными интеграциями.
