Adapty User Acquisition помогает связать расходы на рекламу с доходами от подписок в кампаниях web-to-app, давая вам полное представление об экономике приложения в одном месте.

Чтобы данные о доходах появились в Adapty User Acquisition, сначала нужно включить интеграцию в дашборде Adapty. Передавать API-ключи, токены или идентификаторы не потребуется — достаточно обновить и настроить SDK Adapty.
:::important
User Acquisition доступен при использовании:
- iOS, Android и Flutter SDK версии 3.9.1 или выше.
- React Native и Capacitor SDK версии 3.10.0 или выше.
- Unity SDK версии 3.12.0 или выше.
- Kotlin Multiplatform SDK версии 3.15.0 или выше.
:::
## Прежде чем начать \{#before-you-start\}

Чтобы связать данные о доходах с эффективностью кампаний, позвольте Adapty отслеживать ваши покупки:
- Если у вас **уже реализованы встроенные покупки через Adapty**, на данном этапе ничего делать не нужно.
- Если у вас **ещё нет встроенных покупок и вы хотите использовать Adapty**, выполните шаги из [гайда по быстрому старту](quickstart), чтобы делегировать обработку покупок Adapty.
- Если у вас **уже реализованы встроенные покупки без Adapty** и вы не планируете переходить на Adapty, [установите SDK для вашей платформы в режиме наблюдателя](implement-observer-mode). На данном этапе вам нужно только добавить SDK в проект, активировать его с включённым режимом наблюдателя и передавать транзакции:
Эта настройка позволяет настроить атрибуцию web-to-app:
- Когда пользователи устанавливают приложение, SDK Adapty получает данные об установке из параметров ссылки, и Adapty UA может получить информацию о кампании.
- SDK Adapty знает обо всех событиях, связанных с доходом внутри приложения, и может атрибутировать их к веб-кампаниям.
## Шаг 1. Включите интеграцию User Acquisition \{#step-1-enable-the-user-acquisition-integration\}

Чтобы начать отправку событий дохода в Adapty UA:
1. Перейдите в [Integrations > Adapty](https://app.adapty.io/integrations/user-acquisition) в дашборде Adapty.
2. Включите переключатель.

Как только события начнут поступать, вы увидите следующую информацию по каждому из них:
- Название события
- Статус
- Окружение
- Дата и время

## Шаг 2. Подключите рекламную платформу и добавьте трекинговые ссылки \{#step-2-connect-your-ad-platform-and-add-tracking-links\}

Adapty использует трекинговые ссылки, чтобы связать установки приложения с данными кампаний.
Трекинговую ссылку нужно использовать в качестве целевого URL в каждой рекламной кампании, которую вы хотите отслеживать в Adapty UA.

Если вы запускаете рекламу на нескольких платформах, настройте трекинговые ссылки для каждой из них отдельно.

Adapty работает с рекламными платформами двумя способами:
- **Нативные интеграции (Meta Ads, TikTok Ads).** Adapty подключается напрямую к рекламной платформе. Трекинговые ссылки создаются автоматически, а параметры кампании подставляются динамически в зависимости от того, где используется ссылка. Одну и ту же ссылку можно применять в разных кампаниях, группах объявлений или креативах — Adapty автоматически получит актуальные данные о кампании и расходах на рекламу.
- **Только отслеживающие ссылки (все остальные рекламные платформы).** Adapty не подключается к рекламной платформе. Отслеживающие ссылки создаются вручную, и все параметры кампании необходимо задавать явно при создании ссылки. Данные о расходах на рекламу для этих платформ недоступны.

Чтобы создать отслеживающую ссылку для Meta Ads:
1. Перейдите в [Integrations > Meta](https://app.adapty.io/ua/integrations/facebook/accounts) в дашборде Adapty UA и нажмите **Continue with Facebook**.
<ZoomImage id="ua-connect-meta.webp" width="700px" />

<ZoomImage id="ua-fb-sign.webp" width="500px" />

3. Ознакомьтесь с запрашиваемыми разрешениями и нажмите **Save**.
4. Перейдите на вкладку **Web campaigns** и нажмите **Create campaign**. Выберите приложение и нажмите **Save**.

<ZoomImage id="ua-new-campaign.webp" width="500px" />

5. На вкладке **General** раскройте раздел **iOS** и/или **Android** и вставьте URL-адреса приложений из App Store и/или Google Play. Затем нажмите **Save**.

<ZoomImage id="ua-url.gif" width="900px" />
6. Скопируйте значение поля **Click link** для **одной ссылки** или для платформенной ссылки. Затем откройте своё объявление в Meta Ads Manager и вставьте эту ссылку в качестве URL назначения.

:::important
В поле **Website URL** вставьте `https://api-ua.adapty.io/api/v1/attribution/click`. Остальную часть ссылки вставьте в поле **URL parameters** в разделе **Tracking**. Это поможет вашему объявлению в Meta пройти модерацию. Подробнее — в [рекомендациях по настройке объявлений в Meta Ads Manager](meta-create-campaign).
:::
7. Теперь, когда вы запустите рекламу в Meta Ads, её данные станут доступны для анализа в дашборде Adapty UA.

1. В дашборде Adapty UA перейдите в **Tracking links** через боковое меню. Нажмите **Create link**.

    <ZoomImage id="new-tracking-link.webp" width="700px" />

    Если вы хотите использовать отдельные URL-адреса перенаправления для каждой платформы, снимите флажок **One Link** и укажите ссылки на сторы для каждой платформы вручную.

    - **iOS link** или **Android link** — опциональные версии для конкретных платформ, если вы хотите отдельные ссылки для каждого стора.

## Шаг 3. Запустите кампанию web-to-app и смотрите результаты \{#step-3-launch-your-web-to-app-campaign-and-view-results\}

Как только кампания запущена и пользователи начинают устанавливать приложение, Adapty начинает атрибутировать установки и доходы к вашим кампаниям.

В [дашборде аналитики Adapty UA](ua-analytics) вы увидите метрики на уровне кампании:
- Установки и конверсии
- Доход от подписок и покупок
- Разбивка показателей по рекламной платформе, кампании, группе объявлений и креативу

Метрики появляются сразу после получения событий об установках и доходах из вашего приложения. Данные о расходах на рекламу доступны для платформ с нативными интеграциями.
