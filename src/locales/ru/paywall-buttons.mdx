Кнопки — это интерактивные элементы во Flow Builder, реагирующие на нажатия пользователя. Используйте их для:
- CTA-кнопок покупки, привязанных к продуктам и автоматически обрабатывающих транзакции
- Навигации — перемещения пользователей между экранами (Далее, Назад, Закрыть, Пропустить)
- Служебных ссылок — восстановления покупок, Пользовательского соглашения и Политики конфиденциальности
:::info
Этот раздел описывает новый Flow Builder, который работает с Adapty SDK версии 4.0 и выше.
:::
## Добавление кнопок \{#add-buttons\}

Чтобы добавить кнопку:

1. Нажмите **+** и выберите **Button**.
2. Выберите тип кнопки.

## Типы кнопок \{#button-types\}
### Кнопки покупки \{#purchase-buttons\}

:::link
Чтобы кнопки покупки работали, привяжите продукты к экранам и добавьте элемент **Products**. Подробнее в [гайде](paywall-product-block).
:::

Кнопка покупки запускает встроенную покупку для продукта, который пользователь выбрал на экране. SDK автоматически обрабатывает транзакцию — дополнительный код в приложении не нужен.

Чтобы добавить кнопку покупки:
1. Нажмите **+** и выберите **Button**, затем выберите пресет кнопки.
2. Выделив кнопку, откройте вкладку **Interactions** на правой панели.
3. Нажмите **Add trigger** > **On tap**, затем нажмите **Add action**.
4. Установите **Action** в значение **Purchase**, а **Product** — в `products.selectedProduct`. Переменная `products.selectedProduct` всегда указывает на продукт, выбранный в данный момент на экране.

:::tip
Можно привлечь больше внимания к кнопкам покупки, добавив анимацию. Paywall Builder поддерживает тип анимации **Pulse**.
Настройте стиль анимации в панели **Design**.
:::

### Ссылки \{#links\}

Чтобы соответствовать требованиям сторов, вы можете добавить ссылки на:
- Условия использования
- Политику конфиденциальности
- Восстановление покупок

Чтобы добавить ссылки:
1. Нажмите **+** и выберите **Button > Purchase**. Технически это добавляет строку встроенных кнопок с предустановленными действиями — например, восстановление покупок или открытие URL. Если какие-то из них не нужны, их можно удалить.
2. Теперь настройте действия кнопок:

         style={{

             width: '500px', /* image width */
             display: 'block', /* for alignment */
             margin: '0 auto' /* center alignment */

    />
</Zoom>
- Кнопка **Restore purchases** уже обрабатывает восстановление покупок.
   - Для каждой ссылки:
        1. Нажмите на кнопку, чтобы выделить её, и перейдите на вкладку **Interactions** справа.
        2. Вставьте URL в поле.
        3. По умолчанию URL открывается во встроенном браузере для удобства пользователя. Если нужно открыть ссылку в внешнем браузере, установите флажок **Open in external browser**.

### Закрытие флоу \{#close-flow\}

Кнопка **Close** автоматически закрывает флоу.

Чтобы добавить кнопку закрытия, нажмите **+** и выберите **Button > Close flow**.

<Zoom>
    <img src="/assets/shared/img/close-flow.webp"
         style={{

             width: '500px', /* image width */
             display: 'block', /* for alignment */

:::tip
Используйте позиционирование **Absolute**, чтобы разместить кнопку закрытия в углу экрана.
:::
Вы также можете настроить любую другую кнопку для закрытия флоу с помощью [действий](onboarding-actions).
### Пользовательские кнопки \{#custom-buttons\}

Любую добавленную кнопку можно настроить на выполнение нужного действия при нажатии:
- Перейти на следующий экран
- Показать предупреждение
- Задать [переменную](onboarding-variables)
- [Показать или скрыть элементы экрана](onboarding-element-visibility)
- Открыть URL
- Восстановить покупки
- Выполнить условные действия
