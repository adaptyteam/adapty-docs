Вы можете управлять отображением элемента, добавив к нему условие. Условный элемент виден только тем пользователям, которые соответствуют заданным критериям.

:::important
Если вы показываете или скрываете элемент с помощью [действия](onboarding-actions) **Show** или **Hide**, это действие переопределяет условие **Visibility**, установленное для этого элемента.
Используйте условия **Visibility** для элементов, которые должны всегда отображаться или скрываться по фиксированному критерию. Используйте действия, если видимость должна меняться в зависимости от взаимодействия пользователя — например, показывать кнопку после того, как пользователь ответил на вопрос квиза.
:::

Чтобы добавить условие к элементу:

1. Выберите элемент на канвасе или в панели слоёв.
2. В разделе **Visibility** правой панели выберите **Conditional**.

<ZoomImage id="flow-conditional-visibility.webp" width="500px" alt="Настройки условной видимости" />
3. Настройте условие, выбрав тип свойства на одной из трёх вкладок:
   - **Custom**: Переменные, которые вы создаёте и которыми управляете; их значения можно обновлять через взаимодействия с пользователем. Подробнее — в разделе [Variables](onboarding-variables).
   - **Products**: Свойства продуктов в вашем флоу, например цена или название.
   - **Elements**: Состояния других элементов флоу, например активен ли тогл пробного периода.
4. Введите **Value** для сравнения.
5. Нажмите на оператор, чтобы при необходимости изменить его.
6. (Необязательно) Нажмите **Add condition**, чтобы добавить дополнительные условия. С помощью селектора укажите, должны ли выполняться все условия или хотя бы одно.
