Apple и Google предоставляют возможность отправлять уведомления об обновлениях подписок сразу в момент их возникновения на их серверах. Важно передавать эти события в Adapty, чтобы избежать задержек при обработке статуса подписки и обеспечить наилучший опыт для ваших пользователей.

В большинстве случаев достаточно [включить серверные уведомления App Store](enable-app-store-server-notifications) или [включить уведомления реального времени (RTDN) в Google Play Console](enable-real-time-developer-notifications-rtdn) — никакого кода писать не нужно.

Однако если вы уже обрабатываете эти события и хотите продолжать делать это, убедитесь, что вы пересылаете события в Adapty со своего бэкенда. Это просто: нужно отправить необработанный \(без каких-либо изменений\) payload от Apple или Google в качестве тела запроса на наш сервер, используя тот же URL из настроек.

<TabItem value="go" label="Go" default>    

  ```go showLineNumbers
package main

	"fmt"
	"strings"
	"net/http"
	"io"
)

func main() {
	url := "https://api.adapty.io/api/v1/sdk/apple/webhook/123a258e62fad41bfa734f4b0dbcad456/" // Replace this with the actual URL

	payload := strings.NewReader(`{"latest_receipt":"abc=","notification_type":"INITIAL_BUY"}`) // Ensure valid JSON

	method := "POST"

	client := &http.Client{}
	req, err := http.NewRequest(method, url, payload)
	if err != nil {
		fmt.Println("Error creating request:", err)
		return
	}
	req.Header.Add("Content-Type", "application/json")

	res, err := client.Do(req)
	if err != nil {
		fmt.Println("Error sending request:", err)
		return
	}
	defer res.Body.Close()
}
  ```
