Трекинговые ссылки позволяют отслеживать источники пользователей и связывать установки с рекламными кампаниями.
Когда кто-то кликает по вашему объявлению, Adapty фиксирует клик и затем сопоставляет его с событием установки, переданным SDK. Так вы видите, какие каналы, кампании, группы объявлений и отдельные объявления приносят наибольший доход — на странице [Аналитика](ua-analytics).

Вы можете создать два типа трекинговых ссылок:

- **One link** — универсальная ссылка, которая автоматически определяет платформу пользователя, фиксирует клик и перенаправляет его в App Store или Google Play.

- **Ссылки для конкретного стора** — ссылки, ориентированные на определённую платформу: они фиксируют клик и автоматически перенаправляют пользователей в App Store или Google Play. К ним также можно добавлять параметры отложенных диплинков.

## Создание трекинговых ссылок \{#create-tracking-links\}

Чтобы создать трекинговую ссылку:

1. В дашборде Adapty UA перейдите в раздел **Tracking links** в боковом меню и нажмите **Create link**.

<ZoomImage id="new-tracking-link.webp" width="700px" />

2. Выберите приложение из списка и нажмите **Next**.

<ZoomImage id="ua-choose-app.webp" width="700px" />

3. Заполните параметры ссылки, чтобы привязать её к нужной кампании и объявлению.

| Параметр          | Описание                                                                                         |
|-------------------|--------------------------------------------------------------------------------------------------|
| **Name**          | Внутреннее название трекинговой ссылки.                                                          |
| **Channel**       | Источник трафика: Meta, Reddit, TikTok и т. д. Используется для группировки кампаний в аналитике. |
| **Campaign ID**   | Уникальный идентификатор кампании в вашей рекламной платформе.                                   |
| **Campaign name** | Читаемое название кампании.                                                                      |
| **Ad set ID**     | Уникальный идентификатор группы объявлений в вашей рекламной платформе.                          |
| **Ad set name**   | Название группы объявлений.                                                                      |
| **Ad ID**         | Уникальный идентификатор конкретного рекламного креатива.                                        |
| **Ad name**       | Название рекламного креатива или варианта.                                                       |

4. По умолчанию создаётся One Link. Он автоматически определяет платформу пользователя и после фиксации клика перенаправляет его в App Store или Google Play.

Если вы хотите использовать разные URL перенаправления для каждой платформы, снимите флажок **One Link** и укажите ссылки на сторы для каждой платформы вручную.

<ZoomImage id="one-link.webp" width="700px" />

5. Нажмите **Create**.
6. Откройте страницу трекинговой ссылки и скопируйте **Click link** из одного из разделов:
   - **One link** — используйте эту ссылку для отслеживания кликов и автоматического перенаправления пользователей в нужный стор.
   - **iOS link** или **Android link** — необязательные версии для конкретных платформ, если вы хотите использовать отдельные ссылки для каждого стора.

:::tip
Вы также можете задать дополнительные параметры ссылки для [работы с отложенными данными](ua-deferred-data). Например, реализовать отложенные диплинки.
:::

<ZoomImage id="click-link.webp" width="700px" />

7. Перейдите в свою рекламную платформу и вставьте ссылку в объявление в качестве URL назначения.

Теперь установки приложения будут сопоставляться с объявлениями и кампаниями, из которых они пришли, — и вы сможете оценивать эффективность кампаний на странице **Analytics**.
