Воронки Adapty помогают найти ответы на такие вопросы:

1. Какой процент установок конвертируется в платящих клиентов?
2. Какая доля тех, кто попробовал продукт, стала лояльными пользователями?
3. На каких шагах наибольший отток и куда стоит обратить внимание?
4. Почему клиенты перестают платить?
С помощью графика воронки вы можете глубже изучить поведение пользователей, применяя фильтры и группировки.

Воронки работают с данными, которые мы собираем через SDK и уведомления стора, и не требуют никакой дополнительной настройки с вашей стороны.

:::note
Воронки отражают данные об установках в соответствии с настройкой определения установки в [App Settings](general#4-installs-definition-for-analytics).
:::

### Установки \{#installs\}

Первый столбец (1) — количество установок. Оно отображается в виде абсолютного значения (2) от общего числа установок (не уникальных пользователей), а также как 100% — максимальное входное значение для расчёта последующих конверсий. Если пользователь удалит приложение и установит его снова, это будет считаться двумя отдельными установками.
Серая область рядом отражает параметры перехода между шагами. Процент конверсии на следующий шаг (Displayed paywall) показан на флажке (3). Процент отсева и абсолютное значение потерь отображаются ниже (4).
<ZoomImage id="00416f9-CleanShot_2022-06-23_at_14.02.06.webp" width="700px" alt="Столбец установок с флагом конверсии" />
### Пейвол показан \{#paywall-displayed\}

Второй столбец (5) показывает количество пользователей приложения, которые увидели пейвол хотя бы один раз (6). Учитываются только те установки, которые произошли в выбранный период. Если пользователь увидел пейвол в выбранном периоде, но дата его установки выходит за рамки диапазона, этот просмотр не учитывается.
Здесь также отображается процент таких просмотров от 1-го шага (7). Обратите внимание, что этот процент равен серому флагу (3) первого шага. Это равенство действует только для первых шагов.
Мы собираем данные для этого шага со всех ваших пейволов, которые используют метод `logShowPaywall()`. Поэтому обязательно отправляйте каждый просмотр пейвола в Adapty с помощью этого метода, как описано в [документации](present-remote-config-paywalls#track-paywall-view-events).

Серая область рядом со 2-м столбцом обозначает переход. Процент конверсии на следующий шаг (Trial) отображается на флажке (8). Процент отвала и абсолютное количество ушедших пользователей после пейвола показаны ниже (9).
<ZoomImage id="fb11650-CleanShot_2022-06-23_at_15.54.32.webp" width="700px" alt="Столбец отображаемого пейвола с флагом конверсии" />
### Пробные периоды \{#trials\}

3-й столбец (10) показывает количество пробных периодов, активированных на пейволах пользователями, установившими приложение в выбранный период (11). Если в фильтре выбраны продукты без пробного периода, это значение равно нулю и столбец остаётся пустым.
Обратите внимание на процент триалов, взятых с 1-го шага — это конверсия от установок до триалов (12).
Вы можете заметить, что это значение не совпадает с серым флагом (8) конверсии предыдущего шага. Это потому что мы сравниваем текущее значение с 1-м шагом вверху графика, а с предыдущим шагом — на серых флагах.
Серая область рядом с 3-м столбцом показывает процент конверсии на следующий шаг (Платный), который отображается на флаге (13). Процент оттока и абсолютное значение ушедших пользователей за период триала показаны ниже (14).
<ZoomImage id="7b88909-CleanShot_2022-06-23_at_15.54.32_-_2.webp" width="700px" alt="Столбец триалов с флагом конверсии" />
### Подписки и продления \{#subscriptions-and-renewals\}

В 4-м столбце отображается количество активированных подписок (15). Для продуктов без триала это число включает прямые подписки через пейвол. Для продуктов с триалом — количество триалов, конвертированных в платные подписки. Если у вас есть продукты обоих типов — с триалом и без — отображается их суммарное значение.
Процент вверху показывает конверсию из установок (16).
Процент на сером флажке показывает конверсию к следующему шагу (продление на 2-й период) (17).
Процент и абсолютное значение отвала до продления на 2-й период отображаются под показателем конверсии (18).

<ZoomImage id="d13bf9b-CleanShot_2022-06-23_at_15.54.32-3.webp" width="700px" alt="Столбец подписок и продлений" />
Этот шаг начинает последовательность шагов с похожей структурой. После 2-го продления идёт 3-е, затем 4-е и так далее. Если в истории вашего приложения достаточно данных, с помощью горизонтальной прокрутки можно увидеть десятки периодов. Логика этих шагов остаётся неизменной:

- процент от установок — сверху,
- процент от предыдущего шага — снизу,
- абсолютное количество продлений — сверху,
- абсолютное количество оттока — снизу,
- всплывающее окно с причинами оттока при наведении курсора.
### Причины оттока \{#churn-reasons\}

Adapty показывает статистику *оттока* начиная со стадии Trial и далее. Каждый пользователь, который вошёл в одну стадию, но не перешёл в следующую, засчитывается как случай оттока.

* Если переходу помешало конкретное событие (например, истечение триала или проблема с оплатой), Adapty отображает его причину.

* Статус **unknown** — временный. Он означает, что пользователь ещё не столкнулся с событием, которое позволило бы ему перейти на следующую стадию.
На этапе Trial это обычно означает, что пробный период ещё не завершился. Чаще всего это происходит при просмотре воронок за короткие периоды или за один день, поскольку триалы требуют времени для завершения.

Adapty обновит информацию, как только пользователь конвертируется или отменит триал.

### Таблица, фильтры и экспорт в CSV \{#table-view-filters-and-csv-export\}

График воронки дополнен таблицей с данными — удобным материалом для работы с цифрами.

<ZoomImage id="4787aff-CleanShot_2022-06-23_at_21.01.44.webp" width="700px" alt="Данные воронки в табличном виде" />

Таблица повторяет структуру воронки с небольшими отличиями.
Столбцы отображают данные по всем шагам, кроме шага первой платной подписки.
Вместо него добавлены два отдельных: Install → Paid и Trial → Paid. Они показывают ключевой момент конверсии — когда бесплатный пользователь становится платящим.
Может показаться, что здесь есть чёткое разделение по типу продукта: колонка Install -> Paid содержит только продукты без триалов, а колонка Trial -> Paid — только продукты с триалами. Но это не совсем так. Дело в том, что мы также учитываем пользователей, у которых истёк триал и которые затем покупают продукт с триалом так, как если бы триала не было вовсе.

<ZoomImage id="a9bcbc7-CleanShot_2022-06-23_at_21.29.12.webp" width="700px" alt="Таблица воронки: колонки Install to Paid и Trial to Paid" />
Углубляясь в цифры, вы найдёте в фильтрах мощные инструменты для новых гипотез.
Смело задавайте условия по разным измерениям. Собирайте настоящие инсайты на основе данных.
Варьируйте:

1. Тип продукта — тарифная модель, длительность и т. д.
2. Временной диапазон.
3. Сегментация по странам.
4. Атрибуция трафика.
5. Стор.

Выберите Absolute # (абсолютные значения), Relative % (относительные) или оба варианта, чтобы видеть только нужные данные.

<ZoomImage id="1475e42-CleanShot_2022-06-23_at_21.50.33_-2.webp" width="700px" alt="Фильтры воронки и переключатель абсолютных и относительных значений" />
Наконец, справа на панели управления есть кнопка для экспорта данных воронки в CSV. Затем вы можете открыть файл в Excel, Google Sheets или импортировать в собственную аналитическую систему.
