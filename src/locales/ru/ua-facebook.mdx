Интеграция Adapty UA с Meta позволяет отслеживать и оптимизировать эффективность кампаний в Facebook, Instagram, Messenger и Audience Network.

:::tip
Смотрите наш [гайд по настройке рекламы в Meta Ads Manager](meta-create-campaign).
:::

## Шаг 1. Подключите аккаунт Facebook \{#step-1-connect-your-facebook-account\}

Чтобы подключить Meta Ads к Adapty UA, перейдите в **Integrations > Meta** через левую боковую панель. Доступны два варианта:

- **Continue with Facebook**: подключение через OAuth. Используйте этот вариант, если входите в Meta Ads Manager с личным или бизнес-аккаунтом Facebook.
- **Add system token**: подключение с помощью постоянного токена системного пользователя. Используйте этот вариант, если ваша организация управляет рекламными аккаунтами через системного пользователя Meta Business.

<TabItem value="oauth" label="Continue with Facebook">

:::important
Убедитесь, что ваш аккаунт Facebook имеет доступ к нужным кампаниям и пикселям.
:::

1. Нажмите **Continue with Facebook**.

2. Войдите через аккаунт Facebook и нажмите **Continue**.

<ZoomImage id="ua-fb-sign.webp" width="500px" />

3. Ознакомьтесь с запрашиваемыми разрешениями и нажмите **Save**.

2. Нажмите **Add system token**.

После этого все ваши рекламные аккаунты будут добавлены в Adapty UA. Можно переходить к добавлению кампаний.

## Шаг 2. Добавьте кампании \{#step-2-add-campaigns\}

Чтобы добавить кампанию Meta в Adapty User Acquisition и отслеживать в Adapty работу ваших объявлений:

1. Перейдите на вкладку **Web campaigns** и нажмите **Create campaign**. Выберите приложение и нажмите **Save**.

<ZoomImage id="ua-new-campaign.webp" width="500px" />

2. На вкладке **General** разверните раздел **iOS** и/или **Android** и вставьте URL-адреса приложения из App Store и/или Google Play.

<ZoomImage id="ua-url.gif" width="900px" />

3. Скопируйте значение поля **Click link**. Затем в Meta Ads Manager откройте ваше объявление и вставьте эту ссылку. Это позволит Adapty связать установки и покупки с рекламой в Meta.

<ZoomImage id="ua-lnk.webp" width="900px" />

4. Чтобы отправлять события конверсий обратно в Meta, привяжите пиксели из Meta к кампаниям в Adapty UA. Для этого выберите один из существующих пикселей в выпадающем списке **Pixel**.

:::tip
Вы также можете задать дополнительные параметры ссылки для [работы с отложенными данными](ua-deferred-data). Например, реализовать отложенные диплинки.
:::

## Шаг 3. Настройте маппинг событий \{#step-3-map-events\}

Чтобы отправлять события конверсий обратно в Meta для оптимизации кампаний, настройте маппинг событий в разделе **Events names**. Это позволит Adapty автоматически отправлять события подписок на ваш пиксель Meta, когда пользователи совершают действия в приложении.

В разделе **Events names** включите события, которые хотите отслеживать в Meta Ads Manager. Для каждого включённого события выберите соответствующее событие Meta из выпадающего списка или задайте пользовательское. По умолчанию Adapty сопоставляет события Adapty со стандартными событиями Meta.

Нажмите **Save**, чтобы применить настройки маппинга событий.

## Дополнительная настройка \{#additional-configuration\}

### Дополнительные параметры \{#additional-parameters\}

Поле **Additional parameter** позволяет добавить пользовательские точки данных для анализа за пределами Adapty. Это полезно, когда нужно передать конкретные данные о кампании или пользователе во внешние аналитические инструменты или партнёрам по атрибуции.

В поле **Additional parameter** введите любые пользовательские данные, которые хотите включить в отслеживание атрибуции. Дополнительный параметр будет включён во все данные атрибуции, отправляемые в Meta, и может использоваться для углублённого анализа и оптимизации кампаний.

Например, если вы запускаете несколько вариантов одной кампании, можно добавить `variant=A` или `variant=B`, чтобы различать разные креативные подходы.

:::important
Дополнительные параметры изменяют **Click link**, который вы вставляете в Meta Ads Manager. Если вы уже скопировали эту ссылку туда, а затем добавили пользовательский параметр, убедитесь, что скопировали и вставили обновлённую ссылку с этим параметром.
:::

### Настройки атрибуции \{#attribution-settings\}

Настройки атрибуции определяют, как Adapty сопоставляет действия пользователей с кампаниями Meta Ads. Они задают временны́е окна для детерминированного и вероятностного сопоставления.

Чтобы настроить атрибуцию, перейдите на вкладку **Attribution Settings** в настройках кампании. Там доступны два основных параметра:

- **Deterministic matching window**: используются точные идентификаторы устройства (например, IDFA на iOS или Advertising ID на Android) для точного сопоставления пользователей с кампаниями. Установите значение 168 часов (7 дней) для максимальной точности атрибуции — это значение по умолчанию и рекомендуемое. Когда пользователь нажимает на ваше объявление в Meta и устанавливает приложение в течение этого окна, Adapty может точно атрибутировать установку конкретному клику по объявлению с помощью идентификаторов устройства.

- **Probabilistic matching window**: используется статистическое моделирование и снятие цифрового отпечатка устройства для сопоставления пользователей, когда детерминированное сопоставление невозможно. Установите значение 6 часов для большинства кампаний — это значение по умолчанию, которое хорошо работает в большинстве случаев. Для кампаний с большим объёмом кликов можно сократить до 1–2 часов. Для пользователей, которых нельзя сопоставить детерминированно (из-за настроек конфиденциальности или других факторов), Adapty использует вероятностное сопоставление в рамках этого более короткого окна.

Нажмите **Save**, чтобы применить настройки атрибуции.

### Переопределение дохода \{#revenue-override\}

Если вы отслеживаете события триала и хотите, чтобы Meta атрибутировала им доход, используйте раздел **Revenue override**. Он появляется, когда включено событие **Trial started**.

Для каждого целевого события триала задайте процент от цены подписки, который будет отображаться как доход. Например, при 30% Adapty отправит 30% от цены подписки в качестве значения конверсии для событий триала.

Чтобы добавить переопределение:

1. Включите **Trial started** в разделе **Events names**.
2. В **Revenue override** нажмите **Add override**.
3. Выберите целевое событие и введите процент дохода (0–100).
4. Нажмите **Save**.

<ZoomImage id="revenue-override.webp" width="500px" />

### Отправка всех событий \{#send-all-events\}

По умолчанию Adapty отправляет события на ваш пиксель только для пользователей, атрибутированных к кампании Meta. Включите **Send all events**, чтобы также передавать события от органических и неатрибутированных пользователей на пиксель.

При включении этой опции каждое событие установки и транзакции отправляется на пиксель вне зависимости от атрибуции кампании. Используйте это, чтобы дать Meta более широкие данные о конверсиях для моделирования аудиторий и оптимизации кампаний.

Чтобы включить эту опцию, в настройках кампании выберите **Send all events (forward organic/non-attributed events to this pixel)** и нажмите **Save**.
