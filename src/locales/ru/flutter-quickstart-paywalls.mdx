Чтобы включить встроенные покупки, нужно разобраться в трёх ключевых понятиях:
- [**Products**](product) – всё, что пользователи могут купить (подписки, расходуемые покупки, пожизненный доступ)
- [**Paywalls**](paywalls) — конфигурации, определяющие, какие продукты предлагать. В Adapty пейволы — единственный способ получить продукты, но такой подход позволяет менять предложения, цены и наборы продуктов без изменения кода приложения.
- [**Placements**](placements) – где и когда показывать пейволы в приложении (например, `main`, `onboarding`, `settings`). Вы настраиваете пейволы для плейсментов в дашборде, а затем запрашиваете их по ID плейсмента в коде. Это упрощает запуск A/B-тестов и показ разных пейволов разным пользователям.
Adapty предлагает три способа подключить покупки в вашем приложении. Выберите подходящий в зависимости от требований:
| Реализация             | Сложность  | Когда использовать                                                                                                                                                                                                                                        |
|------------------------|------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Adapty Paywall Builder | ✅ Легко    | Вы [создаёте готовый к покупкам пейвол в визуальном редакторе без кода](quickstart-paywalls). Adapty автоматически его отображает и берёт на себя весь процесс покупки, валидацию чеков и управление подписками. |
| Пейволы, созданные вручную | 🟡 Средне  | Вы реализуете UI пейвола в коде приложения, но всё равно получаете объект пейвола из Adapty, сохраняя гибкость в управлении продуктами. См. [гайд](flutter-quickstart-manual).                                                     |
| Observer mode              | 🔴 Сложно  | У вас уже есть собственная инфраструктура обработки покупок, и вы хотите продолжить её использовать. Обратите внимание, что observer mode имеет ограничения в Adapty. См. [статью](observer-vs-full-mode).                                                 |
:::important
**Шаги ниже показывают, как реализовать пейвол, созданный в Adapty Paywall Builder.**

Если вы не хотите использовать Paywall Builder, ознакомьтесь с [гайдом по обработке покупок в пейволах, созданных вручную](flutter-making-purchases).
:::

Чтобы отобразить пейвол, созданный в Adapty Paywall Builder, в коде вашего приложения нужно лишь:
1. **Получите пейвол**: Получите пейвол из Adapty.
2. **Покажите пейвол — Adapty возьмёт на себя обработку покупок**: Отобразите в приложении контейнер пейвола, который вы получили.
3. **Обработайте действия кнопок**: Свяжите действия пользователя на пейволе с реакцией вашего приложения на них. Например, открывайте ссылки или закрывайте пейвол по нажатию кнопок.

Прежде чем начать, выполните следующие шаги:

2. [Создайте продукты](create-product) в Adapty.
3. [Создайте пейвол и добавьте в него продукты](create-paywall).
4. [Создайте плейсмент и добавьте в него пейвол](create-placement).
5. [Установите и активируйте SDK](sdk-installation-flutter) в коде приложения.
:::tip
Самый быстрый способ выполнить эти шаги — следовать [руководству по быстрому старту](quickstart) или создать пейволы и плейсменты с помощью [Developer CLI](developer-cli-quickstart).
:::
## 1. Получите пейвол \{#1-get-the-paywall\}

Ваши пейволы привязаны к плейсментам, настроенным в дашборде. Плейсменты позволяют показывать разные пейволы разным аудиториям или запускать [A/B-тесты](ab-tests).

Чтобы получить пейвол, созданный в Adapty Paywall Builder, нужно:

1. Получить объект `paywall` по ID [плейсмента](placements) с помощью метода `getPaywall` и проверить, создан ли пейвол в билдере — для этого используйте свойство `hasViewConfiguration`.
2. Создайте представление пейвола с помощью метода `createPaywallView`. Представление содержит элементы UI и стили, необходимые для отображения пейвола.

:::important
Чтобы получить конфигурацию представления, необходимо включить переключатель **Show on device** в Paywall Builder. В противном случае вы получите пустую конфигурацию представления, и пейвол не будет отображён.
:::
```dart showLineNumbers

try {
  final paywall = await Adapty().getPaywall(placementId: "YOUR_PLACEMENT_ID", locale: "en");
  // the requested paywall
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
}

try {
  final view = await AdaptyUI().createPaywallView(
        paywall: paywall,
      );
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```

## 2. Отображение пейвола \{#display-the-paywall\}

Теперь, когда у вас есть конфигурация пейвола, достаточно добавить несколько строк кода, чтобы отобразить его.

Для отображения пейвола используйте метод `view.present()` на объекте `view`, созданном методом `createPaywallView`. Каждый объект `view` можно использовать только один раз. Если нужно показать пейвол снова, вызовите `createPaywallView` ещё раз, чтобы создать новый экземпляр `view`.

```dart showLineNumbers title="Flutter"
try {
  await view.present();
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```
:::tip
Подробнее о том, как отображать пейвол, см. в нашем [гайде](flutter-present-paywalls).
:::
## 3. Обработка действий кнопок \{#handle-button-actions\}

Когда пользователи нажимают кнопки на пейволе, Flutter SDK автоматически обрабатывает покупки и восстановление. Однако другие кнопки имеют пользовательские или предопределённые ID и требуют обработки действий в вашем коде.

Чтобы управлять процессами на экране пейвола или отслеживать их, реализуйте методы `AdaptyUIPaywallsEventsObserver` и установите наблюдатель перед отображением любого экрана. Если пользователь выполнил какое-либо действие, будет вызван `paywallViewDidPerformAction`, и ваше приложение должно отреагировать в зависимости от ID действия.
Например, на вашем пейволе, скорее всего, есть кнопка закрытия и ссылки для открытия (например, пользовательское соглашение и политика конфиденциальности). Поэтому нужно обрабатывать действия с ID `Close` и `OpenUrl`.

:::tip
Читайте наши гайды о том, как обрабатывать [действия кнопок](flutter-handle-paywall-actions) и [события](flutter-handling-events).
:::
Реализуйте наблюдатель как отдельный долгоживущий объект, а не как виджет. Поскольку один глобальный слот наблюдателя разделяется на всё приложение, привязка его к `State` приведёт к утечке экрана (SDK хранит на него сильную ссылку) и к тому, что он будет молча заменён, когда следующий экран зарегистрирует себя. Использование `extends` также наследует поведение SDK по умолчанию, поэтому вы переопределяете только те коллбэки, которые вам нужны.
```dart showLineNumbers title="Flutter"
// A dedicated, long-lived handler for paywall events.
// It does NOT live inside a Widget/State, so it never leaks and is never
// silently replaced when screens are pushed or popped.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // This method is called when user performs an action on the paywall UI
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using url_launcher package
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors
      print('Error launching URL: $e');
    }
  }
}
```

Зарегистрируйте обработчик **один раз** при старте приложения, до того как будет показан любой пейвол:

```dart showLineNumbers title="Flutter"
AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());
```

## Полный пример \{#full-example\}

Вот как все эти шаги можно объединить в вашем приложении.
```dart

void main() {
  // Register a single, long-lived observer once, before any paywall is shown.
  // It is intentionally a plain object (NOT a Widget/State): its lifetime is the
  // whole app, so it never leaks and is never silently replaced when screens are
  // pushed or popped.
  AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());

  runApp(MaterialApp(home: PaywallScreen()));
}

/// A dedicated handler for AdaptyUI paywall events.
///
/// It `extends` [AdaptyUIPaywallsEventsObserver] (rather than being implemented
/// by a `State`), which gives you two things for free:
///   * the SDK's sensible defaults (e.g. auto-dismiss on close/finish), so you
///     only override what you actually care about;
///   * a lifecycle that is independent of the widget tree — there is no strong
///     reference back into a `Widget`, so nothing leaks and there is nothing to
///     unregister.
///
/// Every callback receives the [AdaptyUIPaywallView] it relates to, so handling
/// paywall actions never requires a `BuildContext` or widget state.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // Called when the user performs an action on the paywall UI.
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using the url_launcher package.
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds.
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered.
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs.
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched.
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors.
      print('Error launching URL: $e');
    }
  }
}

class PaywallScreen extends StatefulWidget {
  const PaywallScreen({super.key});

  @override
  State<PaywallScreen> createState() => _PaywallScreenState();
}

class _PaywallScreenState extends State<PaywallScreen> {
  @override
  void initState() {
    super.initState();
    _showPaywallIfNeeded();
  }

  Future<void> _showPaywallIfNeeded() async {
    try {
      final paywall = await Adapty().getPaywall(
        placementId: 'YOUR_PLACEMENT_ID',
      );

      if (!paywall.hasViewConfiguration) return;

      final view = await AdaptyUI().createPaywallView(paywall: paywall);

      await view.present();
    } catch (_) {
      // Handle any errors (network, SDK issues, etc.)
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Adapty Paywall Example')),
      body: Center(
        // Add a button to re-trigger the paywall for testing purposes.
        child: ElevatedButton(
          onPressed: _showPaywallIfNeeded,
          child: const Text('Show Paywall'),
        ),
      ),
    );
  }
}
```
