Прогнозы Adapty помогают ответить на следующие вопросы:

1. Каков прогнозируемый пожизненный доход (LTV) ваших пользовательских когорт?
2. Какие когорты, скорее всего, принесут наибольшую выручку в будущем?
3. Сколько можно вложить, исходя из ожидаемой отдачи?
С помощью Adapty Predictions вы можете принимать решения о доходах и росте на основе данных.

Модель прогнозирования Adapty оценивает долгосрочный потенциал дохода когорт пользователей вашего приложения. Для каждой когорты она прогнозирует, как со временем будут меняться доход, количество платящих подписчиков и средний LTV. Это помогает принимать обоснованные решения в области привлечения пользователей, маркетинговых стратегий и развития продукта.
Adapty предлагает прогноз пожизненной ценности (LTV) и прогноз дохода для когорт платящих подписчиков. Прогнозы отображаются на странице когортного анализа для периодов 3, 6, 9, 12, 18 и 24 месяца после создания когорты.

Для приложений с очень ограниченной историей модель использует средние значения по всем приложениям, поэтому прогнозы для новых приложений могут не полностью отражать поведение их конкретных пользователей.
## Как работает модель \{#how-the-model-works\}

Модель прогнозирования Adapty использует паттерны удержания из исторических данных когорт для прогнозирования будущей выручки и LTV.

Для каждой комбинации приложения и типа подписки модель измеряет, как меняются количество платящих подписчиков и общая выручка от одного периода продления к следующему. На основе прошлых когорт приложения рассчитываются два коэффициента удержания — один для подписчиков, другой для выручки. Затем эти коэффициенты применяются к новым когортам для прогнозирования их роста через 3, 6, 9, 12, 18 и 24 месяца после создания когорты. Все используемые данные полностью анонимизированы.
Модель вычисляет два значения для каждой когорты:

- **Predicted revenue**: суммарная выручка, которую когорта, по прогнозу, принесёт за выбранный горизонт.
- **Predicted LTV**: predicted revenue, делённый на прогнозируемое количество платящих подписчиков в когорте.
### Веса для конкретного приложения и кросс-приложения \{#app-specific-and-cross-app-weights\}

По умолчанию прогнозы для когорты используют веса удержания, полученные из исторических когорт этого же приложения — они отражают характерное поведение его пользователей.
Когда у приложения недостаточно истории для конкретного горизонта прогноза, Adapty использует резервные веса удержания, усреднённые по всем приложениям с тем же типом подписки. Например, 12-месячный прогноз для приложения, которому всего шесть месяцев, строится на основе кросс-приложенческих резервных данных. Этот резервный механизм применяется независимо для каждого горизонта, поэтому одна и та же когорта может использовать собственные веса приложения для 3-месячного прогноза и кросс-приложенческие веса для 12-месячного.
### Доступность и обновления \{#availability-and-updates\}

Прогнозы становятся доступны после того, как когорта завершает первый период продления — как правило, через неделю после создания для еженедельных подписок и примерно через четыре недели для месячных. После этого прогнозы обновляются ежедневно на основе актуальных транзакционных данных, отражая текущее поведение когорты.
### Ограничения \{#limitations\}

- **Качество данных**: нетипичное поведение когорты или когорты с очень небольшим числом платящих пользователей снижают точность. Когорты с менее чем 100 платящими подписчиками исключаются из обучающих данных модели.
- **Новые приложения**: приложения без достаточной истории используют общие веса на основе данных других приложений, которые могут не отражать специфику поведения пользователей конкретного приложения.
- **Возраст когорты**: прогнозы для заданного горизонта скрываются, как только когорта превышает этот горизонт. Например, трёхмесячные прогнозы перестают отображаться по истечении трёх месяцев, а для когорт старше 24 месяцев прогнозы не показываются вовсе.

### Выберите горизонт прогноза \{#select-the-horizon\}

Чтобы изменить горизонт прогноза, выберите нужное значение в выпадающем списке **Predictions**. Доступные варианты: 3, 6, 9, 12, 18 и 24 месяца с момента создания когорты.
### Фильтрация по продукту \{#filter-by-product\}

Вы можете фильтровать прогнозируемый доход и LTV по продукту. По умолчанию прогнозы строятся на основе всех данных о покупках — фильтрация по продукту показывает вклад каждого продукта.

## Когда прогнозы недоступны \{#when-predictions-are-unavailable\}

Если прогноз для когорты не может быть построен, в столбцах Predicted Revenue и Predicted LTV вместо значений отображаются длинные тире (—). Это может происходить по нескольким причинам:
- **Недостаточно времени с момента создания когорты**: прогнозы становятся доступны только после того, как когорта завершит первый период обновления — примерно через одну неделю для еженедельных подписок и около четырёх недель для ежемесячных.
- **Маленький размер когорты**: слишком мало платящих подписчиков для построения надёжного прогноза.
- **Нестандартное поведение когорты**: когорта значительно отклоняется от паттернов, которые ожидает модель. Подождите несколько недель — по мере накопления данных ситуация может измениться.
- **Горизонт превышен**: когорта старше выбранного горизонта прогноза. Например, прогноз на 3 месяца скрывается по истечении трёх месяцев, на 12 месяцев — по истечении двенадцати, а для когорт старше 24 месяцев прогнозы не отображаются вовсе.
:::warning
При включении прогнозов учтите, что данные прогноза по Revenue и LTV могут появиться на дашборде Adapty с задержкой до 24 часов.
:::
