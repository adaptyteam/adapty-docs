Флоу состоит из одного или нескольких экранов. Каждый экран представляет отдельный шаг в пути пользователя — например, пейвол, квиз или слайд с информацией о продукте.

Элементы на каждом экране организованы в иерархию слоёв.
Для управления экранами, слоями и элементами откройте стандартное представление **Screens and Layers**. В нём отображается последовательность экранов и структура слоёв каждого из них.
## Управление экранами \{#manage-screens\}

<ZoomImage id="flow-builder-screens-and-layers-left-sidebar.webp" width="400px" alt="Flow builder screens and layers view (both panels visible)" float="left" />

В верхней части левой панели перечислены все экраны флоу. Каждая запись содержит порядковый номер и миниатюрный предпросмотр.
* **Выбрать экран**: нажмите на экран в списке, чтобы сделать его активным. Визуальный редактор отобразит выбранный экран, а раздел Layers обновится, показывая иерархию слоёв.
* **Добавить экран**: нажмите кнопку <Inline id="plus.svg" alt="Plus" /> вверху раздела Screens, чтобы добавить новый пустой экран во флоу.
* **Открыть библиотеку шаблонов**: нажмите кнопку <Inline id="template-library.svg" alt="Templates" /> вверху раздела Screens, чтобы просмотреть и применить [шаблоны флоу](paywall-builder-templates).
* **Изменить порядок экранов**: перетащите экраны, чтобы изменить их порядок во флоу.
:::important
Если в вашем флоу есть неиспользуемые пустые экраны, опубликовать его не получится. Удалите черновые экраны перед публикацией.
:::
### Действия с экранами \{#screen-actions\}

Нажмите на иконку с тремя точками <Inline id="context.svg" alt="Context" /> на записи экрана, чтобы открыть контекстное меню.

<ZoomImage id="flow-builder-screen-actions.webp" width="600px" alt="Flow builder screen context menu" />
| Действие | Горячая клавиша | Описание |
|----------|-----------------|----------|
| **Play Animation** | | Предпросмотр анимаций, настроенных на этом экране |
| **Copy** | ⌘C / Ctrl+C | Копировать экран в буфер обмена |
| **Paste here** | ⌘V / Ctrl+V | Вставить ранее скопированный экран |
| **Duplicate** | ⌘D / Ctrl+D | Создать копию экрана и добавить её во флоу |
| **Rename** | | Изменить отображаемое имя экрана |
| **Delete** | ⌘⌫ / Ctrl+Del | Удалить экран из флоу |

:::tip
Буфер обмена сохраняется между флоу. Скопируйте экран или элемент из одного флоу, откройте другой и вставьте его туда.
:::
:::warning
Когда вы удаляете экран, любое действие [«Перейти к экрану»](onboarding-navigation-branching), которое ссылалось на него, **теряет цель**, но само действие **не удаляется**. Назначьте новый экран назначения или удалите это действие — иначе вы не сможете [просмотреть или опубликовать флоу](builder-save-publish#publish-a-flow).
:::

## Настройки экрана \{#screen-settings\}

Чтобы просмотреть свойства и настройки активного экрана, нажмите на пустую область в превью экрана. Правая панель переключится в режим настроек экрана.
### Системный интерфейс \{#system-ui\}

<ZoomImage id="builder-screen-settings-system-ui.webp" width="350px" alt="Screen settings — System UI" float="right" />

Управляет тем, как экран взаимодействует с аппаратными элементами устройства.

* **Safe area** добавляет отступы, которые не дают контенту заходить за вырез экрана и системные панели.
* **Status bar** показывает и скрывает системную строку состояния (время, заряд батареи, иконки сигнала).
### Включение экрана в индикатор прогресса \{#include-screen-in-progress-indicator\}

Если вы добавили элемент [Индикатор прогресса](builder-loaders-and-progress-bars#progress-indicators) в флоу, Adapty отображает его на каждом экране.

Снимите флажок **Include screen in progress indicator**, чтобы убрать индикатор прогресса с конкретного экрана. Это удобно для приветственных экранов, финального пейвола или любого шага, который не должен учитываться в прогрессе.
### Макет экрана \{#screen-layout\}

:::link
Полная статья: [Позиционирование элементов](manage-paywall-ui-elements)
:::

<ZoomImage id="builder-screen-settings-layout.webp" width="350px" alt="Screen settings — Layout" float="right" />

Раздел **Layout** определяет, как экран распределяет дочерние элементы. Эти свойства доступны для любого контейнерного элемента.

* **Free**: дочерние элементы позиционируются независимо друг от друга.
* **Vertical**: элементы расставляются сверху вниз, как в flexbox-колонке.
* **Horizontal**: элементы расставляются слева направо, как в flexbox-строке.
Для вертикальных и горизонтальных макетов можно также настроить отступы и выравнивание.

* **Alignment**: Положение элементов по поперечной оси.
* **Gap**: Расстояние между соседними элементами.
* **Distribution**: Распределение пространства между дочерними элементами и вокруг них.

#### RTL-макет \{#rtl-layout\}

Включите чекбокс **Mirror for RTL**, чтобы зеркально отразить макет для систем письма, которые читаются справа налево. Порядок элементов в горизонтальных контейнерах при этом изменится на обратный.
### Фон экрана \{#screen-background\}

:::link
Основная статья: [Фоны](paywall-head-picture)
:::

<ZoomImage id="builder-screen-settings-background.webp" width="350px" alt="Screen settings — Background" float="right" />

**Fill** задаёт [фон экрана](paywall-head-picture) в виде сплошного цвета, градиента, изображения или видео. Фон заполняет весь видимый экран устройства, включая области за вырезом камеры и системными панелями — даже когда включён **Safe area**.

#### Зацикливание фонового видео \{#loop-background-video\}

Включите переключатель **Loop**, чтобы фоновое видео воспроизводилось непрерывно по кругу.

#### Назначение пользовательского media ID \{#assign-a-custom-media-id\}
Как и в случае с [любым изображением или видео](custom-media), вы можете назначить фону экрана пользовательский медиа-ID, чтобы ссылаться на него в вашем SDK.
### Отступы экрана \{#screen-spacing\}

<ZoomImage id="builder-screen-settings-spacing.webp" width="350px" alt="Настройки экрана — Отступы" float="right" />

Задаёт внутренние отступы экрана для каждой стороны (сверху, справа, снизу, слева).
### Прокрутка \{#scroll\}

<ZoomImage id="builder-screen-settings-scroll.webp" width="350px" alt="Screen settings — Scroll" float="right" />

Управляет поведением при переполнении. Включите **Vertical scroll**, чтобы содержимое экрана прокручивалось, если оно выходит за границы видимой области.
### Выбираемые группы \{#selectable-groups\}

:::link
Основная статья: [Выбираемые элементы и группы](flow-selectable-elements)
:::

Раздел **Selectable groups** содержит все выбираемые группы на текущем экране — из [викторин](onboarding-quizzes), [продуктов](paywall-product-block), [вкладок](builder-tabs), [тогглов пробного периода](builder-toggles) или любых [пользовательских выбираемых элементов](flow-selectable-elements#make-an-element-selectable).

Нажмите на запись группы, чтобы переименовать её, изменить тип, просмотреть переменные, которые она предоставляет, или удалить её.
<ZoomImage id="builder-screen-settings-selectable-groups.webp" width="600px" alt="Настройки экрана — группы с выбором" />
## Управление слоями \{#manage-layers\}

Каждый элемент на экране представлен в виде слоя. Раздел «Слои» отображает порядок элементов на активном экране.

:::important
Слои флоу не перекрываются, как слои в графическом редакторе. Вместо этого они представляют отдельные компоненты экрана. Элементы перекрываются *только* если используют [абсолютное или фиксированное позиционирование](manage-paywall-ui-elements). Порядок их наложения определяется свойством `z-index`, а не положением в дереве слоёв.
:::
Древовидная структура отражает связи родительских и дочерних слоёв. Нажмите на стрелку рядом с любым родительским слоем, чтобы развернуть или свернуть его дочерние элементы.

Создавать слои напрямую нельзя. Каждый элемент, добавленный через раздел [Add element](builder-elements), отображается в дереве как новый слой.

<ZoomImage id="flow-builder-layer-focus.webp" width="350px" alt="Flow builder layer view" float="left" />
* **Выбрать слой**: Кликните на слой, чтобы выделить его. Визуальный редактор подсветит соответствующий элемент на холсте, а правая панель отобразит его свойства [дизайна](builder-styling) и [взаимодействия](onboarding-navigation-branching).
* **Изменить порядок слоёв**: Перетащите слои внутри дерева, чтобы изменить их порядок в родительском контейнере. Порядок в дереве соответствует визуальному порядку на экране.
* **Показать или скрыть слой**: Наведите курсор на слой, чтобы справа появилась иконка глаза <Inline id="eye.svg" alt="Eye" />. Нажмите её, чтобы переключить видимость слоя. Скрытые слои остаются в дереве, но не отображаются в визуальном редакторе и на устройстве. Чтобы управлять видимостью с помощью логики во время выполнения, используйте [условную видимость](onboarding-element-visibility).
* **Свернуть все слои**: Нажмите кнопку свернуть <Inline id="collapse.svg" alt="Collapse" /> в правом верхнем углу секции «Layers», чтобы сложить всё дерево.
### Действия со слоями \{#layer-actions\}

Нажмите на иконку с тремя точками <Inline id="context.svg" alt="Context" />, чтобы открыть контекстное меню.

<ZoomImage id="flow-builder-layer-context-menu.webp" width="450px" alt="Flow builder layer context menu" />
| Действие | Горячая клавиша | Описание |
|--------|----------|-------------|
| **Copy** | ⌘C / Ctrl+C | Скопировать слой в буфер обмена |
| **Paste here** | ⌘V / Ctrl+V | Вставить скопированный слой как дочерний |
| **Duplicate** | ⌘D / Ctrl+D | Создать копию слоя в том же контейнере |
| **Rename** | | Изменить отображаемое имя слоя. По умолчанию слои получают имя по содержимому или типу компонента |
| **Delete** | ⌘⌫ / Ctrl+Del | Удалить слой вместе со всеми дочерними элементами |
| **Wrap** | | Обернуть слой в новый контейнер: **Wrap in Horizontal Container** или **Wrap in Vertical Container** |
| **Unwrap / Ungroup** | | Удалить обёртку-контейнер и переместить его дочерние элементы на уровень выше |
| **Move up** | ↑ | Переместить слой на одну позицию вверх в родительском контейнере |
| **Move down** | ↓ | Переместить слой на одну позицию вниз в родительском контейнере |
