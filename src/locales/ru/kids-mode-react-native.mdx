Если ваше приложение на React Native предназначено для детей, вы обязаны соблюдать политики [Apple](https://developer.apple.com/kids/) и [Google](https://support.google.com/googleplay/android-developer/answer/9893335). Если вы используете Adapty SDK, несколько простых шагов помогут настроить его в соответствии с этими политиками и пройти проверку в сторах.

## Что требуется? \{#whats-required\}

Необходимо настроить Adapty SDK так, чтобы отключить сбор:

- [IDFA (Identifier for Advertisers)](https://en.wikipedia.org/wiki/Identifier_for_Advertisers) (iOS)
- [Android Advertising ID (AAID/GAID)](https://support.google.com/googleplay/android-developer/answer/6048248) (Android)
- [IP-адреса](https://www.ftc.gov/system/files/ftc_gov/pdf/p235402_coppa_application.pdf)

Кроме того, рекомендуем осторожно использовать пользовательский ID. ID в формате `<Имя.Фамилия>` однозначно будет расценён как сбор персональных данных — так же, как использование email. Для детского режима лучшая практика — использовать рандомизированные или анонимизированные идентификаторы (например, хешированные ID или UUID, сгенерированные устройством), чтобы обеспечить соответствие требованиям.

## Включение детского режима \{#enabling-kids-mode\}

### Изменения в дашборде Adapty \{#updates-in-the-adapty-dashboard\}

В дашборде Adapty необходимо отключить сбор IP-адресов. Для этого перейдите в [App settings](https://app.adapty.io/settings/general) и нажмите **Disable IP address collection** в разделе **Collect users' IP address**.
