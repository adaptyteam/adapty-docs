Уровни доступа позволяют управлять возможностями пользователей в приложении без жёсткой привязки к конкретным идентификаторам продуктов. Каждый продукт определяет, на какой срок пользователь получает тот или иной уровень доступа. Таким образом, при каждой покупке Adapty предоставляет доступ к приложению на определённый период (для подписок) или навсегда (для покупок с пожизненным доступом).

При создании приложения в дашборде Adapty автоматически создаётся уровень доступа `premium`. Он является уровнем доступа по умолчанию и не может быть удалён.

В одном приложении можно использовать несколько уровней доступа. Вот несколько примеров, когда это бывает полезно:

- В приложении газеты, где подписки на разные тематики продаются независимо друг от друга, можно создать уровни доступа `sports` и `science`.
- В фитнес-приложении, которое предлагает записанные видеотренировки по обычной подписке (используя уровень доступа `premium` по умолчанию), пользователи могут выбрать более дорогой вариант с доступом к живым тренировкам с тренером. В этом случае можно создать уровень `live_coach_access`.
- В приложении для изучения языков можно создать отдельный уровень доступа для каждого доступного языка.

Чтобы начать работу с уровнями доступа в Adapty, перейдите в **[Products](https://app.adapty.io/access-levels)** через главное меню Adapty и выберите вкладку **Access levels**.

Список **Access levels** отображает все уровни доступа: уровень `premium`, который добавляется автоматически, а также все уровни, созданные вами в Adapty.
