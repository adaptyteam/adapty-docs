Уровни доступа позволяют управлять возможностями пользователей в мобильном приложении, не прописывая конкретные ID продуктов в коде. Каждый продукт определяет, на какой срок пользователь получает тот или иной уровень доступа. Таким образом, при каждой покупке Adapty предоставляет доступ к приложению на определённый период (для подписок) или навсегда (для покупок с пожизненным доступом).

При создании приложения в дашборде Adapty автоматически генерируется уровень доступа `premium`. Он является уровнем доступа по умолчанию и не может быть удалён.

В одном приложении может быть несколько уровней доступа. Вот несколько примеров, когда это бывает полезно:

- В газетном приложении, где подписки на разные темы продаются независимо, можно создать уровни доступа вроде `sports` и `science`.
- В фитнес-приложении с записанными видеотренировками по обычной подписке (используя уровень доступа `premium` по умолчанию) пользователи могут выбрать более дорогой вариант с доступом к живым тренировкам с тренером. В этом случае можно создать уровень `live_coach_access`.
- В приложении для изучения языков можно создать отдельный уровень доступа для каждого доступного языка.

:::note
Уровни доступа можно передавать между профилями пользователя или делиться ими — баланс виртуальной валюты так не работает, он привязан к одному профилю. Подробнее см. в разделе [Балансы, профили и устройства](virtual-currency-balance#balances-profiles-and-devices).
:::

Чтобы начать работу с уровнями доступа в Adapty, откройте **[Products](https://app.adapty.io/access-levels)** в главном меню Adapty и перейдите на вкладку **Access levels**.

В списке **Access levels** отображаются все уровни доступа: уровень `premium`, который добавляется автоматически, а также все уровни, созданные вами в Adapty.
