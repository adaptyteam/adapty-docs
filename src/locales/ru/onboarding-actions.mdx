Панель **Interactions** позволяет задавать, как элементы флоу реагируют на события — нажатия, появление элементов, отправку форм. Для каждого события назначается одно или несколько действий: переход между экранами, показ или скрытие элементов, открытие URL, установка переменных и т. д. Используйте условия, чтобы настроить флоу с учётом данных пользователя.

Каждое взаимодействие строится по трёхзвенной цепочке:
1. **Element**: Компонент экрана, с которого начинается взаимодействие — кнопка, вариант ответа в квизе, поле ввода или что-то другое.
2. **Trigger**: Событие, которое запускает логику, — нажатие, появление элемента или отправка формы.
3. **Action**: Задача, которую флоу выполняет в ответ. Один триггер может последовательно запускать несколько действий.
## Настройка взаимодействий \{#set-up-interactions\}

Чтобы настроить взаимодействие:

1. Выберите элемент на экране или на панели **Layers**.
2. Справа переключитесь на панель **Interactions** и нажмите **Add trigger**.

<ZoomImage id="add-trigger.webp" width="500px" alt="Кнопка Add trigger" />

3. В разделе **Button triggers** выберите [тип триггера](#trigger-types).

<ZoomImage id="trigger-type.webp" width="500px" alt="Выбор типа триггера" />

4. Нажмите **Add action**, кликните на название действия и выберите [тип действия](#action-types) из выпадающего списка в окне **Edit action**.
<ZoomImage id="add-action.gif" width="500px" alt="Add action button" />

5. Настройте свойства действия в зависимости от выбранного [типа действия](#action-types).
6. При необходимости нажмите **Add action**, чтобы добавить больше действий для одного триггера.
## Типы триггеров \{#trigger-types\}

На данный момент во флоу можно добавить следующие триггеры:
- **On tap**: Запускает действие по нажатию на элемент. Самый распространённый триггер — используется преимущественно для навигации между экранами.
- **On screen appear**: Запускает действие сразу после загрузки экрана. Используйте его, чтобы воспроизвести анимацию или зафиксировать показ при загрузке экрана.
## Типы действий \{#action-types\}
### Переход к экрану \{#navigate-to-screen\}

Это основное действие для перемещения пользователей между экранами. Оно переводит пользователя на указанный экран назначения.

Для этого действия нужно только задать экран назначения. Если вы хотите включить динамическую навигацию, смотрите раздел [Навигация и ветвление](onboarding-navigation-branching) или секцию [Условные действия](#conditional-actions).

:::important
**Navigate to screen** всегда должен быть последним действием в списке. Любое действие, добавленное после него (например, «Задать переменную»), может не выполниться, так как приложение уже сменит экран.
:::
<ZoomImage id="navigate-to.webp" width="500px" alt="Перейти к экрану" />
### Вернуться назад \{#navigate-back\}

Возвращает пользователя на предыдущий экран в истории навигации, а не на предыдущий экран в последовательности.
### Открыть URL \{#open-url\}

Открывает указанный веб-адрес. Используйте это действие, чтобы перенаправить пользователей на веб-страницы, статьи или профили в социальных сетях за пределами нативных экранов приложения.
Для этого действия можно настроить два параметра:
- **URL address**: укажите URL-адрес. Также можно сделать его динамическим — например, чтобы направлять пользователей на разные страницы в зависимости от ответа на вопрос в квизе или на основе введённых ими данных. Для этого нажмите <Inline id="variable.webp" alt="Variable icon" /> и выберите нужную переменную.
- **Open in external browser**: укажите, где открывать внешние ссылки. По умолчанию они открываются во встроенном браузере, чтобы пользователь оставался в приложении. Установите флажок **Open in external browser**, если хотите открывать ссылки во внешнем браузере.
<ZoomImage id="open-url.webp" width="500px" alt="Открыть URL" />
### Закрыть флоу \{#close-flow\}

Закрывает текущий флоу.
### Показ/скрытие элементов \{#showhide-elements\}

Показывает или скрывает конкретный элемент на экране.

Это действие переопределяет начальное состояние, заданное в **Visibility** в панели **Design**. Если **Visibility** установлен в **Hide**, действие **Show** сделает элемент видимым.
### Показать уведомление \{#show-alert\}

Отображает нативное системное всплывающее окно. Пользователь должен нажать **Ok**, чтобы продолжить.

Для уведомлений необходимо задать **Title** и **Message**. В обоих полях можно использовать переменные, чтобы сделать содержимое динамическим. Для этого нажмите <Inline id="variable.webp" alt="Variable icon" /> и выберите нужную переменную.

<ZoomImage id="show-alert.webp" width="500px" alt="Show alert" />
### Задать переменную \{#set-variable\}

Обновляет значение переменной во флоу. Прежде чем добавлять это действие, создайте переменные в панели **Variables** слева (см. [Переменные](onboarding-variables)).

Нажмите **Add variable** и задайте столько переменных и их значений, сколько нужно.

<ZoomImage id="set-variable.webp" width="500px" alt="Задать переменную" />
### Покупка \{#purchase\}

Запускает флоу покупки прямо из кнопки или другого элемента управления в онбординге. Позволяет пользователям оформить подписку или приобрести продукт, не покидая флоу.
Для этого действия можно настроить два варианта поведения:
- **In-app store**: Инициирует нативную покупку. Укажите в поле **Product** конкретный продукт или `products.selectedProduct`, чтобы использовать текущий выбор пользователя на экране.
- **Web payment**: Направляет пользователя на [веб-пейвол](web-paywall) вместо нативной покупки. Используйте этот вариант, если хотите обрабатывать транзакцию вне приложения, например для веб-подписок.

<ZoomImage id="onboarding-purchase-action.webp" width="500px" alt="Настройка действия покупки в окне Edit action" />
### Произвольное действие

Вызывает именованное произвольное действие, которое обрабатывает ваш код. Используйте его, когда встроенные типы действий не покрывают нужное поведение — например, для запуска внутреннего события, применения скидки или отображения кастомного нативного экрана.

Чтобы настроить произвольное действие:

1. В окне **Edit action** задайте **Action ID** — строку, которую распознает ваше приложение (например, `show_discount`).
2. В коде приложения реализуйте обработчик для этого Action ID. Подробности реализации см. в разделе [Обработка действий пейвола](handle-paywall-actions).
<ZoomImage id="onboarding-custom-action.webp" width="500px" alt="Настройка пользовательского действия с полем Action ID в окне редактирования действия" />
## Условные действия \{#conditional-actions\}

Используйте условные действия, чтобы разветвить флоу на разные пути в зависимости от данных пользователя.

Несколько распространённых сценариев:

- На экране есть квиз, и вы хотите направлять пользователей на разные экраны в зависимости от их ответов. В этом случае добавьте условное действие к кнопке.
- Вы хотите предлагать разные продукты и офферы разным группам пользователей. Разместите их на разных экранах и настройте условия для кнопки навигации.
- Вы хотите пропускать определённые шаги для пользователей, которые уже прошли обучение в предыдущей сессии.
Условные действия работают как цепочка if / else-if / else. Приложение читает правила сверху вниз и останавливается на первом совпадении:
1. **IF**: Флоу проверяет основное условие.
    - Истина? Флоу немедленно выполняет действия из блока THEN и останавливается.
    - Ложь? Флоу переходит к следующему разделу.
2. **ELSE IF**: Здесь можно добавить дополнительные проверки (например, «Если не Premium, находится ли пользователь на Trial?»).
3. **ELSE** (Фолбэк): Если ни одно из правил выше не совпало, флоу выполняет действия из этого финального раздела.
:::warning
- Если правило добавлено, но для него не назначено действие, совпадение с условием не приведёт ни к каким последствиям.
- Если переменная добавлена, но у неё отсутствует оператор или значение, условие считается недействительным.
:::

Для каждого правила выберите переменную для оценки и действие для выполнения. Можно задать несколько действий для одного правила.

:::important
Флоу выполняет только одно правило — первое, которому соответствует условие. Если нужно выполнить и **IF**, и **ELSE IF** одновременно, добавьте оба действия в **IF**.
:::
Чтобы узнать, как сделать элементы выбираемыми и объединить их в группы для использования в условиях, см. [Выбираемые элементы и группы](flow-selectable-elements).
