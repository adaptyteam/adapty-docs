Этот гайд поможет интегрировать Adapty в приложение на Kotlin Multiplatform с помощью LLM. Сначала вы подготовите дашборд Adapty, затем пройдёте каждый этап реализации, отправляя нужные ссылки на документацию в LLM. В конце вы найдёте рекомендации по настройке ИИ-инструментов для работы с документацией Adapty.

:::tip
Скопируйте всю страницу в формате Markdown и вставьте в LLM — нажмите **Copy for LLM** в верхней части страницы или откройте [версию в формате .md](https://adapty.io/docs/ru/adapty-cursor-kmp.md). LLM воспользуется ссылками и чекпоинтами, чтобы провести вас через каждый этап.
:::

## До начала работы: настройка дашборда \{#before-you-start-dashboard-setup\}

Перед написанием кода Adapty требует некоторых настроек в дашборде. Это можно сделать с помощью интерактивного навыка для LLM или вручную через дашборд.

### Вариант со скиллом (рекомендуется) \{#skill-approach-recommended\}

Скилл Adapty CLI позволяет LLM настроить приложение, продукты, уровни доступа, пейволы и плейсменты напрямую — без необходимости открывать дашборд на каждом шаге. Вам нужно только [подключить сторы](integrate-payments) в дашборде.

```
npx skills add adaptyteam/adapty-cli --skill adapty-cli
```

После добавления скилла запустите `/adapty-cli` в агенте. Он проведёт вас через каждый шаг — в том числе подскажет, когда нужно открыть дашборд для подключения сторов.

### Вариант с дашбордом \{#dashboard-approach\}

Если вы предпочитаете настраивать всё вручную, вот что нужно сделать до написания кода. LLM не может получить значения из дашборда за вас — вам придётся предоставить их самостоятельно.

1. **Подключите сторы**: В дашборде Adapty перейдите в **App settings → General**. Подключите App Store и Google Play, если KMP-приложение поддерживает обе платформы. Это обязательно для работы покупок.
   [Подключение сторов](integrate-payments)

2. **Скопируйте публичный SDK-ключ**: В дашборде Adapty перейдите в **App settings → General**, найдите раздел **API keys**. В коде это строка, которую вы передаёте в конфигурационный билдер Adapty.

3. **Создайте хотя бы один продукт**: В дашборде Adapty перейдите на страницу **Products**. В коде продукты не используются напрямую — Adapty передаёт их через пейволы.
   [Добавление продуктов](quickstart-products)

4. **Создайте пейвол и плейсмент**: В дашборде Adapty создайте пейвол на странице **Paywalls**, затем назначьте его плейсменту на странице **Placements**. В коде ID плейсмента — это строка, которую вы передаёте в `Adapty.getPaywall("YOUR_PLACEMENT_ID")`.
   [Создание пейвола](quickstart-paywalls)

5. **Настройте уровни доступа**: В дашборде Adapty настройте их для каждого продукта на странице **Products**. В коде это строка, которую вы проверяете в `profile.accessLevels["premium"]?.isActive`. Стандартный уровень доступа `premium` подходит для большинства приложений. Если платящие пользователи получают доступ к разным функциям в зависимости от продукта (например, план `basic` vs. план `pro`), [создайте дополнительные уровни доступа](assigning-access-level-to-a-product) до начала разработки.

:::tip
Когда у вас есть все пять элементов, можно переходить к написанию кода. Скажите LLM: «Мой публичный SDK-ключ — X, мой ID плейсмента — Y», чтобы он мог сгенерировать корректный код инициализации и получения пейвола.
:::

### Настройте при необходимости \{#set-up-when-ready\}

Эти настройки не нужны для начала разработки, но пригодятся по мере развития интеграции:

- **A/B-тесты**: Настраиваются на странице **Placements**. Изменений в коде не требуется.
  [A/B-тесты](ab-tests)
- **Дополнительные пейволы и плейсменты**: Добавьте больше вызовов `getPaywall` с разными ID плейсментов.
- **Интеграции аналитики**: Настраиваются на странице **Integrations**. Настройка зависит от конкретной интеграции. Смотрите [интеграции аналитики](analytics-integration) и [интеграции атрибуции](attribution-integration).

## Передача документации Adapty в LLM \{#feed-adapty-docs-to-your-llm\}

### Использование Context7 (рекомендуется) \{#use-context7-recommended\}

[Context7](https://context7.com) — это MCP-сервер, который даёт LLM прямой доступ к актуальной документации Adapty. LLM автоматически получает нужные документы на основе вашего запроса — ручная вставка URL не нужна.

Context7 работает с **Cursor**, **Claude Code**, **Windsurf** и другими MCP-совместимыми инструментами. Для настройки выполните:

```
npx ctx7 setup
```

Команда определит ваш редактор и настроит сервер Context7. Для ручной настройки смотрите [репозиторий Context7 на GitHub](https://github.com/upstash/context7).

После настройки ссылайтесь на библиотеку Adapty в промптах:

```
Use the adaptyteam/adapty-docs library to look up how to install the Kotlin Multiplatform SDK
```

:::warning
Даже несмотря на то, что Context7 избавляет от необходимости вручную вставлять ссылки на документацию, порядок реализации важен. Следуйте [пошаговому руководству по реализации](#implementation-walkthrough) ниже строго по шагам, чтобы всё работало корректно.
:::

### Использование документации в виде обычного текста \{#use-plain-text-docs\}

К любой странице документации Adapty можно обратиться как к обычному тексту в формате Markdown. Добавьте `.md` в конец URL или нажмите **Copy for LLM** под заголовком статьи. Например: [adapty-cursor-kmp.md](https://adapty.io/docs/ru/adapty-cursor-kmp.md).

Каждый этап в [пошаговом руководстве по реализации](#implementation-walkthrough) ниже содержит блок «Отправьте LLM» со ссылками `.md` для вставки.

Чтобы передать больше документации сразу, смотрите [индексные файлы и платформенные подборки](#plain-text-doc-index-files) ниже.

## Пошаговое руководство по реализации \{#implementation-walkthrough\}

Оставшаяся часть гайда проведёт вас через интеграцию Adapty в правильном порядке. Каждый этап включает документы для LLM, ожидаемый результат и распространённые проблемы.

### Планирование интеграции \{#plan-your-integration\}

Прежде чем писать код, попросите LLM проанализировать ваш проект и составить план реализации. Если ваш ИИ-инструмент поддерживает режим планирования (как в Cursor или Claude Code), используйте его — LLM сможет изучить структуру проекта и документацию Adapty до написания кода.

Сообщите LLM, какой подход вы используете для покупок — это влияет на то, каким гайдам он должен следовать:

- [**Adapty Paywall Builder**](adapty-paywall-builder): Вы создаёте пейволы в визуальном конструкторе Adapty, и SDK отображает их автоматически.
- [**Пейволы, созданные вручную**](kmp-making-purchases): Вы строите собственный UI пейвола в коде, но используете Adapty для получения продуктов и обработки покупок.
- [**Observer mode**](observer-vs-full-mode): Вы сохраняете существующую инфраструктуру покупок и используете Adapty только для аналитики и интеграций.

Не знаете, что выбрать? Прочитайте [таблицу сравнения в квикстарте](kmp-quickstart-paywalls).

### Установка и настройка SDK \{#install-and-configure-the-sdk\}

Добавьте зависимость Adapty SDK через Gradle и активируйте его с помощью публичного SDK-ключа. Это основа — без неё ничего не работает.

**Гайд:** [Установка и настройка Adapty SDK](sdk-installation-kotlin-multiplatform)

Отправьте LLM:
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ru/sdk-installation-kotlin-multiplatform.md
```

:::tip[Чекпоинт]
- **Ожидаемый результат:** Приложение собирается и запускается. В Logcat (Android) или консоли Xcode (iOS) появляется лог об активации Adapty.
- **Частая проблема:** «Public API key is missing» → убедитесь, что вы заменили плейсхолдер реальным ключом из App settings.
:::

### Идентификация пользователей \{#identify-users\}

Свяжите аккаунты пользователей вашего приложения с профилями Adapty, чтобы покупки сохранялись на всех устройствах.

:::important
Пропустите этот шаг, если в приложении нет аутентификации.
:::

**Гайд:** [Идентификация пользователей](kmp-quickstart-identify)

Отправьте LLM:
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ru/kmp-quickstart-identify.md
```

:::tip[Чекпоинт]
- **Ожидаемый результат:** После вызова `Adapty.identify("your-user-id")` в разделе **Profiles** дашборда отображается ваш пользовательский ID.
- **Частая проблема:** Вызывайте `identify` после активации, но до получения пейволов — иначе покупки могут привязаться к анонимному профилю.
:::

### Подготовка к релизу \{#prepare-for-release\}

Когда интеграция работает в песочнице, пройдитесь по чеклисту релиза, чтобы убедиться, что всё готово для продакшена.

**Гайд:** [Чеклист релиза](release-checklist)

Отправьте LLM:
```
Read these Adapty docs before releasing:
- https://adapty.io/docs/ru/release-checklist.md
```

:::tip[Чекпоинт]
- **Ожидаемый результат:** Все пункты чеклиста подтверждены: подключение сторов, серверные уведомления, флоу покупок, проверка уровней доступа и требования к конфиденциальности.
- **Частая проблема:** Отсутствие серверных уведомлений → настройте App Store Server Notifications в **App settings → iOS SDK** и Google Play Real-Time Developer Notifications в **App settings → Android SDK**.
:::

## Индексные файлы с документацией в виде обычного текста \{#plain-text-doc-index-files\}

Если нужно дать LLM более широкий контекст, выходящий за рамки отдельных страниц, мы предоставляем индексные файлы со всей документацией Adapty:

- [`llms.txt`](https://adapty.io/docs/ru/llms.txt): Список всех страниц со ссылками `.md`. [Формирующийся стандарт](https://llmstxt.org/) для предоставления веб-сайтов LLM. Обратите внимание: для некоторых ИИ-агентов (например, ChatGPT) нужно скачать `llms.txt` и загрузить как файл в чат.
- [`llms-full.txt`](https://adapty.io/docs/ru/llms-full.txt): Вся документация Adapty в одном файле. Очень большой — используйте только когда нужна полная картина.
- Платформенные подборки для Kotlin Multiplatform: [`kmp-llms.txt`](https://adapty.io/docs/ru/kmp-llms.txt) и [`kmp-llms-full.txt`](https://adapty.io/docs/ru/kmp-llms-full.txt) — экономят токены по сравнению с полным сайтом.
