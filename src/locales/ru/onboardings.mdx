Онбординги в Adapty позволяют вашим нетехническим командам создавать привлекательные и гибко настраиваемые онбординг-флоу без написания кода. Наш no-code конструктор помогает разработать серию экранов, которые проводят пользователей через первое знакомство с приложением.

:::important
Онбординги доступны для приложений, использующих Adapty SDK версии v3.8.0+ (iOS, Android, React Native, Flutter), v3.14.0+ (Unity) или v3.15.0+ (Kotlin Multiplatform, Capacitor).
:::

Хороший онбординг знакомит пользователей с вашим приложением:
- Показывает ключевую ценность приложения
- Объясняет основные функции и возможности
- Даёт важные советы по использованию

Решение для онбордингов в Adapty выделяется тем, что предлагает:
- No-code [конструктор онбордингов](design-onboarding), который даёт нетехническим командам полную самостоятельность
- Возможность [персонализировать опыт с помощью интерактивных вопросов и переменных](onboarding-user-engagement)
- Поддержку A/B-тестов для определения наиболее эффективных онбординг-флоу

## Как это работает \{#how-it-works\}

Чтобы запустить онбординг:

1. [Создайте онбординг в no-code редакторе.](design-onboarding)
2. [Создайте плейсмент для онбординга.](create-onboarding#step-2-create-a-placement-for-your-onboarding)
3. Интегрируйте онбординг в свой проект с помощью Adapty SDK:
   - [iOS](ios-onboardings)
   - [Android](android-onboardings)
   - [React Native](react-native-onboardings)
   - [Flutter](flutter-onboardings)
   - [Unity](unity-onboardings)
   - [Kotlin Multiplatform](kmp-onboardings)
   - [Capacitor](capacitor-onboardings)
4. Протестируйте онбординг и выпустите его для пользователей.

Для дальнейшего роста попробуйте более продвинутые подходы к работе с онбордингами:

- Добавьте дополнительные аудитории в плейсмент, чтобы показывать разные онбординги разным группам пользователей.
- Запустите A/B-тесты онбордингов, чтобы найти наиболее эффективный вариант.
- Свяжите онбординги с пейволами и повысьте конверсию.
