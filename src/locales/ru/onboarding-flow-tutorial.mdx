Многоэкранное флоу в Flow Builder — это последовательность экранов, связанных навигационными действиями. Флоу может быть линейным или ветвиться в зависимости от ввода пользователя, собранного на предыдущем экране. Этот гайд охватывает весь процесс — создание экранов, наполнение их контентом, настройку навигации и добавление условных ветвлений — на примере онбординга из четырёх экранов.

В примере используются:
- **Поле ввода имени**, которое передаёт имя пользователя как переменную для персонализации.
- **Квиз с одним вариантом ответа**, от которого зависит, какой экран пользователь увидит следующим.
- **Два разветвляющихся пути** с разным текстом для каждого сегмента аудитории.
- **Пейвол** в качестве финального экрана.

Этот же паттерн применим к любому флоу, который персонализирует контент на основе ввода пользователя.
## Перед началом работы \{#before-you-start\}

- [Создайте продукты](create-product) в дашборде Adapty. В примере используются два — годовая и месячная подписка.
- [Подключите Adapty к App Store и Google Play](integrate-payments).
## 1. Настройте переиспользуемые стили \{#1-set-up-reusable-styles\}

Переиспользуемые стили позволяют применять единую типографику и цвета на всех экранах в один клик. Цветовые стили содержат светлый и тёмный вариант, поэтому флоу автоматически поддерживает обе темы.

Подробные инструкции см. в разделе [Стилизация элементов — Переиспользуемые стили](builder-styling#reusable-styles).

Чтобы настроить стили:
1. В левой панели откройте панель **Styles** <Inline id="styles.svg" alt="Styles" />.
2. На вкладке **Colors** нажмите **<Inline id="plus.svg" alt="Plus" /> Create style** и добавьте цвета, которые будете использовать повторно. Для каждого цвета выберите значение в Light, перейдите на вкладку Dark и выберите значение для тёмной темы.

<ZoomImage id="flow-onboarding-color-styles.webp" width="500px" alt="Colors tab in the Styles panel with Light and Dark variants" />

3. На вкладке **Text** нажмите на существующий стиль, чтобы изменить шрифт, начертание и размер, или нажмите **<Inline id="plus.svg" alt="Plus" /> Create style**, чтобы добавить собственные пресеты.
<ZoomImage id="flow-onboarding-text-styles.webp" width="500px" alt="Вкладка Text в панели Styles с пользовательскими пресетами" />
## 2. Создайте экраны \{#2-create-the-screens\}

Флоу — это последовательность экранов. Сначала настройте первый экран с общей основой — разметкой, фоном и безопасной зоной, — затем продублируйте его для остальных. Так все экраны будут иметь одинаковую основу, и вам нужно будет настроить её только один раз.

Подробнее об управлении экранами см. в разделе [Экраны и слои — Управление экранами](paywall-layout-and-products#manage-screens).

Чтобы настроить экраны:
1. Нажмите на пустую область холста на первом экране, чтобы открыть настройки экрана.
2. В разделе **System UI** отключите **Safe area**, чтобы фоны и элементы, выровненные по краям, могли растягиваться до краёв экрана.
3. В разделе **Fill** выберите тип фона и настройте его — например, **Image** <Inline id="fill-image.svg" alt="Image" />, который будет отображаться за каждым экраном флоу.
4. В разделе **Layout** задайте направление **Vertical** <Inline id="vertical-layout.svg" alt="Vertical" /> и выберите распределение, подходящее для вашего дизайна.
<ZoomImage id="flow-onboarding-screen-settings.webp" width="300px" alt="Настройки экрана: фоновое изображение, отключённая безопасная зона и вертикальная раскладка" />

5. В разделе **Screens** левой панели нажмите на меню из трёх точек <Inline id="context.svg" alt="Контекстное меню" /> на первом экране и выберите **Duplicate**. Повторите, пока не получите четыре экрана — второй путь разветвления будет добавлен позже путём дублирования первого.
6. Переименуйте каждый экран в соответствии с его ролью — в нашем примере: `Welcome`, `Quiz`, `Rock path` и `Paywall`.
<ZoomImage id="flow-onboarding-screens.webp" width="300px" alt="Четыре именованных экрана с общими базовыми настройками" />
## 3. Создайте экран приветствия \{#build-the-introduction-screen\}

Первый экран задаёт тон — заголовок, список функций и призыв к действию, открывающий остальную часть флоу. В нашем примере это экран Welcome.

Нажмите на экран **Welcome** в панели **Screens**, затем добавьте элементы:

1. Добавьте главное изображение. Нажмите **+** > **Media** > **Image**, загрузите изображение и при необходимости настройте отступы.

<ZoomImage id="flow-onboarding-welcome-image.webp" width="700px" alt="Верхняя часть экрана Welcome с главным изображением и заголовком" />
2. Добавьте заголовок: нажмите **+** > **Text**, выберите стиль заголовка из сохранённых текстовых стилей и отредактируйте поле **Content**.

<ZoomImage id="flow-onboarding-welcome-top.webp" width="700px" alt="Top of the Welcome screen with the main image and headline" />

3. Добавьте список функций. Нажмите **+** > **List** > **Icon Cards**, затем отредактируйте иконку и подпись в каждой карточке.

<ZoomImage id="flow-onboarding-welcome-features.webp" width="700px" alt="Icon Cards feature list below the headline" />

4. Добавьте основную кнопку навигации внизу экрана. Действие для неё будет настроено на шаге навигации.
<ZoomImage id="flow-onboarding-welcome.webp" width="700px" alt="Финальный экран приветствия" />
## 4. Создайте экран ввода и квиза \{#build-the-input-and-quiz-screen\}

Второй экран собирает данные от пользователя. В нашем примере он спрашивает имя и предлагает один вопрос с выбором ответа, который определяет, по какому пути пойдёт пользователь дальше.

Подробнее об инпутах и квизах — в статьях [Поля ввода и формы](builder-inputs-and-forms) и [Опросы и квизы](onboarding-quizzes).

Нажмите на экран **Quiz** в панели **Screens**, затем добавьте элементы. Каждая группа на экране — вступление, вопрос + поле ввода, вопрос + квиз — размещена в отдельном Vertical Stack, чтобы связанные элементы оставались визуально сгруппированными.
1. Добавьте заголовок и текст вступления. Нажмите **+** > **Text** > **H1** для заголовка и **+** > **Text** > **Body** для вспомогательного текста.
2. Сгруппируйте вступление. Нажмите **+** > **Layout** > **Vertical Stack**, перетащите новый стек в верхнюю часть дерева слоёв, затем перетащите H1 и Body внутрь него.

<ZoomImage id="flow-onboarding-quiz-intro.gif" width="700px" alt="Заголовок и текст вступления, сгруппированные в Vertical Stack в верхней части дерева слоёв" />
3. Добавьте первый вопрос и поле ввода. Нажмите **+** > **Text** для подписи вопроса, затем **+** > **Inputs** > **Text** для поля.
4. Задайте **Element ID** поля ввода на вкладке **Design** — в нашем примере это `name`. Это делает значение доступным как переменную, на которую могут ссылаться другие экраны.

<ZoomImage id="flow-onboarding-quiz-input-element-id.webp" width="700px" alt="Поле ввода имени с Element ID name, сгруппированное с подписью в вертикальном стеке" />
5. Сгруппируйте подпись и поле ввода в вертикальный стек так же, как и во вводной части.
6. Добавьте второй вопрос и квиз. Нажмите **+** > **Text** для подписи, затем **+** > **Quiz** и выберите пресет макета, например Icon Options. Настройте варианты — в нашем примере `Rock` и `Hip hop`.
7. Сгруппируйте подпись и квиз в вертикальный стек аналогичным образом.
8. Задайте идентификаторы вариантов. Выберите каждый вариант квиза, откройте вкладку **Interactions** и укажите его **Element ID**. Эти идентификаторы используются в условной навигации далее.
<ZoomImage id="flow-onboarding-quiz-option-id.webp" width="700px" alt="Quiz group set to Single choice in Selectable Groups" />

9. Переключите квиз в режим одиночного выбора: нажмите на пустую область холста, чтобы открыть **Screen settings**, прокрутите вниз до **Selectable Groups**, нажмите на название группы квиза и установите тип **Single choice**.

<ZoomImage id="flow-onboarding-quiz-single-choice.webp" width="700px" alt="Quiz group set to Single choice in Selectable Groups" />

10. Добавьте основную кнопку внизу — это кнопка «Далее», которая запускает ветвление.
<ZoomImage id="flow-onboarding-quiz.webp" width="300px" alt="Финальный экран квиза с вступлением, полем для ввода имени и вопросом с одним вариантом ответа" />
## 5. Создайте первую ветку пути \{#build-the-first-branching-path\}

Каждый экран ветки подстраивает контент под определённый сегмент аудитории. В нашем примере ветка Rock содержит контент, ориентированный на рок — плейлисты, исполнителей и рекомендации.

Подробнее о переменных читайте в разделе [Переменные](onboarding-variables).

Чтобы создать экран:
1. На панели **Screens** нажмите на экран **Rock path**.
2. Добавьте заголовок. Поместите курсор в поле **Content** в том месте, где должна появиться персонализация, нажмите на иконку переменной <Inline id="variable.webp" alt="Variable icon" /> и откройте вкладку **Elements**. Выберите экран, на котором находится поле ввода — в нашем примере **Quiz** — затем выберите переменную значения этого поля. Picker разрешит её как `<elementId>.value` — в нашем примере `name.value`. Во время выполнения заголовок обновится с учётом того, что ввёл пользователь.
<ZoomImage id="flow-onboarding-rock-variable.webp" width="700px" alt="Variable picker on the Elements tab showing the Quiz screen and name.value variable" />

3. Добавьте основной текст как дополнительные текстовые элементы, адаптированные под сегмент аудитории для этого пути.
4. Добавьте основную кнопку внизу.

<ZoomImage id="flow-onboarding-rock.webp" width="300px" alt="Final first path screen with personalized headline" />
## 6. Создайте второй разветвлённый путь \{#build-the-second-branching-path\}

Экраны одного пути обычно имеют одинаковый макет — меняется только текст. Продублируйте первый экран пути и обновите его содержимое.

Чтобы продублировать и обновить:

1. На панели **Screens** выберите первый экран пути и нажмите ⌘D / Ctrl+D для дублирования. Копия появится в конце списка экранов.
2. Переименуйте копию — в нашем примере `Hip hop path` — и перетащите её в нужное место в списке экранов, чтобы она стояла рядом с экраном, из которого была продублирована.
<ZoomImage id="flow-onboarding-duplicate.gif" width="700px" alt="Duplicated path screen renamed and updated for the other audience" />

3. Обновите основной текст для другого сегмента аудитории. Персонализированный заголовок продолжает работать — переменная сохраняется.

<ZoomImage id="flow-onboarding-hiphop.webp" width="300px" alt="Duplicated path screen renamed and updated for the other audience" />
## 7. Создайте пейвол \{#build-the-paywall\}

Финальный экран — пейвол, где пользователь может оформить подписку. Подробное описание механики пейвола см. в разделе [Создание базового экрана пейвола](basic-paywall-screen). Версия ниже — сокращённый вариант.

Нажмите на экран **Paywall** в панели **Screens** и добавьте элементы:

1. Добавьте **Horizontal Stack** вверху и поместите внутрь кнопку **Close**. Пресет Close настроен автоматически.
2. Добавьте главное изображение, заголовок (с той же переменной персонализации, что и на экранах пути) и подзаголовок в качестве вспомогательного текста.
<ZoomImage id="flow-onboarding-paywall-top.webp" width="300px" alt="Верхняя часть пейвола с кнопкой закрытия, основным изображением и заголовком" />

3. Добавьте продукты: нажмите **+** > **Products** и выберите **Vertical List**. Назначьте каждой карточке продукт из выпадающего списка на вкладке **Design**.
4. Нажмите на карточку продукта по умолчанию и включите **Set as default product**, чтобы он был выбран при загрузке экрана.

<ZoomImage id="flow-onboarding-paywall-products.gif" width="700px" alt="Список продуктов с одной карточкой, отмеченной как продукт по умолчанию" />
5. Добавьте кнопку покупки. Нажмите **+** > **Buttons** и выберите пресет. На вкладке **Interactions** нажмите **Add trigger** > **On tap** > **Add action** и установите **Action** в **Purchase**, а **Product** — в `products.selectedProduct`.

<ZoomImage id="flow-onboarding-paywall-purchase.webp" width="700px" alt="Top of the paywall with close button, main image, and headline" />
6. Добавьте ссылки в футер. Нажмите **+** > **Buttons** > **Links** — строка содержит Restore Purchases, Terms of Use и Privacy Policy. Ссылка Restore настроена заранее. Для Terms и Privacy выделите текст ссылки в поле **Content**, нажмите на иконку ссылки на панели форматирования и вставьте URL.

<ZoomImage id="flow-onboarding-paywall.webp" width="700px" alt="Финальный пейвол с продуктами, кнопкой покупки и ссылками" />
## 8. Настройте навигацию между экранами \{#wire-navigation-between-the-screens\}

Экраны не связываются друг с другом автоматически. Используйте триггеры **On tap** и действия **Navigate to**, чтобы соединить основную кнопку каждого экрана со следующим экраном. Если экран разветвляется в зависимости от ввода пользователя, вместо этого используется **Conditional action**.

Подробнее о навигации и условных действиях — в разделах [Навигация и взаимодействие](onboarding-navigation-branching) и [Действия — Условные действия](onboarding-actions#conditional-actions).

Чтобы настроить навигацию для примера флоу:
1. **Статическая навигация с начального экрана.** Откройте экран Welcome, выберите основную кнопку и перейдите на вкладку **Interactions**. Нажмите **Add trigger** > **On tap** > **Add action**, установите **Action** в значение **Navigate to** и выберите следующий экран — в нашем примере это экран Quiz.

<ZoomImage id="flow-onboarding-nav-static.gif" width="700px" alt="Static Navigate to action on the Welcome screen's primary button" />
2. **Условная навигация из опроса.** Откройте экран Quiz, выберите кнопку Next и добавьте триггер **On tap** с **Conditional action**. Настройте правило IF/ELSE:
   - В выборе переменных откройте вкладку **Elements**, выберите экран **Quiz** и укажите `quiz.selectedOptionId`.
   - Используйте оператор **Equals** и сравните с ID одного из вариантов — в нашем примере это вариант Rock.
   - **IF** условие выполняется, запустите **Navigate to** и выберите экран первого пути.
   - **ELSE** — запустите **Navigate to** и выберите экран второго пути.
<ZoomImage id="flow-onboarding-nav-conditional.gif" width="700px" alt="Conditional action on the Quiz Next button with IF/ELSE branches" />

3. **Статическая навигация с каждого ответвления на пейвол.** Повторите шаг 1 на каждом экране ответвления, указав пейвол в качестве места назначения.

<ZoomImage id="flow-onboarding-nav-to-paywall.webp" width="700px" alt="Both paths point to the same paywall" />
