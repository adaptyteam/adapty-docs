[Плейсмент](placements) — это конкретное место в вашем мобильном приложении, где можно показать пейвол, онбординг или A/B-тест. Например, выбор подписки может появляться в стартовом флоу, а расходуемый продукт (например, золотые монеты) — когда у пользователя заканчиваются монеты в игре.

В разных плейсментах или для разных сегментов пользователей — в Adapty они называются «аудиториями» — можно показывать одинаковые или разные пейволы, онбординги и A/B-тесты.

Советы по выбору подходящих плейсментов читайте в разделе [Выбор значимых плейсментов](choose-meaningful-placements).

:::tip
Плейсменты также можно создавать программно с помощью [Developer CLI](developer-cli-reference#adapty-placements-create).
:::

:::info
Хотя процесс создания плейсмента одинаков для пейволов и онбордингов, один плейсмент нельзя использовать для обоих, поскольку они отслеживают разные метрики.
:::

## Создание и настройка плейсмента \{#create-and-configure-a-placement\}

1. Перейдите в раздел **[Placements](https://app.adapty.io/placements)** в главном меню Adapty.

2. Нажмите **Create placement**.

<Zoom>
  <img src="/assets/shared/img/create-placement-2.png"
  style={{
    border: '1px solid #727272', /* border width and color */
    width: '700px', /* image width */
    display: 'block', /* for alignment */
    margin: '0 auto' /* center alignment */
  }}
/>
</Zoom>

3. Введите **Placement name** — внутренний идентификатор в дашборде Adapty. При необходимости его можно изменить позже.

4. Введите **Placement ID** — этот идентификатор используется в SDK Adapty для вызова [пейволов](paywalls), [онбордингов](onboardings) и [A/B-тестов](ab-tests) из данного плейсмента. Изменить его после создания нельзя, так как он уникален для каждого плейсмента.

Затем назначьте плейсменту пейвол, онбординг или A/B-тест. Adapty поддерживает [аудитории](audience) — сегменты пользователей на основе [сегментов](segments) — что позволяет показывать разный контент разным группам пользователей. Если таргетинг не нужен, аудитория *All users* по умолчанию охватывает всех.

<AddAudience />

После сохранения и публикации плейсмента всё готово — используйте **Placement ID** в коде приложения, чтобы загрузить и отобразить его.
