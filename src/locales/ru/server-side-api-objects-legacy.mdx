## Объекты \{#objects\}

:::warning

**Вы просматриваете гайд для устаревшего серверного API.**
Актуальная версия описана в разделах [Server-side API V2](ss-authorization) и [Руководство по миграции на Server-side API V2](migration-guide-to-server-side-api-v2).

:::

Все значения даты и времени соответствуют формату [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601), например "2020-01-15T15:10:36.517975+0000".

### CustomerAccessLevel \{#customeraccesslevel\}

Информация об [уровне доступа](access-level) пользователя.

Настройте в дашборде Adapty, какие продукты открывают платные функции вашего приложения. Затем проверяйте в приложении уровень доступа, доступный пользователю, например следующим образом (на Swift):

```swift showLineNumbers title="Swift"
if (profile.paidAccessLevels["premium"]?.isActive == true) {
    /* Grant user access to paid functions of the app */
}
```

### Subscription \{#subscription\}

Информация о подписке вендора. В большинстве случаев использовать этот объект не нужно — предпочтительным способом работы с доступом к приложению является **CustomerPaidAccessLevel**. При использовании этого объекта необходимо реализовать логику обработки для каждого периода подписки (неделя, месяц, год, пожизненный).
