:::warning

**Вы просматриваете гайд по устаревшему серверному API.**
Для последней версии обратитесь к [Server-side API V2](ss-authorization) и [Руководству по миграции на Server-side API V2](migration-guide-to-server-side-api-v2).

:::

С помощью API вы можете:

1. Получить статус подписки пользователя.
2. Активировать подписку для пользователя с [уровнем доступа](access-level).
3. Получить атрибуты пользователя.
4. Установить атрибуты пользователя.

:::note
Получать события подписки через API нельзя, но вы можете использовать [Webhook](webhook) или прямую интеграцию с нужным сервисом.
:::

Для корректной работы с API необходимо использовать уникальный идентификатор для каждого пользователя. Это может быть email, номер телефона или ваш внутренний ID. Без такого идентификатора невозможно распознать одного и того же пользователя на разных платформах.

## Сценарий 1: Синхронизация подписчиков между вебом и мобильным приложением \{#case-1-syncing-subscribers-between-web-and-mobile\}

Какой бы веб-провайдер платежей вы ни использовали — Stripe, ChargeBee или любой другой — вы можете синхронизировать подписчиков. Для этого:

1. _Используйте уникальный идентификатор для пользователей_. Например, email или номер телефона.
2. Проверяйте статус подписки через API.
3. Если пользователь на бесплатном тарифе, покажите ему пейвол на сайте.
4. После успешной оплаты обновите статус подписки в Adapty через API.
5. Ваши подписчики будут автоматически синхронизированы с мобильным приложением.

## Сценарий 2: Выдача подписки \{#case-2-grant-a-subscription\}

:::note
По соображениям безопасности выдать подписку через SDK нельзя.
:::

Представьте ситуацию: вы проводите промо-акцию с пробным периодом на 7 дней и хотите синхронизировать её с мобильным опытом пользователя. Для этого:

1. Получите уникальный идентификатор пользователя.
2. Установите премиум-доступ через платный уровень доступа с помощью API на срок 7 дней.

По истечении 7 дней пользователи, не оформившие подписку, будут автоматически переведены на бесплатный тариф.

## Сценарий 3: Синхронизация атрибутов пользователей и пользовательских свойств \{#case-3-syncing-users-attributes-and-custom-properties\}

Помимо стандартных атрибутов (IDFA, модель устройства и т. д.), у ваших пользователей могут быть собственные. Например, в сервисе изучения языков может понадобиться сохранять количество выученных слов. Для этого:

1. Получите уникальный идентификатор пользователя.
2. Обновите атрибут через API или SDK.

С помощью таких атрибутов можно, например, создать сегмент и запустить A/B-тест.

Подробнее об устаревшем S2S API читайте в разделе [Спецификации API](server-side-api-specs-legacy).
