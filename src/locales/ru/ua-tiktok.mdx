Интеграция Adapty UA с TikTok for Business позволяет отслеживать и оптимизировать эффективность кампаний в TikTok.

:::tip
Смотрите наш [гайд по настройке рекламы в TikTok for Business](tiktok-create-campaign).
:::

## Шаг 1. Подключите аккаунт TikTok \{#step-1-connect-your-tiktok-account\}

1. Перейдите в **Integrations > TikTok Ads** в левом сайдбаре и нажмите **Continue with TikTok**.

<ZoomImage id="ua-connect-tiktok.webp" width="700px" />

2. Войдите в аккаунт TikTok и нажмите **Continue**.

3. Ознакомьтесь с запрашиваемыми разрешениями и нажмите **Save**.

<ZoomImage id="ua-tiktok-sign.webp" width="500px" />

После этого все ваши рекламные аккаунты будут добавлены в Adapty UA. Можно переходить к добавлению кампаний.

## Шаг 2. Добавьте кампании \{#step-2-add-campaigns\}

Чтобы добавить кампанию TikTok for Business в Adapty User Acquisition и отслеживать работу ваших TikTok-объявлений в Adapty:

1. Перейдите на вкладку **Web campaigns** и нажмите **Create campaign**. Выберите приложение и нажмите **Save**.

<ZoomImage id="ua-new-campaign-tiktok.webp" width="500px" />

2. На вкладке **General** разверните раздел **iOS** и/или **Android** и вставьте URL-адреса приложения в App Store и/или Google Play.

<ZoomImage id="ua-url.gif" width="900px" />

3. Скопируйте значение поля **Click link**. Затем при создании объявления в TikTok Ads Manager вставьте это значение в поле **Tracking URL** в разделе **Advanced Settings**. Это позволит Adapty связывать установки и покупки с объявлениями в TikTok.

<ZoomImage id="ua-tiktok-lnk.webp" width="900px" />

4. (Опционально) Чтобы отправлять события конверсии обратно в TikTok, вы можете связать пиксели из TikTok с кампаниями в Adapty UA. Для этого выберите один из существующих пикселей в выпадающем списке **Pixel**.

:::tip
Вы также можете задать дополнительные параметры ссылок для [работы с отложенными данными](ua-deferred-data). Например, можно реализовать отложенные диплинки.
:::

## Шаг 3. Настройте маппинг событий \{#step-3-map-events\}

Чтобы отправлять события конверсии обратно в TikTok для оптимизации кампаний, необходимо настроить маппинг событий в разделе **Events names**. Это позволит Adapty автоматически отправлять события подписки на ваш пиксель TikTok, когда пользователи совершают действия в приложении.

В разделе **Events names** включите события, которые хотите отслеживать в TikTok Ads Manager. Для каждого включённого события выберите соответствующее событие TikTok из выпадающего списка или задайте собственное. По умолчанию Adapty сопоставляет события Adapty со стандартными событиями TikTok.

Нажмите **Save**, чтобы применить настройки маппинга событий.

<ZoomImage id="ua-meta-events.webp" width="900px" />

## Дополнительная настройка \{#additional-configuration\}

### Дополнительные параметры \{#additional-parameters\}

Поле **Additional parameter** позволяет добавлять пользовательские точки данных для анализа за пределами Adapty. Это удобно, когда нужно передать определённые данные кампании или пользователя во внешние инструменты аналитики или партнёрам по атрибуции.

В поле **Additional parameter** введите любые пользовательские данные, которые хотите включить в отслеживание атрибуции. Дополнительный параметр будет включён во все данные атрибуции, отправляемые в TikTok, и может использоваться для расширенного анализа и оптимизации кампаний.

Например, если вы запускаете несколько вариантов одной кампании, можно добавить `variant=A` или `variant=B`, чтобы различать разные креативные подходы.

:::important
Дополнительные параметры изменяют **Click link**, который вы вставляете в TikTok Ads Manager. Если вы уже скопировали эту ссылку и добавили пользовательский параметр позже, убедитесь, что скопировали и вставили обновлённую ссылку с этим параметром.
:::

### Настройки атрибуции \{#attribution-settings\}

Настройки атрибуции определяют, как Adapty сопоставляет действия пользователей с вашими кампаниями в TikTok Ads. Они задают временны́е окна для детерминированного и вероятностного сопоставления.

Чтобы настроить атрибуцию, перейдите на вкладку **Attribution Settings** в настройках кампании. Там вы найдёте два основных параметра:

- **Deterministic matching window**: использует точные идентификаторы устройств (например, IDFA на iOS или Advertising ID на Android) для точного сопоставления пользователей с кампаниями. Установите значение 168 часов (7 дней) для максимальной точности атрибуции — это значение по умолчанию и рекомендуемое. Когда пользователь кликает на ваше объявление в TikTok и устанавливает приложение в этом окне, Adapty может однозначно приписать установку конкретному клику по объявлению с помощью идентификаторов устройства.

- **Probabilistic matching window**: использует статистическое моделирование и цифровой отпечаток устройства для сопоставления пользователей, когда детерминированное сопоставление невозможно. Установите значение 6 часов для большинства кампаний — это значение по умолчанию, которое хорошо работает в большинстве случаев. Для кампаний с большим объёмом кликов можно уменьшить это значение до 1–2 часов. Для пользователей, которых нельзя сопоставить детерминированно (из-за настроек конфиденциальности или других факторов), Adapty использует вероятностное сопоставление в этом более коротком окне.

Нажмите **Save**, чтобы применить настройки атрибуции.

### Переопределение дохода \{#revenue-override\}

Если вы отслеживаете события триала и хотите, чтобы TikTok приписывал им доход, используйте раздел **Revenue override**. Он появляется, когда включено событие **Trial started**.

Для каждого целевого события триала укажите процент от цены подписки, который нужно передавать как доход. Например, при 30% Adapty отправляет 30% от цены подписки в качестве значения конверсии для событий триала.

Чтобы добавить переопределение:

1. Включите **Trial started** в разделе **Events names**.
2. В разделе **Revenue override** нажмите **Add override**.
3. Выберите целевое событие и введите процент дохода (0–100).
4. Нажмите **Save**.

<ZoomImage id="revenue-override.webp" width="500px" />

### Отправка всех событий \{#send-all-events\}

По умолчанию Adapty отправляет события на ваш пиксель только для пользователей, атрибутированных к кампании TikTok. Включите **Send all events**, чтобы также передавать события от органических и неатрибутированных пользователей на пиксель.

При включении этой опции каждое событие установки и транзакции отправляется на пиксель вне зависимости от атрибуции кампании. Используйте это, чтобы предоставить TikTok более широкие данные о конверсиях для моделирования аудитории и оптимизации кампаний.

Чтобы включить эту опцию, в настройках кампании выберите **Send all events (forward organic/non-attributed events to this pixel)** и нажмите **Save**.
