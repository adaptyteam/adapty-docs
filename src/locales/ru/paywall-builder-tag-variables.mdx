Paywall Builder Adapty позволяет настраивать все тексты для ваших продуктов и их офферов. Если вы поддерживаете несколько локалей, настоятельно рекомендуем использовать переменные.

### Как это работает \{#how-it-works\}

Когда вы добавляете теговые переменные из нашего списка в тексты продуктов, SDK подставляет заранее загруженные локализованные данные из сторов вместо тегов. Это гарантирует, что текст на вашем пейволе всегда точно соответствует нужной локали.

**Пример**: Допустим, у вас есть «Premium Subscription», доступная как в США, так и в Испании. В США она может отображаться как «Premium Subscription for $4.99/month», а в Испании — «Suscripción Premium por €4.99/mes».

Теговые переменные позволяют автоматически локализовать эти строки на основе данных непосредственно из стора, обеспечивая точность названий и цен.

### Как использовать теговые переменные \{#how-to-use-tag-variables\}

:::note
Теговые переменные можно использовать только при описании продуктов и офферов в компоненте Product в Paywall Builder.
:::

1. В левой панели Paywall Builder выберите продукт, который хотите настроить.
2. Используйте переменные из [таблицы ниже](paywall-builder-tag-variables#full-list-of-variables) в любых текстовых полях для описания продукта и его офферов.

### Полный список переменных \{#full-list-of-variables\}

| Теговая переменная        | Описание                                                                                                                                                                                                                                              | Пример               |
| :------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------- |
| `<PROD_TITLE/>`           | Локализованное название продукта                                                                                                                                                                                                                      | Premium Subscription |
| `<PROD_PRICE/>`           | Локализованная цена продукта. Для подписок — это цена за один расчётный период.                                                                                                                                                                       | $9.99                |
| `<PROD_PRICE_PER_DAY/>`   | Цена подписки, делённая на количество дней в расчётном периоде. **Возвращает пустую строку для не-подписок.**                                                                                                                                         | $0.33                |
| `<PROD_PRICE_PER_WEEK/>`  | Цена подписки, делённая на количество недель в расчётном периоде. **Возвращает пустую строку для не-подписок.**                                                                                                                                       | $2.33                |
| `<PROD_PRICE_PER_MONTH/>` | Цена подписки, делённая на количество месяцев в расчётном периоде. Если расчётный период меньше месяца, значение приводится к полному месяцу. **Возвращает пустую строку для не-подписок.**                                                           | $9.99                |
| `<PROD_PRICE_PER_YEAR/>`  | Цена подписки, приведённая к полному году использования. Например, цена месячной подписки умножается на 12, а цена годовой остаётся неизменной. **Возвращает пустую строку для не-подписок.**                                                         | $119.88              |
| `<OFFER_PRICE/>`          | Локализованная цена оффера (introductory или promotional). **Применяется только к автовозобновляемым подпискам, возвращает пустую строку, если пользователь не имеет права ни на один оффер.**                                                        | $0.99                |
| `<OFFER_BILLING_PERIOD/>` | Локализованный расчётный период оффера (introductory или promotional). Совпадает с `<OFFER_FULL_DURATION/>` для офферов типа trial и pay-upfront. **Применяется только к автовозобновляемым подпискам, возвращает пустую строку, если пользователь не имеет права ни на один оффер.** | 1 week               |
| `<OFFER_FULL_DURATION/>`  | Локализованная полная продолжительность оффера (introductory или promotional). **Применяется только к автовозобновляемым подпискам, возвращает пустую строку, если пользователь не имеет права ни на один оффер.**                                    | 1 month              |

### Теговые переменные офферов для разных типов офферов \{#offer-tags-for-different-offer-types\}

:::note
Подробнее об офферах и их настройке в Adapty можно узнать [здесь](offers)
:::

Теговые переменные офферов могут быть немного запутанными, поэтому разберём на примере. Допустим, у вас есть еженедельная подписка «Premium Subscription» за $5 с тремя возможными офферами:

- **Pay As You Go**. Первые 3 недели за $3 (списывается еженедельно), затем $5/неделю.
- **Pay Up Front**. Первые 3 недели за $8 (списывается сразу), затем $5/неделю.
- **Free Trial**. Первая неделя бесплатно, затем $5/неделю.

Для этого продукта `<PROD_TITLE/>` будет «Premium Subscription», а `<PROD_PRICE/>` — $5. Значения теговых переменных офферов — в зависимости от того, на какой оффер имеет право пользователь — будут следующими:

| Теговая переменная        | Pay As You Go | Pay Upfront | Free Trial |
| :------------------------ | :------------ | :---------- | :--------- |
| `<OFFER_PRICE/>`          | $3            | $8          | $0         |
| `<OFFER_BILLING_PERIOD/>` | 1 week        | 3 weeks     | 1 week     |
| `<OFFER_FULL_DURATION/>`  | 3 weeks       | 3 weeks     | 1 week     |

Таким образом, для офферов всех типов, кроме «Pay As You Go», значения `<OFFER_BILLING_PERIOD/>` и `<OFFER_FULL_DURATION/>` будут одинаковыми.
