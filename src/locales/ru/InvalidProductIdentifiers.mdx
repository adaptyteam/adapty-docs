Ошибка с кодом 1000, `noProductIDsFound`, означает, что ни один из продуктов, запрошенных на пейволе, недоступен для покупки в App Store, несмотря на то что они там перечислены. Иногда эта ошибка сопровождается предупреждением `InvalidProductIdentifiers`. Если предупреждение появляется без ошибки — просто проигнорируйте его.

Если вы столкнулись с ошибкой `noProductIDsFound`, выполните следующие шаги для её устранения:

## Шаг 1. Проверьте bundle ID \{#step-2-check-bundle-id\}

<ProvideBundleID />

## Шаг 2. Проверьте продукты \{#step-2-check-products\}

1. Перейдите в **App Store Connect** и откройте раздел [**Monetization** → **Subscriptions**](https://appstoreconnect.apple.com/apps/6477523342/distribution/subscriptions) в левом меню.

   margin: '0 auto' /* center alignment */
 }}
  />
</Zoom>

2. Нажмите на название группы подписок. Вы увидите список продуктов в разделе **Subscriptions**.

3. Убедитесь, что тестируемый продукт имеет статус **Ready to Submit**. Если нет — следуйте инструкциям на странице [Продукт в App Store](app-store-products).
    <Zoom>
   <img src="/assets/shared/img/ready-to-submit.webp"

        display: 'block', /* for alignment */
        margin: '0 auto' /* center alignment */

    />
    </Zoom>

4. Сравните идентификатор продукта из таблицы с тем, что указан на вкладке [**Products**](https://app.adapty.io/products) в дашборде Adapty. Если идентификаторы не совпадают, скопируйте идентификатор из таблицы и [создайте продукт](create-product) с этим идентификатором в дашборде Adapty.
