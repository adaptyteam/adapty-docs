Чтобы загрузить [онбординг](onboardings), приложение запрашивает данные его конфигурации у Adapty. Конфигурации онбордингов хранят URL-адреса ваших онбординг-потоков. Если сетевые проблемы нарушают соединение между приложением и серверами Adapty, вы не сможете корректно настроить и отобразить онбординги.

Чтобы получить доступ к данным конфигурации онбордингов в офлайн-режиме, сохраните резервный конфигурационный файл прямо в коде приложения. Вы можете скачать готовые резервные файлы с данными конфигурации для ваших онбордингов и [пейволов](fallback-paywalls) непосредственно из Adapty.

Следуйте инструкциям ниже, чтобы скачать файл и добавить его в код приложения.

:::important
Резервные онбординги **требуют подключения к интернету**, поскольку контент онбордингов всегда хранится онлайн. Резервный файл содержит только конфигурацию онбордингов.

Прочитайте статью [офлайн-режим онбординга](onboarding-offline), чтобы понять, что происходит, когда приложение не может загрузить онбординг.
:::

<details>
   <summary>Перед настройкой резервных онбордингов (нажмите, чтобы развернуть)</summary>

1. Создайте [онбординг](onboardings).
2. [Создайте плейсмент](create-placement) для онбординга, чтобы определить, где в приложении он будет отображаться.

</details>

Adapty автоматически генерирует JSON-конфигурационные файлы для ваших резервных онбордингов — по одному на каждую платформу. Эти файлы также содержат резервные данные для ваших [пейволов](local-fallback-paywalls).

Если в одном плейсменте есть несколько онбордингов или пейволов, в резервной версии будет использоваться вариант с наибольшим весом или наиболее широкой аудиторией. Adapty обновляет эти файлы каждый раз, когда вы вносите изменения в пейволы или онбординги.

Чтобы скачать резервные конфигурации, выполните следующие шаги:

1. Откройте страницу **[Placements](https://app.adapty.io/placements)**.
2. Нажмите кнопку **Fallbacks**.
3. Выберите нужную платформу (*iOS* или *Android*) из выпадающего списка.
4. Выберите версию SDK, чтобы начать загрузку.
