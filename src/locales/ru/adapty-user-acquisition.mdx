Adapty User Acquisition — это решение для атрибуции, которое связывает рекламные кампании с установками приложения и доходами от подписок, объединяя данные из рекламных платформ, трекинговых ссылок и вашего приложения. Оно предоставляет единый маркетинговый аналитический дашборд, в котором собраны все данные о привлечении пользователей.

- Рассчитывайте ROAS (окупаемость рекламных расходов) по всем каналам
- Видьте всю экономику приложения в одном месте
- Получайте точные данные атрибуции для принятия взвешенных решений
- Анализируйте эффективность когорт и поведение пользователей в динамике

## Почему Adapty UA? \{#why-adapty-ua\}

Измерять эффективность привлечения пользователей непросто. Данные зачастую разбросаны по разным платформам, атрибуция усложнилась из-за изменений в политике конфиденциальности, а создание собственных решений требует значительных ресурсов.

Adapty UA предоставляет встроенную атрибуцию и единую аналитику в одном маркетинговом дашборде. Все ваши метрики привлечения — от рекламных расходов до установок и доходов от подписок — автоматически агрегируются и обновляются в реальном времени. Больше не нужно сводить данные из разных таблиц или переключаться между множеством инструментов. Вы можете сосредоточиться на росте приложения, а не на управлении данными.

## Как это работает \{#how-it-works\}

Adapty User Acquisition атрибутирует установки приложения и доходы от подписок к рекламным кампаниям, объединяя данные из рекламных платформ, трекинговых ссылок и вашего приложения.

На верхнем уровне:
- Рекламные платформы предоставляют структуру кампаний и данные о расходах
- Трекинговые ссылки, сгенерированные в Adapty UA, передают контекст кампании с веба до установки приложения
- SDK Adapty отправляет события установки и покупки из вашего приложения

Процесс атрибуции выглядит следующим образом:

1. **Трекинговая ссылка генерируется в Adapty UA и добавляется в рекламную кампанию.** Ссылка содержит параметры кампании: платформу, кампанию, группу объявлений и креатив.

2. **Пользователь нажимает на объявление и устанавливает приложение из стора.** Пользователь перенаправляется через трекинговую ссылку и устанавливает приложение из App Store или Google Play.

3. **Приложение отправляет событие установки в Adapty.** При первом запуске SDK Adapty отправляет событие установки. Adapty извлекает параметры кампании, связанные с этой установкой.

4. **Установка атрибутируется к кампании.** Используя параметры кампании из трекинговой ссылки, Adapty связывает установку с кампанией, которая её сгенерировала.

5. **Рекламные расходы и доходы связываются воедино.** Adapty получает данные о рекламных расходах с поддерживаемых платформ (в настоящее время — Meta Ads и TikTok for Business) и привязывает события подписки и покупки к атрибутированным установкам.

В результате Adapty предоставляет метрики на уровне кампаний: установки, выручку, LTV и ROAS — в едином аналитическом дашборде. Вы можете анализировать когорты, отслеживать эффективность в динамике и принимать решения на основе данных — без ручного сведения информации из разных источников.

:::tip
Трекинговые ссылки также могут содержать пользовательские параметры, что позволяет вашему приложению обрабатывать [отложенные диплинки](ua-deferred-data) и реагировать на данные кампании при обработке события установки.
:::
