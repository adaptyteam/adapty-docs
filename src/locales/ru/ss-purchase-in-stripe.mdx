Выполняет валидацию покупки по переданному токену Stripe с использованием учётных данных Stripe из настроек приложения в дашборде Adapty. Если покупка валидна, история транзакций импортируется из Stripe в профиль Adapty с указанным `customer_user_id`. Если профиль с таким `customer_user_id` не существовал — он будет создан.

В процессе генерируются события профиля, а импортированные транзакции учитываются в MTR.

## Метод и эндпоинт \{#method-and-endpoint\}

```http
POST https://api.adapty.io/api/v1/sdk/purchase/stripe/token/validate/
```

:::warning
Этот запрос требует особых параметров авторизации:

- **Base URL**: https://api.adapty.io/api/v1/sdk/purchase/stripe/token/validate/
- **Authorization**: Запросы к API должны быть аутентифицированы — для этого передайте секретный API-ключ в заголовке **Authorization** в формате `Api-Key {secret_token}`, например `Api-Key secret_live_...`. Найти секретный API-ключ можно в [Adapty Dashboard -> **App Settings** -> вкладка **General** -> раздел API -> **API keys**](https://app.adapty.io/settings/general). Этот ключ секретный, не публикуйте его в открытом доступе.
- **Content-Type header**: API ожидает заголовок **Content-Type** со значением `application/vnd.api+json`.
- **Body**: API ожидает тело запроса в формате JSON.

:::

## Параметры \{#parameters\}

:::warning
Этот запрос требует особых параметров авторизации:

- **Base URL**: https://api.adapty.io/api/v1/sdk/purchase/stripe/token/validate/
- **Authorization**: Запросы к API должны быть аутентифицированы — для этого передайте секретный API-ключ в заголовке **Authorization** в формате `Api-Key {secret_token}`, например `Api-Key secret_live_...`. Этот ключ секретный, не публикуйте его в открытом доступе.
- **Content-Type header**: API ожидает заголовок **Content-Type** со значением `application/vnd.api+json`.

| Параметры          | Тип    | Обязательный      | Nullable           | Описание                                                     |
| :----------------- | :----- | :---------------- | :----------------- | :----------------------------------------------------------- |
| customer\_user\_id | String | :heavy_plus_sign: | :heavy_minus_sign: | ID пользователя в вашей системе. Его можно найти в поле **Customer user ID** на странице дашборда Adapty -> [**Profiles**](https://app.adapty.io/profiles/users) -> конкретный профиль. | stripe\_token      | String | :heavy_plus_sign: | :heavy_minus_sign: | Токен объекта Stripe, представляющего уникальную покупку. Может быть токеном Subscription в Stripe (`sub_XXX`) или Payment Intent (`pi_XXX`). |

## Ошибки \{#errors\}

### 400: Bad request \{#400-bad-request\}

Содержит список ошибок с параметрами.

**Параметры**

| Параметр | Тип     | Описание                                                     |
| -------- | ------- | ------------------------------------------------------------ |
| detail   | String  | Подробное описание ошибки.                                   |
| source   | String  | Объект, содержащий `"pointer"` — ссылку на точное место в документе запроса, вызвавшее проблему. |
| Status   | Integer | HTTP-статус. Всегда `400`.                                   |

**Пример ответа**

```json showLineNumbers
{
  "errors": [
    {
      "detail": "none is not an allowed value",
      "source": {
        "pointer": "/data/attributes/stripe_token"
      },
      "status": "400"
    }
  ]
}
```
