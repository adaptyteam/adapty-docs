Remote Config для онбординга — это инструмент, предоставляющий гибкие возможности настройки. Он позволяет использовать произвольные JSON-данные для точной конфигурации ваших онбордингов. С его помощью можно задавать различные параметры: заголовки, изображения, шрифты, цвета и многое другое. Например, Remote Config можно использовать для передачи дополнительных метаданных.

Прежде чем приступить к настройке онбординга, [создайте онбординг](create-onboarding).

Чтобы начать настройку онбординга с помощью Remote Config:

1. Откройте онбординг в разделе **Onboardings** и нажмите **Edit onboarding**.

2. Перейдите на вкладку **Remote config**.

Remote Config доступен в двух режимах отображения:

- [Таблица](customize-paywall-with-remote-config#table-view-of-the-remote-config)
- [JSON](customize-paywall-with-remote-config#json-view-of-the-remote-config)

Оба режима — **Table** и **JSON** — содержат одни и те же элементы конфигурации. Разница лишь в удобстве: табличный режим предлагает контекстное меню, которое может пригодиться для исправления ошибок локализации.  
Переключаться между режимами можно в любой момент, нажимая на вкладку **Table** или **JSON**.

Независимо от выбранного режима настройки, вы сможете получить доступ к этим данным из SDK через свойство `remoteConfig` объекта `AdaptyOnboarding` и вносить необходимые изменения в онбординг.

Вы можете комбинировать различные параметры и создавать собственные конфигурации.
