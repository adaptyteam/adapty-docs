Валидирует покупку по переданному токену Paddle, используя учётные данные Paddle из [App Settings](https://app.adapty.io/settings/general) в дашборде Adapty. Если покупка действительна, история транзакций импортируется из Paddle в профиль Adapty с указанным `customer_user_id`. Если профиля с таким `customer_user_id` ещё не существует — он будет создан.

## Заголовок запроса \{#request-header\}

:::warning
Этот запрос требует другого набора заголовков. Можно использовать только заголовок **Content-Type**.

Обратите внимание, что `customer_user_id` должен передаваться в теле запроса, а не в заголовке; `profile_id` указывать не нужно вовсе.
:::

| **Заголовок**    | **Описание**                                                 |
| ---------------- | ------------------------------------------------------------ |
| **Content-Type** | Укажите `application/json`, чтобы API обработал запрос.     |

## Метод и эндпоинт \{#method-and-endpoint\}

```http
POST https://api.adapty.io/api/v2/server-side-api/purchase/paddle/token/validate/
```

<TabItem value="curl" label="cURL" default>

curl --location 'https://api.adapty.io/api/v2/server-side-api/purchase/paddle/token/validate/' \

<TabItem value="python" label="Python" default>

```python showLineNumbers

url = "https://api.adapty.io/api/v2/server-side-api/purchase/paddle/token/validate/"

payload = json.dumps({

  "paddle_token": "live_7d279f61a3339fed520f7cd8c08"
})

  "Content-Type": "application/json",
  "Authorization": "Api-Key <YOUR_SECRET_API_KEY>"

response = requests.request("POST", url, headers=headers, data=payload)

```
</TabItem>  
<TabItem value="js" label="JavaScript" default>  

```javascript showLineNumbers
const myHeaders = new Headers();
myHeaders.append("Content-Type", "application/json");
myHeaders.append("Authorization", "Api-Key <YOUR_SECRET_API_KEY>");

const raw = JSON.stringify({
  "customer_user_id": "<YOUR_CUSTOMER_USER_ID>",
  "paddle_token": "live_7d279f61a3339fed520f7cd8c08"
});

const requestOptions = {
  method: "POST",
  headers: myHeaders,
  body: raw,
  redirect: "follow"
};

fetch("https://api.adapty.io/api/v2/server-side-api/purchase/paddle/token/validate/", requestOptions)
  .then((response) => response.text())
  .then((result) => console.log(result))
  .catch((error) => console.error(error));
```
</TabItem>  
</Tabs>

Плейсхолдеры:

- `<YOUR_SECRET_API_KEY>`: Ваш секретный API-ключ для авторизации.
- `<YOUR_CUSTOMER_USER_ID>`: Уникальный идентификатор пользователя в вашей системе.

## Параметры \{#parameters\}

:::

| Параметры          | Тип    | Обязательный      | Nullable           | Описание                                                     |
| :----------------- | :----- | :---------------- | :----------------- | :----------------------------------------------------------- |
| customer\_user\_id | String | :heavy_plus_sign: | :heavy_minus_sign: | Идентификатор вашего пользователя в вашей системе. Его можно найти в поле **Customer user ID** на дашборде Adapty в разделе [Profiles](https://app.adapty.io/profiles/users) на странице конкретного профиля. Чтобы это работало, необходимо <InlineTooltip tooltip="идентифицировать пользователей">[iOS](identifying-users), [Android](android-identifying-users), [React Native](react-native-identifying-users), [Flutter](flutter-identifying-users) и [Unity](unity-identifying-users)</InlineTooltip> в коде мобильного приложения через Adapty SDK. |
| paddle_token       | String | :heavy_plus_sign: | :heavy_minus_sign: | Токен объекта Paddle, представляющего уникальную покупку. Может быть как идентификатором транзакции (txn_...), так и идентификатором подписки (sub_...). |
