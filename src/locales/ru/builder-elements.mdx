Чтобы открыть библиотеку элементов и добавить новый элемент на экран, нажмите кнопку плюса <Inline id="plus.svg" alt="Plus" /> на левой панели или над превью устройства.
Элементы делятся на следующие категории:
- [Базовые](#basics) (контейнеры-лейауты, текст, медиа, списки, бейджи, галочки)
- [Формы и квиз](#forms--quiz) (кнопки, поля ввода, квизы, вкладки)
- [Пейвол и коммерция](#paywall--commerce) (продукты, переключатель пробного периода, взаимодействие с пользователем, обратный отсчёт)
- [Индикаторы прогресса](#progress) и загрузчики

Для каждого элемента Adapty предлагает несколько пресетов — шаблонов с шаблонным контентом или заранее настроенными взаимодействиями.
## Основы \{#basics\}
### Макет \{#layout\}

:::link
Основная статья: [Элементы макета](builder-containers)
:::

Элементы макета — это контейнеры, которые организуют вложенные элементы.

- **Vertical Stack**: располагает дочерние элементы сверху вниз
- **Horizontal Stack**: располагает дочерние элементы слева направо
- **Dividers (horizontal and vertical)**: линии, визуально разделяющие секции контента
- **Carousel**: контейнер с возможностью прокрутки свайпом
- **Bottom Sheet**: панель, выдвигающаяся снизу экрана

<ZoomImage id="layout-element-templates.webp" width="700px" alt="Шаблоны элементов макета" />
:::link
Основная статья: [Текст](onboarding-text)
:::

Текст может быть статическим или содержать [переменные](onboarding-variables) (например, имя пользователя) и [локализованный контент](paywall-localization).


<ZoomImage id="text-templates.webp" width="400px" alt="Text element templates" float="right" />

По умолчанию доступны следующие пресеты текста. Чтобы изменить этот список, добавьте или удалите [сохранённые стили текста](builder-styling#reusable-styles):

- H1
- H2
- H3
- Button Label
- Body
- Caption
- Small Label
### Медиа \{#media\}

:::link
Основная статья: [Изображения, видео и иконки](custom-media)
:::

:::note
В этом разделе описаны медиаэлементы переднего плана. Чтобы заполнить весь экран изображением или видео, измените [фон экрана](paywall-layout-and-products#screen-background).
:::

<ZoomImage id="media-templates.webp" width="200px" alt="Media element templates" float="right"/>

- **Icon**: векторная иконка из встроенной библиотеки с настраиваемыми размером и цветом
- **Image**: изображение — загрузите своё или укажите URL
- **Video**: встроенный видеоплеер для файлов размером до 50 МБ. Поддерживает зацикленное воспроизведение.
### Список \{#list\}

Элементы списка организуют контент в строки и столбцы для отображения однотипно отформатированных данных. Под капотом список — это [стек](manage-paywall-ui-elements#layout).

- **Icon List**: строки с ведущей иконкой и текстовой меткой
- **Timeline**: вертикальная последовательность с соединёнными индикаторами шагов
- **Image List**: строки с ведущим изображением и текстом
- **Icon Cards**: сетка карточек с центрированными иконками
- **Image Cards**: сетка карточек с изображениями
- **Comparison Table**: многоколоночная таблица сравнения функций по тарифам (например, Free vs Pro)
<ZoomImage id="list-templates.webp" width="700px" alt="Шаблоны элемента списка"/>
### Значок \{#badge\}

<ZoomImage id="badge-template.webp" width="200px" alt="Badge templates" float="right"/>

Небольшой наклеечный ярлык для выделения элемента — обычно используется для продвижения скидок или конкретных тарифов (например, «Скидка 5%»). Используйте [абсолютное позиционирование](manage-paywall-ui-elements#absolute), чтобы разместить значок поверх другого элемента.
### Галочки \{#checkmarks\}

Иконки выбора для использования внутри [выбираемых элементов](flow-selectable-elements). Каждый пресет галочки включает состояния «включено» и «выключено», которые автоматически обновляются в зависимости от выбора пользователя.

<ZoomImage id="checkmark-templates.webp" width="700px" alt="Checkmark templates" float="right"/>


- Checkbox
- Circle
- Radiobutton
- Toggle
## Формы и викторины \{#forms--quiz\}
### Кнопки \{#buttons\}

:::link
Основная статья: [Кнопки](paywall-buttons)
:::

Кнопки запускают действия при нажатии — переход на другой экран, открытие URL или закрытие флоу.

<ZoomImage id="button-templates.webp" width="400px" alt="Layout element templates" float="right" />

Каждый пресет — это отправная точка: изменяйте стили и назначайте любое действие. Настроить поведение кнопок можно на вкладке [Interactions](builder-ui#interactions-properties).
- **Base**: обычная кнопка с текстом по центру
- **Right Icon**: кнопка со значком справа
- **With Subtitle**: содержит две строки текста
- **Pulse Animation**: включает анимированный эффект пульсации
- **Purchase**: запускает покупку
- **Secondary Outline**: кнопка с обводкой для второстепенных действий
- **Back**: переход на предыдущий экран
- **Close flow**: выход из флоу
- **View more plans**: показывает дополнительные варианты продуктов
- **Links**: набор кнопок в подвале (Restore, Terms of Service, Privacy Policy)
### Входные поля \{#inputs\}

:::link
Основная статья: [Входные поля и формы](builder-inputs-and-forms)
:::

Входные поля позволяют пользователям вводить данные. Каждое поле применяет соответствующий метод ввода и правила валидации.

<ZoomImage id="input-templates.webp" width="400px" alt="Input templates" float="right" />

- Text
- E-mail
- Password
- Number
- Phone number
- Date
- Time
- Date & Time

Поля Date, Time и Date & Time открывают нативный выбор даты на устройстве (колёсико или календарь) при нажатии.

Используйте [переменные](onboarding-variables) для обработки пользовательского ввода и управления условной логикой.
### Квизы \{#quizzes\}

:::link
Основная статья: [Опросы и квизы](onboarding-quizzes)
:::

Элементы квиза — это экраны с выбором из нескольких вариантов: для опросов, сбора предпочтений и сегментации пользователей. Настройте [взаимодействия](onboarding-navigation-branching), чтобы разветвлять флоу в зависимости от ответа пользователя.

<ZoomImage id="quiz-templates.webp" width="400px" alt="Шаблоны квизов" float="right" />
- **Icon Options**: одноколоночный список с иконками
- **Emoji Options**: одноколоночный список с эмодзи
- **Image Options**: одноколоночный список с изображениями
- **Icon Grid**: многоколоночная сетка с иконками
- **Emoji Grid**: многоколоночная сетка с эмодзи
- **Image Grid**: многоколоночная сетка с фото
- **Rating**: числовая шкала (например, 1–5)
### Вкладки \{#tabs\}

:::link
Основная статья: [Вкладки](builder-tabs)
:::

Вкладки делят раздел экрана на переключаемые панели с контентом. Пользователь выбирает вкладку — и содержимое ниже обновляется. Чаще всего используются для группировки тарифных планов или переключения между ежемесячной и годовой ценой.

<ZoomImage id="tab-templates.webp" width="500px" alt="Шаблоны вкладок" float="right" />

- **Segment control**: переключатель в виде таблетки с закруглёнными углами вокруг выбранной вкладки
- **Button Tabs**: вкладки в виде отдельных кнопок
- **Underline**: текстовые подписи с подчёркиванием у выбранной вкладки
## Пейвол и монетизация \{#paywall--commerce\}
### Продукты \{#products\}

:::link
Основная статья: [Продукты](paywall-product-block)
:::

Элементы продуктов отображают данные о встроенных покупках и управляют выбором продукта. Каждый пресет организует данные продукта в разном расположении. Привяжите свои продукты, чтобы заполнить элементы реальными данными из Adapty.

<ZoomImage id="product-templates.webp" width="400px" alt="Шаблоны продуктов" float="right" />
- **Vertical List**: стековые карточки продуктов
- **Horizontal List**: карточки продуктов в ряд
- **Feature Carousel**: прокручиваемые карточки со списком возможностей
- **Feature Cards**: статичные карточки со списком возможностей
- **Banner List**: компактные строки со встроенными бейджами

Основная статья: [Переключатели](builder-toggles)

<ZoomImage id="toggle-template.webp" width="200px" alt="Toggle template" float="right" />

Переключатель, который меняет отображаемый продукт с обычной цены на бесплатный пробный период. При активации выбор продукта и состояние элементов обновляются автоматически.
### Вовлечённость пользователей \{#user-engagement\}

Готовые блоки с пользовательскими оценками, которые повышают доверие и стимулируют конверсию. Шаблоны — это базовые контейнеры с placeholder-контентом.

<ZoomImage id="user-engagement-templates.webp" width="400px" alt="Шаблоны вовлечённости пользователей" float="right" />

- **Review**: звёздный рейтинг с отзывом и именем автора
- **Rating**: числовая оценка со звёздами
- **App Rating**: крупная оценка со шкалой звёзд и количеством отзывов
- **Social Proof**: стопка аватаров с числом пользователей
### Обратный отсчёт \{#countdown\}

:::link
Основная статья: [Таймер обратного отсчёта](paywall-timer)
:::

<ZoomImage id="countdown-templates.webp" width="300px" alt="Шаблоны таймера обратного отсчёта" float="right" />

Отображает часы, минуты и секунды, отсчитывающие время до нуля. Используйте его, чтобы создать ощущение срочности для ограниченных по времени предложений. При достижении нуля таймер может запускать действия — например, переходить на другой экран или скрывать значок скидки.
- **Inline**: компактное числовое отображение
- **Inline with units**: числовое отображение с подписями единиц
- **Badge**: небольшой таймер-оверлей с подсветкой
- **Blocks**: отдельные карточки для дней, часов, минут и секунд
## Прогресс \{#progress\}

:::link
Основная статья: [Загрузчики и индикаторы прогресса](builder-loaders-and-progress-bars)
:::
### Индикаторы прогресса \{#progress-indicators\}

<ZoomImage id="flow-indicators.webp" width="300px" alt="Варианты индикаторов прогресса" float="right" />

Пошаговые индикаторы прогресса, показывающие, на каком экране многошагового флоу находится пользователь. Удобны для онбординг-последовательностей, где важно показать, сколько шагов осталось.

- **Linear**: единая непрерывная полоса, заполняющаяся по мере продвижения пользователя
- **Segmented**: полоса, разделённая на отдельные сегменты — по одному на каждый шаг
- **Connectors**: пронумерованные маркеры шагов, соединённые короткими линиями-коннекторами
### Загрузчики \{#loaders\}

<ZoomImage id="flow-loaders.webp" width="300px" alt="Loader variants" float="right" />

Анимированные индикаторы загрузки для переходов между экранами. Используйте загрузчики, когда приложение обрабатывает данные пользователя — например, после отправки результатов квиза.

- **Spinner**: вращающийся круговой индикатор
- **Spinner with label**: спиннер с текстовой подписью (например, «Загрузка...»)
- **Loader**: горизонтальная полоса прогресса
