Этот гайд поможет решить типичные проблемы при использовании пейволов, созданных в Adapty Paywall Builder, в Android SDK.

## Получение конфигурации пейвола завершается ошибкой \{#getting-a-paywall-configuration-fails\}

**Проблема**: Метод `getViewConfiguration` не может получить конфигурацию пейвола.

**Причина**: Пейвол не включён для отображения на устройстве в Paywall Builder.

**Решение**: Включите переключатель **Show on device** в Paywall Builder.

## Количество просмотров пейвола слишком велико \{#the-paywall-view-number-is-too-big\}

**Проблема**: Счётчик просмотров пейвола показывает вдвое больше ожидаемого значения.

**Причина**: Возможно, вы вызываете `logShowPaywall` в своём коде, что дублирует счётчик просмотров при использовании Paywall Builder. Для пейволов, созданных с помощью Paywall Builder, аналитика отслеживается автоматически — вызывать этот метод не нужно.

**Решение**: Убедитесь, что не вызываете `logShowPaywall` в своём коде при использовании Paywall Builder.

## Другие проблемы \{#other-issues\}

**Проблема**: Вы столкнулись с другими проблемами, связанными с Paywall Builder, которые не описаны выше.

**Решение**: При необходимости обновите SDK до последней версии, воспользовавшись [гайдами по миграции](android-sdk-migration-guides). Многие проблемы уже исправлены в новых версиях SDK.
