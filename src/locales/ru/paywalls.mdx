В Adapty **пейволы — единственный способ показывать и продавать продукты в приложении**. Это позволяет легко отслеживать, как разные наборы продуктов работают для разных групп пользователей, и управлять отображением каждого продукта.

Adapty предлагает два основных способа создания пейвола:
- **Использовать Adapty Paywall Builder**: вы [создаёте полноценный, готовый к покупкам пейвол в no-code конструкторе](quickstart-paywalls). Adapty автоматически его отображает и берёт на себя весь сложный процесс покупки, валидацию чеков и управление подписками.
- **Создать пейвол вручную**: при ручном создании пейвола доступны следующие варианты:
   - [Использовать Remote Config](customize-paywall-with-remote-config): вместо того чтобы хардкодить значения в коде приложения, вы можете управлять элементами пейвола (текстом, медиа и т.д.) динамически прямо из дашборда — без повторного деплоя приложения.
   - <InlineTooltip tooltip="Делегировать обработку покупок Adapty">[iOS](making-purchases), [Android](android-making-purchases), [React Native](react-native-making-purchases), [Flutter](flutter-making-purchases) и [Unity](unity-making-purchases)</InlineTooltip>: самостоятельная интеграция со Store API — крайне трудоёмкий процесс. Вместо этого можно использовать один метод в Adapty SDK, который берёт на себя всю сложную логику покупок.
