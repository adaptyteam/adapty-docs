Adapty умеет обмениваться данными со сторонними сервисами, чтобы привязывать события подписок к конкретным маркетинговым кампаниям. Это позволяет вам:

* Узнавать, какие маркетинговые стратегии приносят больше всего дохода
* Фильтровать [графики подписок](charts) в Adapty по атрибуции
* Использовать возможности стороннего сервиса для анализа данных о подписках Adapty

Настроить интеграцию можно двумя способами:

* [Встроенная атрибуция](#integrated-attribution) требует минимальной настройки и позволяет Adapty обмениваться данными с 9 популярными платформами.
* [Ручная атрибуция](#manual-attribution) требует самостоятельного получения данных атрибуции из API стороннего сервиса перед их отправкой в Adapty.

:::tip
Подключите [аналитику привлечения пользователей](user-acquisition) для полного обзора экономики вашего приложения.

Adapty UA — удобный веб-дашборд, который объединяет данные из разных источников, чтобы выявлять эффективные стратегии привлечения пользователей.
:::

:::warning
Следите за чистотой данных: избегайте дублирования событий и конфликтов атрибуции. Следуйте советам в разделе **[Предотвращение проблем с данными](#prevent-data-issues)**, чтобы новый источник данных не испортил вашу аналитику.
:::

## Встроенная атрибуция \{#integrated-attribution\}

Adapty предлагает готовую интеграцию атрибуции с 9 популярными сервисами. Эти платформы могут автоматически получать [данные о подписках](events) от Adapty, обрабатывать каждую покупку и возвращать соответствующую атрибуцию.

У каждой платформы свой процесс, но шаги везде примерно одинаково просты:

1. **Настройте автоматический обмен данными.** Авторизуйте Adapty для взаимодействия с выбранной платформой.
2. **Интегрируйте SDK Adapty.** Некоторые платформы требуют дополнительного кода для передачи данных атрибуции.
3. **Отключите другие сервисы передачи событий и источники атрибуции**, чтобы избежать [дублирования событий](#avoid-event-duplication) и [конфликтов данных](#select-a-single-attribution-source).

Подробное описание интеграции для каждой платформы читайте в соответствующем гайде:

- [Adjust](adjust)
- [Airbridge](airbridge)
- [Apple Search Ads](apple-search-ads)
- [AppsFlyer](appsflyer)
- [Asapty](asapty)
- [Branch](branch)
- [Facebook Ads](facebook-ads)
- [Singular](singular)
- [Tenjin](tenjin)

:::note
Если вы хотите, чтобы Adapty расширил этот список, [создайте запрос на функцию](https://adapty.featurebase.app/en?b=6979f233ebd3cffd4f425ba0) и выразите интерес к конкретному сервису.
:::

## Ручная атрибуция \{#manual-attribution\}

Если Adapty не предлагает [встроенную атрибуцию](#integrated-attribution) для вашего сервиса, вам нужно написать собственный код для обмена данными с источником атрибуции.

1. **Получите данные от сервиса атрибуции.** Используйте API сервиса для запроса данных атрибуции.
2. **Создайте словарь с полученными данными атрибуции.**

    Словарь может содержать следующие ключи:

    - `status` (`organic`, `non-organic` или `unknown`)
    - `channel`
    - `campaign`
    - `ad_group`
    - `ad_set`
    - `creative`

    :::important
    * Все ключи необязательны.
    * Adapty игнорирует ключи, которых нет в списке.
    * Значение каждого ключа может содержать не более 50 символов.
    :::

    **Пример**:

    ```swift showLineNumbers title="Swift"
    let attribution = [
        "status": "non_organic",
        "channel": "Google Ads",
        "campaign": "Christmas Sale",
        "ad_group": "ad group 1",
        "ad_set": "ad set 1",
        "creative": "creative id 1"
    ]
    ```

3. **Установите данные атрибуции**:

    Передайте словарь атрибуции в метод `updateAttribution`. После того как значение атрибуции установлено, переопределить его нельзя:

    ```swift showLineNumbers title="Swift"
    Adapty.updateAttribution(attribution, source: "custom") { error in
        if error == nil {
            // successful attribution update
        }
    }
    ```

    **Параметры:**

    - `attribution` (обязательный): словарь с данными атрибуции.
    - `source` (обязательный): источник атрибуции. Установите `.custom`, если ваш провайдер атрибуции не поддерживает [встроенную атрибуцию](#integrated-attribution).

4. **Отключите другие сервисы передачи событий и источники атрибуции**, чтобы избежать [дублирования событий](#avoid-event-duplication) и [конфликтов данных](#select-a-single-attribution-source).

## Предотвращение проблем с данными \{#prevent-data-issues\}

### Используйте единственный источник атрибуции \{#select-a-single-attribution-source\}

Не включайте интеграцию атрибуции с несколькими платформами одновременно. Adapty принимает только один источник атрибуции за раз, и после сохранения значения атрибуции переопределить его невозможно.

Если вы включите несколько источников атрибуции, Adapty выберет источник с наибольшим количеством данных — но не обязательно с наилучшими.

Например, неорганическая [атрибуция Apple Search Ads](apple-search-ads) всегда будет иметь приоритет на iOS. Чтобы отключить атрибуцию Apple Search Ads, откройте [**App Settings** -> вкладку **Apple Search Ads**](https://app.adapty.io/settings/apple-search-ads) и выключите переключатель **Receive Apple Search Ads attribution**.

### Избегайте дублирования событий \{#avoid-event-duplication\}

Если вы используете Adapty для передачи данных о подписках в реальном времени в сервисы атрибуции, **необходимо отключить** другие сервисы, выполняющие ту же функцию. Если вы подключили аккаунт Facebook к AppsFlyer, Adjust или Branch, эти сервисы будут автоматически пересылать ваши события, если вы явно не откажетесь от этого.

Дублирование событий может исказить аналитику и затруднить интерпретацию данных. После настройки передачи событий через Adapty отключите функцию пересылки событий в сторонних сервисах.
