Adapty может обмениваться информацией со сторонними сервисами, чтобы привязывать события подписки к конкретным маркетинговым кампаниям. Это позволяет:

* Выяснить, какие маркетинговые стратегии приносят больше всего дохода
* Фильтровать [графики подписок](charts) Adapty по атрибуции
* Использовать возможности стороннего сервиса для анализа данных подписок Adapty

Настроить это можно одним из двух способов:
* [Интегрированная атрибуция](#integrated-attribution) требует минимальной настройки и позволяет Adapty обмениваться данными с 9 популярными платформами.
* [Ручная атрибуция](#manual-attribution) требует самостоятельного получения данных атрибуции из API сторонних сервисов перед отправкой в Adapty.

:::tip
Включите [Adapty Attribution](user-acquisition) для полного обзора экономики вашего приложения.

Adapty Attribution — это простой в настройке веб-дашборд, который консолидирует данные из разных источников для выявления эффективных стратегий привлечения пользователей.
:::
:::warning
Следите за чистотой данных: избегайте дублирования событий и конфликтов атрибуции. Следуйте рекомендациям из раздела **[Предотвращение проблем с данными](#prevent-data-issues)**, чтобы новый источник данных не засорял вашу аналитику.
:::
## Интегрированная атрибуция \{#integrated-attribution\}

Adapty предлагает готовую интеграцию атрибуции с 9 популярными сервисами. Эти платформы могут автоматически получать [данные о подписках](events) от Adapty, обрабатывать каждую покупку и возвращать соответствующую атрибуцию.

У каждой платформы свой процесс, но шаги везде примерно одинаково просты:
1. **Настройте автоматический обмен данными.** Авторизуйте Adapty для работы с выбранной платформой.
2. **Интегрируйте SDK.** Некоторые платформы требуют дополнительного кода для передачи данных атрибуции.
3. **Отключите другие сервисы отправки событий и источники атрибуции**, чтобы избежать [дублирования событий](#avoid-event-duplication) и [конфликтов данных](#select-a-single-attribution-source).

Ознакомьтесь с гайдом по конкретной платформе для подробного обзора интеграции:
- [Adjust](adjust)
- [Airbridge](airbridge)
- [Apple Search Ads](apple-search-ads)
- [AppsFlyer](appsflyer)
- [Asapty](asapty)
- [Branch](branch)
- [Facebook Ads](facebook-ads)
- [Singular](singular)
- [Tenjin](tenjin)

:::note
Если вы хотите, чтобы Adapty расширил этот список, [создайте запрос на добавление функции](https://adapty.featurebase.app/en?b=6979f233ebd3cffd4f425ba0) и выразите интерес к конкретному сервису.
:::
## Ручная атрибуция \{#manual-attribution\}

Если Adapty не предлагает [интегрированную атрибуцию](#integrated-attribution) с нужным вам сервисом, вам потребуется написать собственный код для обмена данными с источником атрибуции.

1. **Получите данные от сервиса атрибуции**. Используйте API сервиса для запроса данных атрибуции.
2. **Создайте словарь с полученными данными атрибуции.**

    Словарь может содержать следующие ключи:

    - `status` (`organic`, `non-organic` или `unknown`)
    - `channel`
    - `campaign`
    - `ad_group`
    - `ad_set`
    - `creative`
:::important
    * Все ключи необязательны.
    * Adapty игнорирует ключи, которых нет в списке.
    * Значение каждого ключа может содержать не более 50 символов.
    :::

    **Пример**:

    ```swift showLineNumbers title="Swift"
    let attribution = [
        "status": "non_organic",
        "channel": "Google Ads",
        "campaign": "Christmas Sale",
        "ad_group": "ad group 1",
        "ad_set": "ad set 1",
        "creative": "creative id 1"
    ]
    ```

3. **Установите данные атрибуции**:
Передайте словарь атрибуции в метод `updateAttribution`. После того как значение атрибуции установлено, его нельзя переопределить:

    ```swift showLineNumbers title="Swift"
    Adapty.updateAttribution(attribution, source: "custom") { error in
        if error == nil {
            // successful attribution update
        }
    }
    ```

    **Параметры:**

    - `attribution` (обязательный): словарь с данными атрибуции.
    - `source` (обязательный): источник атрибуции. Установите значение `.custom`, если ваш провайдер атрибуции не поддерживает [интегрированную атрибуцию](#integrated-attribution).
4. **Отключите другие сервисы передачи событий и источники атрибуции**, чтобы избежать [дублирования событий](#avoid-event-duplication) и [конфликтов данных](#select-a-single-attribution-source).
## Предотвращение проблем с данными \{#prevent-data-issues\}

### Выберите один источник атрибуции \{#select-a-single-attribution-source\}

Не подключайте интеграцию атрибуции сразу с несколькими платформами. Adapty принимает только один источник атрибуции одновременно, и после сохранения значения атрибуции перезаписать его невозможно.

Если подключено несколько источников атрибуции, Adapty выберет тот, у которого больше данных — но не обязательно лучших.
Например, неорганическая [атрибуция Apple Search Ads](apple-search-ads) всегда имеет приоритет на iOS. Чтобы отключить атрибуцию Apple Search Ads, откройте [**App Settings** -> вкладку **Apple Search Ads**](https://app.adapty.io/settings/apple-search-ads) и переключите тумблер **Receive Apple Search Ads attribution**.
### Избегайте дублирования событий \{#avoid-event-duplication\}

Если вы используете Adapty для передачи данных о подписках в реальном времени в сервисы атрибуции, **необходимо отключить** другие сервисы, выполняющие ту же функцию. Если вы подключили аккаунт Facebook к AppsFlyer, Adjust или Branch, эти сервисы будут автоматически пересылать ваши события, если вы явно не откажетесь от этого.

Дублирование событий может исказить аналитику и затруднить интерпретацию данных. После настройки передачи событий через Adapty отключите функцию пересылки событий в сторонних сервисах.
