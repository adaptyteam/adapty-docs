Интеграция Adapty UA с Google Ads работает в обе стороны: Adapty получает метрики эффективности рекламы из Google и отправляет обратно события конверсий — так у алгоритмов умного назначения ставок появляются реальные данные о доходах для оптимизации.

:::note
Adapty UA поддерживает только **App Campaigns**. Кампании типа Performance Max, Search, Display, Shopping и другие в Adapty UA не отображаются.
:::

## Как это работает \{#how-it-works\}

После подключения аккаунта Google Adapty синхронизирует иерархию аккаунтов, кампании и группы объявлений. Дальше:

- **Метрики поступают в Adapty**: Adapty каждые 2 часа забирает из Google данные о расходах, показах, кликах и конверсиях, и они становятся доступны в аналитике UA.
- **События уходят в Google**: когда пользователь устанавливает приложение или совершает покупку, Adapty отправляет это событие в Google, используя настроенный вами **App Link ID**. Google использует эти события для атрибуции и оптимизации умного назначения ставок.

:::note
Разбивка по странам доступна на уровне кампании и группы объявлений. На уровне объявления и ключевого слова данные по странам рассчитываются пропорционально и могут быть приблизительными.
:::

App Link ID — уникальный идентификатор, который Google генерирует для вашего приложения, когда вы добавляете Adapty в качестве партнёра по измерению конверсий. Вы вводите его в Adapty для установки связи.

## Шаг 1. Подключите аккаунт Google \{#step-1-connect-your-google-account\}

1. В Adapty UA перейдите в **Integrations > Google Ads** на боковой панели и нажмите **Continue with Google**.

<ZoomImage id="ua-sign-in-google-ads.webp" width="700px" />

2. Войдите в аккаунт Google и предоставьте запрошенные разрешения.

После этого Adapty синхронизирует все рекламные аккаунты, доступные из вашего аккаунта Google, включая иерархии управляющих аккаунтов (MCC).

## Шаг 2. Получите App Link ID из Google Ads \{#step-2-get-your-app-link-ids-from-google-ads\}

Нужно сгенерировать link ID для каждой платформы (iOS и Android) в Google Ads. Link ID используется Adapty для отправки событий конверсий в нужное приложение.

1. В Google Ads нажмите значок **Tools** и выберите **Data manager**.
2. Нажмите **+ Connect Product**, выберите **Third-party app analytics** и нажмите **Continue**.
3. Нажмите **Create link ID**.
4. Выберите **Other provider** в списке провайдеров и вставьте **Provider ID** со страницы настройки интеграции в Adapty UA.
5. Выберите **iOS** в качестве платформы и найдите своё приложение через поле **Look up your app**.
6. Нажмите **Create link ID** и скопируйте сгенерированный link ID — это ваш **iOS App Link ID**.
7. Повторите шаги 3–6 для **Android**, чтобы получить **Android App Link ID**.

:::note
Статус вашего link ID будет отображаться как **Unverified**, пока вы не добавите его в Adapty и данные о конверсиях не начнут поступать.
:::

## Шаг 3. Создайте app-кампанию в Adapty UA \{#step-3-create-an-app-campaign-in-adapty-ua\}

1. В Adapty UA перейдите в **Integrations > Google Ads > App campaigns**.
2. Нажмите **Create app campaign**. Adapty автоматически создаст кампанию для вашего текущего приложения.

<ZoomImage id="google-ads-create-ad-campaign.webp" width="700px" />

3. На странице настроек кампании вставьте **iOS App Link ID** и **Android App Link ID**, скопированные из Google Ads.

<ZoomImage id="app-link-id.webp" width="700px" />

4. Нажмите **Save**.

## Шаг 4. Настройте маппинг событий \{#step-4-map-events\}

В разделе **Events names** включите события, которые хотите отправлять в Google. Для каждого включённого события введите или подтвердите соответствующее название события Google.

Adapty отправляет эти события в Google через App Conversion API — установки, запуски триала, покупки, продления и другие.

Нажмите **Save**.

<ZoomImage id="ua-google-ads-events.webp" width="700px" />

## Дополнительная настройка \{#additional-configuration\}

### Переопределение дохода \{#revenue-override\}

Если вы отслеживаете события триала и хотите, чтобы Google атрибутировал им доход, используйте раздел **Revenue override**. Он появляется, когда включено событие **Trial started**.

Для каждой целевой метрики триала задайте процент от цены подписки, который будет передаваться как доход. Например, при 30% Adapty отправит 30% от цены подписки в качестве значения конверсии для событий триала.

Чтобы добавить переопределение:

1. Включите **Trial started** в разделе **Events names**.
2. В **Revenue override** нажмите **Add override**.
3. Выберите целевое событие и введите процент дохода (0–100).
4. Нажмите **Save**.

<ZoomImage id="revenue-override.webp" width="500px" />

### Проверка интеграции \{#test-the-integration\}

Чтобы убедиться в корректности настройки до запуска кампаний, нажмите **Send test event** на странице настроек кампании и выберите тип события. Adapty отправит тестовое событие конверсии в Google, используя настроенные App Link ID, и покажет результат.
