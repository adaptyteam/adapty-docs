Server-Side API v2 от Adapty предоставляет новые возможности и улучшения для более эффективного управления уровнями доступа, транзакциями и профилями пользователей.

## Зачем переходить на v2? \{#why-migrate\}

Вторая версия Server-Side API даёт больше гибкости и функциональности при работе с Adapty:

- **Раздельное управление уровнями доступа**: назначайте уровни доступа пользователям без необходимости указывать детали транзакции — это упрощает обработку компенсаций, бонусов и других нетранзакционных сценариев.
- **Запись разовых покупок**: фиксируйте транзакции для расходуемых покупок, указывая поля, специфичные для таких продуктов.
- **Расширенные сведения о транзакциях:** добавляйте к транзакциям больше данных — возвраты, проблемы с оплатой, причины отмены, продления и т. д.
- **Обновление профиля:** теперь можно не просто добавлять атрибуты, но и обновлять профиль пользователя целиком — например, добавлять метаданные установки или отключать внешнюю аналитику.

Версия 1 по-прежнему поддерживается, однако мы рекомендуем перейти на v2 ради расширенного функционала и дальнейших обновлений.

## Обзор изменений \{#changes-overview\}

| Изменение                              | Необходимые действия                                              |
| ----------------------------------- | ------------------------------------------------------------ |
| **Базовый URL и эндпоинт**           | Базовый URL и все эндпоинты изменились. Обновите конфигурацию согласно описанию эндпоинтов запросов. |
| **Заголовки запросов**                 | <ol><li> Добавьте заголовок `adapty-profile-id` или `adapty-customer-user-id`.</li><li> Добавьте новый заголовок `adapty-platform`.</li></ol> |
| **Структура запросов и ответов**  | Измените параметры в соответствии с описанием каждого запроса и обновите интеграцию для обработки [новых форматов ответов](api-responses). |
| **Изменение управления уровнями доступа** | Старый запрос [Prolong/grant a subscription for a user](server-side-api-specs-legacy#prolonggrant-a-subscription-for-a-user) теперь разделён на три: <ul><li> [Set Transaction](api-adapty/operations/setTransaction): добавление деталей транзакции с предоставлением доступа.</li><li> [Grant Access Level](api-adapty/operations/grantAccessLevel): предоставление или продление доступа без транзакции.</li><li> [Revoke Access Level](api-adapty/operations/revokeAccessLevel): сокращение или отзыв доступа без транзакции.</li></ul> |

### Шаг 1. Настройте заголовки запросов \{#step-1-configure-request-headers\}

Добавьте новые заголовки запросов:

| **Заголовок**                  | **Описание**                                              |
| --------------------------- | ------------------------------------------------------------ |
| **adapty-profile-id**       | (Обязательно, выберите один) Adapty ID профиля пользователя. Отображается в поле **Adapty ID** на странице [дашборд Adapty -> **Profiles**](https://app.adapty.io/profiles/users) -> конкретного профиля. Взаимозаменяем с **adapty-customer-user-id** — используйте любой из них. |
| **adapty-customer-user-id** | <p>(Обязательно, выберите один) ID пользователя в вашей системе. Отображается в поле **Customer user ID** на странице [дашборд Adapty -> **Profiles**](https://app.adapty.io/profiles/users) -> конкретного профиля. Взаимозаменяем с **adapty-profile-id** — используйте любой из них.</p><p>⚠️ Работает только если вы [идентифицируете пользователей](identifying-users) в коде приложения с помощью SDK Adapty.</p> |
| **adapty-platform**         | Укажите платформу приложения. Допустимые значения: `iOS`, `macOS`, `iPadOS`, `visionOS`, `Android`. |

### Шаг 2. Измените запросы управления транзакциями и доступом \{#step-2-change-transaction-and-access-management-requests\}

В версии 1 использовались:

- Запрос [Prolong/Grant a Subscription for a User](server-side-api-specs-legacy#prolonggrant-a-subscription-for-a-user): для записи транзакции и предоставления или сокращения уровня доступа.
- Запрос [Revoke access level](server-side-api-specs-legacy#revoke-subscription-from-a-user): для немедленного отзыва доступа.

Теперь они заменены тремя отдельными запросами, разграничивающими добавление транзакций и управление уровнями доступа:

1. **[Grant Access Level](api-adapty/operations/grantAccessLevel):** используйте этот запрос для продления уровня доступа без привязки к транзакции.
2. **[Revoke Access Level](api-adapty/operations/revokeAccessLevel):** для немедленного отзыва или сокращения доступа.
3. **[Set Transaction](api-adapty/operations/setTransaction):** используйте этот запрос для добавления деталей транзакции в Adapty с предоставлением уровней доступа.

#### Шаг 2.1. Как предоставить уровень доступа \{#step-21-how-to-grant-access-level\}

:::info

Подробное описание см. в запросе [Grant access level](api-adapty/operations/grantAccessLevel).

:::

В версии 1 для предоставления доступа использовался запрос [Prolong/grant a subscription for a user](server-side-api-specs-legacy#prolonggrant-a-subscription-for-a-user). Теперь вы можете предоставить доступ с помощью запроса [Grant access level](api-adapty/operations/grantAccessLevel) без указания деталей транзакции.

- **Эндпоинт:** `https://api.adapty.io/api/v2/server-side-api/grant/access-level/`
- Параметры, которые остаются:
  - **access_level_id**: ранее передавался в эндпоинте. Обязательный.
  - **starts_at**: теперь допускает null.
  - **expires_at**: необязательный для пожизненного доступа, допускает null.

#### Шаг 2.2. Как отозвать или сократить уровень доступа \{#step-22-how-to-revoke-or-shorten-access-level\}

:::info

Подробное описание см. в запросе [Revoke access level](api-adapty/operations/revokeAccessLevel).

:::

В версии 1 для немедленного отзыва доступа использовался запрос [Revoke access level](server-side-api-specs-legacy#revoke-subscription-from-a-user), а для сокращения — запрос [Prolong/Grant a Subscription for a User](server-side-api-specs-legacy#prolonggrant-a-subscription-for-a-user). Теперь для обоих действий можно использовать запрос [Revoke access level](api-adapty/operations/revokeAccessLevel).

- **Эндпоинт:** `https://api.adapty.io/api/v2/server-side-api/purchase/profile/revoke/access-level/`
- Параметры, которые остаются:
  - **access_level_id**: обязательный.
  - **expires_at**: допускает null, если доступ не отзывается немедленно.

:::info

Измените запрос следующим образом:

- **Эндпоинт:** `https://api.adapty.io/api/v2/server-side-api/profile/`
