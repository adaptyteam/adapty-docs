Возвращает данные о конверсии для анализа действий пользователей и оценки эффективности маркетинговых усилий с течением времени.

## Эндпоинт и метод \{#endpoint-and-method\}

```http
POST https://api-admin.adapty.io/api/v1/client-api/metrics/conversion/
```

## Параметры \{#parameters\}

| Название     | Тип                                      | Обязательный       | Описание                                                     |
| ------------ | ---------------------------------------- | ------------------ | ------------------------------------------------------------ |
| filters      | [MetricsFilters](#metricsfilters-object) | :heavy_plus_sign:  | Объект с параметрами фильтрации. Подробности — в таблице ниже. |
| from_period  | String/null                              | :heavy_plus_sign:  | Начальное состояние подписки пользователя до конверсии (например, `null` = установка, `0` = пробный период, `1` = первый платёжный период и т.д.). Допустимые значения см. в разделе [Типы конверсий](#conversion-types). |
| to_period    | String                                   | :heavy_plus_sign:  | Новое состояние подписки пользователя после конверсии (например, `0` = пробный период, `1` = первый платёжный период, `6+` = после шести месяцев и т.д.). Допустимые значения см. в разделе [Типы конверсий](#conversion-types). |
| period_unit  | String                                   | :heavy_minus_sign: | <p>Временной интервал для агрегации аналитических данных — позволяет просматривать результаты, сгруппированные по выбранным периодам: дням, неделям, месяцам и т.д. Возможные значения:</p><ul><li>day</li><li>week</li><li>month (по умолчанию)</li><li>quarter</li><li>year</li></ul> |
| date_type    | String                                   | :heavy_minus_sign: | Определяет, какая дата считается датой присоединения пользователя. Возможные значения:<ul><li>purchase_date (по умолчанию)</li><li>profile_install_date</li></ul> |
| segmentation | String                                   | :heavy_minus_sign: | <p>Устанавливает основу для сегментации. Возможные значения:</p><ul><li>app_id</li><li>period</li><li>renewal_status</li><li>cancellation_reason</li><li>store_product_id</li><li>country</li><li>store</li><li>purchase_container_id</li><li>paywall_id</li><li>audience_id</li><li>placement_id</li><li>attribution_source</li><li>attribution_status</li><li>attribution_channel</li><li>attribution_campaign</li><li>attribution_adgroup</li><li>attribution_adset</li><li>attribution_creative</li><li>duration</li><li>default</li></ul> |
| format       | String                                   | :heavy_minus_sign: | <p>Формат экспортируемого файла. Доступные варианты:</p><ul><li>json</li><li>csv</li></ul> |

## Типы конверсий \{#conversion-types\}

Используйте `from_period` и `to_period` вместе, чтобы указать нужный тип конверсии для анализа. Доступны только переходы, поддерживаемые дашбордом Adapty:

| Конверсия                                                    | from_period | to_period |
| ------------------------------------------------------------ | ----------- | --------- |
| **Установка → Пробный период**<p>Пользователь только что установил приложение (без подписки) и начал бесплатный пробный период.</p> | null        | 0         |
| **Установка → Платная подписка**<p>Пользователь установил приложение и сразу перешёл на платную подписку.</p>        | null        | 1         |
| **Пробный период → Платная подписка**<p>Пользователь перешёл с бесплатного пробного периода на платную подписку.</p> | 0           | 1         |
| **Платная → 2-й период**<p>Пользователь продлил подписку с первого платёжного периода на второй.</p>                 | 1           | 2         |
| **2-й → 3-й период**<p>Пользователь продлил подписку со второго платёжного периода на третий.</p>                    | 2           | 3         |
| **3-й → 4-й период**<p>Пользователь продлил подписку с третьего платёжного периода на четвёртый.</p>                 | 3           | 4         |
| **4-й → 5-й период**<p>Пользователь продлил подписку с четвёртого платёжного периода на пятый.</p>                   | 4           | 5         |
| **Платная → 6 месяцев+**<p>Пользователь оставался на платной подписке шесть месяцев и дольше.</p>                    | 1           | "6+"      |
| **Платная → 1 год+**<p>Пользователь оставался на платной подписке год и дольше.</p>                                  | 1           | "12+"     |
| **Платная → 2 года+**<p>Пользователь оставался на платной подписке два года и дольше.</p>                            | 1           | "24+"     |
