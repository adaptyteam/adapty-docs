Этот гайд поможет решить распространённые проблемы при использовании пейволов, созданных в Adapty Paywall Builder, в React Native SDK.

## Не удаётся получить конфигурацию пейвола \{#getting-a-paywall-configuration-fails\}

**Проблема**: Метод `getPaywallConfiguration` не может получить конфигурацию пейвола.

**Причина**: Пейвол не включён для отображения на устройстве в Paywall Builder.

**Решение**: Включите переключатель **Show on device** в Paywall Builder.

## Слишком большое количество просмотров пейвола \{#the-paywall-view-number-is-too-big\}

**Проблема**: Счётчик просмотров пейвола показывает вдвое больше ожидаемого.

**Причина**: Возможно, в вашем коде вызывается `logShowPaywall`, что дублирует счётчик просмотров при использовании Paywall Builder. Для пейволов, созданных с помощью Paywall Builder, аналитика отслеживается автоматически — использовать этот метод не нужно.

**Решение**: Убедитесь, что `logShowPaywall` не вызывается в вашем коде при использовании Paywall Builder.

## Другие проблемы \{#other-issues\}

**Проблема**: Вы столкнулись с другими проблемами, связанными с Paywall Builder, которые не описаны выше.

**Решение**: При необходимости обновите SDK до последней версии с помощью [гайдов по миграции](react-native-sdk-migration-guides). Многие проблемы уже решены в новых версиях SDK.
