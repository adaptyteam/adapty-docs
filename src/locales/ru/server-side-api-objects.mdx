API Adapty использует JSON-объекты — они помогают понять структуру ответа и интегрировать её в ваш код.

Все значения дат и времени соответствуют формату [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601), например: "2020-01-15T15:10:36.517975+0000".

## Уровень доступа \{#access-level\}

Информация об [уровне доступа](access-level) пользователя.

Уровни доступа позволяют управлять возможностями пользователей в приложении без хардкода конкретных ID продуктов. Каждый продукт определяет, на какой срок пользователь получает определённый уровень доступа. При покупке Adapty предоставляет доступ к приложению на конкретный период (для подписок) или навсегда (для покупок с пожизненным доступом). Кроме того, вы можете [выдать конкретный доступ](api-adapty/operations/grantAccessLevel) пользователю на заданный срок через серверный API.

Доступные действия через серверный API Adapty:

- [Проверить уровень доступа пользователя](api-adapty/operations/getProfile) — получить данные его профиля
- [Выдать конкретный доступ](api-adapty/operations/grantAccessLevel) пользователю без привязки к транзакции
- [Установить транзакцию и выдать уровень доступа](api-adapty/operations/setTransaction) пользователю
- [Отозвать уровень доступа](api-adapty/operations/revokeAccessLevel) у пользователя

<AccessLevel />

:::note

SDK содержит параметр `is_active` для проверки активности подписки, однако серверный API этот параметр не возвращает. Тем не менее статус подписки можно определить в любой момент, проверив, попадает ли текущая дата в диапазон между параметрами `starts_at` и `expires_at`.

:::

## Non Subscription \{#non-subscription\}

Информация о покупках без подписки. Это могут быть разовые (расходуемые) продукты, разблокировки (например, новая карта в игре) и т. д.

Доступные действия через серверный API Adapty:

- [Проверить текущие покупки без подписки пользователя](api-adapty/operations/getProfile) — получить данные его профиля

| Параметр                      | Тип           | Обязательный | Nullable | Описание                                                     |
| :---------------------------- | :------------ | ------------ | -------- | :----------------------------------------------------------- |
| purchase_id                   | String        | Да           | Нет      | Идентификатор покупки в Adapty. Можно использовать, чтобы убедиться, что эта покупка уже обработана, — например, при отслеживании разовых продуктов. |
| store                         | String        | Да           | Нет      | Стор, в котором был куплен продукт. Возможные значения: **app_store**, **play_store**, **stripe**, название вашего [кастомного стора](custom-store). |
| store_product_id              | String        | Да           | Нет      | Идентификатор продукта в сторе (App Store / Google Play / Stripe и т. д.), открывшего данный уровень доступа. |
| store_base_plan_id            | String        | Да           | Да       | [Идентификатор базового плана](https://support.google.com/googleplay/android-developer/answer/12154973) в Google Play Store или [идентификатор цены](https://docs.stripe.com/products-prices/how-products-and-prices-work#use-products-and-prices) в Stripe. |
| store_transaction_id          | String        | Да           | Нет      | Идентификатор транзакции в сторе (App Store / Google Play / Stripe и т. д.). |
| store_original_transaction_id | String        | Да           | Нет      | <p>При продлении подписок формируется цепочка транзакций. Исходная транзакция — самая первая в этой цепочке, и все остальные транзакции связаны с ней. Остальные транзакции в цепочке являются продлениями.</p><p>Если продления не было, `store_original_transaction_id` совпадёт с `store_transaction_id`.</p> |
| purchased_at                  | ISO 8601 date | Да           | Нет      | Дата и время последней покупки уровня доступа.               |
| environment                   | String        | Нет          | Нет      | Среда транзакции, предоставившей уровень доступа. Возможные значения: `Sandbox`, `Production`. |
| is_refund                     | Boolean       | Да           | Нет      | Указывает, был ли продукт возвращён.                         |
| is_consumable                 | Boolean       | Да           | Нет      | Указывает, является ли продукт расходуемым.                  |
