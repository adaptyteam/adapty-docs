Чтобы ваше приложение не блокировалось в Китае и работало эффективно, Adapty предоставляет опцию китайского кластера. Это обеспечивает быстрый и надёжный сервис для пользователей на материковом Китае и помогает соответствовать местным требованиям.

Великий китайский файрвол может существенно влиять на подключение и производительность приложений, использующих серверы за пределами страны.

Однако китайская инфраструктура Adapty позволяет вашему приложению стабильно и надёжно работать для пользователей на материковом Китае.

:::important

В Китае приложениям требуется явное разрешение пользователя на доступ в интернет. До получения этого разрешения **никакие сетевые запросы не будут работать**. Убедитесь, что ваше приложение:
- Запрашивает разрешение на доступ в интернет в нужный момент.
- Обрабатывает случаи, когда разрешение отклонено или ещё не предоставлено.
- Понятно объясняет пользователю, зачем нужен доступ в интернет.
:::

## Шаг 1. Настройте SDK Adapty для региона Китай \{#step-1-configure-adapty-sdk-for-china-region\}

## Шаг 2. Определите, когда использовать серверы в Китае \{#step-2-detect-when-to-use-china-servers\}

Важно динамически выбирать между глобальными и китайскими серверами в зависимости от местоположения пользователя. Вот два подхода:

### Вариант 1: Определение по региону/стране \{#option-1-detect-by-regioncountry\}

<Tabs groupId="current-os" queryString>
<TabItem value="swift" label="iOS">

```swift showLineNumbers
func shouldUseChinaServers() -> Bool {
  Locale.current.regionCode == "CN"
}

// In your configuration
let configurationBuilder = AdaptyConfiguration
    .Builder(withAPIKey: "PUBLIC_SDK_KEY")

if shouldUseChinaServers() {
    configurationBuilder.with(serverCluster: .cn)
}
// other configuration options
```

</TabItem>

<TabItem value="react-native" label="React Native">

```kotlin
// Define the helper function
private fun shouldUseChinaServers() = Locale.getDefault().country == "CN"

// Use it in configuration
private fun setupAdapty() {
    val serverCluster = if (shouldUseChinaServers())
        AdaptyConfig.ServerCluster.CN
    else
        AdaptyConfig.ServerCluster.DEFAULT

    Adapty.activate(
        context,
        AdaptyConfig.Builder("PUBLIC_SDK_KEY")
            // other configuration options
            .withServerCluster(serverCluster)
            .build()
    )
}
```

### Вариант 2: Определение по стору \{#option-2-detect-by-app-store\}

Для приложений, распространяемых через разные сторы, можно определять нужный сервер по источнику установки:

<Tabs groupId="current-os" queryString>
<TabItem value="swift" label="iOS">

```swift showLineNumbers

func shouldUseChinaServers() async -> Bool {
    let code: String?
    
    if #available(iOS 15.0, *) {
        code = await Storefront.current?.countryCode
    } else {
        code = SKPaymentQueue.default().storefront?.countryCode
    }
    
    return code == "CHN"
}

// In your configuration
let configurationBuilder = AdaptyConfiguration
    .Builder(withAPIKey: "PUBLIC_SDK_KEY")

// Only set server cluster if it's China
if await shouldUseChinaServers() {
    configurationBuilder.with(serverCluster: .cn)
}
// other configuration options
```

</TabItem>
