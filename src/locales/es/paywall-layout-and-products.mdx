Un flow está compuesto por una o más pantallas. Cada pantalla representa un paso en el recorrido del usuario — por ejemplo, un paywall, un cuestionario o una diapositiva con información del producto.

Los elementos de cada pantalla se organizan en una jerarquía de capas.
Para gestionar tus pantallas, capas y elementos, abre la vista predeterminada **Screens and Layers**. Aquí verás la secuencia de pantallas y la estructura de capas de cada una.
## Gestionar pantallas \{#manage-screens\}

<ZoomImage id="flow-builder-screens-and-layers-left-sidebar.webp" width="400px" alt="Flow builder screens and layers view (both panels visible)" float="left" />

La sección superior del panel izquierdo muestra todas las pantallas del flow. Cada entrada incluye una etiqueta numerada y una miniatura de vista previa.
* **Seleccionar una pantalla**: Haz clic en una pantalla para activarla. El editor visual muestra la pantalla seleccionada y la sección Layers se actualiza para mostrar su jerarquía de capas.
* **Añadir una pantalla**: Haz clic en el botón <Inline id="plus.svg" alt="Plus" /> en la parte superior de la sección Screens para añadir una nueva pantalla vacía al flow.
* **Abrir la biblioteca de plantillas**: Haz clic en el botón <Inline id="template-library.svg" alt="Templates" /> en la parte superior de la sección Screens para explorar y aplicar [plantillas de flow](paywall-builder-templates).
* **Reordenar pantallas**: Arrastra y suelta las pantallas para cambiar su orden en el flow.
:::important
Si tu flow tiene pantallas vacías sin usar, no podrás publicarlo. Elimina las pantallas en borrador antes de publicar.
:::
### Acciones de pantalla \{#screen-actions\}

Haz clic en el icono de tres puntos <Inline id="context.svg" alt="Context" /> en una entrada de pantalla para abrir el menú contextual.

<ZoomImage id="flow-builder-screen-actions.webp" width="600px" alt="Flow builder screen context menu" />
| Acción | Atajo | Descripción |
|--------|-------|-------------|
| **Play Animation** | | Previsualiza las animaciones configuradas en esta pantalla |
| **Copy** | ⌘C / Ctrl+C | Copia la pantalla al portapapeles |
| **Paste here** | ⌘V / Ctrl+V | Pega una pantalla copiada anteriormente |
| **Duplicate** | ⌘D / Ctrl+D | Crea una copia de la pantalla y la añade al flow |
| **Rename** | | Cambia el nombre visible de la pantalla |
| **Delete** | ⌘⌫ / Ctrl+Del | Elimina la pantalla del flow |

:::tip
El portapapeles se mantiene entre flows. Copia una pantalla o elemento de un flow, abre otro y pégalo ahí.
:::
:::warning
Cuando eliminas una pantalla, cualquier acción [Navegar a la pantalla](onboarding-navigation-branching) que apuntaba a ella **pierde su destino**, pero la propia acción **no se elimina**. Asigna un nuevo destino o elimina la acción; de lo contrario, no podrás [previsualizar ni publicar el flow](builder-save-publish#publish-a-flow).
:::

## Configuración de la pantalla \{#screen-settings\}

Para ver las propiedades y ajustes de la pantalla activa, haz clic en un área vacía de la vista previa de la pantalla. El panel derecho cambiará a la vista de configuración de la pantalla.
### IU del sistema \{#system-ui\}

<ZoomImage id="builder-screen-settings-system-ui.webp" width="350px" alt="Screen settings — System UI" float="right" />

Controla cómo interactúa la pantalla con el hardware del dispositivo.

* **Safe area** añade relleno para que el contenido no quede tapado por el notch ni las barras del sistema.
* **Status bar** muestra u oculta la barra de estado del sistema (hora, batería, iconos de señal).
### Incluir pantalla en el indicador de progreso \{#include-screen-in-progress-indicator\}

Si añades un elemento [Indicador de progreso](builder-loaders-and-progress-bars#progress-indicators) a tu flow, Adapty lo mostrará en todas las pantallas.

Desmarca **Include screen in progress indicator** para quitar el indicador de progreso de una pantalla concreta. Úsalo para limpiar las pantallas de bienvenida, el paywall final o cualquier paso que no quieras registrar como progreso.
### Diseño de pantalla \{#screen-layout\}

:::link
Artículo completo: [Posicionamiento de elementos](manage-paywall-ui-elements)
:::

<ZoomImage id="builder-screen-settings-layout.webp" width="350px" alt="Ajustes de pantalla — Diseño" float="right" />

La sección **Layout** determina cómo la pantalla distribuye sus elementos hijo. Estas propiedades están disponibles en cualquier elemento contenedor.

* **Free**: Los elementos hijo se posicionan de forma independiente.
* **Vertical**: Los elementos se organizan de arriba hacia abajo, como una columna flexbox.
* **Horizontal**: Los elementos se organizan de izquierda a derecha, como una fila flexbox.
Para los layouts vertical y horizontal, también puedes configurar el espaciado y la alineación.

* **Alignment**: Posición del elemento a lo largo del eje transversal.
* **Gap**: Espacio entre elementos adyacentes.
* **Distribution**: Distribución del espacio entre los elementos hijos y alrededor de ellos.

#### Layout RTL \{#rtl-layout\}

Activa la casilla **Mirror for RTL** para invertir el layout en sistemas de escritura que se leen de derecha a izquierda. El orden de los elementos en los contenedores horizontales se invertirá.
### Fondo de pantalla \{#screen-background\}

:::link
Artículo principal: [Fondos](paywall-head-picture)
:::

<ZoomImage id="builder-screen-settings-background.webp" width="350px" alt="Ajustes de pantalla — Fondo" float="right" />

**Fill** establece el [fondo de pantalla](paywall-head-picture) con un color sólido, degradado, imagen o vídeo. El fondo cubre todo el viewport del dispositivo, incluidas las áreas detrás de la muesca y las barras del sistema, incluso cuando **Safe area** está activado.

#### Reproducir vídeo de fondo en bucle \{#loop-background-video\}

Activa el interruptor **Loop** para reproducir el vídeo de fondo de forma continua.

#### Asignar un ID de medio personalizado \{#assign-a-custom-media-id\}
Al igual que con [cualquier imagen o vídeo](custom-media), puedes asignar al fondo de pantalla un ID de contenido multimedia personalizado para referenciarlo en tu SDK.
### Espaciado de pantalla \{#screen-spacing\}

<ZoomImage id="builder-screen-settings-spacing.webp" width="350px" alt="Configuración de pantalla — Espaciado" float="right" />

Ajusta el relleno de pantalla para cada lado (superior, derecho, inferior, izquierdo).
### Scroll \{#scroll\}

<ZoomImage id="builder-screen-settings-scroll.webp" width="350px" alt="Screen settings — Scroll" float="right" />

Controla el comportamiento del desbordamiento. Activa **Vertical scroll** para permitir que el contenido de la pantalla se desplace cuando supere la altura del viewport.
### Grupos seleccionables \{#selectable-groups\}

:::link
Artículo principal: [Elementos y grupos seleccionables](flow-selectable-elements)
:::

La sección **Selectable groups** muestra todos los grupos seleccionables de la pantalla actual — de [cuestionarios](onboarding-quizzes), [productos](paywall-product-block), [pestañas](builder-tabs), [activadores de prueba](builder-toggles) o cualquier [elemento seleccionable personalizado](flow-selectable-elements#make-an-element-selectable).

Haz clic en una entrada de grupo para renombrarlo, cambiar su tipo, ver las variables que expone o eliminarlo.

## Gestionar capas \{#manage-layers\}

Cada elemento de una pantalla se representa como una capa. La sección Layers muestra el orden de los elementos en la pantalla activa.

:::important
Las capas de un flow no se superponen como las capas en un software de diseño gráfico. En su lugar, representan componentes individuales de la pantalla. Los elementos solo se superponen *si* utilizan [posicionamiento absoluto o fijo](manage-paywall-ui-elements). Su orden de apilamiento viene determinado por la propiedad `z-index`, no por su posición en el árbol de capas.
:::
La estructura de árbol refleja las relaciones padre-hijo. Haz clic en la flecha de cualquier capa padre para expandir o contraer sus hijos.

No puedes crear capas directamente. Cada elemento que añadas desde la vista [Añadir elemento](builder-elements) aparece como una nueva capa en el árbol.

<ZoomImage id="flow-builder-layer-focus.webp" width="350px" alt="Flow builder layer view" float="left" />
* **Seleccionar una capa**: Haz clic en una capa para seleccionarla. El editor visual resalta el elemento correspondiente en el lienzo y el panel derecho muestra sus propiedades de [diseño](builder-styling) e [interacción](onboarding-navigation-branching).
* **Reordenar capas**: Arrastra y suelta las capas dentro del árbol para cambiar su orden en el contenedor padre. El orden en el árbol coincide con el orden visual en pantalla.
* **Mostrar u ocultar una capa**: Pasa el cursor sobre una capa para ver el icono de ojo <Inline id="eye.svg" alt="Eye" /> en su lado derecho. Haz clic en él para alternar la visibilidad de la capa. Las capas ocultas permanecen en el árbol pero no aparecen en el editor visual ni en el dispositivo. Para controlar la visibilidad con lógica en tiempo de ejecución, usa la [visibilidad condicional](onboarding-element-visibility).
* **Contraer todas las capas**: Haz clic en el botón de contraer <Inline id="collapse.svg" alt="Collapse" /> en la esquina superior derecha de la sección Layers para plegar todo el árbol.
### Acciones de capa \{#layer-actions\}

Haz clic en el icono de tres puntos <Inline id="context.svg" alt="Context" /> para abrir el menú contextual.

<ZoomImage id="flow-builder-layer-context-menu.webp" width="450px" alt="Flow builder layer context menu" />
| Acción | Atajo | Descripción |
|--------|-------|-------------|
| **Copy** | ⌘C / Ctrl+C | Copiar la capa al portapapeles |
| **Paste here** | ⌘V / Ctrl+V | Pegar una capa copiada previamente como elemento hijo |
| **Duplicate** | ⌘D / Ctrl+D | Crear una copia de la capa en el mismo contenedor |
| **Rename** | | Cambiar el nombre visible de la capa. Por defecto, las capas usan su contenido o tipo de componente como nombre |
| **Delete** | ⌘⌫ / Ctrl+Del | Eliminar la capa y todos sus elementos hijos |
| **Wrap** | | Envolver la capa en un nuevo contenedor: **Wrap in Horizontal Container** o **Wrap in Vertical Container** |
| **Unwrap / Ungroup** | | Eliminar el contenedor envolvente y mover sus hijos un nivel hacia arriba |
| **Move up** | ↑ | Mover la capa una posición hacia arriba en su contenedor padre |
| **Move down** | ↓ | Mover la capa una posición hacia abajo en su contenedor padre |
