El panel **Interactions** te permite definir cómo responden los elementos del flow ante eventos — como toques, aparición de elementos y envío de formularios. Para cada evento, asignas una o más acciones: navegar entre pantallas, mostrar u ocultar elementos, abrir URLs, establecer variables y más. Usa condiciones para personalizar el flow según los datos del usuario.

Cada interacción sigue una cadena de tres partes:
1. **Elemento**: El componente de pantalla que inicia la interacción — un botón, respuesta de quiz, campo de entrada o cualquier otra cosa.
2. **Disparador**: El evento que activa la lógica, como un toque, la aparición de un elemento o el envío de un formulario.
3. **Acción**: La tarea que ejecuta el flow en respuesta. Un solo disparador puede ejecutar varias acciones en secuencia.
## Configurar interacciones \{#set-up-interactions\}

Para configurar una interacción:

1. Selecciona un elemento en la pantalla o en el panel **Layers**.
2. A la derecha, cambia al panel **Interactions** y haz clic en **Add trigger**.

<ZoomImage id="add-trigger.webp" width="500px" alt="Botón Add trigger" />

3. En la sección **Button triggers**, selecciona el [tipo de disparador](#trigger-types).

<ZoomImage id="trigger-type.webp" width="500px" alt="Selección del tipo de disparador" />

4. Haz clic en **Add action**, luego en el nombre de la acción y selecciona un [tipo de acción](#action-types) en el desplegable de la ventana **Edit action**.
<ZoomImage id="add-action.gif" width="500px" alt="Add action button" />

5. Configura las propiedades de la acción según el [tipo de acción](#action-types) que hayas seleccionado.
6. Si lo necesitas, haz clic en **Add action** para añadir más acciones al mismo disparador.
## Tipos de disparadores \{#trigger-types\}

Actualmente, puedes añadir los siguientes disparadores a tu flow:
- **On tap**: Inicia una acción cuando el usuario pulsa un elemento. Es el disparador más habitual; se usa principalmente para navegar entre pantallas.
- **On screen appear**: Inicia una acción en cuanto se carga una pantalla. Úsalo para activar una animación o registrar una impresión cuando se cargue la pantalla.
## Tipos de acciones \{#action-types\}
### Navegar a pantalla \{#navigate-to-screen\}

Esta es la acción principal para mover a los usuarios entre pantallas. Lleva al usuario a una pantalla de destino concreta.

Para esta acción, solo necesitas configurar la pantalla de destino. Si quieres habilitar la navegación dinámica, consulta [Navegación y ramificación](onboarding-navigation-branching) o la sección [Acciones condicionales](#conditional-actions).

:::important
**Navigate to screen** siempre debe ser la última acción de la lista. Cualquier acción colocada después (como "Set Variable") podría no ejecutarse porque la app ya habrá cambiado de pantalla.
:::
<ZoomImage id="navigate-to.webp" width="500px" alt="Navegar a la pantalla" />
### Navegar hacia atrás \{#navigate-back\}

Devuelve al usuario a la pantalla anterior en su historial de navegación, en lugar de la pantalla anterior en la secuencia.
### Abrir URL \{#open-url\}

Abre una dirección web específica. Úsala para enviar a los usuarios a páginas web, artículos o perfiles de redes sociales fuera de las pantallas nativas de tu app.
Para esta acción, puedes configurar dos ajustes:
- **URL address**: Define una dirección URL. Además, puedes hacerla dinámica — por ejemplo, para llevar a los usuarios a páginas distintas según su respuesta en el cuestionario o con los datos que hayan enviado. Para ello, haz clic en <Inline id="variable.webp" alt="Variable icon" /> y selecciona la variable que quieras usar.
- **Open in external browser**: Define dónde quieres abrir los enlaces externos. Por defecto, se abren en un navegador integrado en la app para que los usuarios no la abandonen. Activa la casilla **Open in external browser** si prefieres que se abran en un navegador externo.
<ZoomImage id="open-url.webp" width="500px" alt="Abrir URL" />
### Cerrar flow \{#close-flow\}

Cierra el flow actual.
### Mostrar/ocultar elementos \{#showhide-elements\}

Muestra u oculta un elemento concreto de la pantalla.

Esta acción anula el estado inicial definido en **Visibility** del panel **Design**. Si **Visibility** está configurado como **Hide**, la acción **Show** hará que el elemento aparezca.
### Mostrar alerta \{#show-alert\}

Muestra una ventana emergente nativa del sistema. Los usuarios deben pulsar **Ok** para continuar.

Para las alertas, debes configurar su **Title** y **Message**. En ambos campos puedes usar variables para hacer el contenido dinámico. Para ello, haz clic en <Inline id="variable.webp" alt="Variable icon" /> y selecciona la variable que quieras usar.

<ZoomImage id="show-alert.webp" width="500px" alt="Show alert" />
### Establecer variable \{#set-variable\}

Actualiza el valor de una variable en el flow. Antes de añadir esta acción, crea variables en el panel **Variables** de la izquierda (consulta [Variables](onboarding-variables)).

Haz clic en **Add variable** y define tantas variables y valores como necesites.

<ZoomImage id="set-variable.webp" width="500px" alt="Establecer variable" />
### Compra \{#purchase\}

Activa un flujo de compra directamente desde un botón o interacción en tu onboarding. Úsalo para que los usuarios se suscriban o compren un producto sin salir del flow.
Puedes configurar dos comportamientos para esta acción:
- **In-app store**: Inicia una compra nativa. Establece **Product** en un producto concreto o en `products.selectedProduct` para la selección actual del usuario en la pantalla.
- **Web payment**: Envía al usuario a un [paywall web](web-paywall) en lugar de activar una compra nativa. Úsalo cuando quieras gestionar la transacción fuera de la app, por ejemplo para ofertas de suscripción basadas en web.

<ZoomImage id="onboarding-purchase-action.webp" width="500px" alt="Configuración de la acción de compra en la ventana Editar acción" />
### Personalizado

Activa una acción personalizada con nombre que gestiona el código de tu app. Úsala cuando necesites un comportamiento que los tipos de acción integrados no cubren — por ejemplo, disparar un evento in-app, aplicar un descuento o mostrar una pantalla nativa personalizada.

Para configurar una acción personalizada:

1. En la ventana **Edit action**, asigna un **Action ID** — una cadena que tu app reconocerá (por ejemplo, `show_discount`).
2. En el código de tu app, implementa un handler para este ID de acción. Consulta [Gestionar acciones del paywall](handle-paywall-actions) para ver los detalles de implementación.
<ZoomImage id="onboarding-custom-action.webp" width="500px" alt="Configuración de acción personalizada con el campo ID de acción en la ventana Editar acción" />
## Acciones condicionales \{#conditional-actions\}

Usa las acciones condicionales para dividir el flow en diferentes rutas según los datos del usuario.

Algunos casos de uso habituales son:

- Tienes un cuestionario en la pantalla y quieres llevar a los usuarios a diferentes pantallas según sus respuestas. En este caso, añade una acción condicional a un botón.
- Quieres ofrecer diferentes productos y ofertas a distintos grupos de usuarios. Colócalos en pantallas separadas y configura condiciones en un botón de navegación.
- Quieres saltarte ciertos pasos para los usuarios que ya han completado un tutorial en una sesión anterior de la app.
Las acciones condicionales funcionan como una cadena if / else-if / else. La app lee las reglas de arriba a abajo y se detiene en la primera coincidencia:
1. **IF**: El flow comprueba la condición principal.
    - ¿Es verdadera? El flow ejecuta las acciones THEN inmediatamente y se detiene.
    - ¿Es falsa? El flow salta a la siguiente sección.
2. **ELSE IF**: Aquí puedes añadir comprobaciones adicionales (p. ej., "Si no es Premium, ¿el usuario está en un Trial?").
3. **ELSE** (Fallback): Si ninguna de las reglas anteriores coincidió, el flow ejecuta las acciones de esta sección final.
:::warning
- Si se añade una regla pero no tiene ninguna acción asignada, que se cumpla la condición no produce ningún efecto.
- Si se añade una variable pero le falta un operador o un valor, la condición se considera inválida.
:::

Para cada regla, selecciona una variable que evaluar y una acción que ejecutar. Puedes establecer más de una acción por regla.

:::important
El flow ejecuta solo una regla: la primera que coincide. Si necesitas ejecutar tanto **IF** como **ELSE IF** al mismo tiempo, añade ambas acciones en **IF**.
:::
Para aprender a hacer elementos seleccionables y organizarlos en grupos para usarlos en condiciones, consulta [Elementos seleccionables y grupos](flow-selectable-elements).
