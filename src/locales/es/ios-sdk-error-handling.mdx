El SDK de Adapty tiene su propio wrapper para todo tipo de errores, llamado `AdaptyError`. Básicamente, todos los errores que devuelve el SDK son de tipo `AdaptyError`. Tiene dos propiedades útiles: `originalError` y `adaptyErrorCode`, que se describen a continuación.
**originalError** contiene el error original en caso de que necesites trabajar directamente con él. Puede ser un [SKError](https://developer.apple.com/documentation/storekit/skerror), un [NSError](https://developer.apple.com/documentation/foundation/nserror) o simplemente un [Error](https://developer.apple.com/documentation/swift/error) genérico de Swift. Esta propiedad es opcional, ya que algunos errores pueden generarse directamente desde el SDK —como datos inconsistentes o ausentes— y no tendrán el error original a partir del cual se construyó el wrapper.

**adaptyErrorCode** sirve para gestionar problemas comunes, como:
- credenciales inválidas
- errores de red
- pagos cancelados
- problemas de facturación
- recibo inválido
- y mucho más

Es muy sencillo comprobar el error para códigos específicos y reaccionar en consecuencia.

```swift showLineNumbers title="Swift"
do {
    let info = try await Adapty.makePurchase(product: product)
} catch {
    if error.adaptyErrorCode == .paymentCancelled {
        // purchase was cancelled
        // you can offer discount to your user or remind them later
    }
}
```
:::important
Si estas soluciones no resuelven tu problema, consulta [Otros problemas](#other-issues) para ver los pasos a seguir antes de contactar con soporte y ayudarnos a atenderte de forma más eficiente.
:::

## Otros problemas \{#other-issues\}

Si aún no has encontrado una solución, los siguientes pasos pueden ser:
- **Actualiza el SDK a la versión más reciente**: Siempre recomendamos actualizar a las últimas versiones del SDK, ya que son más estables e incluyen correcciones de problemas conocidos.
- **Contacta al equipo de soporte u obtén ayuda de otros desarrolladores** en el [foro de soporte](https://adapty.featurebase.app/).
- **Contacta al equipo de soporte a través de [support@adapty.io](mailto:support@adapty.io) o por el chat**: Si no estás listo para actualizar el SDK o si la actualización no resolvió el problema, contacta a nuestro equipo de soporte. Ten en cuenta que tu problema se resolverá más rápido si [habilitas el registro detallado](sdk-installation-ios#logging) y compartes los logs con el equipo. También puedes adjuntar fragmentos de código relevantes.
