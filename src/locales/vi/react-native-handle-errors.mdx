Mỗi lỗi được SDK trả về đều có kiểu `AdaptyErrorCode`. Dưới đây là một ví dụ:

:::tip
**Bật verbose logs trước khi debug.** Hầu hết các `AdaptyError` đều bọc một lỗi bên dưới từ StoreKit, Play Billing, network, hoặc backend. Khi bật verbose logs (`adapty.setLogLevel('verbose')` — xem phần Logging cho [React Native thuần](sdk-installation-react-native-pure#logging) hoặc [Expo](sdk-installation-react-native-expo#logging)), lỗi bọc đó sẽ được in ra console, thường cho bạn biết nguyên nhân thực sự.
:::

:::important
Nếu các giải pháp này không giải quyết được vấn đề của bạn, hãy xem phần [Các vấn đề khác](#other-issues) để biết các bước cần thực hiện trước khi liên hệ hỗ trợ, giúp chúng tôi hỗ trợ bạn hiệu quả hơn.
:::

```typescript showLineNumbers
try {
  const params: MakePurchaseParamsInput = {};
  await adapty.makePurchase(product, params);
} catch (error) {
  if (
    error instanceof AdaptyError &&
    error.adaptyCode === getErrorCode(ErrorCode['2'])
  ) {
    // payment cancelled
  }
}
```

## Các vấn đề khác \{#other-issues\}

Nếu bạn vẫn chưa tìm được giải pháp, các bước tiếp theo có thể là:

- **Nâng cấp SDK lên phiên bản mới nhất**: Chúng tôi luôn khuyến nghị nâng cấp lên các phiên bản SDK mới nhất vì chúng ổn định hơn và bao gồm các bản sửa lỗi đã biết.
- **Liên hệ đội hỗ trợ hoặc nhận trợ giúp từ các nhà phát triển khác** trong [diễn đàn hỗ trợ](https://adapty.featurebase.app/).
- **Liên hệ đội hỗ trợ qua [support@adapty.io](mailto:support@adapty.io) hoặc qua chat**: Nếu bạn chưa sẵn sàng nâng cấp SDK hoặc việc nâng cấp không giải quyết được vấn đề, hãy liên hệ đội hỗ trợ của chúng tôi. Lưu ý rằng vấn đề của bạn sẽ được giải quyết nhanh hơn nếu bạn [bật verbose logging](sdk-installation-reactnative) và chia sẻ logs với đội ngũ. Bạn cũng có thể đính kèm các đoạn code liên quan.
