Tích hợp Adapty Attribution với Google Cloud Storage cho phép bạn lưu trữ dữ liệu chiến dịch thu hút người dùng một cách an toàn tại một nơi tập trung. Bạn có thể lưu dữ liệu hiệu suất chiến dịch, dữ liệu attribution, và các sự kiện thu hút người dùng vào bucket Google Cloud Storage của mình dưới dạng file .csv.
Để thiết lập tích hợp này, bạn cần thực hiện một vài bước đơn giản trong Google Cloud Console và Adapty Attribution Dashboard.

:::note
Lịch chạy

Adapty Attribution gửi dữ liệu của bạn đến Google Cloud Storage mỗi 24 giờ lúc 4:00 UTC.
Mỗi file sẽ chứa dữ liệu về các sự kiện được tạo ra trong toàn bộ ngày theo lịch trước đó theo giờ UTC. Ví dụ: dữ liệu được xuất tự động lúc 4:00 UTC ngày 8 tháng 3 sẽ chứa tất cả các sự kiện được tạo vào ngày 7 tháng 3 từ 00:00:00 đến 23:59:59 theo giờ UTC.
:::
## Cách thiết lập tích hợp Google Cloud Storage \{#how-to-set-up-google-cloud-storage-integration\}

### Bước 1. Tạo thông tin xác thực Google Cloud Storage \{#step-1-create-google-cloud-storage-credentials\}

Hướng dẫn này sẽ giúp bạn tạo các thông tin xác thực cần thiết trong Google Cloud Platform Console.

Để Adapty Attribution có thể tải lên các báo cáo dữ liệu thô vào bucket của bạn, cần có khóa của service account cũng như quyền ghi vào bucket tương ứng. Bằng cách cung cấp khóa của service account và cấp quyền ghi vào bucket, bạn cho phép Adapty Attribution truyền dữ liệu thô một cách an toàn và hiệu quả từ nền tảng của mình sang môi trường lưu trữ của bạn.
:::warning
Xin lưu ý rằng chúng tôi chỉ hỗ trợ xác thực bằng Service Account HMAC key, do đó bạn cần đảm bảo rằng Service Account HMAC key của mình có các quyền "Storage Object Viewer", "Storage Legacy Bucket Writer" và "Storage Object Creator" để có thể truy cập đúng cách vào Google Cloud Storage.
:::

### Bước 2. Cấu hình tích hợp trong Adapty Attribution \{#step-2-configure-integration-in-adapty-attribution\}

1. Truy cập [**Integrations** -> **Google Cloud Storage**](https://app.adapty.io/ua/integrations/google-cloud-storage)
2. Bật toggle **Export install events to Google Cloud Storage**
3. Điền vào các trường bắt buộc để thiết lập kết nối giữa Google Cloud Storage và Adapty Attribution:
| Trường                                    | Mô tả                                                                                                                                                                                                       |
|:------------------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Google Cloud service account key file** | File JSON key riêng tư đã tải xuống [JSON key file](ua-google-cloud-storage#step-1-create-google-cloud-storage-credentials).                                                          |
| **Google Cloud bucket name**              | Tên bucket trong Google Cloud Storage nơi bạn muốn lưu trữ dữ liệu. Tên này phải là duy nhất trong môi trường Google Cloud Storage và không được chứa khoảng trắng.                                       |
| **Folder inside the bucket**              | Tên thư mục bên trong bucket nơi bạn muốn lưu trữ dữ liệu. Tên này phải là duy nhất trong bucket và có thể dùng để sắp xếp dữ liệu. Trường này không bắt buộc điền.                                       |

## Xuất dữ liệu thủ công \{#manual-data-export\}

Ngoài tính năng tự động xuất dữ liệu sự kiện sang Google Cloud Storage, Adapty UA còn cung cấp tính năng xuất file thủ công. Với tính năng này, bạn có thể chọn một ngày cụ thể cho dữ liệu thu hút người dùng và xuất thủ công sang bucket GCS của mình. Điều này giúp bạn kiểm soát tốt hơn dữ liệu cần xuất và thời điểm xuất.

## Cấu trúc bảng \{#table-structure\}

Trong tích hợp Google Cloud Storage, Adapty Attribution cung cấp một bảng để lưu trữ dữ liệu lịch sử cho các sự kiện cài đặt. Bảng chứa thông tin về hồ sơ người dùng, doanh thu và lợi nhuận, cửa hàng gốc, cùng nhiều điểm dữ liệu khác.

:::warning
Lưu ý rằng cấu trúc này có thể mở rộng theo thời gian — khi chúng tôi hoặc các bên thứ ba mà chúng tôi hợp tác giới thiệu dữ liệu mới. Hãy đảm bảo rằng code xử lý dữ liệu của bạn đủ linh hoạt và chỉ phụ thuộc vào các trường cụ thể, không phụ thuộc vào toàn bộ cấu trúc.
:::

Dưới đây là cấu trúc bảng cho các sự kiện:
| Cột                      | Mô tả                                              |
|--------------------------|----------------------------------------------------|
| `adapty_profile_id`      | Mã định danh hồ sơ người dùng Adapty               |

| `created_at`             | Thời điểm tạo bản ghi (ISO 8601)                   |

| `store`                  | Cửa hàng ứng dụng (`ios`, `android`)               |

| `ip_address`             | Địa chỉ IP của client                              |
| `idfa`                   | iOS Identifier for Advertisers                     |
| `idfv`                   | iOS Identifier for Vendors                         |
| `gaid`                   | Google Advertising ID (Android)                    |
| `android_id`             | ID thiết bị Android                                |
| `app_set_id`             | Android App Set ID                                 |
| `channel`                | Kênh attribution                                   |
| `campaign_id`            | Mã định danh chiến dịch                            |
| `campaign_name`          | Tên chiến dịch                                     |
| `adset_id`               | Mã định danh nhóm quảng cáo                        |
