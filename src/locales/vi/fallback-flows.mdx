Để duy trì trải nghiệm người dùng mượt mà, bạn cần thiết lập **phiên bản dự phòng** cho các [flow](adapty-flow-builder) của mình.
Khi ứng dụng của bạn yêu cầu một flow, Adapty SDK liên hệ với máy chủ của chúng tôi để tải cấu hình. Nếu thiết bị không thể kết nối với Adapty (sự cố mạng, máy chủ gián đoạn), SDK sẽ chuyển sang dùng dữ liệu cục bộ:

- Nếu người dùng đã xem flow đó một lần, SDK sẽ phục vụ bản đã lưu trong cache.
- Nếu chưa có cache, SDK sẽ tải file cấu hình dự phòng được đóng gói sẵn trong ứng dụng.

Adapty tự động tạo các file dự phòng này. Gói dự phòng của flow được dùng chung với paywall — một file JSON duy nhất cho mỗi nền tảng chứa các biến thể dự phòng cho cả hai. SDK đọc phần nào cần thiết tùy theo ngữ cảnh.
:::important
Flow fallback được tích hợp trong gói **Adapty SDK 4.0+**. Nếu bạn chọn phiên bản SDK cũ hơn trong hộp thoại tải xuống, file sẽ chỉ chứa các biến thể paywall và onboarding — không có flow. Hãy đảm bảo ứng dụng của bạn đang dùng phiên bản SDK hỗ trợ flow trước khi sử dụng flow fallback.
:::
## Trước khi bắt đầu \{#before-you-start\}

1. Tạo một [flow](adapty-flow-builder) trong Flow Builder.
2. [Tạo một placement](create-placement) cho flow đó.

## Tải file dự phòng \{#download-the-fallback-file\}

1. Mở trang **[Placements](https://app.adapty.io/placements)**.
2. Nhấn nút **Fallbacks** ở góc trên bên phải.
3. Chọn nền tảng mục tiêu từ dropdown.
4. Chọn phiên bản SDK khớp với phiên bản bạn đã tích hợp trong app. Chọn **Adapty SDK v4.0.0 and higher** (hoặc tùy chọn mới hơn) để nhận bundle bao gồm cả flow.

Trình duyệt sẽ tải xuống một file JSON cho mỗi nền tảng — ví dụ: `ios_4_0_0_fallback.json`.

## Sau khi tải xuống \{#after-the-download\}

Thêm file vào code ứng dụng của bạn, sau đó làm theo hướng dẫn cài đặt theo từng nền tảng. Các API dùng để tải paywall dự phòng cũng sẽ tải flow dự phòng khi ứng dụng của bạn sử dụng phiên bản SDK hỗ trợ flow:

- [iOS](ios-use-fallback-paywalls)
- [Android](android-use-fallback-paywalls)
- [React Native](react-native-use-fallback-paywalls)
- [Capacitor](capacitor-use-fallback-paywalls)
## Hạn chế \{#limitations\}

Các flow dự phòng được mã hóa cứng và lưu trữ cục bộ, do đó chúng không mang đầy đủ khả năng động của các flow trực tiếp:
- **Một biến thể cho mỗi placement.** Nếu một placement có nhiều hơn một flow (các đối tượng khác nhau, các biến thể A/B test), file dự phòng sẽ sử dụng biến thể có trọng số cao nhất, hoặc đối tượng rộng nhất.
- **Không có A/B testing.** Một A/B test flow đang chạy sẽ được xử lý trên server; fallback luôn phục vụ một biến thể duy nhất đã được chọn.
- **Không có cập nhật từ xa.** Việc cập nhật fallback yêu cầu phải phát hành ứng dụng mới. Đối với các cập nhật runtime mà bạn thường thực hiện qua remote config, hãy đẩy chúng qua flow trực tiếp thay thế.
- **Chỉ có locale mặc định.** Fallback sử dụng locale `en`; các biến thể đã được bản địa hóa không được đóng gói kèm theo.
