ようこそ！アプリ内課金をもっと手軽にするお手伝いをします 🚀

Adapty Unity SDK は、アプリ内課金の面倒な処理をすべて引き受けることで、あなたが最も得意なこと――素晴らしいゲームの開発――に集中できるよう設計されています。私たちが代わりに対応する内容はこちらです：

- 購入処理、レシート検証、サブスクリプション管理をすぐに使える状態で提供
- アプリのアップデートなしにペイウォールを作成・テスト可能
- ゼロ設定で詳細な購入分析を取得 - コホート、LTV、チャーン、ファネル分析を含む
- アプリセッションやデバイスをまたいでユーザーのサブスクリプション状態を常に最新の状態に維持
- たった1行のコードでマーケティングアトリビューションや分析サービスとアプリを連携

:::note
コードに入る前に、Adapty を App Store Connect および Google Play Console と連携し、ダッシュボードでプロダクトを設定する必要があります。まずは[クイックスタートガイド](quickstart)ですべての設定を完了させてください。
:::

## 主要な概念 \{#main-concepts\}

コードに入る前に、Adapty の仕組みを支える重要な概念を理解しておきましょう。

Adapty のアプローチの優れた点は、アプリ内にハードコードするのはプレースメントだけという点です。プロダクト、ペイウォールのデザイン、価格、オファーなど、それ以外のすべては Adapty ダッシュボードからアプリのアップデートなしに柔軟に管理できます：

1. [**プロダクト**](product) - アプリ内で購入可能なすべてのもの――サブスクリプション、消耗型アイテム、永続アクセスなど。

2. [**ペイウォール**](paywalls) - Adapty からプロダクトを取得し、その機能をフルに活用するための唯一の手段です。異なるプロダクトの組み合わせが収益化指標にどう影響するかを追跡しやすくするため、このように設計されています。Adapty のペイウォールは、プロダクトの特定のセットとそれに付随するビジュアル設定の両方として機能します。

3. [**プレースメント**](placements) - ユーザーのジャーニーの中でペイウォールを表示する戦略的なポイントです。プレースメントは収益化戦略における「どこで」「いつ」に相当します。よく使われるプレースメントの例：
   - `main` - メインのペイウォール表示場所
   - `onboarding` - ユーザーのオンボーディングフロー中に表示
   - `settings` - アプリの設定からアクセス可能

   最初のインテグレーションでは `main` や `onboarding` などの基本的なものから始め、その後[アプリの他のどこでユーザーが購入しやすいか](choose-meaningful-placements)を検討してみてください。

4. [**プロファイル**](profiles-crm) - ユーザーがプロダクトを購入すると、そのプロファイルに**アクセスレベル**が付与され、有料機能へのアクセス制御に使用します。
