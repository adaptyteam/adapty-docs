Adapty Attributionは、広告プラットフォーム・トラッキングリンク・アプリのデータを組み合わせることで、広告キャンペーンとアプリインストールおよびサブスクリプション収益を紐づけるアトリビューションソリューションです。すべての獲得データを一か所に集約した統合マーケティング分析ダッシュボードを提供します。

- すべてのチャネルにわたるROAS（広告費用対効果）を算出
- アプリ全体の収益状況を一か所で把握
- より良い意思決定のための正確なアトリビューションデータを取得
- コホートのパフォーマンスとユーザー行動を時系列で分析

:::tip
Adapty のアトリビューション機能についてもっと詳しく知りたい方は、[お問い合わせください](https://calendly.com/tnurutdinov-adapty/30min)。
:::

## Adapty アトリビューションを選ぶ理由 \{#why-choose-adapty-attribution\}

ユーザー獲得のパフォーマンス計測は難しいものです。データは複数のプラットフォームに分散し、プライバシー規制の変化でアトリビューションが困難になり、独自ソリューションの構築には多大な時間がかかります。

Adapty Attribution は、組み込みのアトリビューションと統合されたアナリティクスを1つのマーケティングダッシュボードで提供します。広告費からインストール数、サブスクリプション収益まで、すべての獲得指標が自動的に統合され、リアルタイムで更新されます。スプレッドシート間でデータを照合したり、複数のツールを切り替えたりする必要はもうありません。データ管理システムではなく、アプリの成長に集中できます。

## 仕組み \{#how-it-works\}

Adapty Attribution は、広告プラットフォーム、トラッキングリンク、アプリからのデータを組み合わせて、アプリのインストールおよびサブスクリプション収益を広告キャンペーンにアトリビュートします。

大まかな流れは以下のとおりです：
- 広告プラットフォームがキャンペーン構成と広告費用を提供する
- Adapty Attribution で生成されたトラッキングリンクが、ウェブからアプリインストールへのキャンペーンのコンテキストを伝える
- Adapty SDK がアプリからインストールおよび収益イベントを送信する

アトリビューションのフローは以下のように機能します：

1. **トラッキングリンクがAdapty アトリビューションで生成され、広告キャンペーンに追加されます。** リンクには、プラットフォーム、キャンペーン、広告セット、クリエイティブなどのキャンペーンパラメーターが含まれています。

2. **ユーザーが広告をクリックし、アプリストアからアプリをインストールします。** ユーザーはトラッキングリンク経由でリダイレクトされ、App StoreまたはGoogle Playからアプリをインストールします。

3. **アプリがAdaptyにインストールイベントを送信します。** 初回起動時に、AdaptyのSDKがインストールイベントを送信します。Adaptyはこのインストールに関連するキャンペーンパラメーターを抽出します。

4. **インストールがキャンペーンにアトリビューションされます。** トラッキングリンクのキャンペーンパラメータを使用して、Adaptyはインストールをそれを生み出したキャンペーンに紐付けます。

5. **広告費用と収益が連携されます。** Adaptyは対応する広告プラットフォーム（現時点では Meta Ads と TikTok for Business）から広告費用データを取得し、サブスクリプションや購入イベントをアトリビューション済みのインストールに紐付けます。

その結果、Adapty はインストール数、収益、LTV、ROAS などのキャンペーンレベルの指標を一元化されたアナリティクスダッシュボードで提供します。コホートを分析したり、時系列でパフォーマンスを追跡したり、データドリブンな最適化の意思決定を行ったりできます。異なるソースのデータを手動で突き合わせる必要はありません。

:::tip
トラッキングリンクにカスタムパラメーターを含めることもできます。これにより、アプリがインストールイベントを処理する際に [ディファードディープリンク](ua-deferred-data) の処理やキャンペーンデータへの対応が可能になります。
:::
