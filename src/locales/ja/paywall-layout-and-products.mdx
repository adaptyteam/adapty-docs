フローは1つ以上の画面で構成されます。各画面はユーザーのジャーニーにおける1つのステップを表します。たとえば、ペイウォール、クイズ、プロダクト情報のスライドなどです。

各画面上の要素はレイヤー階層として整理されています。
スクリーン、レイヤー、要素を管理するには、デフォルトの **Screens and Layers** ビューを開きます。スクリーンの順番と各スクリーンのレイヤー構造が表示されます。
## 画面の管理 \{#manage-screens\}

<ZoomImage id="flow-builder-screens-and-layers-left-sidebar.webp" width="400px" alt="Flow builder screens and layers view (both panels visible)" float="left" />

左パネルの上部には、フロー内のすべての画面が一覧表示されます。各エントリには番号付きのラベルとサムネイルプレビューが表示されます。
* **スクリーンを選択する**: スクリーンのエントリーをクリックするとアクティブになります。ビジュアルエディターに選択したスクリーンが表示され、下のレイヤーセクションがそのレイヤー階層に更新されます。
* **スクリーンを追加する**: スクリーンセクション上部の <Inline id="plus.svg" alt="Plus" /> ボタンをクリックして、フローに新しい空のスクリーンを追加します。
* **テンプレートライブラリを開く**: スクリーンセクション上部の <Inline id="template-library.svg" alt="Templates" /> ボタンをクリックして、[フローテンプレート](paywall-builder-templates)を閲覧・適用します。
* **スクリーンを並び替える**: スクリーンのエントリーをドラッグ＆ドロップして、フロー内の順序を変更します。
:::important
フローに未使用の空のスクリーンがある場合、公開できません。公開前にドラフトのスクリーンを削除してください。
:::
### スクリーンアクション \{#screen-actions\}

スクリーンエントリの三点アイコン <Inline id="context.svg" alt="Context" /> をクリックすると、コンテキストメニューが開きます。

<ZoomImage id="flow-builder-screen-actions.webp" width="600px" alt="Flow builder screen context menu" />
| アクション | ショートカット | 説明 |
|--------|----------|-------------|
| **Play Animation** | | このスクリーンに設定されているアニメーションをプレビューする |
| **Copy** | ⌘C / Ctrl+C | スクリーンをクリップボードにコピーする |
| **Paste here** | ⌘V / Ctrl+V | コピーしたスクリーンを貼り付ける |
| **Duplicate** | ⌘D / Ctrl+D | スクリーンのコピーを作成してフローに追加する |
| **Rename** | | スクリーンの表示名を変更する |
| **Delete** | ⌘⌫ / Ctrl+Del | スクリーンをフローから削除する |

:::tip
クリップボードはフローをまたいで保持されます。あるフローでスクリーンや要素をコピーし、別のフローを開いて貼り付けることができます。
:::
:::warning
スクリーンを削除すると、そのスクリーンを指していた [Navigate to Screen](onboarding-navigation-branching) アクションは**ターゲットを失います**が、アクション自体は**削除されません**。新しい遷移先を設定するか、アクションを削除してください。そうしないと、[フローのプレビューや公開](builder-save-publish#publish-a-flow)ができなくなります。
:::
## スクリーン間の移動 \{#navigate-between-screens\}

:::link

リスト内のスクリーンの順序だけでは、ナビゲーションは決まりません。スクリーンを接続するには、要素のインタラクションを使用してください。ボタンを設定して、ユーザーを別のスクリーンに移動させます。

## スクリーン設定 \{#screen-settings\}

アクティブなスクリーンのプロパティと設定を表示するには、スクリーンプレビューの空白部分をクリックしてください。右パネルがスクリーン設定ビューに切り替わります。

### システムUI \{#system-ui\}

<ZoomImage id="builder-screen-settings-system-ui.webp" width="350px" alt="Screen settings — System UI" float="right" />

スクリーンがデバイスのハードウェアとどのように連携するかを制御します。

* **Safe area** は、ノッチやシステムバーからコンテンツを保護するパディングを追加します。
* **Status bar** は、システムのステータスバー（時刻、バッテリー、信号アイコン）の表示と非表示を切り替えます。

### プログレスインジケーターへのスクリーンの含め方 \{#include-screen-in-progress-indicator\}

フローに[プログレスインジケーター](builder-loaders-and-progress-bars#progress-indicators)要素を追加すると、Adaptyはすべてのスクリーンに表示します。

**Include screen in progress indicator** のチェックを外すと、特定のスクリーンからプログレスインジケーターが非表示になります。ウェルカムスクリーン、最終的なペイウォール、または進行状況として登録したくないステップに使用してください。

### スクリーンレイアウト \{#screen-layout\}

:::link
詳細記事: [要素の配置](manage-paywall-ui-elements)
:::

<ZoomImage id="builder-screen-settings-layout.webp" width="350px" alt="スクリーン設定 — レイアウト" float="right" />

**Layout** セクションでは、スクリーン内の子要素の配置方法を設定します。これらのプロパティはすべてのコンテナ要素で利用できます。

* **Free**: 子要素が独立して配置されます。
* **Vertical**: 要素がフレックスボックスのカラムのように上から下へ並びます。
* **Horizontal**: 要素がフレックスボックスの行のように左から右へ並びます。
縦・横レイアウトでは、スペーシングと配置も設定できます。

* **Alignment**：クロス軸に沿った要素の位置。
* **Gap**：隣接する要素間のスペース。
* **Distribution**：子要素の間および周囲のスペースの配分。

#### RTL レイアウト \{#rtl-layout\}

**Mirror for RTL** チェックボックスをオンにすると、右から左に読む言語向けにレイアウトを左右反転できます。水平コンテナ内の要素の順序が逆になります。
### スクリーンの背景 \{#screen-background\}

:::link
メイン記事：[背景](paywall-head-picture)
:::

<ZoomImage id="builder-screen-settings-background.webp" width="350px" alt="スクリーン設定 — 背景" float="right" />

**Fill** は[スクリーンの背景](paywall-head-picture)をソリッドカラー、グラデーション、画像、または動画に設定します。背景はノッチやシステムバーの後ろのエリアも含め、デバイスのビューポート全体を埋めます（**Safe area** が有効な場合でも同様です）。

#### 背景動画のループ再生 \{#loop-background-video\}

**Loop** トグルを有効にすると、背景動画が連続してループ再生されます。

#### カスタムメディア ID の割り当て \{#assign-a-custom-media-id\}
[カスタム画像や動画](custom-media)と同様に、スクリーンの背景にカスタムメディアIDを割り当て、SDK内で参照することができます。
### スクリーンの余白 \{#screen-spacing\}

<ZoomImage id="builder-screen-settings-spacing.webp" width="350px" alt="Screen settings — Spacing" float="right" />

各辺（上、右、下、左）のスクリーンパディングを調整します。

### スクロール \{#scroll\}

<ZoomImage id="builder-screen-settings-scroll.webp" width="350px" alt="Screen settings — Scroll" float="right" />

オーバーフローの動作を制御します。**Vertical scroll** を有効にすると、スクリーンコンテンツがビューポートの高さを超えたときにスクロールできるようになります。

### 選択可能なグループ \{#selectable-groups\}

:::link
関連記事：[選択可能な要素とグループ](flow-selectable-elements)
:::

**Selectable groups** セクションには、現在の画面上のすべての選択可能なグループが一覧表示されます。[クイズ](onboarding-quizzes)、[プロダクト](paywall-product-block)、[タブ](builder-tabs)、[トライアルトグル](builder-toggles)、または任意の[カスタム選択可能要素](flow-selectable-elements#make-an-element-selectable)が対象です。

グループのエントリをクリックすると、名前の変更、タイプの変更、公開されている変数の確認、または削除ができます。
<ZoomImage id="builder-screen-settings-selectable-groups.webp" width="600px" alt="画面設定 — 選択可能グループ" />
## レイヤーを管理する \{#manage-layers\}

スクリーン上の各要素はレイヤーとして表示されます。**Layers** セクションには、アクティブなスクリーン上の要素の順序が表示されます。

:::important
フローのレイヤーは、グラフィックデザインソフトウェアのレイヤーのように重なり合うものではありません。代わりに、スクリーンの個々のコンポーネントを表しています。要素が重なり合うのは、[絶対配置または固定配置](manage-paywall-ui-elements)を使用している場合のみです。重なり順は、レイヤーツリー内の位置ではなく、`z-index` プロパティによって決まります。
:::
ツリー構造は親子関係を表しています。親レイヤーの矢印をクリックすると、子要素を展開または折りたたむことができます。

レイヤーを直接作成することはできません。[要素を追加](builder-elements)ビューから追加したすべての要素は、ツリー内の新しいレイヤーとして表示されます。

<ZoomImage id="flow-builder-layer-focus.webp" width="350px" alt="Flow builder layer view" float="left" />
* **レイヤーを選択する**: レイヤーをクリックして選択します。ビジュアルエディターはキャンバス上の対応する要素をハイライトし、右パネルにはその[デザイン](builder-styling)と[インタラクション](onboarding-navigation-branching)のプロパティが表示されます。
* **レイヤーを並べ替える**: ツリー内でレイヤーをドラッグ＆ドロップすると、親コンテナ内の順序を変更できます。ツリー内の順序は画面上の表示順序と一致します。
* **レイヤーの表示・非表示を切り替える**: レイヤーにカーソルを合わせると、右側に目 <Inline id="eye.svg" alt="Eye" /> アイコンが表示されます。クリックするとレイヤーの表示/非表示を切り替えられます。非表示のレイヤーはツリーに残りますが、ビジュアルエディターやデバイス上には表示されません。実行時にロジックで表示を制御するには、[条件付き表示](onboarding-element-visibility)を使用してください。
* **すべてのレイヤーを折りたたむ**: Layers セクションの右上にある折りたたみ <Inline id="collapse.svg" alt="Collapse" /> ボタンをクリックすると、ツリー全体を折りたたみます。
### レイヤーのアクション \{#layer-actions\}

三点アイコン <Inline id="context.svg" alt="Context" /> をクリックしてコンテキストメニューを開きます。

<ZoomImage id="flow-builder-layer-context-menu.webp" width="450px" alt="Flow builder layer context menu" />
| アクション | ショートカット | 説明 |
|--------|----------|-------------|
| **コピー** | ⌘C / Ctrl+C | レイヤーをクリップボードにコピーする |
| **ここに貼り付け** | ⌘V / Ctrl+V | コピー済みのレイヤーを子要素として貼り付ける |
| **複製** | ⌘D / Ctrl+D | 同じコンテナ内にレイヤーのコピーを作成する |
| **名前を変更** | | レイヤーの表示名を変更する。デフォルトでは、コンテンツまたはコンポーネントの種類が名前として使われる |
| **削除** | ⌘⌫ / Ctrl+Del | レイヤーとその子要素をすべて削除する |
| **ラップ** | | レイヤーを新しいコンテナで囲む: **Wrap in Horizontal Container** または **Wrap in Vertical Container** |
| **アンラップ / グループ解除** | | ラップしているコンテナを削除し、子要素を一段階上に移動する |
| **上に移動** | ↑ | 親コンテナ内でレイヤーを1つ上の位置に移動する |
| **下に移動** | ↓ | 親コンテナ内でレイヤーを1つ下の位置に移動する |
