フローは1つ以上のスクリーンで構成されます。各スクリーンはユーザーのジャーニーにおける1つのステップを表します。たとえば、ペイウォール、クイズ、プロダクト情報のスライドなどがこれにあたります。

各スクリーン上のエレメントはレイヤー階層で管理されます。
スクリーン、レイヤー、要素を管理するには、デフォルトの **Screens and Layers** ビューを開きます。スクリーンの順序と各スクリーンのレイヤー構造が表示されます。
## 画面の管理 \{#manage-screens\}

<ZoomImage id="flow-builder-screens-and-layers-left-sidebar.webp" width="400px" alt="Flow builder screens and layers view (both panels visible)" float="left" />

左パネルの上部セクションには、フロー内のすべての画面が一覧表示されます。各エントリには番号付きのラベルとサムネイルプレビューが表示されます。
* **スクリーンを選択する**: スクリーンのエントリをクリックするとアクティブになります。ビジュアルエディタに選択したスクリーンが表示され、下部の「レイヤー」セクションがそのレイヤー階層を反映して更新されます。
* **スクリーンを追加する**: 「スクリーン」セクション上部の <Inline id="plus.svg" alt="Plus" /> ボタンをクリックして、フローに新しい空のスクリーンを追加します。
* **テンプレートライブラリを開く**: 「スクリーン」セクション上部の <Inline id="template-library.svg" alt="Templates" /> ボタンをクリックして、[フローテンプレート](paywall-builder-templates)を閲覧・適用します。
* **スクリーンを並べ替える**: スクリーンのエントリをドラッグ＆ドロップして、フロー内の順番を変更します。
:::important
フローに未使用の空のスクリーンがある場合、公開できません。公開前にドラフトのスクリーンをすべて削除してください。
:::
### スクリーンのアクション \{#screen-actions\}

スクリーンのエントリにある三点アイコン <Inline id="context.svg" alt="Context" /> をクリックすると、コンテキストメニューが開きます。

<ZoomImage id="flow-builder-screen-actions.webp" width="600px" alt="Flow builder screen context menu" />
| アクション | ショートカット | 説明 |
|--------|----------|-------------|
| **Play Animation** | | このスクリーンで設定されたアニメーションをプレビューする |
| **Copy** | ⌘C / Ctrl+C | スクリーンをクリップボードにコピーする |
| **Paste here** | ⌘V / Ctrl+V | コピー済みのスクリーンを貼り付ける |
| **Duplicate** | ⌘D / Ctrl+D | スクリーンのコピーを作成してフローに追加する |
| **Rename** | | スクリーンの表示名を変更する |
| **Delete** | ⌘⌫ / Ctrl+Del | スクリーンをフローから削除する |

:::tip
クリップボードはフローをまたいで保持されます。あるフローでスクリーンや要素をコピーし、別のフローを開いて貼り付けることができます。
:::
:::warning
スクリーンを削除すると、そのスクリーンを指していた [Navigate to Screen](onboarding-navigation-branching) アクションは**ターゲットを失います**が、アクション自体は**削除されません**。新しい移動先を設定するか、アクションを削除してください。そうしないと、[フローのプレビューまたは公開](builder-save-publish#publish-a-flow)ができなくなります。
:::

## スクリーン設定 \{#screen-settings\}

アクティブなスクリーンのプロパティと設定を表示するには、スクリーンプレビューの空白部分をクリックしてください。右パネルがスクリーン設定ビューに切り替わります。

### システムUI \{#system-ui\}

<ZoomImage id="builder-screen-settings-system-ui.webp" width="350px" alt="Screen settings — System UI" float="right" />

スクリーンがデバイスのハードウェアとどのように連携するかを制御します。

* **Safe area** は、ノッチやシステムバーからコンテンツを保護するパディングを追加します。
* **Status bar** は、システムのステータスバー（時刻、バッテリー、信号アイコン）の表示と非表示を切り替えます。

### プログレスインジケーターへのスクリーンの含め方 \{#include-screen-in-progress-indicator\}

フローに[プログレスインジケーター](builder-loaders-and-progress-bars#progress-indicators)要素を追加すると、Adaptyはすべてのスクリーンに表示します。

**Include screen in progress indicator** のチェックを外すと、特定のスクリーンからプログレスインジケーターが非表示になります。ウェルカムスクリーン、最終的なペイウォール、または進行状況として登録したくないステップに使用してください。

### スクリーンレイアウト \{#screen-layout\}

:::link
詳細記事: [要素の配置](manage-paywall-ui-elements)
:::

<ZoomImage id="builder-screen-settings-layout.webp" width="350px" alt="スクリーン設定 — レイアウト" float="right" />

**Layout** セクションでは、スクリーンが子要素をどのように配置するかを設定します。これらのプロパティは、すべてのコンテナ要素で利用できます。

* **Free**: 子要素が独立して配置されます。
* **Vertical**: 要素がフレックスボックスの列のように上から下へ並びます。
* **Horizontal**: 要素がフレックスボックスの行のように左から右へ並びます。
縦方向・横方向のレイアウトでは、スペーシングと配置も設定できます。

* **Alignment**：クロス軸方向の要素の位置。
* **Gap**：隣接する要素間のスペース。
* **Distribution**：子要素の間および周囲のスペースの分配方法。

#### RTL レイアウト \{#rtl-layout\}

**Mirror for RTL** チェックボックスをオンにすると、右から左に読む文字体系に合わせてレイアウトをミラーリングできます。横方向コンテナ内の要素の順序が反転します。
### スクリーンの背景 \{#screen-background\}

:::link
関連記事：[背景](paywall-head-picture)
:::

<ZoomImage id="builder-screen-settings-background.webp" width="350px" alt="スクリーン設定 — 背景" float="right" />

**Fill** は[スクリーンの背景](paywall-head-picture)を単色、グラデーション、画像、動画に設定します。背景は、**Safe area** が有効な場合でも、ノッチやシステムバーの後ろを含むデバイスのビューポート全体を覆います。

#### 背景動画のループ再生 \{#loop-background-video\}

**Loop** トグルを有効にすると、背景動画を繰り返し再生します。

#### カスタムメディア ID の割り当て \{#assign-a-custom-media-id\}
[カスタムメディア](custom-media)と同様に、スクリーンの背景にカスタムメディアIDを割り当て、SDK内で参照することができます。
### スクリーンの余白 \{#screen-spacing\}

<ZoomImage id="builder-screen-settings-spacing.webp" width="350px" alt="Screen settings — Spacing" float="right" />

各辺（上、右、下、左）のスクリーンパディングを調整します。

### スクロール \{#scroll\}

<ZoomImage id="builder-screen-settings-scroll.webp" width="350px" alt="Screen settings — Scroll" float="right" />

オーバーフローの動作を制御します。**Vertical scroll** を有効にすると、スクリーンコンテンツがビューポートの高さを超えたときにスクロールできるようになります。

### セレクタブルグループ \{#selectable-groups\}

:::link
メイン記事：[セレクタブル要素とグループ](flow-selectable-elements)
:::

**Selectable groups** セクションには、現在の画面上のすべてのセレクタブルグループが表示されます。[クイズ](onboarding-quizzes)、[プロダクト](paywall-product-block)、[タブ](builder-tabs)、[トライアルトグル](builder-toggles)、または任意の[カスタムセレクタブル要素](flow-selectable-elements#make-an-element-selectable)から作成されたグループが対象です。

グループのエントリをクリックすると、名前の変更、タイプの変更、公開している変数の確認、または削除ができます。
<ZoomImage id="builder-screen-settings-selectable-groups.webp" width="600px" alt="画面設定 — 選択可能なグループ" />
## レイヤーの管理 \{#manage-layers\}

スクリーン上の各要素はレイヤーとして表示されます。「Layers」セクションには、アクティブなスクリーン上の要素の順序が表示されます。

:::important
フローのレイヤーは、グラフィックデザインソフトのレイヤーのように重なり合うわけではありません。各レイヤーはスクリーンの個々のコンポーネントを表しています。要素が重なり合うのは、[絶対配置または固定配置](manage-paywall-ui-elements)を使用している場合のみです。重なり順は、レイヤーツリー内の位置ではなく、`z-index` プロパティによって決まります。
:::
ツリー構造は親子関係を表しています。親レイヤーの矢印をクリックすると、子要素の展開・折りたたみができます。

レイヤーを直接作成することはできません。[要素の追加](builder-elements)画面から追加した要素はすべて、ツリー内に新しいレイヤーとして表示されます。

<ZoomImage id="flow-builder-layer-focus.webp" width="350px" alt="Flow builder layer view" float="left" />
* **レイヤーを選択する**: レイヤーをクリックすると選択できます。ビジュアルエディタはキャンバス上の対応する要素をハイライトし、右パネルにその[デザイン](builder-styling)と[インタラクション](onboarding-navigation-branching)のプロパティが表示されます。
* **レイヤーを並べ替える**: ツリー内でレイヤーをドラッグ＆ドロップして、親コンテナ内の順序を変更します。ツリー内の順序は画面上の表示順序と一致します。
* **レイヤーの表示・非表示を切り替える**: レイヤーにカーソルを合わせると、右側に目 <Inline id="eye.svg" alt="Eye" /> アイコンが表示されます。クリックするとレイヤーの表示・非表示を切り替えられます。非表示のレイヤーはツリーに残りますが、ビジュアルエディタや端末上には表示されません。実行時にロジックで表示を制御するには、[条件付き表示](onboarding-element-visibility)を使用してください。
* **すべてのレイヤーを折りたたむ**: Layers セクションの右上にある折りたたみ <Inline id="collapse.svg" alt="Collapse" /> ボタンをクリックすると、ツリー全体を折りたたむことができます。
### レイヤーのアクション \{#layer-actions\}

三点アイコン <Inline id="context.svg" alt="Context" /> をクリックすると、コンテキストメニューが開きます。

<ZoomImage id="flow-builder-layer-context-menu.webp" width="450px" alt="Flow builder layer context menu" />
| アクション | ショートカット | 説明 |
|--------|----------|-------------|
| **Copy** | ⌘C / Ctrl+C | レイヤーをクリップボードにコピーする |
| **Paste here** | ⌘V / Ctrl+V | コピーしたレイヤーを子要素として貼り付ける |
| **Duplicate** | ⌘D / Ctrl+D | 同じコンテナ内にレイヤーのコピーを作成する |
| **Rename** | | レイヤーの表示名を変更する。デフォルトでは、レイヤーのコンテンツまたはコンポーネントタイプが名前として使用される |
| **Delete** | ⌘⌫ / Ctrl+Del | レイヤーとそのすべての子要素を削除する |
| **Wrap** | | レイヤーを新しいコンテナでラップする: **Wrap in Horizontal Container** または **Wrap in Vertical Container** |
| **Unwrap / Ungroup** | | ラッピングコンテナを削除し、その子要素を一階層上に移動する |
| **Move up** | ↑ | 親コンテナ内でレイヤーを一つ上に移動する |
| **Move down** | ↓ | 親コンテナ内でレイヤーを一つ下に移動する |
