バーチャル通貨のクレジットを販売するには、まずバーチャル通貨を一度定義し、プロダクトにリンクします。

この記事で説明する2つのステップで設定します。プロダクトをリンクすると、購入または更新のたびに自動的にクレジットが付与されます。

## バーチャル通貨を作成する \{#create-a-virtual-currency\}

Adapty ダッシュボードで **Products** > [**Virtual currency**](https://app.adapty.io/virtual-currency) を開きます。

<ZoomImage id="virtual-currency-tab.webp" width="700px" alt="The Virtual currency tab under Products, showing the empty state and the New virtual currency button" />

バーチャル通貨を作成するには：

1. **New virtual currency** をクリックします。ドロワーが **General** ステップで開きます。
2. 通貨の詳細を入力します：
    - **Code**：アプリ内で使用する、`COINS` のような一意の**固定**識別子です。サーバーサイド API で通貨を識別し、各ユーザーの残高を追跡します。
    
ラテン文字、数字、アンダースコアを使用し、最大32文字。小文字は自動的に大文字に変換されます。
    - **Name**: `Gold` など、表示名を設定します。後から変更可能です。
    - **Description**: 通貨に関する任意のメモです。
3. **Continue** をクリックして **Link Products** ステップに進みます。

:::important
通貨の **Code** は作成後に変更できません。**Name** は後から編集できます。
:::

新しい通貨を追加すると、すべてのユーザープロファイルの初期残高は0になります。

<ZoomImage id="new-currency-step-1.webp" width="500px" alt="新しい仮想通貨ドロワーの「General」ステップ。Code、Name、Descriptionフィールドが表示されています" />

## プロダクトをリンクする \{#link-products\}

バーチャル通貨にプロダクトをリンクすると、購入またはサブスクリプションの更新時にその通貨のクレジットが付与されます。1つのプロダクトを複数の通貨にリンクしたり、1つの通貨を複数のプロダクトにリンクしたりすることができます。プロダクトは **Link Products** ステップで今すぐリンクするか、後から通貨を編集する際に追加することもできます。

**Link Products** ステップでは、**Associated products** セクションにその通貨を付与するプロダクトの一覧が表示されます。プロダクトを追加するには、**Add associated products** をクリックしてプロダクトを選択してください。

各付与は残高を置き換えるのではなく加算されるため、異なるプロダクトからのクレジットが累積されます。クレジットの設定はプロダクトタイプによって異なります：

- **サブスクリプションプロダクト**には、3つのクレジット設定があります：
    - **Credit per cycle**（必須）：トライアルから有料への切り替え時を含む、各更新時に付与されるクレジット数。
    - **Credit on trial start**（任意）：無料トライアル開始時に一度だけ付与される別枠のクレジット数。
    - **Credits expire at the end of each billing cycle**（トグル）：オンにすると、各更新時に未使用クレジットが0にリセットされてから、新しいサイクルのクレジットが付与されます。オフにすると、クレジットはサイクルをまたいで積み上がり、有効期限はありません。
- **買い切り購入プロダクト**：**Credit amount**（購入ごとに付与されるクレジット数）を設定します。買い切り購入のクレジットに有効期限はありません。

**Save** をクリックして、プロダクトリンクとともに通貨を作成します。

プロダクトリンクは今後の購入にのみ適用されます。新たにリンクされたプロダクトは、次回の購入または更新時からクレジットが付与されます。Adapty は現在のサブスクリプション登録者に対して自動的に通貨を付与しません。登録者はサブスクリプションを更新する必要があります。リンクを削除すると、以降の付与は停止しますが、すでに付与済みのクレジットはそのまま残ります。

有効期限トグルがオンの場合、期限付きクレジットはサブスクリプションのライフサイクルに従います：

- 請求の再試行中やグレース期間中、Adaptyは新しいクレジットを付与しません。
- キャンセル後も、有料期間の終了まではクレジットが維持されます。
- サブスクリプションが期限切れになると、Adaptyは残りのクレジットを0にリセットします。

<ZoomImage id="new-currency-step-2.webp" width="500px" alt="「Link Products」ステップ。買い切り購入プロダクトとサブスクリプションプロダクトにそれぞれのクレジット設定がリンクされている" />

## 次のステップ \{#next-steps\}

通貨を作成してプロダクトを紐付けると、購入時に自動でクレジットが付与されます。ここからできること：

- 各ユーザーの残高は[バーチャル通貨残高](virtual-currency-balance)ページで確認できます。
- 残高の付与・消費・読み取りは、サーバーサイド API を通じてランタイムで行えます：[バーチャル通貨トランザクションの作成](api-adapty/operations/createVirtualCurrencyTransaction)と[バーチャル通貨残高の一覧取得](api-adapty/operations/listVirtualCurrencyBalances)。実際の使用例については、[バーチャル通貨クイックスタート](virtual-currency-quickstart)を参照してください。
