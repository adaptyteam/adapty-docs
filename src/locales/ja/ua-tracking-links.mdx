トラッキングリンクを使うと、ユーザーの流入元を計測し、インストールを広告キャンペーンに紐づけることができます。
ユーザーが広告をクリックすると、Adapty はそのクリックを記録し、後で SDK から送信されたインストールイベントと照合します。これにより、[Analytics ページ](ua-analytics)でどのチャネル・キャンペーン・広告セット・広告が最も多くの収益をもたらしているかを確認できます。

作成できるトラッキングリンクは次の 2 種類です：

- **One link** — ユーザーのプラットフォームを自動検出し、クリックを記録して App Store または Google Play にリダイレクトするユニバーサルリンクです。
- **ストア固有リンク** — プラットフォームに応じたリンクで、クリックを記録しつつ App Store または Google Play へ自動でリダイレクトします。遅延ディープリンクのパラメーターを追加することもできます。
## トラッキングリンクを作成する \{#create-tracking-links\}

トラッキングリンクを作成するには：

1. Adapty Attribution ダッシュボードのサイドバーメニューから **Tracking links** に移動し、**Create link** をクリックします。

<ZoomImage id="new-tracking-link.webp" width="700px" />

2. リストからアプリを選択し、**Next** をクリックします。

<ZoomImage id="ua-choose-app.webp" width="700px" />

3. トラッキングしたいキャンペーンや広告に合わせて、リンクのパラメーターを入力します。
| パラメータ | 説明 |
|-----------|------|
| **Name** | トラッキングリンクの内部名称。 |
| **Channel** | Meta、Reddit、TikTok などのトラフィックの流入元。アナリティクスでキャンペーンをグループ化するために使用されます。 |
| **Campaign ID** | 広告プラットフォームにおけるキャンペーンの一意識別子。 |
| **Campaign name** | キャンペーンのわかりやすい名称。 |
| **Ad set ID** | 広告プラットフォームにおける広告セット（広告グループ）の一意識別子。 |
| **Ad set name** | 広告セットの名称。 |
| **Ad ID** | 個々の広告クリエイティブの一意識別子。 |
| **Ad name** | 広告クリエイティブまたはバリエーションの名称。 |
4. デフォルトでは、One Link が作成されます。これはユーザーのプラットフォームを自動的に検出し、クリックを追跡した後、App Store または Google Play にリダイレクトします。

プラットフォームごとに別々のリダイレクト URL を使用したい場合は、**One Link** チェックボックスをオフにして、プラットフォーム固有のストアリンクを手動で入力してください。

<ZoomImage id="one-link.webp" width="700px" />
5. **Create**をクリックします。
6. トラッキングリンクのページを開き、いずれかのセクションから**クリックリンク**をコピーします：
   - **One link** – クリックを計測し、ユーザーを自動的に適切なストアにリダイレクトするリンクです。
   - **iOS link** または **Android link** — ストアごとに別々のリンクを使いたい場合のオプションです。

:::tip
[遅延データを扱う](ua-deferred-data)ために追加のリンクパラメーターを設定することもできます。たとえば、遅延ディープリンクを実装できます。
:::

<ZoomImage id="click-link.webp" width="700px" />
7. 広告プラットフォームにアクセスして、広告のリンクを広告の遷移先 URL として貼り付けてください。

これにより、アプリのインストールがどの広告やキャンペーンから来たかと照合され、**Analytics** ページでキャンペーンの効果を測定できるようになります。
