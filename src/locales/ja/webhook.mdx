Webhookは、[イベント](webhook-event-types-and-fields#webhook-event-types)に関するリアルタイム通知を受け取る効率的な方法です。特にサブスクリプションや購入の変化を追跡するのに役立ちます。これにより、サブスクライバーの状態を監視し、適切に対応できます。

常時ポーリングが必要なAPIリクエストとは異なり、Webhookは一度設定するだけで、イベント発生時にHTTP経由でデータを自動送信します。

Webhookを統合することで、次のことが可能になります：

- バックエンドシステムでサブスクリプションと購入を追跡する。
- サブスクリプションのライフサイクルに基づいてプロセスやワークフローを自動化する。
- アプリのメリットを伝えたり、解約の判断に対応したり、課金の問題を処理したりして、サブスクライバーとのエンゲージメントを高める。
- ユーザー行動の詳細な分析を行う。

**連携の特徴**

| 連携の特徴 | 説明 |
| :------------------------- | :---------------------------------------------------------- |
| スケジュール | リアルタイム更新 |
| データの方向 | 一方向のデータ送信：AdaptyからあなたのサーバーへAへ |
| Adaptyの連携フロー | イベントはAdaptyサーバーが受信次第送信されます |

## Webhookに送信されるイベント \{#events-sent-to-webhook\}

Webhookに送信できるすべてのイベントタイプは、[Webhookイベントタイプとフィールド](webhook-event-types-and-fields)のページで確認できます。

すべてのイベントをWebhookに送信することも、一部だけを選択することもできます。どのイベントが必要かどうかは、[イベントフロー](event-flows)のページを参照して判断してください。不要なイベントタイプは、[Webhook連携の設定](set-up-webhook-integration#configure-webhook-integration-in-the-adapty-dashboard)時に無効化できます。また、AdaptyデフォルトのイベントIDを独自のIDに置き換えることも可能です。
