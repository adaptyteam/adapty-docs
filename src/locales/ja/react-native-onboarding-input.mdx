ユーザーがクイズの質問に回答したり、入力フィールドにデータを入力したりすると、`onStateUpdatedAction` メソッドが呼び出されます。フィールドの種類をコード内で保存・処理できます。

例：

```javascript
// フルスクリーン表示
const unsubscribe = view.registerEventHandlers({
  onStateUpdated(action, meta) {
    // データを処理する 
  },
});

// 埋め込みウィジェット
<AdaptyOnboardingView
  onboarding={onboarding}
  eventHandlers={{
    onStateUpdated(action, meta) {
      // データを処理する 
    },
  }}
/>
```

アクションのフォーマットは[こちら](https://react-native.adapty.io/types/onboardingstateupdatedaction)をご覧ください。

## ユースケース \{#use-cases\}

### ユーザープロファイルをデータで拡充する \{#enrich-user-profiles-with-data\}

入力データをユーザープロファイルにすぐに紐づけて、同じ情報を二度聞かないようにするには、アクションを処理する際に入力データで[ユーザープロファイルを更新](react-native-setting-user-attributes)する必要があります。

たとえば、`name` というIDのテキストフィールドにユーザー名を入力してもらい、その値をユーザーの名（first name）として設定したい場合や、`email` フィールドにメールアドレスを入力してもらう場合、アプリのコードは次のようになります：

```javascript showLineNumbers
// フルスクリーン表示
const unsubscribe = view.registerEventHandlers({
  onStateUpdated(action, meta) {
    // ユーザーの設定や回答を保存する
    if (action.elementType === 'input') {
      const profileParams = {};
      
      // elementIdを適切なプロファイルフィールドにマッピングする
      switch (action.elementId) {
        case 'name':
          if (action.value.type === 'text') {
            profileParams.firstName = action.value.value;
          }
          break;
        case 'email':
          if (action.value.type === 'email') {
            profileParams.email = action.value.value;
          }
          break;
      }
      
      // 更新するデータがある場合はプロファイルを更新する
      if (Object.keys(profileParams).length > 0) {
        adapty.updateProfile(profileParams).catch(error => {
          // エラーを処理する
        });
      }
    }
  },
});

// 埋め込みウィジェット
<AdaptyOnboardingView
  onboarding={onboarding}
  eventHandlers={{
    onStateUpdated(action, meta) {
      // ユーザーの設定や回答を保存する
      if (action.elementType === 'input') {
        const profileParams = {};
        
        // elementIdを適切なプロファイルフィールドにマッピングする
        switch (action.elementId) {
          case 'name':

              profileParams.firstName = action.value.value;
            }
            break;
          case 'email':
            if (action.value.type === 'email') {
              profileParams.email = action.value.value;
            }
            break;
        }
        
        // 更新するデータがある場合はプロファイルを更新する
        if (Object.keys(profileParams).length > 0) {
          adapty.updateProfile(profileParams).catch(error => {
            // エラーを処理する
          });
        }
      }
    },
  }}
/>
```

### 回答をもとにペイウォールをカスタマイズする \{#customize-paywalls-based-on-answers\}

オンボーディングのクイズを活用して、オンボーディング完了後にユーザーへ表示するペイウォールをカスタマイズすることもできます。

たとえば、スポーツの経験についてユーザーに質問し、ユーザーグループごとに異なるCTAやプロダクトを表示できます。

1. オンボーディングビルダーで[クイズを追加](onboarding-quizzes)し、各選択肢に意味のあるIDを割り当てます。

<ZoomImage id="experience.webp" />

2. IDをもとにクイズの回答を処理し、ユーザーに[カスタム属性を設定](react-native-setting-user-attributes)します。

```javascript showLineNumbers
// フルスクリーン表示
const unsubscribe = view.registerEventHandlers({
  onStateUpdated(action, meta) {
    // クイズの回答を処理してカスタム属性を設定する
    if (action.elementType === 'select') {
      const profileParams = {};
      
      // クイズの回答をカスタム属性にマッピングする
      switch (action.elementId) {
        case 'experience':
          // カスタム属性'experience'に選択した値（beginner、amateur、pro）を設定する
          profileParams.codableCustomAttributes = {
            experience: action.value.value
          };
          break;
      }
      
      // 更新するデータがある場合はプロファイルを更新する
      if (Object.keys(profileParams).length > 0) {
        adapty.updateProfile(profileParams).catch(error => {
          // エラーを処理する
        });
      }
    }
  },
});

// 埋め込みウィジェット
<AdaptyOnboardingView
  onboarding={onboarding}
  eventHandlers={{
    onStateUpdated(action, meta) {
      // クイズの回答を処理してカスタム属性を設定する
      if (action.elementType === 'select') {
        const profileParams = {};
        
        // クイズの回答をカスタム属性にマッピングする
        switch (action.elementId) {
          case 'experience':
            // カスタム属性'experience'に選択した値（beginner、amateur、pro）を設定する
            profileParams.codableCustomAttributes = {
              experience: action.value.value
            };
            break;
        }
        
        // 更新するデータがある場合はプロファイルを更新する
        if (Object.keys(profileParams).length > 0) {
          adapty.updateProfile(profileParams).catch(error => {
            // エラーを処理する
          });
        }
      }
    },
  }}
/>
```

3. カスタム属性の値ごとに[セグメントを作成](segments)します。
4. [プレースメント](placements)を作成し、作成した各セグメントに[オーディエンス](audience)を追加します。
5. アプリのコードでそのプレースメントの[ペイウォールを表示](react-native-paywalls)します。オンボーディングにペイウォールを開くボタンがある場合は、[そのボタンのアクションへの応答](react-native-handling-onboarding-events#opening-a-paywall)としてペイウォールのコードを実装してください。
