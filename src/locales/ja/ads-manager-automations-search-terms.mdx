Discovery および Search Match キャンペーンは検索語句データを生成します。そのデータを整理されたキーワードリストに変換するには、レポートをダウンロードし、語句をフィルタリングして、広告グループに手動で追加する必要があります。検索語句の自動化はこの作業を自動的に行います。語句が設定した条件を満たすと、ルールは設定したアクションに基づいてその語句を処理します。

検索語句ルールのアクションタイプは2種類あります。
- **Add as keyword**: ターゲットの広告グループにその語句を完全一致キーワードとして追加し、オプションでソースキャンペーンでは除外キーワードに設定することで重複出費を防ぎます。
- **Add as negative keyword**: プロモーションなしでその語句を直接除外します。ディスカバリーキャンペーンやサーチマッチキャンペーンから無関係または費用対効果の低い検索語句を除外するために使用します。
**Add as keyword** の典型的なユースケース: Discovery キャンペーンや Search Match キャンペーンで実際のユーザーの検索クエリを収集し、ルールを使ってパフォーマンスが一定のしきい値を超えた語句を検出して、完全一致キーワードとして Probing キャンペーンに追加しながら、元のキャンペーンではネガティブキーワードに設定します。Probing キャンペーンとは、プロモートするキーワードを管理された入札額でテストするために専用に作成された Apple Search Ads キャンペーンのことです。

**Add as negative keyword** の典型的なユースケース: インプレッション数が多いのにタップが一度もないような、頻繁に表示されても全くコンバージョンしない語句を自動でネガティブキーワードに設定し、無駄な予算消費を防ぎます。
## 検索語句オートメーションルールを作成する \{#create-a-search-term-automation-rule\}

検索語句オートメーションルールはテンプレートから作成することも、最初から手動で作成することもできます。

:::note
ルールを作成する前に、ディスカバリーキャンペーンまたはSearch Matchキャンペーンがアクティブに実行されており、検索語句データを収集していることを確認してください。完全一致のみのキャンペーンは検索語句レポートを生成しないため、ルールが処理する対象がなくなります。
:::

### テンプレートからルールを作成する \{#from-a-template\}

テンプレートからルールを作成するには：
1. 左サイドバーで **Automations** に移動し、**Templates** をクリックします。
   <ZoomImage id="asa-rule-templates.webp" width="700px" alt="Automation rule templates" />
2. テンプレートを選択して **Next** をクリックします。
   <ZoomImage id="asa-search-template.webp" width="700px" alt="Choose a template" />
3. あらかじめ入力された設定を確認・調整します。
   - **Rule name**: テンプレート名と現在の日付が自動的に設定されます。
   - **Apply to**: ルールが検索語を探すキャンペーングループ、アプリ、キャンペーン、または広告グループを選択します。
   - **Conditions**: 必要に応じて事前設定された条件を変更します。
   - **Actions**: 必要に応じてターゲットの広告グループ、CPT 入札額、ネガティブキーワードのスコープを調整します。
   - **Schedule**: ルールの実行頻度を設定します。
   <ZoomImage id="asa-templated-rule-search.webp" width="700px" alt="Templated rule settings" />
4. **Save** をクリックしてルールを有効化します。
### 手動で作成する \{#manually\}

カスタム検索語句の自動化ルールをゼロから作成するには：
1. 左側のサイドバーで **Automations** に移動し、**Create rule** をクリックして、ルールタイプとして **Search terms** を選択します。
   <ZoomImage id="asa-create-rule.webp" width="700px" alt="Create rule button" />
2. ルールの目的を識別できるわかりやすい **Rule name** を入力します。
3. **Apply to** セクションで、ルールが検索キーワードを検索するキャンペーングループ、アプリ、キャンペーン、または広告グループを選択します。
   <ZoomImage id="asa-search-rule-apply-to.gif" width="700px" alt="Apply to selection" />
4. **Add condition** をクリックして、リストから[指標](adapty-ads-manager-metrics)を選択します。
指標はアカウントの通貨で選択した期間について計算されます。データはほぼリアルタイムで更新されるため、ルールは常に最新のパフォーマンスデータを使用します。
   <ZoomImage id="asa-search-condition.webp" width="700px" alt="Add condition" />
5. 期間（例：直近3日間や直近7日間）を設定し、比較演算子を選択して、しきい値を入力します。
6. 条件を追加するには、**Add condition** をクリックし、左側で **And** または **Or** 演算子を選択します。
   <ZoomImage id="search-combine.gif" width="700px" alt="Combine conditions" />
7. **Action** セクションで、検索キーワードが条件を満たした場合の動作を選択します。
**キーワードとして追加**

マッチする検索語句を、ターゲット広告グループに完全一致キーワードとして追加します。
- **Target ad groups**: ターゲットとする広告グループを選択します。ディスカバリーパイプラインを構築する場合は、Probing またはその他の構造化キャンペーン内の広告グループを選択してください。
   - **CPT bid**: プロモートするキーワードごとに初期のタップ単価（CPT）を設定します。オプション：広告グループのデフォルト入札額、検索語句の現在の CPT、または指定した値。
   - **Skip if keyword already exists**: 有効にすると、ターゲット広告グループにすでに存在する語句をスキップします。
   - **Add as negative**: 同じ語句を除外キーワードとして追加し、同じトラフィックに二重で費用を払わないようにします。
     - **Scope**: 除外キーワードを追加する広告グループを選択します。
:::tip
**Add as negative** を同じルールで有効にすると、キーワードをストラクチャードキャンペーンに昇格させながら、ソースで除外設定も一括で行えます。これにより、ディスカバリーキャンペーンをクリーンに保ちつつ、キーワードファネルを自動的に構築できます。
:::

**Add as negative keyword**

マッチした検索語句を昇格させずに除外します。

- **Scope**: 除外キーワードを追加する広告グループまたはキャンペーンを選択します。
- **Match type**: **Exact** または **Broad** を選択します。
DiscoveryおよびMax Conversionキャンペーンから、関連性の低い検索語句や品質の低い検索語句を除外する場合に使用します。たとえば、インプレッションが50件以上あってタップ数が0の語句を自動的に除外するといった使い方ができます。

   <ZoomImage id="asa-search-rule-action.webp" width="700px" alt="Rule actions" />
8. **Schedule** セクションでは：
   - 頻度を選択します：**Every day**、**Every 2 days**、**Every week** など。
   - 実行時刻を選択します（すべての時刻はUTCです）。
ルールはUTC時刻で設定したスケジュールに従って実行されます。通常は数分以内に完了し、その後はLogsおよびメインダッシュボードで変更内容を確認できます。
   <ZoomImage id="asa-schedule.webp" width="700px" alt="スケジュール設定" />
9. **Save** をクリックしてルールを作成します。

ルールが実行されたら、**Automations** → **Logs** に移動し、対象ルールのエントリを開きます。実行が成功した場合、評価された各検索語句がソースキャンペーン、ターゲット広告グループ、アクション結果、否定結果とともに一覧表示されます。語句が表示されない場合は、条件が満たされていません。しきい値を見直すか、ルックバックウィンドウを延長してください。
## ベストプラクティス \{#best-practices\}
- **ディスカバリーまたはサーチマッチキャンペーンをソースとして使用する**: これらのキャンペーンタイプは実際のユーザークエリを収集するため、ルールが評価できる検索語のプールが大きくなります。
- **しきい値をルックバックウィンドウに合わせる**: 7日間で2件以上のダウンロードが妥当な出発点です。ウィンドウを長くする（14〜30日）とハードルが実質的に下がり、コンバージョン頻度が低い語でも通過できるようになります。流入量の多いアプリでは、ウィンドウを短くしてしきい値を引き上げてください。
- **プロモート時は必ずソースでネガティブ設定する**: プロービングキャンペーンに語をキーワードとして追加したとき、ディスカバリーでネガティブ設定しないと、両方のキャンペーンが同じクエリで競合します。同じルール内で **Add as negative** を有効にしてください。
- **ターゲットの広告グループを意図的に選ぶ**: プロモートした語は、広範なキャンペーンではなく特定のプロービング広告グループにルーティングします。これによりキーワード構造が整理され、パフォーマンス分析がしやすくなります。
- **実行後はログを確認する**: ログタブでどの語がプロモートされ、どこに追加されたかを確認します。初期段階ではセットアップ後にルールを手動で実行し、期待どおりに動作しているか検証してください。ログの読み方については[オートメーション](ads-manager-automations#explore-logs)を参照してください。
- **プロモートしたキーワードにキーワードルールが作用するまで時間を置く**: 同じキャンペーンでキーワードルールも使用している場合、新しくプロモートしたキーワードを除外するか、ルールが適用されるまで1〜2日待ちましょう。パフォーマンス履歴がない新しい語にキーワードルールが発動すると、コンバージョンする前に入札が削られてしまうことがあります。
- **インプレッションは多いのにタップがゼロの語にはネガティブキーワードとして追加する**: ディスカバリーやMax Conversionキャンペーンでは、無関係な検索語が表示されることがよくあります。「インプレッション > 50 かつ タップ = 0」というルールを設定することで、こうした語を自動的にネガティブ設定し、無駄なインプレッションがこれ以上積み重ならないようにできます。
## 検索語句をエクスポートする \{#export-search-terms\}

検索語句テーブルをCSVとしてエクスポートするには、テーブル上部のダウンロードアイコンをクリックして **Export current page** または **Export all pages** を選択します。

**Export all pages** は全ページにわたるすべての検索語句を1つのファイルにダウンロードします。進捗モーダルがダウンロードを追跡し、いつでもキャンセルできます。

テーブルは表示されている状態、つまり選択して表示している列でエクスポートされます。
