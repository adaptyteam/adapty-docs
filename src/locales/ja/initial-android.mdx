## 初期連携のチェックリスト \{#checklist-for-the-initial-integration\}

- [ ] Adaptyでアカウントを作成し、モバイルアプリ名とカテゴリを入力すると、Adaptyプラットフォーム内でアプリが設定されます。
- [ ] Google Cloud ConsoleでDeveloper APIを[有効化する](enabling-of-devepoler-api)
- [ ] Google Cloud Consoleで[サービスアカウントを作成する](create-service-account)
- [ ] Google Play Consoleで[サービスアカウントに権限を付与する](grant-permissions-to-service-account)
- [ ] Google Cloud Consoleで[サービスアカウントキーファイルを生成する](create-service-account-key-file)
- [ ] Adapty ダッシュボードで[Google Play連携を設定する](google-play-store-connection-configuration)
- [ ] Google Play Consoleで[リアルタイム デベロッパー通知（RTDN）を有効化する](enable-real-time-developer-notifications-rtdn)
- [ ] Adapty SDKをインストールして設定する（必要なフレームワーク向けのSDKを1つ以上インストールしてください）
  - [ ] [Android向けAdapty SDKをインストールする](sdk-installation-android)
  - [ ] [React Native向けAdapty SDKをインストールする](sdk-installation-reactnative)
  - [ ] [Flutter向けAdapty SDKをインストールする](sdk-installation-flutter)
  - [ ] [Kotlin Multiplatform向けAdapty SDKをインストールする](sdk-installation-kotlin-multiplatform)
  - [ ] [Unity向けAdapty SDKをインストールする](sdk-installation-unity)
- [ ] アプリケーションをビルドして実行する。スナップショットやサンドボックス環境での実行で問題ありません。

:::note
変更が反映されるまで最低24時間かかりますが、[回避策](https://stackoverflow.com/a/60691844)があります。[Google Play Console](https://play.google.com/apps/publish/)で任意のアプリケーションを開き、**Monetize**セクションで**Products** → **Subscriptions**/**In-app products**に進みます。任意のプロダクトの説明を変更して保存してください。これで動作するようになります。アプリ内の変更は元に戻して構いません。
:::

初期連携が完了したら、[Adaptyの機能を使い始める](product)ことができます。

ペイウォールやプロダクトをモバイルアプリに表示したり、アナリティクスを動作させたりするには、アプリのコードを変更する必要があります。具体的には、少なくとも[ペイウォールを表示する](android-quickstart-paywalls)ことが必要で、ペイウォールビルダーで作成されていないペイウォールを使用している場合は、アプリ内で[購入処理を実装する](android-making-purchases)必要があります。

:::danger
アプリのリリース前にリリースチェックリストを確認してください

アプリケーションをリリースする前に、[リリースチェックリスト](release-checklist)を必ずしっかりと確認してください。このチェックリストは、すべての必要な手順が完了していることを確認し、連携の成功を評価するための基準を提供します。
:::
