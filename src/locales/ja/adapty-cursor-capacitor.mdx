このガイドでは、AIコーディングツールを使ってCapacitorアプリにAdaptyを段階的に統合する方法を説明します。適切なAdaptyドキュメントを正しい順序でAIに提供していきます。

<SkillPromo />
## はじめる前に：ダッシュボードの設定 \{#before-you-start-dashboard-setup\}

AdaptyはSDKコードを書く前に、いくつかのダッシュボード設定が必要です。インタラクティブなLLMスキルを使うか、ダッシュボードから手動で設定できます。

### スキルを使う方法（推奨） \{#skill-approach-recommended\}

Adapty CLIスキルを使うと、LLMがアプリ、プロダクト、アクセスレベル、ペイウォール、プレースメントを直接設定できます。各ステップでダッシュボードを開く必要がありません。[ストアの接続](integrate-payments)だけダッシュボードで行う必要があります。

```
npx skills add adaptyteam/adapty-cli --skill adapty-cli
```

スキルを追加したら、エージェントで `/adapty-cli` を実行してください。ストアの接続が必要なタイミングも含め、各ステップをガイドしてくれます。

### ダッシュボードを使った設定方法 \{#dashboard-approach\}

コードを書く前に手動で設定を行う場合、事前に準備が必要なものがあります。LLM がダッシュボードの値を自動で調べることはできないため、あなた自身が提供する必要があります。

1. **アプリストアを連携する**: Adapty ダッシュボードで **App settings → General** に移動します。Capacitor アプリが両方のプラットフォームを対象とする場合は、App Store と Google Play の両方を連携してください。これは購入機能を動作させるために必要です。
   [アプリストアを連携する](integrate-payments)
2. **公開 SDK キーをコピーする**: Adapty ダッシュボードで **App settings → General** に移動し、**API keys** セクションを確認します。コード内では、この文字列を `adapty.activate()` に渡します。

3. **プロダクトを少なくとも 1 つ作成する**: Adapty ダッシュボードで **Products** ページに移動します。コード内でプロダクトを直接参照する必要はありません — Adapty はペイウォールを通じてプロダクトを提供します。
   [プロダクトを追加する](quickstart-products)
4. **ペイウォールとプレースメントを作成する**: Adapty ダッシュボードの **Paywalls** ページでペイウォールを作成し、**Placements** ページでプレースメントに割り当てます。コード内では、プレースメント ID が `adapty.getFlow()` に渡す文字列です。
   [ペイウォールを作成する](quickstart-paywalls)
5. **アクセスレベルを設定する**: Adapty ダッシュボードの **Products** ページで、プロダクトごとに設定します。コード内では `profile.accessLevels['premium']?.isActive` で確認します。デフォルトの `premium` アクセスレベルは、ほとんどのアプリで利用できます。購入ユーザーがプロダクトによって異なる機能にアクセスできる場合（たとえば `basic` プランと `pro` プランなど）、コーディングを始める前に[追加のアクセスレベルを作成](assigning-access-level-to-a-product)してください。
:::tip
5つすべてが揃ったら、コードを書く準備完了です。LLMに「パブリックSDKキーはX、プレースメントIDはY」と伝えることで、正しい初期化とフロー取得コードを生成できます。
:::
### 準備ができたら設定すること \{#set-up-when-ready\}

コーディングを始める前に必須ではありませんが、統合が成熟するにつれて必要になります：

- **A/B テスト**: **Placements** ページで設定します。コードの変更は不要です。
  [A/B テスト](ab-tests)
- **追加のペイウォールとプレースメント**: 異なるプレースメントIDで `getPaywall` の呼び出しを追加します。
- **アナリティクス連携**: **Integrations** ページで設定します。セットアップは連携先によって異なります。[アナリティクス連携](analytics-integration)と[アトリビューション連携](attribution-integration)を参照してください。

## AdaptyドキュメントをLLMに提供する \{#feed-adapty-docs-to-your-llm\}

### Context7を使う（推奨）

[Context7](https://context7.com)は、LLMが最新のAdaptyドキュメントに直接アクセスできるMCPサーバーです。何を質問するかに応じて、LLMが自動的に適切なドキュメントを取得するため、URLを手動で貼り付ける必要はありません。

Context7は**Cursor**、**Claude Code**、**Windsurf**、その他のMCP対応ツールで動作します。セットアップするには、次のコマンドを実行してください：

```
npx ctx7 setup
```

これにより、エディタが検出され、Context7サーバーが設定されます。手動セットアップについては、[Context7 GitHubリポジトリ](https://github.com/upstash/context7)を参照してください。
設定が完了したら、プロンプトでAdaptyライブラリを参照してください：

```
Use the adaptyteam/adapty-docs library to look up how to install the Capacitor SDK
```

:::warning
Context7によってドキュメントのリンクを手動で貼り付ける手間は省けますが、実装の順序は重要です。すべてが正しく動作するよう、以下の[実装手順](#implementation-walkthrough)にしたがって順番に進めてください。
:::
### プレーンテキストのドキュメントを使う \{#use-plain-text-docs\}

Adaptyのドキュメントはプレーンテキストのマークダウンとして取得できます。URLの末尾に `.md` を追加するか、記事タイトルの下にある **Copy for LLM** をクリックしてください。例：[adapty-cursor-capacitor.md](https://adapty.io/docs/ja/adapty-cursor-capacitor.md)

以下の[実装ウォークスルー](#implementation-walkthrough)の各ステージには、貼り付け用の `.md` リンクが含まれた「LLMに送るもの」ブロックがあります。

一度に複数のドキュメントが必要な場合は、以下の[インデックスファイルとプラットフォーム別サブセット](#plain-text-doc-index-files)を参照してください。

## 実装ウォークスルー \{#implementation-walkthrough\}

このガイドの残りでは、実装順序に沿ってAdapty統合を進めていきます。各ステージには、LLMに送るドキュメント、完了時に確認できること、よくある問題が含まれています。

### インテグレーションを計画する \{#plan-your-integration\}

コーディングを始める前に、LLM にプロジェクトを分析させて実装計画を立ててもらいましょう。AI ツールにプランニングモードがある場合（Cursor や Claude Code のプランモードなど）、コードを書く前に LLM がプロジェクト構造と Adapty のドキュメントを両方確認できるよう、そのモードを活用してください。

購入に使っているアプローチを LLM に伝えましょう。これによって参照すべきガイドが変わります：
- [**Adapty フロービルダー**](adapty-flow-builder): Adapty のノーコードビルダーでフローを作成し、SDK が自動的にレンダリングします。
- [**手動で作成したペイウォール**](capacitor-making-purchases): 独自のペイウォール UI をコードで構築しながら、プロダクトの取得と購入処理には Adapty を使用します。
- [**オブザーバーモード**](observer-vs-full-mode): 既存の購入インフラをそのまま使用し、Adapty はアナリティクスとインテグレーションにのみ利用します。

どれを選べばよいかわからない場合は、[クイックスタートの比較表](capacitor-quickstart-paywalls)をご覧ください。
### SDKのインストールと設定 \{#install-and-configure-the-sdk\}

npm を使って Adapty SDK の依存関係を追加し、公開 SDK キーで有効化します。これがすべての基盤となります。これなしには何も動作しません。

**ガイド:** [Adapty SDK のインストールと設定](sdk-installation-capacitor)
:::info
このウォークスルーは Adapty Capacitor SDK v4（ベータ版）を対象としています — [クイックスタート](capacitor-quickstart-paywalls)で紹介している API です。v4 はプレリリース版のため、最新の安定版 3.x をインストールするのではなく、LLM が正確なバージョン（`npm install @adapty/capacitor@4.0.0-beta.2`）を固定するように注意してください。[SDK 4.0 インストールセクション](sdk-installation-capacitor#adapty-sdk-40-beta)と[移行ガイド](migration-to-capacitor-sdk-v4)をご参照ください。
:::

これを LLM に送信してください：
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ja/sdk-installation-capacitor.md
```
:::tip[チェックポイント]
- **確認事項：** iOS・Android 両方でアプリがビルド・起動できること。コンソールに Adapty のアクティベーションログが表示されること。
- **よくある問題：** "Public API key is missing" → **App settings** のキーをプレースホルダーから実際のキーに置き換えたか確認してください。
:::

### ユーザーを識別する \{#identify-users\}

購入が複数のデバイスをまたいで維持されるよう、アプリのユーザーアカウントを Adapty プロファイルに紐付けます。

:::important
アプリに認証機能がない場合は、このステップをスキップしてください。
:::

**ガイド:** [ユーザーを識別する](capacitor-quickstart-identify)

以下を LLM に送信してください：
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ja/capacitor-quickstart-identify.md
```
:::tip[Checkpoint]
- **期待される動作:** `adapty.identify()` を呼び出した後、ダッシュボードの **Profiles** セクションにカスタムユーザー ID が表示される。

## プレーンテキストのドキュメントインデックスファイル \{#plain-text-doc-index-files\}

LLM に個別のページを超えたより広いコンテキストを提供したい場合は、Adapty のすべてのドキュメントをリスト化または結合したインデックスファイルを用意しています：
- [`llms.txt`](https://adapty.io/docs/ja/llms.txt): `.md` リンク付きの全ページ一覧です。LLM がウェブサイトにアクセスしやすくするための[新興標準](https://llmstxt.org/)です。一部の AI エージェント（例：ChatGPT）では、`llms.txt` をダウンロードしてチャットにファイルとしてアップロードする必要があります。
- [`llms-full.txt`](https://adapty.io/docs/ja/llms-full.txt): Adapty ドキュメントサイト全体を 1 つのファイルにまとめたものです。非常に大きいため、全体像が必要な場合にのみ使用してください。
- Capacitor 専用の [`capacitor-llms.txt`](https://adapty.io/docs/ja/capacitor-llms.txt) と [`capacitor-llms-full.txt`](https://adapty.io/docs/ja/capacitor-llms-full.txt): サイト全体と比べてトークンを節約できるプラットフォーム固有のサブセットです。
