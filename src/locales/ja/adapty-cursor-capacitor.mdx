このガイドでは、AIコーディングツールを使ってAdaptyをCapacitorアプリに段階的に統合する手順を説明します。適切なAdaptyドキュメントを正しい順序でAIツールに渡していきます。

<SkillPromo />
## はじめる前に：ダッシュボードの設定 \{#before-you-start-dashboard-setup\}

AdaptyはSDKコードを書く前に、いくつかのダッシュボード設定が必要です。インタラクティブなLLMスキルを使うか、ダッシュボードから手動で設定できます。

### スキルを使う方法（推奨） \{#skill-approach-recommended\}

Adapty CLIスキルを使うと、LLMがアプリ、プロダクト、アクセスレベル、ペイウォール、プレースメントを直接設定できます。各ステップでダッシュボードを開く必要がありません。[ストアの接続](integrate-payments)だけダッシュボードで行う必要があります。

```
npx skills add adaptyteam/adapty-cli --skill adapty-cli
```

スキルを追加したら、エージェントで `/adapty-cli` を実行してください。ストアの接続が必要なタイミングも含め、各ステップをガイドしてくれます。

### ダッシュボードを使う方法 \{#dashboard-approach\}

コードを書く前に手動で設定したい場合は、以下の情報が必要です。LLMはダッシュボードの値を自動で調べることができないので、ご自身で用意してください。

1. **アプリストアを接続する**: Adapty ダッシュボードで **App settings → General** に移動します。Capacitor アプリが両方のプラットフォームを対象としている場合は、App Store と Google Play の両方を接続してください。これは課金を機能させるために必須です。
   [アプリストアを接続する](integrate-payments)
2. **公開SDKキーをコピーする**: Adapty ダッシュボードで **App settings → General** に移動し、**API keys** セクションを見つけます。コードでは、これが `adapty.activate()` に渡す文字列です。

3. **プロダクトを少なくとも1つ作成する**: Adapty ダッシュボードで **Products** ページに移動します。コードからプロダクトを直接参照する必要はありません — Adapty がペイウォールを通じてプロダクトを配信します。
   [プロダクトを追加する](quickstart-products)
4. **ペイウォールとプレースメントを作成する**: Adapty ダッシュボードの **Paywalls** ページでペイウォールを作成し、**Placements** ページでプレースメントに割り当てます。コード内では、プレースメント ID が `adapty.getFlow()` に渡す文字列になります。
   [ペイウォールを作成する](quickstart-paywalls)
5. **アクセスレベルの設定**: Adapty ダッシュボードの **Products** ページで、プロダクトごとに設定します。コード内では `profile.accessLevels['premium']?.isActive` で確認します。デフォルトの `premium` アクセスレベルはほとんどのアプリで機能します。課金ユーザーがプロダクトによって異なる機能にアクセスできる場合（例：`basic` プランと `pro` プラン）は、コーディングを始める前に[追加のアクセスレベルを作成](assigning-access-level-to-a-product)してください。
:::tip
5つすべて揃ったらコーディングを始めましょう。LLM に「パブリック SDK キーは X、プレースメント ID は Y です」と伝えると、正しい初期化とフロー取得のコードを生成してもらえます。
:::
### 準備ができたら設定すること \{#set-up-when-ready\}

コーディングを始める前に必須ではありませんが、統合が成熟するにつれて必要になります：

- **A/B テスト**: **Placements** ページで設定します。コードの変更は不要です。
  [A/B テスト](ab-tests)
- **追加のペイウォールとプレースメント**: 異なるプレースメントIDで `getPaywall` の呼び出しを追加します。
- **アナリティクス連携**: **Integrations** ページで設定します。セットアップは連携先によって異なります。[アナリティクス連携](analytics-integration)と[アトリビューション連携](attribution-integration)を参照してください。

## AdaptyドキュメントをLLMに提供する \{#feed-adapty-docs-to-your-llm\}

### Context7を使う（推奨）

[Context7](https://context7.com)は、LLMが最新のAdaptyドキュメントに直接アクセスできるMCPサーバーです。何を質問するかに応じて、LLMが適切なドキュメントを自動的に取得します。URLを手動で貼り付ける必要はありません。

Context7は**Cursor**、**Claude Code**、**Windsurf**、およびその他のMCP対応ツールで動作します。セットアップするには、以下を実行してください。

```
npx ctx7 setup
```

エディタを自動検出してContext7サーバーを設定します。手動セットアップについては、[Context7 GitHubリポジトリ](https://github.com/upstash/context7)をご覧ください。
設定が完了したら、プロンプトでAdaptyライブラリを参照してください：

```
Use the adaptyteam/adapty-docs library to look up how to install the Capacitor SDK
```

:::warning
Context7によってドキュメントリンクを手動で貼り付ける必要はなくなりますが、実装の順序は重要です。すべてが正しく動作するよう、以下の[実装手順](#implementation-walkthrough)をステップごとに確認してください。
:::
### プレーンテキストのドキュメントを使う \{#use-plain-text-docs\}

Adaptyのドキュメントはプレーンテキストのマークダウンとして取得できます。URLの末尾に `.md` を追加するか、記事タイトルの下にある **Copy for LLM** をクリックしてください。例：[adapty-cursor-capacitor.md](https://adapty.io/docs/ja/adapty-cursor-capacitor.md)

以下の[実装ウォークスルー](#implementation-walkthrough)の各ステージには、貼り付け用の `.md` リンクが含まれた「LLMに送るもの」ブロックがあります。

一度に複数のドキュメントが必要な場合は、以下の[インデックスファイルとプラットフォーム別サブセット](#plain-text-doc-index-files)を参照してください。

## 実装ウォークスルー \{#implementation-walkthrough\}

このガイドの残りでは、実装順序に沿ってAdapty統合を進めていきます。各ステージには、LLMに送るドキュメント、完了時に確認できること、よくある問題が含まれています。

### インテグレーションを計画する \{#plan-your-integration\}

コードに入る前に、LLM にプロジェクトを分析させてから実装計画を立てましょう。AI ツールがプランニングモードをサポートしている場合（Cursor や Claude Code のプランモードなど）、LLM がコードを書く前にプロジェクトの構造と Adapty ドキュメントの両方を読めるよう、そのモードを使いましょう。

課金に使っているアプローチを LLM に伝えてください。参照すべきガイドが変わります：
- [**Adapty Flow Builder**](adapty-flow-builder): Adaptyのノーコードビルダーでフローを作成し、SDKが自動的にレンダリングします。
- [**手動作成のペイウォール**](capacitor-making-purchases): 独自のペイウォールUIをコードで構築しながら、プロダクトの取得と購入処理にAdaptyを使用します。
- [**オブザーバーモード**](observer-vs-full-mode): 既存の購入インフラをそのまま使用し、分析と連携のみにAdaptyを利用します。

どれを選べばよいか迷っていますか？[クイックスタートの比較表](capacitor-quickstart-paywalls)をご覧ください。
### SDKのインストールと設定 \{#install-and-configure-the-sdk\}

npm を使って Adapty SDK の依存関係を追加し、公開 SDK キーで有効化します。これがすべての基盤となります — これなしには何も動作しません。

**ガイド:** [Adapty SDK のインストールと設定](sdk-installation-capacitor)
:::info
このウォークスルーは Adapty Capacitor SDK v4（ベータ版）を対象としています — [クイックスタート](capacitor-quickstart-paywalls)で説明している API です。v4 はプレリリース版のため、最新の安定版 3.x がインストールされないよう、LLM が正確なバージョンを指定する（`npm install @adapty/capacitor@4.0.0-beta.1`）ようにしてください。[SDK 4.0 インストールセクション](sdk-installation-capacitor#adapty-sdk-40-beta)と[移行ガイド](migration-to-capacitor-sdk-v4)もご参照ください。
:::

以下を LLM に送信してください：
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ja/sdk-installation-capacitor.md
```

- **注意点:** 「Public API key is missing」が表示される場合は、プレースホルダーを **App settings** の実際のキーに置き換えたか確認してください。

### ユーザーを識別する \{#identify-users\}

アプリのユーザーアカウントを Adapty プロファイルに紐付けることで、デバイスをまたいで購入情報を引き継げます。

:::important
アプリに認証機能がない場合、この手順はスキップしてください。
:::

**ガイド:** [ユーザーを識別する](capacitor-quickstart-identify)

LLM に送信するプロンプト:
```
Read these Adapty docs before writing code:
- https://adapty.io/docs/ja/capacitor-quickstart-identify.md
```
:::tip[チェックポイント]
- **期待される結果:** `adapty.identify()` を呼び出した後、ダッシュボードの **Profiles** セクションにカスタムユーザーIDが表示されます。
- **注意点:** 匿名プロファイルへの帰属を避けるため、アクティベーション後、ペイウォールを取得する前に `identify` を呼び出してください。
:::

サンドボックスでのインテグレーションが正常に動作したら、リリースチェックリストを確認して、本番環境への準備が整っているかチェックしましょう。

**ガイド：** [リリースチェックリスト](release-checklist)

以下をLLMに送信してください：
```
Read these Adapty docs before releasing:
- https://adapty.io/docs/ja/release-checklist.md
```
:::tip[Checkpoint]
- **期待される状態:** チェックリストのすべての項目が確認済み: ストア接続、サーバー通知、購入フロー、アクセスレベルチェック、プライバシー要件。

:::
## プレーンテキストのドキュメントインデックスファイル \{#plain-text-doc-index-files\}

LLM により広いコンテキストを提供したい場合は、Adapty のすべてのドキュメントをリストアップまたは統合したインデックスファイルをホストしています。
- [`llms.txt`](https://adapty.io/docs/ja/llms.txt): `.md` リンク付きで全ページを一覧表示します。LLM がウェブサイトにアクセスしやすくするための[新興標準](https://llmstxt.org/)です。一部の AI エージェント（ChatGPT など）では、`llms.txt` をダウンロードしてチャットにファイルとしてアップロードする必要があります。
- [`llms-full.txt`](https://adapty.io/docs/ja/llms-full.txt): Adapty ドキュメントサイト全体を 1 つのファイルにまとめたものです。非常に大きいため、全体像が必要な場合にのみ使用してください。
- Capacitor 専用の [`capacitor-llms.txt`](https://adapty.io/docs/ja/capacitor-llms.txt) と [`capacitor-llms-full.txt`](https://adapty.io/docs/ja/capacitor-llms-full.txt): サイト全体と比べてトークンを節約できる、プラットフォーム固有のサブセットです。
