ようこそ！アプリ内課金をもっと簡単に 🚀
Adapty Flutter SDKは、アプリ内課金の複雑さを解消し、あなたが最も得意なこと—素晴らしいアプリを作ること—に集中できるように設計されています。Adaptyが代わりに処理する内容は以下のとおりです：
- 購入処理、レシートの検証、サブスクリプション管理をすぐに使える状態で提供
- アプリのアップデートなしでフローやペイウォールを作成・テスト可能
- 初期設定なしで詳細な購入分析を取得 — コホート、LTV、チャーン、ファネル分析を標準搭載
- アプリのセッションやデバイスをまたいでユーザーのサブスクリプション状態を常に最新の状態に維持
- マーケティングのアトリビューションや分析サービスとの連携をわずか1行のコードで実現
:::note
コードに入る前に、AdaptyをApp Store ConnectおよびGoogle Play Consoleと連携し、ダッシュボードでプロダクトを設定する必要があります。まずは[クイックスタートガイド](quickstart)を参照して、すべての設定を完了させてください。
:::
## はじめに \{#get-started\}

<SkillPromo />

このインテグレーションガイドで説明する内容は以下のとおりです。
1. [SDKのインストールと設定](sdk-installation-flutter): SDKをプロジェクトの依存関係として追加し、コード内で有効化します。
2. [フローで購入を有効にする](flutter-quickstart-paywalls): ユーザーがプロダクトを購入できるよう購入フローを設定します。独自のUIを構築する場合は、[ペイウォールを手動で実装する](flutter-quickstart-manual)を参照してください。
3. [サブスクリプションステータスの確認](flutter-check-subscription-status): ユーザーのサブスクリプション状態を自動的に確認し、有料コンテンツへのアクセスを管理します。
4. [ユーザーの識別（オプション）](flutter-quickstart-identify): ユーザーをAdaptyのプロファイルに関連付け、デバイスをまたいでデータが一貫して保存されるようにします。

## 主要コンセプト \{#main-concepts\}

コードを書く前に、Adapty の仕組みを支える主要なコンセプトを確認しておきましょう。

Adapty のアプローチの優れた点は、アプリにハードコードするのはプレースメントだけという点です。それ以外のすべて（プロダクト、ペイウォールのデザイン、価格、オファー）は、アプリのアップデートなしに Adapty ダッシュボードから柔軟に管理できます。

1. [**プロダクト**](product) - アプリ内で購入できるもの全般。サブスクリプション、消耗型アイテム、永続アクセスなどが含まれます。
2. **フローまたはペイウォール** - プロダクトと設定をまとめて、プレースメントにアタッチします。2種類あります：
   - **[フロー](adapty-flow-builder)** - ペイウォールビルダーで作成するビジュアルなノーコードインターフェース。AdaptyがUIをレンダリングし、購入処理も行います。
   - **[ペイウォール](paywalls)** - ビジュアル設定なし。自分のコードでUIを構築し、`makePurchase` を自分で呼び出します。[ペイウォールを手動で実装する](flutter-quickstart-manual)を参照してください。

   SDKコードでは、どちらも同じ `getFlow` メソッドで取得します。
3. [**プレースメント**](placements) - フローまたはペイウォールを表示するユーザージャーニー上の戦略的なポイントです。プレースメントはマネタイズ戦略の「どこで」「いつ」を決めるものです。代表的なプレースメントには次のものがあります：
   - `main` - 主要なペイウォールの表示場所
   - `onboarding` - ユーザーのオンボーディングフロー中に表示
   - `settings` - アプリの設定画面からアクセス可能

   最初の連携では `main` や `onboarding` などの基本的なプレースメントから始め、その後は[アプリ内でユーザーが購入意欲を持ちやすい場所を考えてみましょう](choose-meaningful-placements)。
4. [**プロファイル**](profiles-crm) - ユーザーがプロダクトを購入すると、そのプロファイルに**アクセスレベル**が付与され、有料機能へのアクセスを管理するために使用します。
