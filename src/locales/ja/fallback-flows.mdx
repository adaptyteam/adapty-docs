スムーズなユーザー体験を維持するために、[フロー](adapty-flow-builder)の**フォールバックバージョン**を設定することが重要です。
アプリケーションがフローをリクエストすると、Adapty SDK はサーバーに接続してその設定を取得します。デバイスが Adapty に到達できない場合（ネットワーク障害やサーバー障害など）、SDK はローカルデータにフォールバックします：

- ユーザーがすでに一度フローを表示したことがある場合、SDK はキャッシュされたコピーを提供します。
- キャッシュが存在しない場合、SDK はアプリにバンドルされたフォールバック設定ファイルを読み込みます。

Adapty はこれらのフォールバックファイルを自動的に生成します。フローのフォールバックバンドルはペイウォールと共有されており、プラットフォームごとに 1 つの JSON ファイルに両方のフォールバックバリアントが含まれています。SDK は必要なセクションを読み取ります。
:::important
フローのフォールバックは **Adapty SDK 4.0以上** のバンドルに含まれています。ダウンロードダイアログで以前のSDKバージョンを選択した場合、ファイルにはペイウォールとオンボーディングのバリエーションのみが含まれ、フローは含まれません。フローのフォールバックを使用する前に、アプリがフロー対応のSDKリリースを使用していることを確認してください。
:::
## 始める前に \{#before-you-start\}

1. フロービルダーで[フロー](adapty-flow-builder)を作成します。
2. フロー用の[プレースメントを作成](create-placement)します。

## フォールバックファイルのダウンロード \{#download-the-fallback-file\}

1. **[Placements](https://app.adapty.io/placements)** ページを開きます。
2. 右上の **Fallbacks** ボタンをクリックします。
3. ドロップダウンからターゲットプラットフォームを選択します。
4. アプリに組み込まれているSDKのバージョンに合ったSDKバージョンを選択します。フローを含むバンドルを受け取るには、**Adapty SDK v4.0.0 and higher**（またはそれ以降のオプション）を選択してください。

ブラウザがプラットフォームごとにJSONファイルをダウンロードします。例：`ios_4_0_0_fallback.json`

## ダウンロード後の手順 \{#after-the-download\}

ファイルをアプリのコードに追加し、プラットフォーム固有のセットアップガイドに従ってください。ペイウォールのフォールバックを読み込む API は、アプリがフロー対応 SDK リリースになった時点でフローのフォールバックも読み込みます。

- [iOS](ios-use-fallback-paywalls)

## 制限事項 \{#limitations\}

フォールバックフローはハードコードされてローカルに保存されるため、ライブフローの完全な動的機能は利用できません。
- **プレースメントごとに1つのバリアント。** プレースメントに複数のフロー（異なるオーディエンス、A/B テストのバリアント）がある場合、フォールバックファイルはウェイトが最も高いバリアント、または最も広いオーディエンスのバリアントを使用します。
- **A/B テストなし。** ライブフローの A/B テストはサーバー側で解決されます。フォールバックは常に1つの選択されたバリアントを提供します。
- **リモート更新なし。** フォールバックの更新には新しいアプリリリースが必要です。リモートコンフィグで行うような実行時の更新は、代わりにライブフロー経由で行ってください。
- **デフォルトロケールのみ。** フォールバックは `en` ロケールを使用します。ローカライズされたバリアントはバンドルされません。
