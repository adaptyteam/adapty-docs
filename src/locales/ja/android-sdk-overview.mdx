ようこそ！アプリ内課金をもっと簡単にするお手伝いをします 🚀

Adapty Android SDK は、アプリ内課金の煩わしさを取り除き、本当に大切なこと——素晴らしいアプリの開発——に集中できるよう設計されています。以下のことをすべて代わりに処理します：

- 購入処理、レシート検証、サブスクリプション管理をすぐに利用可能
- アプリのアップデートなしでペイウォールの作成・テストが可能
- ゼロ設定で詳細な購入分析——コホート、LTV、チャーン、ファネル分析を含む
- アプリセッションやデバイスをまたいでユーザーのサブスクリプション状態を常に最新に保つ
- たった1行のコードでマーケティングアトリビューションや分析サービスと連携

:::note
コードに入る前に、Adapty と Google Play Console を連携し、ダッシュボードでプロダクトをセットアップする必要があります。まず[クイックスタートガイド](quickstart)を確認して、すべての設定を済ませてください。
:::

## 主要コンセプト \{#main-concepts\}

コードに入る前に、Adapty を動かす主要なコンセプトを理解しておきましょう。

Adapty のアプローチの優れた点は、アプリにハードコードするのはプレースメントだけという点です。それ以外のすべて——プロダクト、ペイウォールのデザイン、価格、オファー——は、アプリのアップデートなしに Adapty ダッシュボードから柔軟に管理できます：

1. [**プロダクト**](product) - アプリ内で購入可能なもの全て——サブスクリプション、消耗型アイテム、永続アクセスなど。

2. [**ペイウォール**](paywalls) - Adapty からプロダクトを取得し、その機能を最大限に活用する唯一の方法です。異なるプロダクトの組み合わせが収益化指標にどう影響するかを追跡しやすくするため、このように設計されています。Adapty のペイウォールは、プロダクトの特定のセットとそれに付随するビジュアル設定の両方として機能します。

3. [**プレースメント**](placements) - ペイウォールを表示したい、ユーザージャーニー上の戦略的なポイントです。プレースメントは収益化戦略の「どこで」「いつ」を担います。よくあるプレースメントの例：
   - `main` - メインのペイウォール表示箇所
   - `onboarding` - ユーザーオンボーディングフロー中に表示
   - `settings` - アプリの設定からアクセス可能

   最初のインテグレーションでは `main` や `onboarding` などの基本から始めて、その後[アプリ内のどこでユーザーが購入に前向きになりそうか考えてみましょう](choose-meaningful-placements)。

4. [**プロファイル**](profiles-crm) - ユーザーがプロダクトを購入すると、プロファイルに**アクセスレベル**が割り当てられ、有料機能へのアクセス管理に使用します。
