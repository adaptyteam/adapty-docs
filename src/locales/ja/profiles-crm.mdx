プロファイルは、ユーザー向けのCRMです。プロファイルを使うと、以下のことができます：

1. プロファイルID、カスタマーユーザーID、メールアドレス、またはトランザクションIDで特定のユーザーを検索できます。
2. 請求の問題、グレース期間、その他の[イベント](events)を含む、ユーザーのイベントタイムラインを確認できます。
3. サブスクリプションの状態、総収益/収益額などのユーザープロパティを分析できます。
4. ユーザーにサブスクリプションを付与できます。

## ユーザーの検索 \{#finding-users\}

プロファイルリストでは、以下の方法で特定のユーザーを検索できます：

- **プロファイルID**：Adaptyがユーザーに割り当てた内部識別子（Adapty IDとも呼ばれます）。
- **カスタマーユーザーID**：設定している場合、アプリ内でのユーザーの識別子。
- **メールアドレス**：カスタム属性として送信されている場合、ユーザーのメールアドレス。
- **トランザクションID**：購入時のストアトランザクションID。

任意の行をクリックすると、ユーザーの全プロファイルが表示されます。

SDKを使って、追加のユーザープロパティをAdaptyに送信できます。

デフォルトでAdaptyが設定する項目：

| プロパティ | 説明 |
| ---------------- | ------------------------------------------------------------ |
| Customer user ID | お使いのシステムにおけるエンドユーザーの識別子。 |
| Adapty ID | エンドユーザーのAdapty内部識別子。プロファイルIDとも呼ばれます。 |
| IDFA | Appleがユーザーのデバイスに割り当てる広告識別子。iOS 14以降ではApp Tracking Transparency（ATT）の許可が必要です。Androidでは利用できません。 |
| 国 | エンドユーザーの国。 |
| OS | エンドユーザーが使用しているオペレーティングシステム。 |
| デバイス | エンドユーザーに表示されるデバイスのモデル名。 |
| インストール日 | ユーザーがAdaptyに最初に記録された日付：<ul><li>ユーザーが作成された日付。</li><li>Adaptyを導入する前にアプリをインストールしていたユーザーの場合、インストール日は最初のトランザクションの日付になります。</li><li>該当する場合、過去データのインポート時に指定された日付。</li></ul> |
| 作成日時 | ユーザーが作成された日付。 |

少なくとも内部ユーザーIDまたはユーザーのメールアドレスを送信してください。これにより、プロファイルリストでこれらの識別子を使ってユーザーを検索できるようになります。

SDKをインストールすると、Adaptyは支払いキューからユーザーイベントを自動的に収集し、ユーザープロファイルに表示します。上記の表の属性は自動的に収集されるため、手動で送信する必要はありません。

## サブスクリプションの付与 \{#granting-a-subscription\}

プロファイルから、有効なサブスクリプションを延長したり、購入なしにユーザーにアクセスレベルへの永続アクセスを付与したりできます。

<Zoom>
  <img src="/assets/shared/img/b1d74fd-edit_paid_access_level.webp"
  style={{
    border: '1px solid #727272',
    width: '700px',
    display: 'block',
    margin: '0 auto'
  }}
/>
</Zoom>

この機能が特に役立つ場面：
- 請求やサポートの問題が発生したユーザーへの補償。
- 手動プロモーションやベータプログラムの実施。
- 実際の購入なしにサブスクリプションのフローをテストする場合。

アクセスを付与するには、ユーザーのプロファイルを開き、**Access levels** セクションに移動して **Edit** をクリックします。有効期限を設定して保存します。有効期限は将来の日付である必要があり、一度設定すると短縮することはできません。有効なサブスクリプションの有効期限を変更しても、継続中の支払いには影響しません。

:::note
アクセスを付与しても、App StoreやGoogle Playの購入イベントは作成されません。ユーザーのイベントフィードや分析データは、実際の購入フローとは異なります。
:::

[Grant access level](api-adapty/operations/grantAccessLevel) APIメソッドを使って、プログラム的にアクセスを付与することもできます。

### アクセス共有の履歴 \{#access-sharing-history\}

アクセスレベルが共有または移転された場合、ユーザーのプロファイルには接続されたプロファイルへのリンクが表示されます。アクセスを共有したプロファイル、またはアクセスを受け取ったプロファイルです。接続されたプロファイルを確認するには、ユーザーの **Profile** でアクセスレベルの隣にあるリンクをクリックします。
