Apple Ads Managerでは、キャンペーンのパフォーマンスとユーザー行動を測定するための包括的な指標が提供されています。

## パフォーマンス \{#performance\}
| 指標 | 説明 |
|--------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Spend | 広告のタップごとのコストの合計。 |
| Impressions | 指定期間内にApp Storeの検索結果でスポンサー広告が表示された回数。 |
| CPM | 広告1,000インプレッションあたりの平均支払額。平均CPM = Spend / (Impressions / 1000) ※ CPT課金モデルのApp Store検索結果キャンペーンでは、実効CPMが表示されます。 |
| Taps | 指定期間内にユーザーが広告をタップした回数。 |
| CPT | 広告1タップあたりの平均支払額。平均CPT = Spend / Taps |
| TTR | 広告がタップされた回数をインプレッションの合計数で割った値。TTR = Taps / Impressions × 100% |
| Downloads (Total) | 指定期間内に広告からのタップスルーおよびビュースルーによる新規ダウンロードと再ダウンロードの合計数。 |
| Downloads (View-Through) | 広告を24時間以内に閲覧したがタップしなかったユーザーによるダウンロードと再ダウンロードの数。 |
| Downloads (Tap-Through) | 30日以内に広告をタップしたユーザーによる新規ダウンロードと再ダウンロードの合計数。 |
| Avg CPA (Total) | 合計平均獲得単価（CPA）は、キャンペーンの総支出を、指定期間内に広告の閲覧またはタップによって発生したダウンロード数で割った値。 |
| Avg CPA (Tap-Through) | タップスルーの平均獲得単価（CPA）は、キャンペーンの総支出を、指定期間内のタップスルーダウンロード数で割った値。 |
| Download Rate (Total) | 広告の閲覧またはタップによって発生したダウンロードの合計を、指定期間内のタップ総数で割った値。計算式：Taps > 0 の場合 (Total Downloads / Taps) × 100%、そうでない場合は 0% |
| Download Rate (Tap-Through) | 広告タップによって発生したダウンロード数を、指定期間内のタップ総数で割った値。計算式：Taps > 0 の場合 (Tap-Through Downloads / Taps) × 100%、そうでない場合は 0% |
| DPM (Total) | Downloads per Mille（DPM）は1,000インプレッションあたりのダウンロード数。計算式：Impressions > 0 の場合 (Total Downloads / Impressions) × 1000、そうでない場合は 0 |
## コンバージョン \{#conversions\}

:::note
Revenue、ARPU、ARPPU、ARPAS、ROAS、ROI はコホート指標としても利用でき、ユーザーグループの時系列分析に活用できます。
:::
| 指標 | 説明 |
|--------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Conversions | Conversionsは、レポート期間中のコンバージョンイベントの合計数です。計算式：トライアル開始数 + サブスクリプション開始数 + 非サブスクリプション数 |
| Conversion CR | Conversion CR（コンバージョン率）は、総ダウンロード数のうちコンバージョンに至った割合です。計算式：総ダウンロード数 > 0 の場合は（コンバージョン数 / 総ダウンロード数）× 100%、それ以外は 0% |
| Cost per Conversion | Cost per Conversionは、総広告費をコンバージョン数で割った値です。計算式：コンバージョン数 > 0 の場合は 広告費 / コンバージョン数、それ以外は 0 |
| Revenue | Revenueは、選択した期間内にアプリでの購入・更新・その他の収益化コンバージョンから得られた総収益額です（ストア手数料控除前）。 |
| ROAS | ROAS（広告費回収率）は、広告による収益を広告費で割ってパーセントで表したものです。計算式：広告費 > 0 の場合は（収益 / 広告費）× 100%、それ以外は 0% |
| ROI | ROI（投資収益率）は、広告費に対する純利益を計算します。計算式：広告費 > 0 の場合は（（収益 − 広告費）/ 広告費）× 100%、それ以外は 0% |
| ARPU | ARPU（ユーザー平均収益）は、ユーザー1人あたりの平均収益です。総収益をユニークユーザー数で割って算出します。例：収益 $60,000 / 5,000ユーザー = ARPU $12。このARPUをインストール単価（CPI）と比較することで、マーケティングキャンペーンの効果を評価できます。 |
| ARPPU | ARPPU（課金ユーザー平均収益）は、課金ユーザー1人あたりの平均収益です。総収益をユニーク課金ユーザー数で割って算出します。例：収益 $60,000 / 1,000課金ユーザー = ARPPU $60。課金顧客が平均的にどれだけの収益をもたらすかを把握するのに役立ちます。 |
| ARPAS | ARPASは、アクティブサブスクライバー1人あたりの平均収益です。総収益 / アクティブサブスクライバー数で算出します。ここでいうサブスクライバーとは、トライアル期間またはサブスクリプションを有効化したユーザーを指します。例：収益 $60,000 / 1,500サブスクライバー = ARPAS $40。 |
| Installs | Installsは、初めてアプリをインストールしたユーザー数と、既存ユーザーによる再インストール数の合計です。同一ユーザーが複数の端末にインストールした場合もカウントされます。なお、ダウンロードが途中で中断された場合や、完了前にキャンセルされたインストールはカウントされません。 |
| Installs CR | Installs CR（コンバージョン率）は、総ダウンロード数のうちアプリをインストールしたユーザーの割合です。計算式：総ダウンロード数 > 0 の場合は（インストール数 / 総ダウンロード数）× 100%、それ以外は 0% |
| CPI | CPI（インストール単価）は、Adaptyが計測したインストール1件あたりのコストです。計算式：インストール数 > 0 の場合は 広告費 / インストール数、それ以外は 0 |
| Trials | Trialsは、レポート期間中に新たに開始されたトライアルサブスクリプションの数です。 |
| Trial CR | Trial CR（コンバージョン率）は、総ダウンロード数のうちトライアルを開始した割合です。計算式：総ダウンロード数 > 0 の場合は（トライアル数 / 総ダウンロード数）× 100%、それ以外は 0% |
| Cost per Trial | Cost per Trialは、総広告費を新規トライアル開始数で割った値です。計算式：トライアル数 > 0 の場合は 広告費 / トライアル数、それ以外は 0 |
| Trials converted | Trials convertedは、レポート期間内にトライアルサブスクリプションから有料サブスクリプションへ正常に転換した件数です。 |
| Trial converted CR | Trial converted CR（コンバージョン率）は、トライアルサブスクリプションのうち有料サブスクリプションへ転換した割合です。計算式：トライアル数 > 0 の場合は（転換済みトライアル数 / トライアル数）× 100%、それ以外は 0% |
| Cost per Trial converted | Cost per Trial convertedは、総広告費を同期間内の転換済みトライアル数で割った値です。計算式：転換済みトライアル数 > 0 の場合は 広告費 / 転換済みトライアル数、それ以外は 0 |
| Subscriptions | Subscriptionsは、レポート期間中の新規サブスクリプション登録数（トライアルなし）の合計です。 |
| Subscription CR | Subscription CR（コンバージョン率）は、総ダウンロード数のうち無料トライアルなしで有料サブスクリプションを開始した割合です。計算式：総ダウンロード数 > 0 の場合は（サブスクリプション数 / 総ダウンロード数）× 100%、それ以外は 0% |
| Cost per Subscription | Cost per Subscriptionは、総広告費を新規サブスクリプション開始数で割った値です。計算式：サブスクリプション数 > 0 の場合は 広告費 / サブスクリプション数、それ以外は 0 |
| Non-subscriptions started | Non-subscriptions startedは、サブスクリプション以外のアプリ内買い切り購入の合計件数です。 |
| Non-subscription CR | Non-subscription CR（コンバージョン率）は、総ダウンロード数のうち非サブスクリプション購入を行った割合です。計算式：総ダウンロード数 > 0 の場合は（非サブスクリプション開始数 / 総ダウンロード数）× 100%、それ以外は 0% |
| Cost per Non-subscription | Cost per Non-subscriptionは、総広告費を非サブスクリプション購入数で割った値です。計算式：非サブスクリプション開始数 > 0 の場合は 広告費 / 非サブスクリプション開始数、それ以外は 0 |
## 高度なダウンロード \{#advanced-downloads\}
| 指標 | 説明 |
|--------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 新規ダウンロード（合計） | レポート期間内のタップスルーおよびビュースルーによる新規ダウンロードの合計数。 |
| 新規ダウンロード（ビュースルー） | 広告を見たがタップせず、かつ過去にアプリをダウンロードしたことがないユーザーによる新規ダウンロード。24時間のウィンドウ内でカウントされます。 |
| 新規ダウンロード（タップスルー） | 広告をタップし、過去にアプリをダウンロードしたことがないユーザーによる新規ダウンロード。30日間のアトリビューションウィンドウ内でカウントされます。 |
| 新規ダウンロード割合（タップスルー） | タップによる総ダウンロード数のうち、新規ダウンロードが占める割合を示します（30日間のアトリビューションウィンドウ内で広告をタップしたユーザーが対象）。 |
| 再ダウンロード（合計） | レポート期間内のタップスルーおよびビュースルーによる再ダウンロードの合計数。 |
| 再ダウンロード（ビュースルー） | 24時間のウィンドウ内で広告を見たがタップしなかったユーザーによる再ダウンロード。広告を視聴後、アプリをダウンロードして削除し、同じデバイスまたは別のデバイスで再度ダウンロードした場合にカウントされます。 |
| 再ダウンロード（タップスルー） | 30日間のアトリビューションウィンドウ内で広告をタップしたユーザーによる再ダウンロード。広告をタップ後、アプリをダウンロードして削除し、同じデバイスまたは別のデバイスで再度ダウンロードした場合にカウントされます。 |
| 再ダウンロード割合（タップスルー） | タップによる総ダウンロード数のうち、再ダウンロードが占める割合を示します。 |
## インサイト \{#insights\}
| 指標 | 説明 |
|--------|------------------------------------------------------------------------------------------------------------------------------------------|
| インプレッションシェア | インプレッションシェアとは、同一の検索キーワードにおける総インプレッション数に対して、自社広告が獲得したインプレッションの割合です。 |
| ランク | ランク（現在）は、特定の国や地域において、選択した検索キーワードのインプレッションシェアにおけるアプリの現在の順位です。 |
| 検索人気度 | 検索人気度（現在）は、国や地域ごとの検索キーワードの人気度です。1〜5のスケールで評価され、5が最も検索ボリュームが高いことを示します。 |
