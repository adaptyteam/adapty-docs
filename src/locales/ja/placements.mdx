Adapty のプレースメントシステムを使うと、オンボーディングフローやアプリ設定画面など、ユーザーのアプリ内ジャーニーのさまざまな地点で[フロー](adapty-flow-builder)、[ペイウォール](paywalls)、[オンボーディング](onboardings)、[A/B テスト](ab-tests)を作成・配信できます。これらの地点を**プレースメント**と呼びます。

アプリ内の各プレースメントでは、特定のユーザーグループ（[オーディエンス](audience)）向けに複数のフロー、ペイウォール、オンボーディング、または A/B テストを同時に管理できます。また、新しいアプリバージョンをリリースすることなく、フロー・ペイウォール・オンボーディングを随時差し替えることも可能です。

モバイルアプリにハードコードするのは、プレースメント ID だけです。

## プレースメント一覧 \{#placements-list\}

プレースメントには 3 種類あります：
- **フロープレースメント**
- **ペイウォールプレースメント**
- **オンボーディングプレースメント**

すべてのプレースメントを確認するには、Adapty メインメニューから **Placements** に移動してください。**Paywalls**、**Onboardings**、**Flows** のタブに分類されて表示されます。

各タブでは、フロー・ペイウォール・オンボーディング・A/B テストを表示できるユーザージャーニー上のさまざまな地点を一覧で確認できます。リストの各アイテムが個別のプレースメントに対応しており、管理や変更を簡単に行えます。

プレースメントの詳細を編集したり、特定のオーディエンス向けにフロー・ペイウォール・オンボーディング・A/B テストを紐付けたり、不要なプレースメントを削除したりできます。テーブル内の数値は、各プレースメントが有効化されてからのアナリティクスデータを反映しています。

ここからできること：

- [新しいプレースメントを作成する](create-placement)
- [既存のプレースメントを編集する](edit-placement)
- [既存のプレースメントを削除する](delete-placement)
- ローカルフォールバックの[フロー](fallback-flows)、[ペイウォール](fallback-paywalls)、またはオンボーディングをダウンロードする。これらのファイルは、ユーザーがアプリを開いた際に Adapty バックエンドへの接続がない場合（インターネット未接続、またはバックエンド障害の稀なケース）かつデバイスにキャッシュがない場合に使用されます。**Fallbacks** のダウンロードはフローとペイウォールの両方をカバーしており、ダウンロードダイアログで選択する SDK バージョンによって、バンドルにフロー（SDK 4.0 以降）が含まれるか、ペイウォールのみが含まれるかが決まります。
