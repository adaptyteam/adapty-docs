要加载[用户引导](onboardings)，您的应用程序需要向 Adapty 请求其配置数据。用户引导配置存储了您的用户引导流程的 URL。如果网络问题导致您的应用程序与 Adapty 服务器之间的连接中断，您将无法正确配置和显示用户引导。

要在离线状态下访问用户引导配置数据，请在应用程序代码中存储一个备用配置文件。您可以直接从 Adapty 下载包含用户引导和[付费墙](fallback-paywalls)配置数据的现成备用文件。

请按照以下说明下载文件并将其添加到您的应用程序代码中。

:::important
备用用户引导**需要网络连接**，因为用户引导内容始终存储在线上。备用文件仅存储用户引导的配置信息。

请阅读[用户引导离线模式](onboarding-offline)文章，了解当应用程序无法加载用户引导时会发生什么。
:::

<details>
   <summary>配置备用用户引导前的准备工作（点击展开）</summary>

1. 创建一个[用户引导](onboardings)。
2. 为用户引导[创建一个版位](create-placement)，以确定用户引导在应用中出现的位置。

</details>

Adapty 会自动为您的备用用户引导生成 JSON 配置文件，每个平台各一份。这些文件同时包含您的[付费墙](local-fallback-paywalls)的备用数据。

如果单个版位包含多个用户引导或付费墙，备用版本将包含权重最高或受众最广的实验变体。每当您修改付费墙或用户引导时，Adapty 会自动更新这些文件。

请按照以下步骤下载您的备用配置：

1. 打开 **[Placements](https://app.adapty.io/placements)** 页面。
2. 点击 **Fallbacks** 按钮。
3. 从下拉菜单中选择您的目标平台（*iOS* 或 *Android*）。
4. 选择您的 SDK 版本以开始下载。
