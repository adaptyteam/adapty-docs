### ARPPU

每付费用户平均收入（ARPPU）数据图表显示每位付费用户的平均收入。它展示付费客户产生的实际收入除以客户数量，并扣除退款后的结果。

### 计算方式 \{#calculation\}

:::warning
以下计算器**不考虑**[应用商店佣金和税费](how-adapty-analytics-works#tax-and-commission-logic)。请将结果与您的**毛收入**计算结果进行对比。
:::

ARPPU 指标用总收入除以付费用户数得出。在下方互动计算器中输入购买、退款和付费用户的相关数据，即可计算毛 ARPPU：
<CompoundCalculator client:load
    heading="ARPPU"
    formuLatex="\frac{\sum P_i \times Q_i - D}{U_p}"
    variables={[
        {
            nameInTheFormula: "P",
            variableName: "price",
            variableDescription: "产品价格",
            variableValue: 10
        },
        {
            nameInTheFormula: "Q",
            variableName: "qty",
            variableDescription: "已购产品数量",
            variableValue: 1,
            isInteger: true
        },
        {
            nameInTheFormula: "D",
            variableName: "refunds",
            variableDescription: "退款金额",
            variableValue: 35,
            global: true
        },
        {
            nameInTheFormula: "Up",
            variableName: "users",
            variableDescription: "付费用户数",
            variableValue: 16,
            global: true,
            isInteger: true
        }
    ]}
    rowFormula="price * qty"
    resultFormula="(_sum - refunds) / users"
    defaultRows={[
        { price: 10, qty: 5 },
        { price: 50, qty: 10 },
        { price: 100, qty: 1 }
    ]}
/>
例如，如果你的应用在某段时间内产生了 $10,000 的收入，且同期有 500 名付费用户，则 ARPPU = $10,000 / 500 = $20。这意味着在所选时间段内，每位付费用户平均贡献了 $20 的收入。
需要注意的是，显示的 ARPPU 值是总收入除以付费用户总数所得的结果。该计算在扣除渠道手续费之前进行，且退款金额不计入收入。由于同一用户在整个周期内可能多次付款，ARPPU 的总体值可能高于日均值。这一数据图表能帮助你深入了解付费用户的创收能力，从而优化定价和订阅策略，实现收入最大化。
:::note
退款会减少计算中的收入部分，从而导致 ARPPU 值降低。此外，分母中的用户数包含了后续购买被退款的用户，这会进一步拉低平均值。该数据图表反映的是扣除退款后的真实平均收入。
:::

### ARPPU 数据图表的用途 \{#arppu-chart-usage\}

Adapty 中的 ARPPU 数据图表是一款强大的工具，可帮助企业了解每位付费用户产生的收入。通过分析该数据图表随时间的变化，企业可以识别哪些渠道、网络和活动能够吸引最有价值的客户。这些信息可用于优化营销策略，并通过定向高价值客户来提升收入。借助 Adapty 的 ARPPU 数据图表，企业可以做出数据驱动的决策，从而改善盈利状况并推动长期成功。
