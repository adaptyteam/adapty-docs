用户画像是面向你的用户的 CRM 系统。通过用户画像，你可以：

1. 通过用户画像 ID、客户用户 ID、邮箱或交易 ID 查找特定用户。
2. 查看用户的事件时间线，包括账单问题、宽限期及其他[事件](events)。
3. 分析用户属性，如订阅状态、总收入/收益等。
4. 为用户授予订阅。

## 查找用户 \{#finding-users\}

在用户画像列表中，你可以通过以下方式搜索特定用户：

- **Profile ID**：Adapty 为用户分配的内部标识符（也称为 Adapty ID）。
- **Customer user ID**：你的应用为用户设置的标识符（如已设置）。
- **Email**：用户的邮箱地址（如已作为自定义属性发送）。
- **Transaction ID**：购买时的应用商店交易 ID。

点击任意一行即可打开该用户的完整画像。

你可以通过 SDK 向 Adapty 发送额外的用户属性。

Adapty 默认设置以下属性：

| 属性 | 描述 |
| ---------------- | ------------------------------------------------------------ |
| Customer user ID | 你系统中终端用户的标识符。 |
| Adapty ID | Adapty 内部的终端用户标识符，即 Profile ID。 |
| IDFA | Apple 分配给用户设备的广告标识符。在 iOS 14+ 上需要 App Tracking Transparency（ATT）权限，Android 不支持。 |
| Country | 终端用户所在国家/地区。 |
| OS | 终端用户使用的操作系统。 |
| Device | 终端用户可见的设备型号名称。 |
| Install date | 用户首次在 Adapty 中被记录的日期：<ul><li>用户创建的日期。</li><li>如果用户在你集成 Adapty 之前已安装应用，则安装日期以其第一笔交易日期为准。</li><li>如适用，以历史数据导入时提供的日期为准。</li></ul> |
| Created at | 用户创建的日期。 |

请至少发送你的内部用户 ID 或用户邮箱，这样你才能在用户画像列表中通过这些标识符找到对应用户。

安装 SDK 后，Adapty 会自动从支付队列中收集用户事件并显示在用户画像中。上表中的属性均会自动采集，无需手动发送。

## 授予订阅 \{#granting-a-subscription\}

在用户画像中，你可以延长用户的有效订阅，或为用户授予某个访问等级的永久授权——无需用户进行实际购买。

<Zoom>
  <img src="/assets/shared/img/b1d74fd-edit_paid_access_level.webp"
  style={{
    border: '1px solid #727272',
    width: '700px',
    display: 'block',
    margin: '0 auto'
  }}
/>
</Zoom>

此功能最适合用于：
- 在账单或客服问题后对用户进行补偿。
- 运行手动促销活动或内测计划。
- 无需真实购买即可测试订阅流程。

要授予访问权限，请打开用户画像，进入 **Access levels** 部分，点击 **Edit**，设置到期日期后保存。到期日期必须是未来的时间，且一旦设置后不能减少。调整有效订阅的到期日期不会影响正在进行的付款。

:::note
授予访问权限不会创建 App Store 或 Google Play 的购买事件。用户的事件记录和分析数据将与真实购买流程有所不同。
:::

你也可以通过 [Grant access level](api-adapty/operations/grantAccessLevel) API 方法以编程方式授予访问权限。
