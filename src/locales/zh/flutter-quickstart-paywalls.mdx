要启用应用内购买，你需要了解三个关键概念：
- [**产品**](product) – 用户可以购买的任何内容（订阅、消耗型商品、永久授权）
- [**付费墙**](paywalls) 是定义向用户展示哪些产品的配置。在 Adapty 中，付费墙是获取产品的唯一方式，这种设计让你无需修改应用代码，就能随时调整产品组合、定价和搭配方式。
- [**版位**](placements) – 应用中展示付费墙的位置和时机（例如 `main`、`onboarding`、`settings`）。你在看板中为版位配置付费墙，然后在代码里通过版位 ID 获取对应内容。这样便于运行 A/B 测试，并向不同用户展示不同的付费墙。
Adapty 为您提供了三种在应用中开启购买功能的方式。请根据您的应用需求选择其中一种：
| 实现方式 | 复杂度 | 适用场景 |
|------------------------|------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Adapty 付费墙编辑工具 | ✅ 简单 | 你在[无代码编辑工具中创建完整的、可直接购买的付费墙](quickstart-paywalls)。Adapty 自动渲染付费墙，并在后台处理所有复杂的购买流程、收据验证和订阅管理。 |
| 手动创建付费墙 | 🟡 中等 | 你在应用代码中自行实现付费墙 UI，但仍从 Adapty 获取付费墙对象，以保持产品组合的灵活性。请参阅[指南](flutter-quickstart-manual)。 |
| 观察者模式 | 🔴 复杂 | 你已有自己的购买处理基础设施，并希望继续使用。请注意，观察者模式在 Adapty 中存在一定限制。请参阅[文章](observer-vs-full-mode)。 |
:::important
**以下步骤说明如何在应用中展示使用 Adapty 付费墙编辑工具创建的付费墙。**

如果你不想使用付费墙编辑工具，请参阅[手动创建付费墙的购买处理指南](flutter-making-purchases)。
:::

要展示在 Adapty 付费墙编辑工具中创建的付费墙，你只需在应用代码中完成以下操作：
1. **获取付费墙**：从 Adapty 获取付费墙。
2. **展示付费墙，Adapty 负责处理购买流程**：在应用中显示你获取到的付费墙容器。
3. **处理按钮操作**：将用户与付费墙的交互行为与应用的响应逻辑关联起来。例如，当用户点击按钮时打开链接或关闭付费墙。
## 开始之前 \{#before-you-start\}

开始之前，请先完成以下步骤：

1. 在 Adapty 看板中将你的应用连接至 [App Store](initial_ios) 和/或 [Google Play](initial-android)。
2. 在 Adapty 中[创建产品](create-product)。
3. [创建付费墙并向其添加产品](create-paywall)。
4. [创建版位并将付费墙添加到版位中](create-placement)。
5. 在应用代码中[安装并激活 Adapty SDK](sdk-installation-flutter)。
:::tip
完成这些步骤最快的方式是参照[快速入门指南](quickstart)，或使用 [Developer CLI](developer-cli-quickstart) 创建付费墙和版位。
:::
## 1. 获取付费墙 \{#1-get-the-paywall\}

你的付费墙与在看板中配置的版位相关联。通过版位，你可以为不同的目标受众展示不同的付费墙，或运行 [A/B 测试](ab-tests)。

要获取在 Adapty 付费墙编辑工具中创建的付费墙，你需要：

1. 通过 `getPaywall` 方法，使用[版位](placements) ID 获取 `paywall` 对象，并通过 `hasViewConfiguration` 属性检查该付费墙是否由编辑工具创建。
2. 使用 `createPaywallView` 方法创建付费墙视图。该视图包含显示付费墙所需的 UI 元素和样式。

:::important
要获取视图配置，必须在付费墙编辑工具中开启 **Show on device** 开关。否则，将获得空的视图配置，付费墙将无法显示。
:::
```dart showLineNumbers

try {
  final paywall = await Adapty().getPaywall(placementId: "YOUR_PLACEMENT_ID", locale: "en");
  // the requested paywall
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
}

try {
  final view = await AdaptyUI().createPaywallView(
        paywall: paywall,
      );
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```

## 2. 展示付费墙 \{#display-the-paywall\}

现在，有了付费墙配置之后，只需添加几行代码即可展示付费墙。

要展示付费墙，请在 `createPaywallView` 方法创建的 `view` 上调用 `view.present()` 方法。每个 `view` 只能使用一次。如果需要再次展示付费墙，请重新调用 `createPaywallView` 来创建新的 `view` 实例。

```dart showLineNumbers title="Flutter"
try {
  await view.present();
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```
:::tip
有关如何展示付费墙的更多详情，请参阅我们的[指南](flutter-present-paywalls)。
:::
## 3. 处理按钮事件 \{#handle-button-actions\}

当用户点击付费墙中的按钮时，Flutter SDK 会自动处理购买和恢复操作。但其他按钮具有自定义或预定义的 ID，需要在代码中处理相应的事件。

要控制或监听付费墙界面上的操作，请实现 `AdaptyUIPaywallsEventsObserver` 方法，并在显示任何界面之前设置观察者。当用户执行某个操作时，`paywallViewDidPerformAction` 将被调用，你的应用需要根据事件 ID 做出相应处理。
例如，你的付费墙可能有一个关闭按钮和需要打开的 URL（如使用条款和隐私政策）。因此，你需要响应 ID 为 `Close` 和 `OpenUrl` 的操作。

:::tip
请参阅我们的指南，了解如何处理按钮[操作](flutter-handle-paywall-actions)和[事件](flutter-handling-events)。
:::
将 Observer 实现为专用的长生命周期对象，而非 widget。由于整个应用共享同一个全局 Observer 插槽，将其绑定到 `State` 会导致页面泄漏（SDK 会持有对它的强引用），且当下一个页面注册自身时会被静默替换。使用 `extends` 还能继承 SDK 的默认行为，让你只需覆盖自己关心的回调即可。
```dart showLineNumbers title="Flutter"
// A dedicated, long-lived handler for paywall events.
// It does NOT live inside a Widget/State, so it never leaks and is never
// silently replaced when screens are pushed or popped.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // This method is called when user performs an action on the paywall UI
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using url_launcher package
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors
      print('Error launching URL: $e');
    }
  }
}
```

在应用启动时、任何付费墙展示之前，**注册一次**处理程序：

```dart showLineNumbers title="Flutter"
AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());
```

## 完整示例 \{#full-example\}

以下是如何将所有步骤整合到你的应用中的完整示例。
```dart

void main() {
  // Register a single, long-lived observer once, before any paywall is shown.
  // It is intentionally a plain object (NOT a Widget/State): its lifetime is the
  // whole app, so it never leaks and is never silently replaced when screens are
  // pushed or popped.
  AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());

  runApp(MaterialApp(home: PaywallScreen()));
}

/// A dedicated handler for AdaptyUI paywall events.
///
/// It `extends` [AdaptyUIPaywallsEventsObserver] (rather than being implemented
/// by a `State`), which gives you two things for free:
///   * the SDK's sensible defaults (e.g. auto-dismiss on close/finish), so you
///     only override what you actually care about;
///   * a lifecycle that is independent of the widget tree — there is no strong
///     reference back into a `Widget`, so nothing leaks and there is nothing to
///     unregister.
///
/// Every callback receives the [AdaptyUIPaywallView] it relates to, so handling
/// paywall actions never requires a `BuildContext` or widget state.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // Called when the user performs an action on the paywall UI.
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using the url_launcher package.
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds.
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered.
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs.
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched.
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors.
      print('Error launching URL: $e');
    }
  }
}

class PaywallScreen extends StatefulWidget {
  const PaywallScreen({super.key});

  @override
  State<PaywallScreen> createState() => _PaywallScreenState();
}

class _PaywallScreenState extends State<PaywallScreen> {
  @override
  void initState() {
    super.initState();
    _showPaywallIfNeeded();
  }

  Future<void> _showPaywallIfNeeded() async {
    try {
      final paywall = await Adapty().getPaywall(
        placementId: 'YOUR_PLACEMENT_ID',
      );

      if (!paywall.hasViewConfiguration) return;

      final view = await AdaptyUI().createPaywallView(paywall: paywall);

      await view.present();
    } catch (_) {
      // Handle any errors (network, SDK issues, etc.)
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Adapty Paywall Example')),
      body: Center(
        // Add a button to re-trigger the paywall for testing purposes.
        child: ElevatedButton(
          onPressed: _showPaywallIfNeeded,
          child: const Text('Show Paywall'),
        ),
      ),
    );
  }
}
```
