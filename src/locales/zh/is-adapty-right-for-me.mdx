Adapty 是一个面向移动应用的应用内购买平台。它涵盖订阅、一次性购买和消耗品——从购买处理和收据验证到分析、A/B 测试和集成，一应俱全。

以下是 Adapty 在不同场景下的应用方式。

## 我正在发布一款包含应用内购买的新应用

无论您计划销售订阅、一次性购买还是消耗品，Adapty 覆盖完整技术栈：

- **7 个平台的 SDK**：iOS、Android、React Native、Flutter、Unity、Kotlin Multiplatform 和 Capacitor。
- **购买处理**：带续期和重试逻辑的订阅、一次性购买、消耗品以及收据验证——全部由 Adapty 托管。
- **无代码付费墙构建器**：无需编写 UI 代码即可设计和发布付费墙。
- **从第一天起的分析**：用户到来后即可追踪收入、试用、转化等指标。

准备好开始了吗？请参照[快速入门指南](quickstart)。

## 我需要 A/B 测试、分析和集成

Adapty 帮助您优化现有业务：

- **A/B 测试**：测试不同价格、付费墙设计、试用时长和促销优惠，找到最高转化率的方案。使用 [Growth Autopilot](autopilot) 基于竞品和市场分析获取数据驱动的测试计划。
- **分析图表**：追踪 MRR、LTV、流失率、留存率及数十项其他指标。
- **受众细分**：针对特定用户群组提供定制付费墙和优惠。
- **远程付费墙配置**：无需发布新应用版本即可迭代付费墙。
- **第三方集成**：将购买事件发送至您团队已在使用的 Amplitude、AppsFlyer、Adjust、Mixpanel 等工具。

探索 [A/B 测试](ab-tests)、[分析](analytics)、[分析服务集成](analytics-integration)或[归因服务集成](attribution-integration)。

## 我想借助 LLM 实现应用内购买

Adapty 的文档已针对 Cursor、Claude、ChatGPT 等 AI 编程助手进行优化。每个页面均提供纯 Markdown 格式，并为每个平台提供分步 LLM 辅助实现指南：

- **开箱即用的指南**：将指南发送给您的 LLM，让它引导您完成每个实现阶段。
- **Markdown 访问**：在任意文档 URL 后添加 `.md`，或点击**复制供 LLM 使用**以获取纯文本版本。
- **Context7 MCP 支持**：将 Adapty 文档直接连接到您的 LLM 驱动 IDE。

选择您的平台并开始：[借助 AI 集成 Adapty](adapty-cursor)。

## 我想运营和优化 Apple 广告系列

如果您正在投放 Apple Search Ads，Adapty 的 Apple 广告管理器可将广告系列效果与收入指标直接关联——无需 MMP：

- **实时效果数据**：追踪广告系列、广告组和关键词。
- **端到端收入追踪**：从搜索到安装、试用、订阅直至 LTV 的完整链路追踪。
- **AI 预测与建议**：预测回报并获取扩量建议。
- **规则自动化**：保持您的 CPA 和 ROAS 目标稳定。

从 [Apple 广告管理器](adapty-ads-manager)开始。

## 我想追踪用户来源

Adapty 用户获取是一个内置归因解决方案，将广告支出与应用安装和订阅收入关联起来：

- **统一营销仪表板**：在一处查看所有渠道的 ROAS、安装量和收入。
- **内置归因**：无需依赖外部 MMP 即可将广告活动与安装和收入关联。
- **追踪链接**：在 Adapty 中生成链接并添加到广告活动中，实现精准归因。
- **延迟深链接**：即使用户点击时尚未安装应用，也可在安装后将其引导至正确内容。
- **队列分析**：分析获客效果和用户行为随时间的变化。

了解更多关于 [Adapty 用户获取](adapty-user-acquisition)的信息。

## 我想在无需发布新版本的情况下快速迭代

Adapty 集成后，大部分日常工作在仪表板中完成——无需发布新的应用版本：

- **无代码付费墙构建器**：在可视化编辑器中设计和更新付费墙，立即发布更改。
- **仪表板中的 A/B 测试**：启动实验、调整定价和更换优惠，无需修改代码。
- **仪表板分析**：实时监控收入、流失率、试用和转化。
- **Slack 和邮件报告**：获取对您团队重要指标的自动更新。

探索[付费墙构建器](adapty-paywall-builder)或查看[分析](charts)。

## 我在 Web 端销售，需要一个移动应用

如果您的用户已通过网站付费，而您正在添加移动应用，Adapty 可跨平台同步购买：

- **Stripe 和 Paddle 集成**：自动将 Web 购买同步至 Adapty。
- **Web 到移动同步**：在 Web 上付费的用户可在应用中获得访问权限，反之亦然。
- **统一跨平台分析**：在一个仪表板中查看 Web 和移动收入。

设置 [Stripe 集成](stripe)、[Paddle 集成](paddle)，或了解如何[同步 Web 和移动订阅者](sync-subscribers-from-web)。

## 我正在从其他工具迁移

Adapty 让从其他订阅平台迁移变得简单：

- **迁移指南**：从其他订阅平台迁移的分步说明。
- **观察者模式**：保留现有计费代码，通过[观察者模式](observer-vs-full-mode)逐步采用 Adapty——先从分析和 A/B 测试开始，准备好后再扩展。
- **历史数据导入**：将现有交易历史导入 Adapty，保持分析数据完整。

了解[迁移至 Adapty](migrate-to-adapty-from-another-solutions)和[导入历史数据](importing-historical-data-to-adapty)。
