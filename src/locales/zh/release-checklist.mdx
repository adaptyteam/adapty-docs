我们非常高兴您决定使用 Adapty！希望集成过程一切顺利。本指南将引导您完成确保应用准备好在商店发布所需的各个步骤，让您确信变现流程运行正常。

## 起飞前必备事项 \{#pre-flight-essentials\}

开始验证前您需要准备：

- 一台配置了沙盒账号的真实设备

- 访问 Adapty 看板的权限

- 访问 App Store Connect / Google Play Console 的权限

:::note
虽然沙盒购买可以在模拟器上运行，但要完整测试所有流程（包括支付对话框和生物识别提示），仍需要真实设备。
:::

## 通用验证项目 \{#universal-validations\}

- [ ] **商店连接**：确保您已将 Adapty 连接到 App Store 和/或 Google Play：
    - [ ] [App Store](initial_ios)
    - [ ] [Google Play](initial-android)
- [ ] **订阅事件传递**：确认服务器通知已正确配置：
    - [ ] [App Store 服务器通知](enable-app-store-server-notifications)
    - [ ] [实时开发者通知（RTDN）](enable-real-time-developer-notifications-rtdn)
- [ ] **用户画像识别**：验证用户识别逻辑，确保购买记录映射到正确的用户画像：
    - [ ] [检查应用代码中的识别逻辑是否与您的使用场景相匹配](ios-quickstart-identify)
    - [ ] [确保您了解在用户画像之间共享付费访问权限的父级/继承逻辑](sharing-paid-access-between-user-accounts)
- [ ] **优惠活动**：如果应用中包含 App Store 促销活动，请确保您已将[应用内购买密钥](app-store-connection-configuration#step-4-for-trials-and-special-offers--set-up-promotional-offers)添加到主字段以及 **App Store promotional offers** 部分。
- [ ] **数据收集**：确保符合隐私合规要求：
    - [ ] 如果您需要遵守 GDPR 或 CCPA 等隐私法规，或您的应用面向儿童，请控制是否[启用 IDFA 和 IP 的收集与共享](sdk-installation-ios#data-policies)。
    - [ ] 如果您的应用使用了 AppTrackingTransparency，请确保您已[将授权状态发送给 Adapty](ios-deal-with-att)。
- [ ] **隐私标签**：[了解更多](apple-app-privacy)关于 Adapty 收集的数据以及您需要为审核设置的标志。

## 购买验证 \{#purchase-validations\}

<SupportForum />

在正式上线前，请确保应用内购买功能正常运行，并且您的付费墙已准备好通过商店审核。

验证应用内购买的方式取决于您的实现方式：

- 您展示的是在 Adapty 付费墙编辑工具中创建的付费墙
- 您实现了自己的付费墙，并在其中使用 `makePurchase` 方法处理购买
- 您以观察者模式使用 Adapty（无论是使用付费墙编辑工具还是自定义付费墙）

<Tabs groupId="paywall" queryString>
<TabItem value="builder" label="Adapty Paywall Builder" default>

**目标**：Adapty 渲染付费墙，用户可以购买产品，访问权限解锁，且恢复购买流程正常运行。

- [ ] 您的应用从您将要上线的同一版位[展示付费墙](ios-present-paywalls)。
    - [ ] 付费墙正常显示在屏幕上。如果加载时间过长（例如，您或用户网络不稳定），请考虑[调整获取策略](get-pb-paywalls#fetch-paywall-designed-with-paywall-builder)。
    - [ ] 付费墙与预期的实验变体相符（如适用，包括目标受众/语言区域）。如有需要，您可以[更改目标受众优先级](change-audience-priority)。
- [ ] 产品和价格在付费墙上正确显示。请注意，Apple 的 API 在测试期间（尤其是在不同区域配置下）可能偶尔提供不准确的价格，因此请优先测试购买流程的功能性而非价格准确性，因为 Adapty 不会影响商店价格。
- [ ] 沙盒购买成功完成，并收到购买成功的回调。
- [ ] 访问权限已解锁并持续有效。确认[付费访问权限是基于当前 Adapty 用户画像授予的](ios-check-subscription-status#connect-profile-with-paywall-logic)。
    - [ ] 购买完成后，Adapty 用户画像具有有效的访问等级。
    - [ ] 当用户画像包含相应访问等级时，付费功能解锁（而不仅仅依赖购买回调）。
- [ ] 恢复购买功能正常运行。当您重新安装应用或在新设备上安装时，自动恢复购买功能按照[共享付费访问](sharing-paid-access-between-user-accounts)设置运行。如果您没有任何后端身份验证，无论该设置如何，购买都会自动恢复。在其他情况下，确保用户在重新安装应用后能够恢复购买。
- [ ] 商店审核要求：
    - [ ] 付费墙上有 **Restore purchases** 按钮。您可以在付费墙编辑工具中添加该按钮，点击后将自动处理购买恢复。
    - [ ] 服务条款和隐私政策可从付费墙界面访问，且点击这些链接会在浏览器中打开。

</TabItem>

**目标**：您负责渲染 UI；Adapty 处理购买、用户画像更新和恢复操作。

- [ ] 产品 ID 未硬编码在您的应用代码中。您只硬编码[版位](placements) ID。
- [ ] 您的应用从您将要上线的同一版位[获取产品](fetch-paywalls-and-products)。
    - [ ] 产品列表加载成功。如果加载时间过长（例如，您或用户网络不稳定），请考虑[调整获取策略](fetch-paywalls-and-products#fetch-paywall-information)。
    - [ ] 获取的产品与预期的实验变体相符（如适用，包括目标受众/语言区域）。如有需要，您可以[更改目标受众优先级](change-audience-priority)。
- [ ] 产品和价格在付费墙上正确显示。请注意，Apple 的 API 在测试期间（尤其是在不同区域配置下）可能偶尔提供不准确的价格，因此请优先测试购买流程的功能性而非价格准确性，因为 Adapty 不会影响商店价格。
- [ ] 使用 [makePurchase](making-purchases) 的沙盒购买成功完成：
    - [ ] 购买成功的结果已正确处理。
    - [ ] 待处理/失败/取消等情况均得到了妥善处理。
- [ ] 如果您[使用了远程配置](present-remote-config-paywalls)，其值已正确传递到您的付费墙。
- [ ] 当付费墙展示时，调用了 [`logShowPaywall` 方法](present-remote-config-paywalls#track-paywall-view-events)。
- [ ] 沙盒购买成功完成，并收到购买成功的回调。
- [ ] 访问权限已解锁并持续有效。确认[付费访问权限是基于当前 Adapty 用户画像授予的](ios-check-subscription-status#connect-profile-with-paywall-logic)。
    - [ ] 购买完成后，Adapty 用户画像具有有效的访问等级。
    - [ ] 当用户画像包含相应访问等级时，付费功能解锁（而不仅仅依赖购买回调）。
- [ ] 恢复购买功能正常运行。当您重新安装应用或在新设备上安装时，自动恢复购买功能按照[共享付费访问](sharing-paid-access-between-user-accounts)设置运行。如果您没有任何后端身份验证，无论该设置如何，购买都会自动恢复。在其他情况下，确保用户在重新安装应用后能够恢复购买。
- [ ] 商店审核要求：
    - [ ] **Restore purchases** 按钮可访问，并[处理恢复操作](restore-purchase)。
    - [ ] 服务条款和隐私政策可从付费墙界面访问，且点击这些链接会在浏览器中打开。

<TabItem value="observer" label="Observer mode">

**目标**：您自行处理购买、用户画像更新和恢复操作；Adapty 接收交易报告。

- [ ] **您的应用使用自己的购买流程完成购买**（StoreKit / BillingClient / 后端）：
    - [ ] 沙盒购买在商店 UI 中成功完成。
    - [ ] 待处理/失败/取消等情况均在您的应用中得到了妥善处理。
- [ ] **交易已上报给 Adapty**：
    - [ ] 观察者模式已在[您的应用代码中启用](implement-observer-mode)。
    - [ ] 购买记录出现在 Adapty 事件动态中。
    - [ ] 续订、取消和退款随时间推移得到正确反映（如适用）。
- [ ] **付费墙展示已被追踪**。当付费墙显示时，调用了 [`logShowPaywall` 方法](present-remote-config-paywalls#track-paywall-view-events)。
- [ ] **恢复购买功能适用于您的实现**。重新安装应用或切换设备后，访问权限能正确恢复。
- [ ] **商店审核要求**：
    - [ ] **Restore purchases** 操作可访问，并触发您的恢复流程。
    - [ ] 服务条款和隐私政策可从付费墙或购买界面访问，且在浏览器中打开。
