按钮是 Flow 编辑工具中响应用户点击的交互元素。常见用途包括：
- 购买行动号召按钮——关联产品并自动处理交易
- 导航——在屏幕之间引导用户（下一步、返回、关闭、跳过）
- 实用链接——恢复购买、服务条款和隐私政策
:::info
本节介绍全新的 Flow Builder，需配合 Adapty SDK 4.0 或更高版本使用。
:::
## 添加按钮 \{#add-buttons\}

添加任意按钮的步骤：

1. 点击 **+** 并选择 **Button**。
2. 选择按钮类型。

## 按钮类型 \{#button-types\}
### 购买按钮 \{#purchase-buttons\}

:::link
要让购买按钮正常工作，需要将产品绑定到页面并添加 **Products** 元素。请参阅[操作指南](paywall-product-block)。
:::

购买按钮会对用户在页面上选中的产品发起应用内购买。SDK 会自动处理交易，无需在应用代码中手动处理购买逻辑。

添加购买按钮的步骤：
1. 点击 **+** 并选择 **Button**，然后选择一个按钮预设。
2. 选中按钮后，在右侧面板中打开 **Interactions** 标签页。
3. 点击 **Add trigger** > **On tap**，然后点击 **Add action**。
4. 将 **Action** 设置为 **Purchase**，将 **Product** 设置为 `products.selectedProduct`。`products.selectedProduct` 变量始终指向当前屏幕上已选中的产品。

:::tip
你可以通过为购买按钮添加动画来吸引更多注意力。付费墙编辑工具目前支持 **Pulse** 动画类型。
在 **Design** 面板中配置动画样式。
:::

### 链接 \{#links\}

为符合部分商店的要求，您可以添加以下链接：
- 服务条款
- 隐私政策
- 购买恢复

添加链接的步骤：
1. 点击 **+**，选择 **Button > Purchase**。这会添加一行内联按钮，其中预设了恢复购买或打开 URL 等操作。如果您不需要某些按钮，可以将其删除。
2. 接下来，设置按钮操作：
- **Restore purchases** 按钮已处理购买恢复功能。
   - 对于每个链接：
        1. 点击按钮选中它，然后切换到右侧的 **Interactions** 标签页。
        2. 将 URL 粘贴到字段中。
        3. 默认情况下，URL 会在应用内浏览器中打开，以提供流畅的用户体验。如果希望在外部浏览器中打开，请勾选 **Open in external browser** 复选框。

### 关闭流程 \{#close-flow\}

**Close** 按钮可自动关闭当前流程。

要添加关闭按钮，点击 **+** 并选择 **Button > Close flow**。

:::tip
使用 **Absolute** 定位可将关闭按钮放置在屏幕角落。
:::
您也可以通过[操作](onboarding-actions)将任意其他按钮配置为关闭流程。
### 自定义按钮 \{#custom-buttons\}

你添加的每个按钮都可以配置点击后执行的操作：
- 跳转到下一个页面
- 显示弹窗提示
- 设置[变量](onboarding-variables)
- [显示或隐藏页面元素](onboarding-element-visibility)
- 打开 URL
- 恢复购买
- 执行条件操作
