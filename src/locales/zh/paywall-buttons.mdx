按钮是流程编辑器中响应用户点击的交互元素。你可以将其用于：
- 购买行动号召（CTA）——关联产品并自动处理交易
- 页面导航——在屏幕之间跳转（下一步、返回、关闭、跳过）
- 实用链接——恢复购买、服务条款和隐私政策
:::info
本节介绍新版流程编辑器，该编辑器需配合 Adapty SDK 4.0 或更高版本使用。
:::
## 添加按钮 \{#add-buttons\}

要添加任意按钮：

1. 点击 **+** 并选择 **Button**。
2. 选择按钮类型。

## 按钮类型 \{#button-types\}
### 购买按钮 \{#purchase-buttons\}

:::link
要让购买按钮正常工作，需要将产品绑定到页面并添加 **Products** 元素。请参阅[指南](paywall-product-block)。
:::

购买按钮会针对用户在页面上选中的产品发起应用内购买。SDK 会自动处理交易，因此无需在应用代码中手动处理购买逻辑。

要添加购买按钮：
1. 点击 **+** 并选择 **Button**，然后选择一个按钮预设。
2. 选中该按钮后，在右侧面板中打开 **Interactions** 标签页。
3. 点击 **Add trigger** > **On tap**，然后点击 **Add action**。
4. 将 **Action** 设置为 **Purchase**，将 **Product** 设置为 `products.selectedProduct`。`products.selectedProduct` 变量始终指向当前屏幕上选中的产品。

:::tip
你可以通过添加动画效果让购买按钮更吸引眼球。付费墙编辑工具目前支持 **Pulse** 动画类型。
在 **Design** 面板中配置动画样式。
:::

### 链接 \{#links\}

:::important
**Terms of Use** 和 **Privacy Policy** 按钮内置了 **Open URL** 操作，请在此处设置目标 URL。若 Open URL 为空或[内联链接](onboarding-text#inline-link)未填写，将无法预览和发布。
:::

为满足某些应用商店的要求，您可以添加以下链接：
- 服务条款
- 隐私政策
- 购买恢复
添加链接：
1. 点击 **+**，选择 **Button > Links**。这会添加一行内联按钮，包含预设操作：恢复购买或打开 URL。如果不需要其中某些按钮，可在图层面板中删除。

<Zoom>
    <img src="/assets/shared/img/add-links.webp"
         style={{
             border: '1px solid #727272', /* border width and color */
             width: '500px', /* image width */
             display: 'block', /* for alignment */
             margin: '0 auto' /* center alignment */
         }}
    />
</Zoom>
2. 接下来，设置按钮操作：

   - **Restore purchases** 按钮已自动处理恢复购买的逻辑。
   - 对于其余每个链接：
        1. 点击该按钮将其选中，然后切换到右侧的 **Interactions** 标签页。
        2. 将 URL 粘贴到输入框中。
        3. 默认情况下，URL 会在应用内浏览器中打开，以提供流畅的用户体验。如果希望跳转到外部浏览器，请勾选 **Open in external browser** 复选框。

### 关闭流程 \{#close-flow\}

**关闭** 按钮会自动关闭流程。

要添加关闭按钮，点击 **+** 并选择 **Button > Close flow**。

<Zoom>
    <img src="/assets/shared/img/close-flow.webp"
         style={{

             width: '500px', /* image width */
             display: 'block', /* for alignment */
             margin: '0 auto' /* center alignment */
         }}
    />
</Zoom>

:::tip
使用 **Absolute** 定位将关闭按钮放置在屏幕角落。
:::
您也可以通过[操作](onboarding-actions)将任意其他按钮配置为关闭流程。
### 自定义按钮 \{#custom-buttons\}

你添加的每个按钮都可以配置点击后执行的操作：
- 跳转到下一个页面
- 显示弹窗提示
- 设置[变量](onboarding-variables)
- [显示或隐藏页面元素](onboarding-element-visibility)
- 打开 URL
- 恢复购买
- 执行条件操作
