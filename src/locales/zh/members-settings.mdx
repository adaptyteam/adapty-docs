Adapty 看板成员系统允许您为每位成员授予不同级别的 Adapty 访问权限，并指定其可访问的应用。

### 角色 \{#roles\}

Adapty 看板中为成员提供以下角色：

| 角色        | 访问账单 | 添加新成员 | 修改任意内容 | 访问所有版块 |
|-------------|----------|------------|--------------|--------------|
| Owner       | ✅        | ✅          | ✅            | ✅            |
| Admin       | ❌        | ✅          | ✅            | ✅            |
| Developer   | ❌        | ❌          | ✅            | ❌            |
| Viewer      | ❌        | ❌          | ❌            | ✅            |
| Support     | ❌        | ❌          | ❌            | ❌            |
| ASA manager | ❌        | ❌          | ❌            | ❌            |

- **Owner：** Owner 是 Adapty 账户的原始创建者，拥有最高级别的访问权限和控制权。Owner 可完全访问 Adapty 账单，能够管理支付信息和订阅计划。此外，只有 Owner 和 Admin 可以为新成员指定应用访问权限。每个 Adapty 账户只能有一位 Owner。
- **Admin：** 拥有 Admin 角色的成员可完全访问所选应用。他们可以执行各种管理任务，包括创建和修改付费墙、进行 A/B 测试、分析数据以及管理这些应用内的成员。
- **Developer：** 拥有 Developer 角色的成员可完全访问除数据分析和账户成员以外的所有实体。他们无法访问任何账单设置。该角色适用于负责搭建付费墙、A/B 测试及其他实体并将 Adapty 集成到应用中，但不应查看财务数据的人员。
- **Viewer：** 拥有 Viewer 角色的成员对所选应用拥有只读访问权限。他们可以查看信息，但无法创建或修改付费墙、A/B 测试及其他功能，也无法邀请新用户、创建新应用或更改应用设置。
- **Support：** 拥有 Support 角色的成员只能访问所选应用中的用户画像。但他们无法执行添加新成员或访问 Adapty 其他版块等操作。该角色特别适合需要协助客户处理订阅相关咨询或问题排查的支持团队或个人。
- **ASA manager：** 拥有 ASA manager 角色的成员只能访问 [Apple Ads Manager](adapty-ads-manager) 看板。

### 添加成员 \{#add-a-member\}

在 Adapty 中，您可以免费邀请任意数量的团队成员。

:::note
您只能邀请尚未在 Adapty 中注册的电子邮件地址。如果您的同事已有独立账户，请邀请其使用其他电子邮件地址，或联系 Adapty 支持团队删除其现有账户。
:::

添加团队成员的步骤：

1. 点击右上角的 **Account**，然后打开 **Members** 标签页。
2. 点击 **Invite member**。
    <Zoom>
        <img src="/assets/shared/img/invite-member.webp"
        style={{

        width: '700px', /* image width */

        margin: '0 auto' /* center alignment */

    </Zoom>
3. 输入成员的电子邮件地址。
4. 从列表中选择一个[角色](#roles)。
5. 选择要授予访问权限的应用。
6. （可选）启用 **Always allow access to new apps**，以自动授予对未来新增应用的访问权限。
7. 点击 **Save**。
