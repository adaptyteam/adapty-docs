当您的公司被收购、出售应用或重组业务实体时，需要将应用转移给其他所有者。转移过程涉及协调 Adapty、App Store Connect 和 Google Play Console 中的变更，以确保服务不中断。

## 转移应用所有权 \{#transfer-app-ownership\}

请先完成应用商店的转移，然后再在 Adapty 中转移应用。此顺序可确保在整个转移过程中购买功能持续正常运行。

:::note
在转移过程中，请勿删除或重新创建产品。在验证转移成功完成之前，请勿更改产品 ID。
:::

### App Store (iOS) 转移 \{#app-store-ios-transfer\}

:::important
App Store Connect API 密钥（Issuer ID、Key ID、.p8 文件）的作用范围是账户级别，而非应用级别。转移完成后，您必须从新所有者的账户生成新的 API 密钥，并在 Adapty 中进行更新。
:::

1. **新所有者：** 如果尚未注册，请在 [app.adapty.io](https://app.adapty.io) 创建 Adapty 账户。
2. **原所有者：** 按照 Apple 的[转移指南](https://developer.apple.com/help/app-store-connect/transfer-an-app/overview-of-app-transfer)，在 App Store Connect 中发起应用转移。
3. **新所有者：** 在 App Store Connect 中接受转移请求。
4. **原所有者：** 发送邮件至 [support@adapty.io](mailto:support@adapty.io)，申请在 Adapty 中转移应用。请提供应用名称和新所有者的电子邮件地址。
5. **新所有者：** 在 Adapty 中收到应用后，完成 [App Store 集成指南](initial_ios)，在您的账户下生成并配置所有凭据。

### Google Play (Android) 转移 \{#google-play-android-transfer\}

1. **新所有者：** 如果尚未注册，请在 [app.adapty.io](https://app.adapty.io) 创建 Adapty 账户。
2. **双方所有者：** 确保两个 Google Play 开发者账户均已完全注册。
3. **原所有者：** 通过 Google Play Console 或 Google Play 开发者支持提交转移申请。Google 可能会要求提供额外文件，例如 DUNS 编号、合同或销售证明。
4. **新所有者：** 审核并批准转移申请。
5. **Google：** Google 支持团队处理转移请求，通常需要几个工作日，但实际时间可能因账户验证、订阅复杂性和支付设置而有所不同。
6. **原所有者：** Google 完成转移后，发送邮件至 [support@adapty.io](mailto:support@adapty.io)，申请在 Adapty 中转移应用。请提供应用名称和新所有者的电子邮件地址。
7. **新所有者：** 在 Adapty 中收到应用后，完成 [Google Play 集成指南](initial-android)，在您的账户下生成并配置所有凭据。

转移内容包括用户、订阅、统计数据、评分和应用商店页面。现有订阅者的订阅连续性将得到保障，但付款将在转移完成后才切换到新所有者的商户账户。转移前的付款报告和订单将保留在原账户中。请参阅 Google 的[转移指南](https://support.google.com/googleplay/android-developer/answer/6230247)了解详细要求。

## 风险缓解与时机选择 \{#risk-mitigation-and-timing\}

**转移期间持续正常运行的功能：**
- 购买功能（应用专属共享密钥用于验证收据）
- 续订功能（共享密钥保持有效）
- 现有订阅者的访问权限
- SDK 持续正常运行

**暂时停止运行的功能：**
- App Store Connect API 调用（在配置新密钥之前）
- 服务器通知（在重新配置端点之前）
- 凭据转换期间，分析数据可能出现缺口

**推荐时机：**
- 在低流量时段完成转移（主要用户时区的凌晨 3 点至 6 点）
- 新所有者应在接受应用商店转移后立即准备好配置凭据
- 在接受转移与完成 Adapty 集成之间预留 15 至 30 分钟

**完成转移后：**
- 立即测试收据验证
- 监控自动续订成功率，持续 48 小时
- 验证服务器通知是否已送达您的系统
- 检查新购买记录是否被正确追踪

## 验证转移是否成功完成 \{#verify-transfer-completed-successfully\}

在完成 Adapty 和应用商店的转移后：

1. **检查看板访问权限：** 新所有者应能在其 Adapty 看板中看到该应用。
2. **验证 API 密钥连接：** 检查新的 App Store Connect API 密钥或 Google Play 服务账户是否在 Adapty 中成功连接。
3. **测试 SDK 连接：** 运行您的应用，验证 Adapty SDK 初始化时是否无报错。
