**Interactions** 面板用于定义流程元素如何响应事件——例如点击、元素出现以及表单提交。对于每个事件，你可以指定一个或多个动作：在屏幕之间导航、显示或隐藏元素、打开 URL、设置变量等。使用条件可以根据用户数据自定义流程。

每个交互都遵循三段式链：
1. **Element（元素）**：触发交互的页面组件——按钮、测验选项、输入框或其他任何组件。
2. **Trigger（触发器）**：激活逻辑的事件，例如点击、元素出现或表单提交。
3. **Action（动作）**：流程响应时执行的任务。一个触发器可以按顺序执行多个动作。
## 设置交互 \{#set-up-interactions\}

设置交互的步骤如下：

1. 在画布上或 **Layers** 面板中选中一个元素。
2. 在右侧切换到 **Interactions** 面板，点击 **Add trigger**。

<ZoomImage id="add-trigger.webp" width="500px" alt="Add trigger button" />

3. 在 **Button triggers** 区域，选择[触发类型](#trigger-types)。

<ZoomImage id="trigger-type.webp" width="500px" alt="Trigger type selection" />

4. 点击 **Add action**，点击动作名称，然后在 **Edit action** 窗口的下拉菜单中选择[动作类型](#action-types)。
<ZoomImage id="add-action.gif" width="500px" alt="Add action button" />

5. 根据你选择的[操作类型](#action-types)配置操作属性。
6. 如有需要，点击 **Add action** 添加更多针对同一触发器的操作。
## 触发器类型 \{#trigger-types\}

目前，你可以在流程中添加以下触发器：
- **On tap**：当用户点击某个元素时触发操作。这是最常用的触发器，主要用于在屏幕之间进行导航。
- **On screen appear**：在屏幕加载后立即触发操作。可用于在屏幕加载时播放动画或记录展示事件。
## 操作类型 \{#action-types\}
### 导航至屏幕 \{#navigate-to-screen\}

这是用于在屏幕之间移动用户的主要操作，会将用户带到指定的目标屏幕。

对于此操作，只需设置目标屏幕即可。如果想启用动态导航，请参阅[导航与分支](onboarding-navigation-branching)或[条件操作](#conditional-actions)部分。

:::important
**Navigate to screen** 应始终是列表中的最后一个操作。放在它之后的任何操作（如"Set Variable"）可能不会执行，因为应用已经切换了屏幕。
:::
<ZoomImage id="navigate-to.webp" width="500px" alt="导航到屏幕" />
### 返回上一页 \{#navigate-back\}

将用户导航回其历史记录中的上一个页面，而不是序列中的上一个页面。
### 打开 URL \{#open-url\}

打开指定网页地址。可用于将用户引导至应用外部的网页、文章或社交媒体主页。
对于此操作，您可以配置两项设置：
- **URL address**：设置一个 URL 地址。此外，您还可以使其动态化——例如，根据用户的问卷回答或提交的数据将其导航到不同页面。为此，请点击 <Inline id="variable.webp" alt="Variable icon" /> 并选择要使用的变量。
- **Open in external browser**：定义外部链接的打开方式。默认情况下，链接会在应用内浏览器中打开，以便用户留在应用内。如需在外部浏览器中打开链接，请勾选 **Open in external browser** 复选框。
<ZoomImage id="open-url.webp" width="500px" alt="打开 URL" />
### 关闭流程 \{#close-flow\}

关闭当前流程。
### 显示/隐藏元素 \{#showhide-elements\}

显示或隐藏屏幕上的特定元素。

此操作会覆盖 **Design** 面板中 **Visibility** 设置的初始状态。如果 **Visibility** 设置为 **Hide**，则 **Show** 操作将使其显示。
### 显示提示框 \{#show-alert\}

显示一个原生系统弹出窗口。用户必须点击 **Ok** 才能继续操作。

对于提示框，你需要设置其 **Title** 和 **Message**。在这两个字段中，你都可以使用变量来让内容动态化。方法是点击 <Inline id="variable.webp" alt="Variable icon" /> 并选择要使用的变量。

<ZoomImage id="show-alert.webp" width="500px" alt="Show alert" />
### 设置变量 \{#set-variable\}

在流程中更新某个变量的值。添加此操作前，请先在左侧的 **Variables** 面板中创建变量（参见[变量](onboarding-variables)）。

点击 **Add variable**，按需设置变量及其对应的值。

<ZoomImage id="set-variable.webp" width="500px" alt="Set variable" />
### 购买 \{#purchase\}

直接从用户引导中的按钮或交互触发购买流程。使用此功能，让用户无需离开当前流程即可完成订阅或购买产品。
您可以为此操作配置两种行为：
- **应用内商店**：发起原生购买。将 **Product** 设置为指定产品，或设置为 `products.selectedProduct` 以使用用户在当前界面的选择。
- **网页支付**：将用户引导至[付费墙网页](web-paywall)，而非触发原生购买。当您希望在应用外处理交易（例如基于网页的订阅优惠）时，可使用此选项。

<ZoomImage id="onboarding-purchase-action.webp" width="500px" alt="编辑操作窗口中的购买操作配置" />
### 自定义 \{#custom\}

触发一个由应用代码处理的自定义命名动作。当内置动作类型无法满足需求时使用此选项，例如触发应用内事件、应用折扣或显示自定义原生页面。

设置自定义动作的步骤：

1. 在 **Edit action** 窗口中，指定一个 **Action ID**——应用识别该动作所用的字符串（例如 `show_discount`）。
2. 在应用代码中，为该 Action ID 实现对应的处理逻辑。详情请参阅[处理付费墙动作](handle-paywall-actions)。
<ZoomImage id="onboarding-custom-action.webp" width="500px" alt="编辑操作窗口中带有 Action ID 字段的自定义操作配置" />
## 条件操作 \{#conditional-actions\}

使用条件操作可以根据用户数据将流程分流到不同的路径。

常见使用场景包括：

- 屏幕上有一个测验，你想根据用户的回答将其引导到不同的页面。此时，可以给按钮添加一个条件操作。
- 你想为不同用户群体提供不同的产品和优惠。将它们放在不同的屏幕上，并为导航按钮设置条件。
- 你想跳过已经在上一个应用会话中完成教程的用户的某些步骤。
条件动作的运行方式类似 if / else-if / else 链。应用从上到下逐条读取规则，遇到第一个匹配项即停止：
1. **IF**：流程检查主条件。
    - 结果为 True？流程立即执行 THEN 动作并停止。
    - 结果为 False？流程跳至下一节。
2. **ELSE IF**：可在此添加额外检查（例如，"如果不是 Premium 用户，是否处于试用期？"）。
3. **ELSE**（兜底）：如果以上所有规则均未匹配，流程将执行此最终节中的动作。
:::warning
- 如果添加了规则但未分配任何动作，匹配该条件后将不执行任何操作。
- 如果添加了变量但缺少运算符或值，该条件将被视为无效。
:::

对于每条规则，选择要评估的变量和要执行的动作。每条规则可以设置多个动作。

:::important
流程只执行一条规则——即第一条匹配的规则。如果需要同时执行 **IF** 和 **ELSE IF**，请将两个动作都添加到 **IF** 中。
:::
要了解如何使元素可选并将其组织成组以在条件中使用，请参阅[可选元素与分组](flow-selectable-elements)。
