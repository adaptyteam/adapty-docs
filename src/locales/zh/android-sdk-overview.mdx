## 主要概念 \{#main-concepts\}

在开始写代码之前，先来了解一下 Adapty 的核心概念。

Adapty 方案的优雅之处在于：只有版位是硬编码在应用里的。其他所有内容——产品、付费墙设计、定价和优惠——都可以在 Adapty 看板上灵活管理，无需更新应用：

1. [**产品**](product) - 应用内所有可购买的内容，包括订阅、消耗型商品或永久授权。
2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其价值的唯一方式。我们如此设计，是为了更方便地追踪不同产品组合对你的变现数据图表的影响。Adapty 中的付费墙既是你的产品集合，也包含与之对应的视觉配置。
3. [**版位**](placements) - 用户旅程中你想要展示付费墙的关键节点。版位定义了变现策略中的"在哪里"和"什么时候"。常见的版位包括：
   - `main` - 主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 从应用设置页面进入

   首次集成时，先从 `main` 或 `onboarding` 等基础版位开始，之后再[思考应用中还有哪些地方的用户可能准备好购买](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 当用户购买产品后，其用户画像将被赋予一个**访问等级**，用于定义对付费功能的访问权限。
