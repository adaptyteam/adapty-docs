欢迎！我们致力于让应用内购买变得轻松愉快 🚀

我们构建了 Adapty React Native SDK，旨在消除应用内购买的烦恼，让您专注于最擅长的事情——打造优秀的应用。以下是我们为您处理的内容：

- 开箱即用地处理购买、收据验证和订阅管理
- 无需更新应用即可创建和测试付费墙
- 零配置获取详细的购买分析数据——包含同期群、LTV、流失率和漏斗分析
- 跨应用会话和设备始终保持用户订阅状态最新
- 仅需一行代码即可将您的应用与营销归因和分析服务集成

无论您的应用是使用 **Expo** 还是**纯 React Native** 构建的，Adapty SDK 均完全支持两种环境。

:::note
在深入代码之前，您需要将 Adapty 与 App Store Connect 和 Google Play Console 集成，然后在看板中设置产品。请查看我们的[快速入门指南](quickstart)，先完成所有配置。
:::

## 核心概念 \{#main-concepts\}

在深入代码之前，让我们先熟悉一下使 Adapty 运作的关键概念。

Adapty 方法的优雅之处在于，只有版位是硬编码在您的应用中的。其他所有内容——产品、付费墙设计、定价和优惠——都可以从 Adapty 看板灵活管理，无需更新应用：

1. [**产品**](product) - 应用中可供购买的任何内容——订阅、消耗型商品或永久授权。

2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其效力的唯一方式。我们之所以这样设计，是为了更轻松地追踪不同产品组合对您的变现数据图表的影响。Adapty 中的付费墙既代表您特定的产品组合，也包含与之对应的视觉配置。

3. [**版位**](placements) - 用户旅程中您希望展示付费墙的战略节点。将版位视为您变现策略的"在哪里"和"何时"。常见版位包括：
   - `main` - 您的主要付费墙位置
   - `onboarding` - 在用户引导流程中显示
   - `settings` - 可从应用设置中访问

   首次集成时从 `main` 或 `onboarding` 等基础版位开始，然后[思考应用中用户可能准备好购买的其他位置](choose-meaningful-placements)。

4. [**用户画像**](profiles-crm) - 当用户购买产品时，其用户画像会被分配一个**访问等级**，您可以用它来定义对付费功能的访问权限。
