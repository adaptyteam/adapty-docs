欢迎！我们致力于让应用内购买变得轻松愉快 🚀

我们打造 Adapty React Native SDK，就是为了让应用内购买不再繁琐，让你专注于最擅长的事——打造出色的应用。以下是我们为你处理的一切：
- 开箱即用地处理购买、收据验证和订阅管理
- 无需更新应用即可创建和测试流程与付费墙
- 零配置获取详细的购买分析——包含同期群、LTV、流失率和漏斗分析
- 跨会话和设备始终保持用户订阅状态最新
- 只需一行代码即可将应用与营销归因和分析服务集成

无论你的应用是基于 **Expo** 还是**纯 React Native** 构建，Adapty SDK 均完全支持这两种环境。
:::note
在深入代码之前，你需要先将 Adapty 与 App Store Connect 和 Google Play Console 集成，然后在看板中配置产品。请先查看我们的[快速入门指南](quickstart)，完成所有配置。
:::

## 核心概念 \{#main-concepts\}

在深入代码之前，让我们先熟悉一下使 Adapty 运作的关键概念。

Adapty 方法的优雅之处在于，只有版位是硬编码在您的应用中的。其他所有内容——产品、付费墙设计、定价和优惠——都可以从 Adapty 看板灵活管理，无需更新应用：

1. [**产品**](product) - 应用中可供购买的任何内容——订阅、消耗型商品或永久授权。

2. **流程或付费墙** - 与配置绑定的产品，挂载在一个版位上。有两种形式：
   - **[流程](adapty-flow-builder)** - 在流程编辑工具中构建的可视化无代码界面。Adapty 负责渲染 UI 并替你处理购买逻辑。
   - **[付费墙](paywalls)** - 无可视化配置；你需要在自己的代码中构建 UI，并自行调用 `makePurchase`。详见[手动实现付费墙](react-native-quickstart-manual)。

   在 SDK 代码中，两者均通过同一个 `getFlow` 方法获取。
3. [**版位**](placements) - 用户旅程中你希望展示流程或付费墙的关键节点。版位解决的是变现策略中"在哪里"和"什么时候"的问题。常见版位包括：
   - `main` - 主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 从应用设置中访问

   初次集成时先从 `main` 或 `onboarding` 等基础版位入手，之后再[考虑应用中哪些地方的用户更有购买意愿](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 当用户购买产品时，其用户画像会被分配一个**访问等级**，您可以用它来定义对付费功能的访问权限。
