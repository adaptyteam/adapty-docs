Discovery 和 Search Match 广告活动会生成搜索词数据。将这些数据转化为结构化的关键词列表，需要手动下载报告、筛选搜索词并将其添加到广告组。搜索词自动化可自动完成这一过程：当某个搜索词满足您设置的条件时，规则将根据您配置的操作对其执行相应动作。

搜索词规则有两种操作类型：
- **Add as keyword**：将该词条提升为目标广告组中的精确匹配关键词，并可选择在来源广告系列中将其设为否定关键词，以避免重复消耗预算。
- **Add as negative keyword**：直接将该词条设为否定关键词，而不进行提升。适用于从 Discovery 和 Search Match 广告系列中过滤掉不相关或低效的搜索词条。
**添加为关键词**的典型用例：让 Discovery 或 Search Match 广告系列收集真实用户查询词，然后使用规则检测超过效果阈值的词语，并将其作为精确匹配关键词推送到 Probing 广告系列中——同时在来源处将其否定。Probing 广告系列是专门用于以可控出价测试推广关键词的 Apple Search Ads 广告系列。

**添加为否定关键词**的典型用例：如果某个词语出现频率较高但从未带来转化（例如，展示次数高但点击次数为零），则自动将其否定，以避免在该词语上浪费预算。
## 创建搜索词自动化规则 \{#create-a-search-term-automation-rule\}

您可以通过模板或手动从头创建搜索词自动化规则。

:::note
在创建规则之前，请确保您已有正在运行并收集搜索词数据的 Discovery 或 Search Match 广告活动。仅使用精准匹配的广告活动不会生成搜索词报告，因此规则将无数据可操作。
:::
### 从模板创建 \{#from-a-template\}

要从模板创建规则：
1. 在左侧边栏中，前往 **Automations** 并点击 **Templates**。
   <ZoomImage id="asa-rule-templates.webp" width="700px" alt="自动化规则模板" />
2. 选择一个模板并点击 **Next**。
   <ZoomImage id="asa-search-template.webp" width="700px" alt="选择模板" />
3. 查看并调整预填写的设置：
   - **Rule name**：自动设置为模板名称和当前日期。
   - **Apply to**：选择规则应在其中查找搜索词的广告系列组、应用、广告系列或广告组。
   - **Conditions**：根据需要修改预配置的条件。
   - **Actions**：根据需要调整目标广告组、CPT 出价和否定关键词范围。
   - **Schedule**：设置规则的运行频率。
   <ZoomImage id="asa-templated-rule-search.webp" width="700px" alt="模板规则设置" />
4. 点击 **Save** 以激活规则。
### 手动创建 \{#manually\}

要从头创建自定义搜索词自动化规则：
1. 在左侧边栏中，进入 **Automations**，点击 **Create rule**，然后选择 **Search terms** 作为规则类型。
   <ZoomImage id="asa-create-rule.webp" width="700px" alt="Create rule button" />
2. 输入描述性的 **Rule name** 以标识规则的用途。
3. 在 **Apply to** 部分，选择应让规则搜索搜索词的广告系列组、应用、广告系列或广告组。
   <ZoomImage id="asa-search-rule-apply-to.gif" width="700px" alt="Apply to selection" />
4. 点击 **Add condition**，然后从列表中选择一个[数据图表](adapty-ads-manager-metrics)。
数据图表按照您账户货币计算所选时间范围内的数据。数据近实时更新，因此规则始终使用最新的效果数据。
   <ZoomImage id="asa-search-condition.webp" width="700px" alt="添加条件" />
5. 设置时间段（例如，过去 3 天或过去 7 天），选择比较运算符，并输入阈值。
6. 如需添加更多条件，请点击 **Add condition** 并在左侧选择 **And** 或 **Or** 运算符。
   <ZoomImage id="search-combine.gif" width="700px" alt="组合条件" />
7. 在 **Action** 部分，选择当搜索词满足条件时所执行的操作：
**Add as keyword**

将匹配的搜索词作为精确匹配关键词添加到目标广告组。
- **Target ad groups**：选择接收推广关键词的广告组。如需构建探索渠道，请在 Probing 或其他结构化广告系列中选择广告组。
   - **CPT bid**：为每个推广关键词设置初始每次点击出价。选项：广告组默认出价、搜索词的当前 CPT，或指定值。
   - **Skip if keyword already exists**：启用后，跳过目标广告组中已存在的词条。
   - **Add as negative**：将相同词条添加为否定关键词，以避免为相同流量重复付费。
     - **Scope**：选择添加否定关键词的广告组。
:::tip
启用同一规则中的 **Add as negative** — 一步将词提升至结构化广告系列，同时在来源处将其否定。这可保持您的 Discovery 广告系列整洁，并自动构建关键词漏斗。
:::

**Add as negative keyword**

否定匹配的搜索词，但不对其进行推广。

- **Scope**：选择要添加否定关键词的广告组或广告系列。
- **Match type**：选择 **Exact** 或 **Broad**。
使用此操作可从 Discovery 和 Max Conversion 广告系列中排除不相关或低质量的搜索词。例如：如果某个词的展示次数超过 50 次且点击次数为 0，则自动将其排除。

   <ZoomImage id="asa-search-rule-action.webp" width="700px" alt="规则操作" />
8. 在 **Schedule** 部分：
   - 选择频率：**Every day**、**Every 2 days**、**Every week** 等。
   - 选择运行时间（所有时间均为 UTC）。
规则按 UTC 时间在计划时间运行。执行通常在几分钟内完成，之后您可以在 Logs 和主看板中查看变更。
   <ZoomImage id="asa-schedule.webp" width="700px" alt="Schedule settings" />
9. 点击 **Save** 以创建规则。

规则运行后，前往 **Automations** → **Logs** 并打开您的规则对应的条目。成功运行后会列出每个已评估的搜索词及其来源广告系列、目标广告组、操作结果和否定结果。如果没有出现任何词条，则说明条件未满足——请检查您的阈值或延长回顾窗口期。
## 最佳实践 \{#best-practices\}
- **使用 Discovery 或 Search Match 广告系列作为来源**：这些广告系列类型会收集真实用户查询，为您的规则提供大量可供评估的搜索词。
- **将阈值与回溯窗口匹配**：在 7 天内达到 2 次或以上下载量是一个合理的起始点。较长的窗口期（14–30 天）会降低实际门槛——搜索词可以通过较低频率的转化。对于高流量应用，请缩短窗口期并提高阈值。
- **推广时务必在来源处添加否定词**：如果您在 Probing 广告系列中将某个词添加为关键词，但未在 Discovery 中将其否定，则两个广告系列将竞争相同的查询。请在同一规则中启用 **Add as negative**。
- **有针对性地选择目标广告组**：将推广的词导入特定的 Probing 广告组，而非宽泛的广告系列。这样可以保持关键词结构清晰，并使效果分析更加便捷。
- **每次运行后检查日志**：查看 Logs 标签页，确认哪些词已被推广及其目标位置。在初期阶段，设置完成后手动运行规则以验证其行为是否符合预期。请参阅[自动化](ads-manager-automations#explore-logs)了解如何读取日志。
- **在关键词规则对已推广关键词生效前留出时间**：如果您在同一广告系列中使用关键词规则，请排除新推广的关键词，或等待一两天后再让规则对其执行。关键词规则可能会对没有效果历史的新词触发，并在其产生转化之前削减出价。
- **对曝光量高但点击为零的词使用"添加为否定关键词"**：Discovery 和 Max Conversion 广告系列经常出现不相关的搜索词。设置"展示次数 > 50 且点击次数 = 0"的规则可自动捕获这些词，并在其积累更多无效曝光之前将其否定。
## 导出搜索词 \{#export-search-terms\}

要将搜索词表格导出为 CSV 文件，请点击表格上方的下载图标，然后选择 **Export current page** 或 **Export all pages**。

**Export all pages** 会将所有页面的搜索词下载到单个文件中。下载进度弹窗会追踪下载状态，您可以随时取消。

表格将按照其在看板上的显示形式导出，包含您选择展示的列。

<ZoomImage id="export-search-terms.webp" width="700px" alt="导出搜索词" />
