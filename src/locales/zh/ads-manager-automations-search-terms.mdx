探索活动（Discovery）和搜索匹配（Search Match）活动会生成搜索词数据。将这些数据转化为结构化的关键词列表，需要下载报告、筛选词条，并手动将其添加到广告组中。搜索词自动化可自动完成这一过程：当某个词满足您设定的条件时，规则会将其作为精确匹配关键词提升到目标广告组，在来源处将其添加为否定关键词，或同时执行两项操作。

典型使用场景：让探索活动或搜索匹配活动收集真实用户查询词，然后使用规则检测超过性能阈值的词条，将其作为精确匹配关键词推送到试探活动（Probing campaign）中，同时在来源处否定它们。试探活动是一种专门用于在受控出价下测试提升关键词的 Apple Search Ads 活动。

## 创建搜索词自动化规则 \{#create-a-search-term-automation-rule\}

您可以从模板创建搜索词自动化规则，也可以从头手动创建。

:::note
在创建规则之前，请确保您已有正在运行并收集搜索词数据的探索活动或搜索匹配活动。仅限精确匹配的活动不会生成搜索词报告，因此规则将无数据可操作。
:::

### 从模板创建 \{#from-a-template\}

从模板创建规则：

1. 在左侧边栏中，前往 **Automations** 并点击 **Templates**。
   <ZoomImage id="asa-rule-templates.webp" width="700px" alt="Automation rule templates" />
2. 选择一个模板并点击 **Next**。
   <ZoomImage id="asa-search-template.webp" width="700px" alt="Choose a template" />
3. 查看并调整预填的设置：
   - **Rule name**：自动设置为模板名称和当前日期。
   - **Apply to**：选择规则应在哪些活动组、应用、活动或广告组中查找搜索词。
   - **Conditions**：根据需要修改预配置的条件。
   - **Actions**：根据需要调整目标广告组、CPT 出价和否定关键词范围。
   - **Schedule**：设置规则的运行频率。
   <ZoomImage id="asa-templated-rule-search.webp" width="700px" alt="Templated rule settings" />
4. 点击 **Save** 以激活规则。

### 手动创建 \{#manually\}

从头手动创建自定义搜索词自动化规则：

1. 在左侧边栏中，前往 **Automations**，点击 **Create rule**，然后选择 **Search terms** 作为规则类型。
   <ZoomImage id="asa-create-rule.webp" width="700px" alt="Create rule button" />
2. 输入描述性的 **Rule name**，以标识规则的用途。
3. 在 **Apply to** 部分，选择规则应在哪些活动组、应用、活动或广告组中查找搜索词。
   <ZoomImage id="asa-search-rule-apply-to.gif" width="700px" alt="Apply to selection" />
4. 点击 **Add condition** 并从列表中选择一个[数据图表](adapty-ads-manager-metrics)。

   数据图表按照您账户货币的所选时间范围计算。数据近实时更新，因此规则始终使用最新的效果数据。
   <ZoomImage id="asa-search-condition.webp" width="700px" alt="Add condition" />
5. 设置时间段（例如，过去 3 天或过去 7 天），选择比较运算符，并输入阈值。
6. 要添加更多条件，点击 **Add condition** 并在左侧选择 **And** 或 **Or** 运算符。
   <ZoomImage id="search-combine.gif" width="700px" alt="Combine conditions" />
7. 在 **Actions** 部分，配置当搜索词满足条件时发生的操作：

   - **Action**：始终为 **Add as keyword**。将匹配的搜索词作为精确匹配关键词提升。
   - **Target ad groups**：选择接收提升关键词的广告组。要构建探索流程，请选择试探活动或其他结构化活动中的广告组。
   - **CPT bid**：为每个提升的关键词设置初始每次点击费用出价。可以是广告组默认出价、搜索词当前 CPT，或特定值。
   - **Skip if keyword already exists**：启用后，规则将跳过目标广告组中已存在的关键词。
   - **Add as negative**：将相同词条作为否定关键词添加，以避免为相同流量重复付费。
     - **Scope**：选择添加否定关键词的具体广告组。

   :::tip
   在同一规则中同时启用两项操作——将词条提升到结构化活动并在来源处否定。这可以保持您的探索活动整洁，并自动构建关键词漏斗。
   :::
   <ZoomImage id="asa-search-rule-action.webp" width="700px" alt="Rule actions" />
8. 在 **Schedule** 部分：
   - 选择频率：**Every day**、**Every 2 days**、**Every week** 等。
   - 选择运行时间（所有时间均为 UTC）。

   规则按 UTC 计划时间运行。执行通常在几分钟内完成，之后您可以在日志和主看板中查看更改。
   <ZoomImage id="asa-schedule.webp" width="700px" alt="Schedule settings" />
9. 点击 **Save** 以创建规则。

规则运行后，前往 **Automations** → **Logs** 并打开您规则的条目。成功运行后，将列出每个已评估的搜索词及其来源活动、目标广告组、操作结果和否定结果。如果没有词条显示，说明条件未满足——请查看您的阈值或延长回溯窗口。

## 最佳实践 \{#best-practices\}

- **使用探索活动或搜索匹配活动作为来源。** 这些活动类型会收集真实的用户查询词，为您的规则提供大量可评估的搜索词。
- **将阈值与回溯窗口相匹配。** 7 天内 2 次或以上下载是一个合理的起点。更长的窗口（14–30 天）会降低有效门槛——词条可能以不频繁的转化通过。对于高流量应用，请缩短窗口并提高阈值。
- **提升时始终在来源处否定。** 如果您在试探活动中将某词条添加为关键词，但未在探索活动中否定它，两个活动将竞争相同的查询词。请在同一规则中启用 **Add as negative**。
- **谨慎选择目标广告组。** 将提升的词条路由到特定的试探广告组，而非宽泛的活动。这样可以保持关键词结构整洁，并使效果分析更容易。
- **每次运行后查看日志。** 查看日志选项卡，确认哪些词条被提升到何处。在初期，设置完成后手动运行规则以验证其行为是否符合预期。有关如何阅读日志，请参阅[自动化](ads-manager-automations#explore-logs)。
- **给提升的关键词留出时间，然后再让关键词规则对其操作。** 如果您在同一活动中使用关键词规则，请排除新提升的关键词，或在规则对其运行前等待一两天。关键词规则可能会对没有效果历史记录的新词条触发，并在其转化前削减出价。
