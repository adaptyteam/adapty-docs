Apple Ads Manager 提供全面的数据图表，用于衡量广告系列效果和用户行为。

## 效果 \{#performance\}

| 数据图表 | 描述 |
|--------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Spend | 用户每次点击您的广告所产生的费用总和。 |
| Impressions | 在报告时间段内，您的赞助广告在 App Store 搜索结果中显示的次数。 |
| CPM | 每千次广告展示的平均费用。平均 CPM = 花费 / (展示次数 / 1000)。注意：对于采用 CPT 定价模型的 App Store 搜索结果广告系列，此处显示的是有效 CPM。 |
| Taps | 在报告时间段内，用户点击广告的总次数。 |
| CPT | 每次点击广告的平均费用。平均 CPT = 花费 / 点击次数 |
| TTR | 用户点击广告的次数除以广告获得的总展示次数。TTR = 点击次数 / 展示次数 × 100% |
| Downloads (Total) | 在报告时间段内，通过点击和浏览产生的新下载量与重新下载量的总和。 |
| Downloads (View-Through) | 在 24 小时窗口内查看过您的广告但未点击的用户产生的下载量和重新下载量。 |
| Downloads (Tap-Through) | 在 30 天窗口内点击过您的广告的用户产生的新下载量和重新下载量总数。 |
| Avg CPA (Total) | 总平均每次获取费用（CPA）是广告系列总花费除以在报告时间段内通过浏览或点击广告产生的总下载量。 |
| Avg CPA (Tap-Through) | 点击型平均每次获取费用（CPA）是广告系列总花费除以报告时间段内点击型下载量。 |
| Download Rate (Total) | 通过浏览或点击广告产生的总下载量除以报告时间段内的总点击次数。公式：如果点击次数 > 0，则为（总下载量 / 点击次数）× 100%，否则为 0% |
| Download Rate (Tap-Through) | 通过点击广告产生的总下载量除以报告时间段内的总点击次数。公式：如果点击次数 > 0，则为（点击型下载量 / 点击次数）× 100%，否则为 0% |
| DPM (Total) | 每千次展示下载量（DPM）是每千次展示产生的下载量。公式：如果展示次数 > 0，则为（总下载量 / 展示次数）× 1000，否则为 0 |

## 转化 \{#conversions\}

:::note
收入、ARPU、ARPPU、ARPAS、ROAS 和 ROI 也可作为同期群数据图表用于对用户群体进行基于时间的分析。
:::

| 数据图表 | 描述 |
|--------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Revenue | 收入是指在选定时间范围内，通过应用内购买、续订或其他变现转化所产生的总金额（扣除商店佣金前）。 |
| ROAS | ROAS（广告支出回报率）是广告收入除以广告花费，以百分比表示。公式：如果花费 > 0，则为（收入 / 花费）× 100%，否则为 0% |
| ROI | ROI（投资回报率）衡量相对于花费的净利润。公式：如果花费 > 0，则为（（收入 − 花费）/ 花费）× 100%，否则为 0% |
| ARPU | ARPU（每用户平均收入）是每位用户的平均收入，计算方式为总收入除以唯一用户数。例如：$60000 收入 / 5000 用户 = $12 ARPU。将此值与每次安装费用（CPI）进行比较，有助于了解营销活动的效果。 |
| ARPPU | ARPPU（每付费用户平均收入）是每位付费用户的平均收入，计算方式为总收入除以唯一付费用户数。例如：$60000 收入 / 1000 名付费用户 = $60 ARPPU。它帮助您了解付费用户平均产生多少收入。 |
| ARPAS | ARPAS 是每位活跃订阅者的平均收入，计算方式为总收入 / 活跃订阅者数量。订阅者是指激活了试用期或订阅的用户。例如：$60000 收入 / 1500 名订阅者 = $40 ARPAS。 |
| Installs | 安装量是首次安装该应用的用户总数，以及现有用户的重新安装次数。包括同一用户在不同设备上的多次安装。请注意，未完成的下载或在完成前取消的安装不计入安装量。 |
| Installs CR | 安装转化率（CR）是总下载量中完成安装的用户百分比。公式：如果总下载量 > 0，则为（安装量 / 总下载量）× 100%，否则为 0% |
| CPI | CPI（每次安装费用）是 Adapty 记录的每次安装费用。公式：如果安装量 > 0，则为花费 / 安装量，否则为 0 |
| Trials | 试用量是报告时间段内新发起的试用订阅数量。 |
| Trial CR | 试用转化率（CR）是总下载量中开始试用的百分比。公式：如果总下载量 > 0，则为（试用量 / 总下载量）× 100%，否则为 0% |
| Cost per Trial | 每次试用费用是您的总花费除以新增试用开始次数。公式：如果试用量 > 0，则为花费 / 试用量，否则为 0 |
| Trials converted | 转化的试用量是报告时间段内成功转化为付费订阅的试用订阅数量。 |
| Trial converted CR | 试用转化率（CR）是转化为付费订阅的试用订阅百分比。公式：如果试用量 > 0，则为（转化的试用量 / 试用量）× 100%，否则为 0% |
| Cost per Trial converted | 每次试用转化费用是您的总花费除以同期内转化的试用数量。公式：如果转化的试用量 > 0，则为花费 / 转化的试用量，否则为 0 |
| Subscriptions | 订阅量是报告时间段内新增订阅注册数量（不含试用）。 |
| Subscription CR | 订阅转化率（CR）是总下载量中开始付费订阅（不含免费试用）的百分比。公式：如果总下载量 > 0，则为（订阅量 / 总下载量）× 100%，否则为 0% |
| Cost per Subscription | 每次订阅费用是您的总花费除以新增订阅数量。公式：如果订阅量 > 0，则为花费 / 订阅量，否则为 0 |
| Non-subscriptions started | 非订阅开始量是非订阅类型的一次性应用内购买总数量。 |
| Non-subscription CR | 非订阅转化率（CR）是总下载量中在应用内完成非订阅购买的百分比。公式：如果总下载量 > 0，则为（非订阅开始量 / 总下载量）× 100%，否则为 0% |
| Cost per Non-subscription | 每次非订阅费用是您的总花费除以非订阅购买数量。公式：如果非订阅开始量 > 0，则为花费 / 非订阅开始量，否则为 0 |

## 高级下载数据 \{#advanced-downloads\}

| 数据图表 | 描述 |
|--------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| New Downloads (Total) | 报告时间段内，点击型和浏览型新下载量的总和。 |
| New Downloads (View-Through) | 在 24 小时窗口内，查看过您的广告但未点击且此前未下载过您应用的用户产生的新下载量。 |
| New Downloads (Tap-Through) | 点击过您的广告且此前未下载过您应用的用户产生的新下载量，在 30 天归因窗口内统计。 |
| New Download Share (Tap-Through) | 显示点击型总下载量中新下载量的占比百分比（来自在 30 天归因窗口内点击过您广告的用户）。 |
| Redownloads (Total) | 报告时间段内，点击型和浏览型重新下载量的总和。 |
| Redownloads (View-Through) | 在 24 小时窗口内，查看过您的广告但未点击的用户产生的重新下载量。当用户下载您的应用、将其删除并在同一设备或不同设备上查看广告后再次下载时计入统计。 |
| Redownloads (Tap-Through) | 在 30 天归因窗口内，点击过您的广告的用户产生的重新下载量。当用户下载您的应用、将其删除并在同一设备或不同设备上点击广告后再次下载时计入统计。 |
| Redownloads Share (Tap-Through) | 显示点击型总下载量中重新下载量的占比百分比。 |

## 洞察 \{#insights\}

| 数据图表 | 描述 |
|--------|------------------------------------------------------------------------------------------------------------------------------------------|
| Impression Share | 展示份额是您的广告获得的展示次数占相同搜索词总展示次数的百分比。 |
| Rank | 排名（当前）是您的应用在特定国家或地区针对所选搜索词的展示份额排名中的当前位置。 |
| Search Popularity | 搜索热度（当前）是基于国家或地区的搜索词热度排名，评级为 1–5，5 表示搜索量最高。 |
