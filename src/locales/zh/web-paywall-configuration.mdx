在 **Web paywall** 页面点击 **Create web paywall** 后，您将被重定向到一个独立页面，用于设置网页付费墙的设计和支付方式。

## 设置支付方式 \{#set-up-a-payment-method\}

首先，您需要连接一个用于处理购买的支付提供商。可用选项如下：

- Stripe
- Paddle
- Braintree
- Paypal
- Solidgate

:::important
为确保 Adapty 中网页付费墙分析数据的准确追踪，您需要在 Adapty 中[添加产品](product)，并填写对应的 Stripe/Paddle/其他支付提供商的产品 ID。
:::

要设置支付提供商：
1. 在网页付费墙列表页面，点击 **Settings** 并切换到 **Integrations** 选项卡。
2. 选择一个支付提供商，并按照屏幕上的集成说明操作。

3. ⚠️ 如果您选择 Stripe，请确保使用的是 **Test Mode** 环境中的密钥，尽管界面显示的是 **Sandbox**。否则您的网页付费墙将无法正常工作。Stripe 中的 **Sandboxes** 目前尚不支持。

:::important
要在 Stripe 中使用 Apple Pay，您需要在 Stripe 设置中验证付费墙域名：
1. 前往 [Payment method domain settings](https://dashboard.stripe.com/settings/payment_method_domains) 并点击 **Add a new domain**。
2. 添加 `app.funnelfox.com` 以及您的个人付费墙子域名（格式类似 `paywalls-....fnlfx.com`）。要查找您的子域名，请在网页付费墙创建页面进入 **Settings > Domains**，复制 **Hosted subdomain** 的值。

要在 Paddle 中使用 Apple Pay，请在 Paddle 设置中验证付费墙域名：
1. 在 Paddle 控制台中，前往 **Checkout > Website approval** 并点击 **Add a new domain**。
2. 添加 `app.funnelfox.com` 以及您的个人付费墙子域名（格式类似 `paywalls-....fnlfx.com`）。要查找您的子域名，请在网页付费墙创建页面进入 **Settings > Domains**，复制 **Hosted subdomain** 的值。

Paddle 的审批流程为人工审核，您需要等待域名状态从 `Pending` 变为 `Approved`。
:::

## 创建并配置网页付费墙 \{#create-and-configure-a-web-paywall\}
1. 在网页付费墙列表页面，点击 **Create a paywall**。
2. 输入付费墙名称，然后点击 **Create**。

   <Zoom>
   <img src="/assets/shared/img/web-paywall-configuration-2.png"
   style={{
   border: '1px solid #727272', /* border width and color */
   width: '700px', /* image width */
   display: 'block', /* for alignment */
   margin: '0 auto' /* center alignment */
   }}
   />
   </Zoom>
3. 您将被重定向到一个基础模板，其中包含两个订阅选项和 Apple Pay 购买按钮。

   第一个屏幕列出订阅计划。第二和第三个屏幕是结账屏幕，每个屏幕对应您提供的一个计划。如果您只有一个计划，请删除多余的屏幕；如果有更多计划，则需要复制结账屏幕。

   用户成功购买后看到的最后一个屏幕，需要清晰地告知用户可以返回您的应用。

   <Zoom>
   <img src="/assets/shared/img/web-paywall-configuration-10.gif"
   style={{
   border: '1px solid #727272', /* border width and color */
   width: '700px', /* image width */
   display: 'block', /* for alignment */
   margin: '0 auto' /* center alignment */
   }}
   />
   </Zoom>
4. 设置计划列表：添加或删除计划和价格。屏幕上显示的所有价格和计划均非动态加载，因此需要手动配置。
   <Zoom>
   <img src="/assets/shared/img/web-paywall-configuration-8.gif"
   style={{
   border: '1px solid #727272', /* border width and color */
   width: '700px', /* image width */
   display: 'block', /* for alignment */
   margin: '0 auto' /* center alignment */
   }}
   />
   </Zoom>
5. 为每个计划添加或配置结账屏幕。建议在每个结账屏幕上添加总金额，以便用户在点击购买按钮之前了解需要支付的费用。
6. 在结账屏幕上，已有 Apple Pay 按钮。要使其正常工作，请在每个屏幕上配置以下内容：
   1. **Product type**：选择是否要添加试用期或折扣。
   2. **Trial period**：输入试用期时长。
   3. **Product**：从您的支付提供商中选择产品。
      :::important
      请确保该产品已添加到 Adapty。否则，购买结果将设置为默认值。
      :::
   4. **Subscription discount**：可选，从您的支付提供商中选择优惠券。
   <Zoom>
   <img src="/assets/shared/img/web-paywall-configuration-6.png"
   style={{
   border: '1px solid #727272', /* border width and color */
   width: '700px', /* image width */
   display: 'block', /* for alignment */
   margin: '0 auto' /* center alignment */
   }}
   />
   </Zoom>
7. 现在，您需要将计划与结账屏幕关联。在计划选择屏幕上，点击 **Continue** 按钮，并为每个计划选择目标屏幕。
   <Zoom>
   <img src="/assets/shared/img/web-paywall-configuration-9.png"
   style={{
   border: '1px solid #727272', /* border width and color */
   width: '700px', /* image width */
   display: 'block', /* for alignment */
   margin: '0 auto' /* center alignment */
   }}
   />
   </Zoom>

付费墙准备好后，您需要获取其链接，以便在 Adapty 中激活该付费墙。获取链接的方式取决于您是进行测试还是在生产环境中发布：

1. **沙盒测试**：点击右上角的 **Preview**，复制链接。
2. **生产环境**：点击右上角的 **Publish**。点击 **Home**，从 **URL** 列中复制链接。
