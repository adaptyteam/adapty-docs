[版位](placements)是您移动应用中的特定位置，可用于展示付费墙、用户引导或 A/B 测试。例如，订阅选择页面可能出现在启动流程中，而消耗型商品（如金币）可能在游戏中用户金币耗尽时弹出。

您可以在不同版位或针对不同用户群组（在 Adapty 中称为"目标受众"）展示相同或不同的付费墙、用户引导或 A/B 测试。

请阅读[选择有意义的版位](choose-meaningful-placements)部分，获取选择合适版位的建议。

:::tip
您也可以使用 [Developer CLI](developer-cli-reference#adapty-placements-create) 以编程方式创建版位。
:::

:::info
尽管付费墙和用户引导的版位创建流程相似，但由于它们处理不同的数据指标，您无法为两者创建同一个版位。
:::

## 创建并配置版位 \{#create-and-configure-a-placement\}

1. 从 Adapty 主菜单进入 **[Placements](https://app.adapty.io/placements)**。

2. 点击 **Create placement**。

<Zoom>
  <img src="/assets/shared/img/create-placement-2.png"
  style={{
    border: '1px solid #727272', /* border width and color */
    width: '700px', /* image width */
    display: 'block', /* for alignment */
    margin: '0 auto' /* center alignment */
  }}
/>
</Zoom>

3. 输入 **Placement name**。这是 Adapty 看板中的内部标识符，您可以在之后进行编辑。

4. 输入 **Placement ID**。您将在 Adapty SDK 中使用此 ID 来调用版位对应的[付费墙](paywalls)、[用户引导](onboardings)和 [A/B 测试](ab-tests)。由于每个版位的 ID 是唯一的，创建后无法修改。

接下来，为版位分配一个付费墙、用户引导或 A/B 测试。Adapty 支持[目标受众](audience)——基于[市场细分](segments)的用户群组——因此您可以向不同用户群体展示不同内容。如果不需要定向投放，默认的 *All users* 目标受众可覆盖所有用户。

<AddAudience />

版位保存并发布后，您已准备好一切所需——在您的应用代码中使用 **Placement ID** 来获取并展示对应内容。
