迁移分为三个步骤：

1. 切换到 Adapty SDK。
2. 更改 [Apple](enable-app-store-server-notifications)/ [Google](enable-real-time-developer-notifications-rtdn) 服务器到服务器通知 webhook。
3. （可选）[将历史数据导入 Adapty](importing-historical-data-to-adapty) 以即时获取统计数据。

让我们快速了解每个部分。

:::info
您的订阅用户将自动完成迁移

所有曾经激活过订阅的用户，一旦打开集成了 Adapty SDK 的新版本应用，即会自动完成迁移。订阅状态验证和高级访问权限将自动恢复。
:::

### 安装 Adapty SDK \{#installing-adapty-sdk\}

在您的应用中为对应平台安装 Adapty SDK（[iOS](sdk-installation-ios)、[Android](sdk-installation-android)、[React Native](sdk-installation-reactnative)、[Flutter](sdk-installation-flutter)、[Kotlin Multiplatform](sdk-installation-kotlin-multiplatform)、[Unity](sdk-installation-unity)），并用 Adapty SDK 中相应的方法替换旧有逻辑。您需要替换的核心功能包括：

- 检查[访问等级](access-level)以开放受限内容；
- 发起购买；
- 恢复购买；
- 获取/设置用户信息。

:::tip
正在从其他订阅服务提供商切换？请参阅我们的详细迁移指南：

 - [从 RevenueCat 迁移](migration-from-revenuecat)（约 20 分钟）
 :::

### 更改 Apple 服务器通知 \{#changing-apple-server-notifications\}

Apple 和 Google 通过 [App Store 服务器通知](enable-app-store-server-notifications)向我们发送用户订阅在应用外发生的事件（续订、取消、暂停、退款等）。

Adapty 可以在没有此 URL 的情况下运行，但功能将受到限制。例如，向第三方服务的[集成](events)会有延迟，订阅数据分析不会实时更新，付费墙 A/B 测试数据也不会准确。

从旧系统切换时，有时您希望两个系统同时运行一段时间。在这种情况下，您可以使用我们的[原始事件转发](enable-app-store-server-notifications#raw-events-forwarding)功能，其中 Adapty 作为旧系统的代理服务器。

### 将历史数据迁移到 Adapty \{#move-historical-data-to-adapty\}

迁移历史数据是可选的，不会影响订阅用户的当前状态。但有以下几个理由说明这样做更好：

1. **数据分析将立即正常运行**。Adapty 通过原始交易 ID 匹配订阅用户，对于未经 Adapty SDK 上报的 Apple webhook 事件，我们不会重复计数（技术上无法实现）。
2. **已有数据将得以保留**。您将拥有所有包含用户属性的 Adapty 用户画像，并可在[市场细分](segments)和[用户画像/CRM](profiles-crm) 中使用。

请参阅我们的[教程](importing-historical-data-to-adapty)，了解如何向我们发送历史数据。
