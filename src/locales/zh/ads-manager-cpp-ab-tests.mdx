CPP A/B 测试允许你在 Apple Ads 中对比不同的自定义产品页（CPP）。你可以选择 2 到 4 个产品页，[Apple Ads Manager](adapty-ads-manager) 会在它们之间分配流量、收集表现数据，并告诉你哪个页面的转化效果最好。
您可以将**默认产品页面**作为其中一个实验变体，从而测试自定义页面是否比当前默认页面表现更好。
## 前提条件 \{#prerequisites\}

在创建 CPP A/B 测试之前，请确认以下几点：

- **已连接 Apple Ads Manager**：如尚未完成，请按照[设置指南](adapty-ads-manager-get-started)操作。
- **来源广告组有流量**：待测广告组必须至少运行 28 天，且在此期间有展示量、点击量和安装量。Apple Ads Manager 会根据这些历史数据估算测试时长和所需样本量。
- **至少有一个自定义产品页面**：请先在 App Store Connect 中创建 CPP。Apple Ads Manager 会自动读取这些页面。
## 创建 CPP A/B 测试 \{#create-a-cpp-ab-test\}

要创建测试，在左侧边栏中进入 **CPP A/B Tests**，然后点击 **Create A/B Tests**。

<ZoomImage id="cpp-ab-tests-create.webp" width="900px" alt="CPP A/B Tests page with the Create A/B Tests button" />

向导共分四个步骤：**Ad Group(s)**、**Ad Creative(s)**、**Testing Method** 和 **Review**。
### 1. 广告组 \{#1-ad-groups\}

输入**测试名称**，然后点击 **Select Ad Group** 选择要测试 CPP 的广告组。你最多可以从同一广告系列中选择四个广告组，但前提是计划在这些广告组中测试同一个广告素材。如需比较多个 CPP，请选择一个广告组。

<ZoomImage id="cpp-ab-tests-step-ad-group.webp" width="700px" alt="创建 CPP A/B 测试向导：广告组步骤" />
### 2. 广告素材 \{#2-ad-creatives\}

选择要对比的 CPP。你可以加入**默认产品页面**（标记为 **Control**）和最多三个**自定义产品页面**，共 2 到 4 个实验变体。

- **默认产品页面**：点击 **+ Add Default**，将现有的默认产品页面作为对照组变体。
- **自定义产品页面**：点击 **+ Select CPP**，从 App Store Connect 中选择自定义产品页面。

<ZoomImage id="cpp-ab-tests-step-creatives.gif" width="700px" alt="Create CPP A/B Test wizard: Ad Creative(s) step" />
### 3. 测试方法 \{#3-testing-method\}

配置测试的运行方式。Apple Ads Manager 会自动计算**预计测试时长**、**开始时间**和**结束时间**——每次修改以下三项设置时，这些值都会自动更新。

<ZoomImage id="cpp-ab-tests-step-method.webp" width="700px" alt="Create CPP A/B Test wizard: Testing Method step with switch preset, precision, and confidence dropdowns" />

#### 切换时间预设 \{#switch-time-preset\}

系统在各实验变体之间轮换的频率。如果所选频率对于当前流量水平过高，系统会自动降级处理。
| 间隔 | 典型流量水平 | 时间槽时长 | 典型测试周期 |
|--------------|----------------------------------------|---------------|---------------------|
| **每小时** | 高（每天 5,000+ 次展示） | 7 小时 | 数天 |
| **每天** | 正常 | 24 小时 | 数周 |
| **每周** | 低（每天少于 400 次展示） | 7 天 | 数月 |

**时间槽**是系统在考虑切换到下一个实验变体之前，当前实验变体运行的基础时长。
#### 期望精度 \{#desired-precision\}

测试能可靠检测到的最小转化率差异。可选项：**1%**、**2%**、**3%**、**4%**、**5%**。默认值：**5%**。精度设为 1% 时，可以发现细微差异，但需要更多数据，测试周期也更长；精度设为 5% 时，测试完成更快，但只能捕捉较大的差异。
| 精度 | 使用场景 |
|-----------|-----------------------------------------------------------------------------------|
| 1–2% | 你预期各 CPP 之间的差异较小，且广告组流量较高。 |
| 3–4% | 适合大多数测试的均衡默认值。 |
| 5% | 你预期会有明显的优胜者，并希望尽快得出结果。 |

#### 置信水平 \{#confidence-level\}
您希望对结果是真实的而非随机噪声有多大把握。选项：**80%**、**85%**、**90%**、**95%**、**99%**。默认值：**90%**。置信度越高，所需数据越多。
| 置信度 | 权衡取舍 |
|------------|---------------------------------------------------------------------------------|
| 80–85%     | 完成速度更快，但结果为噪音的概率更高。                      |
| 90%        | 大多数测试的推荐默认值。                                             |
| 95–99%     | 最为保守。需要最多数据和最长的测试时间。                 |
### 4. 检查

确认摘要内容——已选择的广告组、素材、测试方式、时长、精度和置信水平——然后点击 **Start CPP A/B Tests**。

启动测试后，系统会为每个实验变体克隆广告组，激活第一个实验变体，测试状态将在几分钟内变为 **Running**。

<ZoomImage id="cpp-ab-tests-step-review.webp" width="700px" alt="Create CPP A/B Test wizard: Review step with test summary" />
## 监控运行中的测试 \{#monitor-a-running-test\}

要查看测试列表，请在左侧边栏中点击 **CPP A/B Tests**。页面顶部的四个选项卡按状态筛选测试：

- **Live**：当前正在运行的测试。
- **Completed**：已完成的测试。
- **Draft**：尚未启动的测试。
- **Archive**：不再需要在主视图中显示的历史测试。

每张测试卡片会显示测试名称、状态、切换间隔、目标精度以及已运行时长。点击 **View metrics** 可展开实验变体表格。
<ZoomImage id="cpp-ab-tests-tabs.webp" width="900px" alt="CPP A/B 测试列表，包含实时、已完成、草稿和归档选项卡" />
### 实验变体性能 \{#variant-performance\}

实验变体表格对比了测试中所有实验变体的表现：
| 列名 | 说明 |
|--------------------------|-----------------------------------------------------------------------------------------------------|
| **Variant Name** | 正在测试的 CPP。实验变体 A 始终是您添加的第一个实验变体。 |
| **Confidence Level** | 该实验变体距离所需样本量的接近程度，以 0 到 100 的百分比表示。 |
| **Impressions** | Apple 为该实验变体展示广告的次数。 |
| **TTR** | 点击率：点击次数除以展示次数。 |
| **Tap → Download CR** | 从点击到下载的转化率。 |
| **CPT** | 每次点击的平均费用。 |
| **Avg CPA (Tap-Through)** | 基于点击下载的每次获客平均费用。 |
| **Spend** | 归因于该实验变体的总花费。 |
| **Revenue** | 归因于该实验变体的总收入。 |
| **ROAS** | 广告支出回报率：收入除以花费。 |
在每个实验变体积累到足够可比的展示次数之前，Apple Ads Manager 不会高亮显示优胜者。数据仍在收集过程中时，表格上方会显示如下提示横幅：**Winner highlighting is paused — variants don't have comparable impressions yet.**

<ZoomImage id="cpp-ab-tests-variant-table.webp" width="900px" alt="展开的测试行及实验变体性能表格" />
### 详细数据图表 \{#detailed-metrics\}

如需深入了解测试情况，点击 **View metrics** 打开详细数据图表页面。该页面包含同期群留存曲线、ARPPU 对比，以及按以下两个部分分组的数据图表表格：

- **漏斗顶部 · Apple Search Ads**：各实验变体的 TTR、下载率、CPM、CPT 和平均 CPA。
- **漏斗底部 · 变现**：各实验变体的付费用户数、付费转化率、每付费用户成本、ARPPU、收入和 ROAS。
**获胜者** 列显示各项数据图表中领先的实验变体。只有当某个实验变体在主要数据图表上领先，且置信度达到至少 95% 时，才会被标记为整体获胜者。

有关数据图表定义，请参阅 [Apple Ads Manager 中的数据图表](adapty-ads-manager-metrics)。

<ZoomImage id="cpp-ab-tests-detailed-metrics.webp" width="900px" alt="Detailed metrics page with cohort retention chart, ARPPU comparison, and per-variant metrics" />
## 停止测试 \{#stop-a-test\}

你可以随时停止测试。测试状态将标记为 **Stopped**，原始广告组将被恢复，克隆的广告组将被暂停。

停止正在运行的测试：

1. 点击左侧边栏中的 **CPP A/B Tests**。
2. 在测试卡片上点击 **Stop A/B test**，或打开测试后点击 **Stop Test**。
3. 在 **Stop A/B Test?** 对话框中确认操作。

:::important
停止测试是不可逆的——无法恢复。已收集的结果仍可在 **Completed** 标签页中查看。
:::
## 测试状态 \{#test-statuses\}

每个测试都会经历一组固定的状态：
| 状态 | 含义 |
|---------------|------------------------------------------------------------------------------------------|
| **Draft** | 测试已创建但尚未启动，仍可进行编辑。 |
| **Starting** | 正在配置——系统正在克隆广告组并创建广告。 |
| **Running** | 测试已上线，实验变体轮流展示并收集数据图表。 |
| **Completed** | 预定时长已到或已达到置信度，原广告组已恢复。 |
| **Stopped** | 您手动停止了测试，原广告组已恢复。 |
| **Failed** | 配置失败或连续出错次数过多，您可以重新启动失败的测试。 |
## 工作原理 \{#how-it-works\}

Apple Ads Manager 使用 **Ad Group Switch** 方法：
1. 测试开始时，系统会按实验变体数量克隆源广告组。每个克隆指向不同的 CPP（其中一个可以是默认页面）。
2. 每次只运行一个克隆。系统按固定周期（每小时、每天或每周）轮换当前活跃的克隆。
3. 原始广告组在测试期间暂停，测试结束后恢复至之前的状态。
4. Apple Ads Manager 收集每个实验变体的展示量、点击量和下载量，并追踪各实验变体距离统计显著样本量的进度。
5. 当每个实验变体均获得足够数据，或达到预定时长时，测试自动结束。
## 测试运行期间的注意事项 \{#what-to-expect-while-a-test-runs\}

以下是关于运行中的测试在看板上表现的一些须知事项：
- **实验变体切换不遵循固定时钟**：切换间隔只是基准参考，Apple Ads Manager 会自动调整时机，确保每个实验变体获得公平的展示量。如果某个变体展示量偏少，它可能会比预定时长多运行一段时间。
- **结束时间可能向后推移**：如果某个变体在预定结束前数据量不足，测试会自动延长以继续收集点击数据。新的结束时间会显示在测试卡片上。
- **测试结束后，原始广告组将恢复**：所有克隆的广告组都会暂停，源广告组恢复到测试前的状态。测试结果可在 **Completed** 标签页中查看。
