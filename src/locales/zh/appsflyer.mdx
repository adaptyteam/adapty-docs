[AppsFlyer](https://www.appsflyer.com/) 是领先的移动归因和营销分析平台，作为第三方服务，它能够收集并整理营销活动的数据，帮助企业在一处清晰地了解各活动的效果。

Adapty 提供完整的数据集，让你能在一处追踪来自各应用商店的[订阅事件](events)。借助 Adapty，你可以轻松了解订阅用户的行为和偏好，并据此进行精准有效的沟通。因此，本集成方案让你能在 AppsFlyer 中追踪订阅事件，精确分析各营销活动带来的收益。

Adapty 与 AppsFlyer 的集成主要通过以下两种方式运作。

1. **从 AppsFlyer 接收归因数据**  
      [在应用代码中完成将 AppsFlyer 归因数据发送至 Adapty 的配置](appsflyer#connect-your-app-to-appsflyer)后，Adapty 将开始接收来自 AppsFlyer 的归因数据，你可以在用户画像页面方便地查看这些数据。

## 配置设置 \{#set-up-configuration\}

### 将 Adapty 连接到 AppsFlyer \{#connect-adapty-to-appsflyer\}

   对于 iOS，需在 App Store Connect 中找到 **Apple ID**（打开 [App Store Connect](https://appstoreconnect.apple.com/) 中的应用页面，进入 **General** 板块的 **App Information** 页，在屏幕左下方找到 **Apple ID**）。

另外需注意，AppsFlyer 不支持超过 26 小时的历史事件。因此，如果某事件的时间超过 26 小时，Adapty 仍会将其发送到 AppsFlyer，但事件的日期和时间将被替换为当前时间戳。

### 将应用连接到 AppsFlyer \{#connect-your-app-to-appsflyer\}

完成上述步骤后，调用 `updateAttribution` 方法保存归因数据，并使用 `Adapty.setIntegrationIdentifier()` 设置集成标识符。

## 事件结构 \{#event-structure\}

Adapty 通过 POST 请求以 JSON 格式向以下地址发送所选事件：
- API v2：`https://api2.appsflyer.com/inappevent/{app_id}`
- API v3：`https://api3.appsflyer.com/inappevent/{app_id}`（推荐）

每个事件的结构如下：

```json
{
  "appsflyer_id": "1699887556000-6192770",
  "eventName": "subscription_renewed",
  "eventTime": "2024-03-01 12:00:00",
  "eventValue": "{\"af_content_id\":\"yearly.premium.6999\",\"af_order_id\":\"GPA.3383-4699-1373-07113\",\"store_country\":\"US\",\"profile_country\":\"US\",\"af_content_type\":\"in_app\",\"af_revenue\":\"9.9900\",\"af_currency\":\"USD\",\"af_quantity\":\"1\"}",
  "os": "17.0.1",
  "bundleIdentifier": "com.example.app",
  "customer_user_id": "user_12345",
  "eventCurrency": "USD",
  "ip": "192.168.100.1",
  "advertising_id": "00000000-0000-0000-0000-000000000000",
  "idfa": "00000000-0000-0000-0000-000000000000",
  "idfv": "00000000-0000-0000-0000-000000000000",
  "att": "3"
}
```

各参数说明如下：

| 参数                 | 类型     | 说明                                                      |
|:-------------------|:-------|:--------------------------------------------------------|
| `appsflyer_id`     | String | AppsFlyer ID（通过 SDK 获取）。                               |
| `eventName`        | String | AppsFlyer 事件名称（从 Adapty 事件映射而来）。                      |
| `eventTime`        | String | 事件发生的日期和时间（UTC，格式为 `YYYY-MM-DD HH:MM:SS`）。             |
| `eventValue`       | String | 包含事件详情的 JSON 字符串（见下文）。                                 |
| `os`               | String | 操作系统版本。                                                 |
| `bundleIdentifier` | String | 应用的 Bundle ID / 包名。                                    |
| `customer_user_id` | String | 用户的 Customer User ID。                                  |
| `eventCurrency`    | String | 货币代码（例如 "USD"）。                                        |
| `ip`               | String | 用户的 IP 地址。                                             |
| `advertising_id`   | String | **仅 Android**。Google 广告 ID。                            |
| `idfa`             | String | **仅 iOS**。广告主标识符（IDFA）。                                |
| `idfv`             | String | **仅 iOS**。供应商标识符（IDFV）。                                |
| `att`              | String | **仅 iOS**。App 跟踪透明度状态（例如 "3" 表示已授权）。                   |

`eventValue` 参数是一个 JSON 编码的字符串，包含以下字段：

| 参数                | 类型     | 说明                    |
|:------------------|:-------|:----------------------|
| `af_content_id`   | String | 商店中的产品 ID。            |
| `af_order_id`     | String | 原始交易 ID。              |
| `store_country`   | String | 商店用户所在国家/地区代码。        |
| `profile_country` | String | 基于用户 IP 的国家/地区代码。     |
| `af_content_type` | String | 有收益时始终为 `in_app`。     |
| `af_revenue`      | String | 收益金额，保留 4 位小数。        |
| `af_currency`     | String | 货币代码。                 |
| `af_quantity`     | String | 有收益时始终为 `1`。          |

## 问题排查 \{#troubleshooting\}

### 收益数据不一致 \{#revenue-discrepancy\}

如果 Adapty 和 AppsFlyer 之间的收益数据存在差异，通常是因为并非所有用户都在使用包含 Adapty SDK 的应用版本。为确保数据一致性，你可以强制用户更新到包含 Adapty SDK 的版本。

### 集成数据缺失 \{#missing-integration-data\}

如果事件发送失败，通常是由于集成数据缺失。请检查以下事项：
- 你的应用已安装 AppsFlyer SDK。
- 你正在调用 `getAppsFlyerUID` 方法。

### 身份验证失败 \{#authentication-failure\}

如果控制台出现 `Failed to authenticate` 错误，可能是 AppsFlyer 版本与凭据版本不匹配导致的。

请参阅[迁移指南](switch-from-appsflyer-s2s-api-2-to-3)，或从[此处](https://hq1.appsflyer.com/security-center/api-tokens)替换为有效凭据。
