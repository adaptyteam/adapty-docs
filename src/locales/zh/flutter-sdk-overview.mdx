欢迎使用！我们致力于让应用内购买变得轻松愉快 🚀
我们打造了 Adapty Flutter SDK，让应用内购的繁琐工作不再困扰你，让你专注于真正重要的事——构建出色的应用。以下是我们为你处理的一切：

- 开箱即用地处理购买、凭证验证和订阅管理
- 无需更新应用即可创建和测试付费墙
- 零配置获取详细的购买分析数据——包含同期群、LTV、流失率和漏斗分析
- 跨会话、跨设备始终保持用户订阅状态最新
- 只需一行代码，即可将应用与营销归因及分析服务集成
:::note
在深入了解代码之前，你需要先将 Adapty 与 App Store Connect 和 Google Play Console 集成，然后在看板中配置产品。请先查阅我们的[快速入门指南](quickstart)完成所有配置。
:::

## 主要概念 \{#main-concepts\}

在深入代码之前，我们先来了解 Adapty 的核心概念。

Adapty 方案的优雅之处在于：只有版位需要硬编码到应用中。其他所有内容——产品、付费墙设计、定价和优惠——都可以在 Adapty 看板上灵活管理，无需更新应用：

1. [**产品**](product) - 应用中任何可供购买的内容，包括订阅、消耗型商品或永久授权。
2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其功能的唯一途径。我们如此设计，是为了更方便地追踪不同产品组合对变现数据图表的影响。Adapty 中的付费墙既代表一组特定的产品，也包含与之对应的视觉配置。
3. [**版位**](placements) - 用户旅程中你希望展示付费墙的关键位置。版位回答的是变现策略中"在哪里"和"什么时候"的问题。常见版位包括：
   - `main` - 主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 从应用设置中访问

   初次集成时先从 `main` 或 `onboarding` 等基础版位入手，之后再[思考应用中还有哪些地方的用户可能已准备好付费](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 用户购买产品后，其用户画像会被分配一个**访问等级**，用于控制对付费功能的访问权限。
