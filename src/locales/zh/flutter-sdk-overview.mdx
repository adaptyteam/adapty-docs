我们打造了 Adapty Flutter SDK，让应用内购买不再令你头疼，助你专注于最擅长的事——构建出色的应用。以下是我们为你处理的一切：
- 开箱即用，轻松处理购买、收据验证和订阅管理
- 无需更新应用，直接创建和测试流程与付费墙
- 零配置即可获取详细的购买分析数据，涵盖同期群、LTV、流失率及漏斗分析
- 跨会话、跨设备实时同步用户订阅状态
- 只需一行代码，即可将应用与营销归因和分析服务集成

## 主要概念 \{#main-concepts\}

在深入代码之前，我们先来了解一下 Adapty 的核心概念。

Adapty 方案的妙处在于：只有版位是在应用中硬编码的，其他所有内容——产品、付费墙设计、定价和优惠——都可以在 Adapty 看板上灵活管理，无需更新应用：

1. [**产品**](product) - 应用中可供购买的任何内容——订阅、消耗型商品或永久授权。
2. **流程或付费墙** - 将产品与配置打包后绑定到版位。有两种形式：
   - **[流程](adapty-flow-builder)** - 在 Flow Builder 中构建的可视化无代码界面。Adapty 负责渲染 UI 并处理购买流程。
   - **[付费墙](paywalls)** - 无可视化配置；你需要在自己的代码中构建 UI，并自行调用 `makePurchase`。参见[手动实现付费墙](flutter-quickstart-manual)。

   在 SDK 代码中，两者均通过同一个 `getFlow` 方法获取。
3. [**版位**](placements) - 用户旅程中展示流程或付费墙的关键节点，可以理解为变现策略的"在哪里"和"什么时候"。常见版位包括：
   - `main` - 主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 从应用设置中进入

   初次集成时，从 `main` 或 `onboarding` 等基础版位入手，然后再[思考应用中还有哪些地方的用户可能已准备好进行购买](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 当用户购买产品后，其用户画像将被分配一个**访问等级**，您可以用它来定义对付费功能的访问权限。
