我们打造了 Adapty Flutter SDK，让应用内购买不再繁琐，让你专注于最擅长的事——开发出色的应用。以下是我们为你处理的一切：

- 开箱即用，处理购买流程、收据验证和订阅管理
- 无需更新应用，即可创建和测试付费墙
- 零配置获取详细的购买数据分析，包含同期群、LTV、流失率和漏斗分析
- 跨会话、跨设备始终保持用户订阅状态最新
- 只需一行代码，即可将应用与营销归因和数据分析服务集成

## 主要概念 \{#main-concepts\}

在深入代码之前，先来了解一下 Adapty 的核心概念。

Adapty 方案的精妙之处在于：你的应用中只需硬编码版位，其他所有内容——产品、付费墙设计、定价和优惠——都可以直接在 Adapty 看板上灵活管理，无需更新应用：

1. [**产品**](product) - 应用中可供购买的任何内容，包括订阅、消耗型商品或永久授权。
2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其功能的唯一途径。我们之所以这样设计，是为了更方便地追踪不同产品组合对变现数据图表的影响。Adapty 中的付费墙既代表一组特定的产品，也包含与之对应的视觉配置。
3. [**版位**](placements) - 用户旅程中展示付费墙的关键节点，即变现策略的"在哪里"和"什么时候"。常见版位包括：
   - `main` - 主付费墙位置
   - `onboarding` - 用户引导流程中展示
   - `settings` - 可从应用设置中访问

   首次集成时，先从 `main` 或 `onboarding` 等基础版位入手，再[考虑应用中还有哪些地方的用户可能已准备好付费](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 当用户购买产品后，其用户画像会被分配一个**访问等级**，您可以通过它来定义对付费功能的访问权限。
