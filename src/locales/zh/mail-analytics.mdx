Adapty Mail 会追踪每一条 SES 投递事件以及每一条归因于邮件点击的收入事件。数据图表以**每个实验变体行**的形式在两处展示：
- **Flows 页面**：在触发器视图中，每个市场细分行内联显示，无论该行内容是活动还是 A/B 测试。
- **A/B 测试页面**：仅适用于 A/B 测试——在这里可以并排比较各实验变体。

两个视图的数据图表集完全相同。没有独立的分析看板；分析数据直接显示在对应行的旁边。
## 每个实验变体跟踪的数据 \{#whats-tracked-per-variation\}

每个实验变体行会显示以下原始计数：
- **发送量**：已发送至 SES 的邮件数。
- **送达量**：SES 确认送达收件箱的邮件数。
- **退信量**：SES 报告的退信数。硬退信与软退信不作区分，均计为一次**退信**。
- **打开量**：像素加载次数。Apple Mail 隐私保护会在 iOS 15 及以上版本预加载图片，导致该数据虚高——建议以点击量和收入作为更可靠的参考指标。
- **点击量**：邮件正文中的链接点击次数。
- **退订量**：通过页脚链接或 `List-Unsubscribe` 标头触发的退订次数。
- **购买量**：已归因的购买事件数，按计划邮件去重统计。同一封计划邮件的收件人多次购买仅计为一次。
- **收入**：订阅开始、续订及一次性购买的归因收入总和（USD）。
## 派生指标 \{#derived-rates\}

每个比率均由原始计数计算得出。分母因指标而异——请根据你所衡量的漏斗步骤选择合适的分母。
| 比率          | 计算公式                |
| ------------- | ---------------------- |
| 送达率      | 送达数 / 发送数     |
| 退回率   | 退回数 / 发送数        |
| 打开/发送     | 打开数 / 发送数          |
| 打开/送达      | 打开数 / 送达数     |
| 点击/发送    | 点击数 / 发送数         |
| 点击/送达     | 点击数 / 送达数    |
| 点击/打开    | 点击数 / 打开数         |
| 购买/发送    | 购买数 / 发送数      |
| 购买/送达     | 购买数 / 送达数 |
| 购买/打开    | 购买数 / 打开数      |
| 购买/点击   | 购买数 / 点击数     |
| 退订/发送    | 退订数 / 发送数         |
| 退订/送达     | 退订数 / 送达数    |
| 退订/打开    | 退订数 / 打开数      |
| 收入/发送      | 收入 / 发送数        |
## 收入归因 \{#revenue-attribution\}

收入归因通过追踪链接的**最后一次点击**来实现：

1. 当收件人点击邮件中的任意链接时，Adapty Mail 会将该 `scheduled_email_id` 短暂存储到对应的用户画像中。
2. 如果此后收到一个购买事件，且该事件尚无归因记录，Adapty Mail 会将已存储的 `scheduled_email_id` 回填到该交易中——前提是购买时间戳晚于点击时间。
3. 没有前置追踪点击的购买行为将保持未归因状态。
跟踪参数为 `scheduled_email_id`。结账 URL 还通过 `{email}` 和 `{external_profile_id}` 占位符携带收件人的身份信息，以便 Web 付费墙对流程进行个性化处理——这是独立于归因的另一套机制。详见[设置结账](mail-checkout)。
## 限制 \{#limitations\}
- **没有统一看板**：数据图表以内联方式显示在 Flows 编辑器和 A/B Tests 页面中，不支持跨项目或跨活动的汇总视图。
- **A/B Tests 页面隐藏活动行**：单活动流程行以内部 A/B 测试包装器的形式存储，但 A/B Tests 页面会将其过滤掉。如需查看相关数据图表，请打开对应的流程。
- **没有日期筛选**：计数是从该实验变体首次收到 SES 事件起累积的，无法在 UI 中缩小到"最近 7 天"。
- **没有按收件人下钻**：分析视图不显示单个用户的邮件历史记录。
- **不区分软退信和硬退信**：所有退信，无论是临时的还是永久的，都会合并为一个退信计数。
- **购买数统计的是唯一计划邮件数，而非收件人数**：在同一序列中匹配多封邮件的收件人，可能在该实验变体的汇总数据中被计入多次。
- **最终一致性，而非实时**：数据从 ClickHouse 事件表中聚合而来。新事件通常在几分钟内出现，但不提供流式实时保证。
