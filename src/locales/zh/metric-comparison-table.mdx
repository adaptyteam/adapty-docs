本文概述了 Adapty Analytics 中可用的数据图表。您可以通过本文了解每个数据图表的含义，以及它与相关数据图表的区别。

如需深入了解 Adapty 处理分析数据的方式，请参阅 [Adapty 分析的工作原理](how-adapty-analytics-works)。

:::note
本文不涵盖 [Adapty User Acquisition](adapty-user-acquisition) 数据图表。请阅读 [UA analytics](ua-analytics)，了解广告活动数据图表（包括 Spend、CPI、ROAS、CTR 等）的详细信息。
:::

## 全局数据图表 \{#global-metrics\}

全局数据图表用于追踪您整个应用在所有版位和付费墙中的表现。

### 收入 \{#revenue\}

这些数据图表衡量应用产生了多少收入以及收入来源。

| 数据图表 | 描述 | 主要区别 |
|--------|-------------|----------------|
| [Revenue](revenue) | 订阅和一次性购买的总收入，扣除退款 | 实际产生的收入。根据[数据图表控制项](controls-filters-grouping-compare-proceeds)，可显示毛收入、扣除佣金后的收入，或扣除税费和佣金后的收入 |
| [MRR](mrr) | 活跃订阅的月度经常性收入 | 应用可预测的月度收入。不含一次性购买和非周期性订阅 |
| [ARR](arr) | 活跃订阅的年度经常性收入 | 计算方式与 MRR 相同，但按年度计算。适用于预测年度收入 |
| [ARPU](arpu) | 每位用户的平均收入 | 将收入除以用户总数（含付费用户和非付费用户）。反映每位用户平均产生的收入 |
| [ARPPU](arppu) | 每位付费用户的平均收入 | 仅统计在所选时间段内进行过购买的用户（含已退款交易）。始终高于 ARPU |
| [LTV（生命周期价值）](ltv) | 付费客户的收入除以同期群中付费客户数量 | 付费客户随时间产生的实现价值。与 ARPPU（单一时间段）不同，LTV 体现的是整个客户关系周期内的总收入。可按续订次数或日历天数查看 |
| [预测 LTV](predicted-ltv-and-revenue) | 同期群中每位用户的预估生命周期价值 | 前瞻性预估。与已实现的 LTV 不同，通过对交易模式应用梯度提升算法来预测未来价值。可选 3、6、9 或 12 个月 |
| [预测收入](predicted-ltv-and-revenue) | 同期群预计产生的总收入估算 | 前瞻性预估。与已实现的收入不同，预测同期群在所选时间范围内将产生的总收入。每日更新 |
| [Non-subscriptions](non-subscriptions) | 应用内购买数量：消耗型商品、非消耗型商品和非续订订阅 | 不含自动续订订阅 |
| [Refund events](refund-events) | 退款购买或订阅的数量 | 按退款日期归因，而非原始购买日期 |
| [Refund money](refund-money) | 所选时间段内的退款总金额 | 退款的财务影响。在扣除渠道费用前计算。与 Refund events（计数）不同，此项显示的是货币金额 |

### 订阅者与转化 \{#subscribers-and-conversion\}

这些数据图表追踪用户如何进入应用并经历漏斗各阶段。

| 数据图表 | 描述 | 主要区别 |
|--------|-------------|----------------|
| [Installs](installs) | 该时间段内的应用安装次数 | 根据[安装定义](general#4-installs-definition-for-analytics)，统计以下之一：<br /> • 设备安装次数（重新安装应用的用户会被重复计入）<br /> • 唯一用户数（仅统计设置了 `customer_user_id` 的用户。匿名用户完全不计入——如果没有已识别用户，则计数为 0） |
| [New trials](new-trials) | 该时间段内激活的试用 | 统计每次试用开始，即使在您查看数据图表时该试用已到期或已转为付费 |
| [Active trials](active-trials) | 尚未到期的试用数量 | 仅统计在该时间段结束时仍处于活跃状态的试用 |
| [New subscriptions](reactivated-subscriptions) | 该时间段内首次激活的订阅 | 不含续订和重新激活 |
| [Active subscriptions](active-subscriptions) | 尚未到期的付费订阅数量 | 不含试用和已取消续订的订阅 |
| [Install to trial](analytics-conversion#install---trial) | 开始试用的安装用户百分比 | 分母包含所有安装用户，而非仅付费墙查看者，因此该比率可能低于 Paywall view to trial。如果应用未记录付费墙查看，两个数据图表也可能出现差异。这可能发生在自定义付费墙未调用 `logShowPaywall`，或用户通过[推广应用内购买](https://developer.apple.com/documentation/storekit/supporting-promoted-in-app-purchases-in-your-app)开始试用的情况下。 |
| [Paywall view to trial](analytics-conversion#paywall-view---trial) | 查看付费墙后开始试用的用户百分比 | 仅统计查看过付费墙的用户，因此该比率可能高于 Install to trial |
| [Trial to paid](analytics-conversion#trial---paid) | 试用用户购买订阅的百分比 | 衡量试用质量和转化效率。与 Install to paid 不同，仅关注完成试用的用户 |
| [Install to paid](analytics-conversion#install---paid) | 购买首个订阅的安装用户百分比 | 统计所有安装用户，而非仅付费墙查看者。该比率可能低于 Paywall view to paid。包含直接购买和试用转付费的转化 |
| [Paywall view to paid](analytics-conversion#paywall-view---paid) | 最终购买订阅的付费墙查看者百分比 | 仅统计查看过付费墙的用户，因此该比率可能高于 Install to paid。包含先完成试用的用户 |

### 留存率与订阅续订 \{#retention-and-subscription-renewal\}

这些数据图表追踪应用留住付费订阅者的效果。

| 数据图表 | 描述 | 主要区别 |
|--------|-------------|----------------|
| [Retention](analytics-retention) | 每个计费周期（第 1 次续订、第 2 次续订等）后剩余的原始订阅者比例 | 从首次付款起追踪订阅者。与下方按周期对比的数据图表不同，始终与原始群体进行比较，让您一目了然地看到全貌 |
| [Paid to 2nd period](analytics-conversion#paid---2nd-period) | 首次订阅者续订第二个周期的百分比 | 衡量两个相邻特定周期之间的过渡。与 Retention 不同，聚焦于最关键的单次续订——第一次续订 |
| [2nd to 3rd period](analytics-conversion#2nd-period---3rd-period) | 从第 2 个周期续订到第 3 个周期的百分比 | 反映首次续订后的早期留存稳定性 |
| [3rd to 4th period](analytics-conversion#3rd-period---4th-period) | 从第 3 个周期续订到第 4 个周期的百分比 | 中期留存指标 |
| [4th to 5th period](analytics-conversion#4th-period---5th-period) | 从第 4 个周期续订到第 5 个周期的百分比 | 长期忠诚度指标 |
| [6 Months+](analytics-conversion#6-months-) | 首次订阅者保持订阅超过 6 个月的百分比 | 衡量日历时间，而非续订次数。年度订阅者即使没有续订，在 6 个月时也被视为留存 |
| [1 Year+](analytics-conversion#1-year-) | 首次订阅者保持订阅超过 12 个月的百分比 | 年度留存里程碑 |
| [2 Years+](analytics-conversion#2-years-) | 首次订阅者保持订阅超过 24 个月的百分比 | 长期留存里程碑 |

### 流失 \{#churn\}

这些数据图表衡量应用流失的订阅者和试用用户数量。

| 数据图表 | 描述 | 主要区别 |
|--------|-------------|----------------|
| [Trials renewal cancelled](trials-renewal-cancelled) | 用户已禁用自动续订的试用 | 用户可继续使用试用直至到期，但不会自动转为付费。与 Subscriptions renewal cancelled 不同，适用于尚未付费的试用用户 |
| [Expired (churned) trials](expired-churned-trials) | 已到期的试用——用户已失去对高级功能的访问权限 | 用户已失去访问权限。按到期日期归因，即使用户在上一个周期已取消续订。可按原因分组（主动取消与计费问题） |
| [Subscriptions renewal cancelled](cancelled-subscriptions) | 用户已禁用自动续订的订阅 | 用户在周期结束前仍有访问权限。是流失风险信号，而非实际流失——用户可能在周期到期前重新启用自动续订 |
| [Churned (expired) subscriptions](churned-expired-subscriptions) | 已到期的订阅——用户已失去对高级功能的访问权限 | 实际流失。用户已失去访问权限。按到期日期归因，即使用户在上一个周期已取消续订。可按原因分组（主动取消与计费问题） |

### 计费问题与收入恢复 \{#billing-issues-and-revenue-recovery\}

这些数据图表追踪应用从计费问题导致的收入损失中恢复的效率。

| 数据图表 | 描述 | 主要区别 |
|--------|-------------|----------------|
| [Grace period](grace-period) | 因计费失败而进入宽限期的订阅 | 包含超出宽限期且已失去访问权限的用户 |
| [Grace period to paid](analytics-conversion#grace-period---paid) | 在宽限期结束前成功续订的宽限期用户百分比 | 一个比率（%）。回答"有多少比例的宽限期用户完成了恢复？" |
| [Grace period converted](grace-period-converted) | 成功续订的宽限期订阅绝对数量 | 与 Grace period to paid 统计相同的事件，但以计数而非百分比形式显示 |
| [Grace period converted revenue](grace-period-converted-revenue) | 来自宽限期恢复的收入 | 宽限期功能的财务影响 |
| [Billing issue](billing-issue) | 进入计费问题状态的订阅 | 在宽限期到期后开始计算。与 Grace period 不同，仅统计已失去高级访问权限的用户 |
| [Billing issue to paid](analytics-conversion#billing-issue---paid) | 在计费周期结束前成功续订的计费问题用户百分比 | 一个比率（%）。回答"有多少比例的计费问题用户完成了恢复？" |
| [Billing issue converted](billing-issue-converted) | 成功续订的计费问题订阅绝对数量 | 成功续订的计费问题订阅计数。与 Billing issue to paid 统计相同的事件，但以计数而非百分比形式显示 |
| [Billing issue converted revenue](billing-issue-converted-revenue) | 来自计费问题恢复的收入 | 计费问题恢复的财务影响 |

## 付费墙、版位和用户引导数据图表 \{#paywall-placement-and-onboarding-metrics\}

这些数据图表针对各个[付费墙](paywall-metrics)、[版位](placement-metrics)和[用户引导](onboarding-metrics)进行计算。它们衡量特定付费墙或版位的表现，而非整个应用的表现。**关联全局数据图表**列显示了全局分析部分中对应的数据图表。

| 数据图表 | 描述 | 主要区别 | 关联全局数据图表 |
|--------|-------------|----------------|---------------|
| [Proceeds](paywall-metrics#proceeds) | 单个版位扣除税费和佣金后的收入 | 等同于扣除税费和佣金后的 [Revenue](revenue) | [Revenue](revenue) |
| [ARPPU](paywall-metrics#arppu) | 该付费墙或版位的每位付费用户平均收入 | 与全局 ARPPU 计算方式相同，但范围限定于单个付费墙或版位 | [ARPPU](arppu) |
| [ARPAS](paywall-metrics#arpas) | 收入除以活跃订阅者数量（试用和付费） | 包含试用用户。与 ARPPU 不同，反映整个订阅者群体的收入潜力 | — |
| [Views](paywall-metrics#views) | 付费墙或版位的总显示次数 | 统计每次显示。同一用户查看同一付费墙两次计为 2 次浏览 | — |
| [Unique views](paywall-metrics#unique-views) | 查看付费墙或版位的唯一用户数 | 无论用户查看多少次，每位用户仅计一次。与 Views 不同，衡量的是覆盖范围而非参与频率 | — |
| [CR to purchases](paywall-metrics#cr-to-purchases) | 购买次数除以总浏览次数 | 以总浏览次数（包含同一用户的重复浏览）作为分母 | [Paywall view to paid](analytics-conversion#paywall-view---paid) |
| [Unique CR to purchases](paywall-metrics#unique-conversion-rate-cr-to-purchases) | 购买次数除以唯一浏览次数 | 以唯一浏览次数作为分母。由于重复浏览者仅计一次，该比率高于非唯一 CR | [Paywall view to paid](analytics-conversion#paywall-view---paid) |
| [CR to trials](paywall-metrics#unique-cr-to-trials) | 开始试用次数除以总浏览次数 | 衡量付费墙将浏览转化为试用的效率 | [Paywall view to trial](analytics-conversion#paywall-view---trial) |
| [Unique CR to trials](paywall-metrics#unique-cr-to-trials) | 开始试用次数除以唯一浏览次数 | 计算方式与 CR to trials 相同，但以唯一浏览者作为分母 | [Paywall view to trial](analytics-conversion#paywall-view---trial) |
| [Purchases](paywall-metrics#purchases) | 该付费墙的总交易次数：新购买、试用转化、升级、降级和续订订阅 | 不含续订 | [Revenue](revenue) |
| [Trials](paywall-metrics#trials) | 通过该付费墙激活的总试用次数 | 仅限于该付费墙的范围 | [New trials](new-trials) |
| [Trials canceled](paywall-metrics#trials-canceled) | 用户已禁用自动续订的试用数量 | 仅限于该付费墙试用的范围 | [Trials renewal cancelled](trials-renewal-cancelled) |
| [Refund rate](paywall-metrics#refund-rate) | 退款次数除以首次购买次数（不含续订） | 一个比率（%），而非计数。将退款归一化为购买数量 | [Refund events](refund-events)（计数，非比率） |
| [Completions](onboarding-metrics#completions--unique-completions) | 用户从第一屏到最后一屏完成用户引导流程的次数 | 仅限版位和用户引导。统计每次完成，包含同一用户的重复完成 | — |
| [Unique completions](onboarding-metrics#completions--unique-completions) | 完成用户引导流程的唯一用户数 | 仅限版位和用户引导。每位用户仅计一次。与 Completions 不同，衡量实际完成流程的用户数量 | — |
| [Unique completions rate](onboarding-metrics#unique-completions-rate) | 唯一完成次数除以唯一浏览次数 | 仅限版位和用户引导。衡量用户引导的有效性：开始流程的用户中，有多少比例实际完成了 | — |
