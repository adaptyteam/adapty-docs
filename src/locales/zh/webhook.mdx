Webhook 是接收[事件](webhook-event-types-and-fields#webhook-event-types)实时通知的高效方式，尤其适用于追踪订阅和购买变化。这使您能够监控订阅者状态并做出相应响应。

与需要持续轮询的 API 请求不同，Webhook 只需配置一次，即可在事件发生时通过 HTTP 自动发送数据。

集成 Webhook 后，您可以：

- 在后端系统中追踪订阅和购买情况。
- 根据订阅生命周期自动化流程和工作流。
- 通过提醒订阅者应用福利、处理退订决策以及解决账单问题来维系用户关系。
- 对用户行为进行详细分析。

**集成特性**

| 集成特性 | 描述 |
| :------------------------- | :---------------------------------------------------------- |
| 调度方式 | 实时更新 |
| 数据方向 | 单向数据传输：从 Adapty 到您的服务器 |
| Adapty 集成流程 | 事件由 Adapty 服务器在收到后立即发送 |

## 发送到 Webhook 的事件 \{#events-sent-to-webhook\}

您可以在 [Webhook 事件类型与字段](webhook-event-types-and-fields) 页面查看所有可发送到 Webhook 的事件类型。

您可以将所有事件发送到 Webhook，或仅选择其中部分事件。请参阅我们的[事件流](event-flows)页面，以决定哪些事件是必要的。您可以在[设置 Webhook 集成](set-up-webhook-integration#configure-webhook-integration-in-the-adapty-dashboard)时禁用不需要的事件类型。在该页面，您还可以根据需要将 Adapty 默认事件 ID 替换为您自己的事件 ID。
