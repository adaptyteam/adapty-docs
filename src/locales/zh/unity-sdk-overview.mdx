## 主要概念 \{#main-concepts\}

在深入代码之前，先来了解一下让 Adapty 运转起来的几个核心概念。

Adapty 的设计理念在于：只有版位需要硬编码到应用中。其他所有内容——产品、付费墙设计、定价和优惠——都可以在 Adapty 看板上灵活管理，无需更新应用：

1. [**产品**](product) - 应用内所有可购买的内容，包括订阅、消耗型商品和永久授权。
2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其功能的唯一方式。我们这样设计，是为了更方便地追踪不同产品组合对你的变现数据图表的影响。Adapty 中的付费墙既是你的产品的特定组合，也包含与之对应的视觉配置。
3. [**版位**](placements) - 用户旅程中你希望展示付费墙的关键节点。版位解决的是变现策略中"在哪里"和"什么时候"的问题。常见版位包括：
   - `main` - 主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 可从应用设置页进入

   首次集成时，先从 `main` 或 `onboarding` 等基础版位入手，之后再[考虑应用中其他用户可能有购买意愿的位置](choose-meaningful-placements)。
4. [**用户画像**](profiles-crm) - 当用户购买产品后，其用户画像会被分配一个**访问等级**，您可以通过它来定义对付费功能的访问权限。
