Adapty SDK 包含两个核心模块，可无缝集成到您的移动应用中：

- **Core Adapty**：这是 Adapty 在您应用中正常运行所必需的基础 SDK。
- **AdaptyUI**：如果您使用 [Adapty 付费墙编辑工具](adapty-paywall-builder)（一款友好的无代码工具，可轻松创建跨平台付费墙），则需要此可选模块。

## 系统要求 \{#requirements\}

虽然 SDK 核心模块在技术上支持 iOS 13.0+，但实际使用中 iOS 15.0+ 是必要条件，原因如下：
- 所有 StoreKit 2 功能需要 iOS 15.0+
- AdaptyUI 模块仅支持 iOS 15.0+

## 安装 Adapty SDK \{#install-adapty-sdk\}

[![Release](https://img.shields.io/github/v/release/adaptyteam/AdaptySDK-iOS.svg?style=flat&logo=apple)](https://github.com/adaptyteam/AdaptySDK-iOS/releases)

<Tabs>
<TabItem value="spm" label="Swift Package Manager（推荐）" default>
在 Xcode 中，前往 **File** -> **Add Package Dependency...**。请注意，不同 Xcode 版本中添加 Package 依赖项的步骤可能有所不同，如有需要请参考 Xcode 文档。

1. 输入代码仓库 URL：
   ```
   https://github.com/adaptyteam/AdaptySDK-iOS.git
   ```
2. 选择版本（推荐选择最新稳定版本），然后点击 **Add Package**。
3. 在 **Choose Package Products** 窗口中，选择所需的模块：
   - **Adapty**（核心模块）
   - **AdaptyUI**（可选 - 仅在您计划使用付费墙编辑工具时需要）
   :::note
   注意：
     - 若要启用[儿童模式](kids-mode)，请选择 **Adapty_KidsMode** 代替 **Adapty**。
     - 不要选择列表中的其他 Package——您不需要它们。
   :::
4. 点击 **Add Package** 完成安装。
5. **验证安装：** 在项目导航栏中，您应该能在 **Package Dependencies** 下看到"Adapty"（如果已选择，还会看到"AdaptyUI"）。

</TabItem>
   
<TabItem value="cocoapods" label="CocoaPods（旧版）" default>

:::info

CocoaPods 目前处于维护模式，官方已停止开发。我们建议切换至 [Swift Package Manager](sdk-installation-ios#install-adapty-sdk)。

:::

1. 将 Adapty 添加到您的 `Podfile`。选择所需的模块：

    1. **Adapty** 是必选模块。
    2. **AdaptyUI** 是可选模块，仅在您计划使用 [Adapty 付费墙编辑工具](adapty-paywall-builder) 时需要。

   ```shell showLineNumbers title="Podfile"
   pod 'Adapty'
   pod 'AdaptyUI'  # optional module needed only for Paywall Builder
   ```

2. 运行：

   ```sh showLineNumbers title="Shell"
   pod install
   ```

这将为您的应用创建一个 `.xcworkspace` 文件。之后的所有开发请使用此文件。
</TabItem>
</Tabs>

## 激活 Adapty SDK 的 Adapty 模块 \{#activate-adapty-module-of-adapty-sdk\}

在您的应用代码中激活 Adapty SDK。

:::note
Adapty SDK 在您的应用中只需激活一次。
:::

<GetKey />

<Tabs groupId="current-os" queryString>
<TabItem value="swiftui" label="SwiftUI">

```swift showLineNumbers
import SwiftUI
import Adapty

@main
struct YourApp: App {
  init() {
    // Configure Adapty SDK
    let configurationBuilder = AdaptyConfiguration
      .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY") // Get from Adapty dashboard
  
   Adapty.logLevel = .verbose // recommended for development and the first production release
    
    let config = configurationBuilder.build()
    
    // Activate Adapty SDK asynchronously
    Task {
      do {
        try await Adapty.activate(with: config)
      } catch {
        // Handle error appropriately for your app
        print("Adapty activation failed: ", error)
      }
    }
    
    var body: some Scene {
      WindowGroup {
        // Your content view
      }
    }
  }
}
```

</TabItem>

## 可选配置 \{#optional-setup\}

### 日志记录 \{#logging\}

#### 配置日志系统 \{#set-up-the-logging-system\}

Adapty 会记录错误和其他重要信息，帮助您了解应用的运行状况。以下是可用的日志级别：

| 级别       | 描述                                                         |
| ---------- | ------------------------------------------------------------ |
| `error`    | 仅记录错误                                                   |
| `warn`     | 记录错误以及来自 SDK 的非严重性警告信息                      |
| `info`     | 记录错误、警告以及各种信息性消息                             |
| `verbose`  | 记录调试时可能有用的所有附加信息，例如函数调用、API 查询等   |

```swift showLineNumbers
 let configurationBuilder = AdaptyConfiguration
         .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY") 
         .with(logLevel: .verbose) // recommended for development
```

#### 重定向日志系统消息 \{#redirect-the-logging-system-messages\}

如果您需要将 Adapty 的日志消息发送到您的系统或保存到文件中，请使用 `setLogHandler` 方法并在其中实现自定义日志逻辑。该处理程序会接收包含消息内容和严重性级别的日志记录。

```swift showLineNumbers title="Swift"
Adapty.setLogHandler { record in
    writeToLocalFile("Adapty \(record.level): \(record.message)")
}
```

### 数据政策 \{#data-policies\}

Adapty 不会存储您用户的个人数据，除非您明确发送，但您可以实施额外的数据安全政策，以符合应用商店或国家/地区的相关规定。

#### 禁用 IDFA 收集与共享 \{#disable-idfa-collection-and-sharing\}

在激活 Adapty 模块时，将 `idfaCollectionDisabled` 设置为 `true` 以禁用 IDFA 的收集和共享。

当您的应用不需要 IDFA 时，可使用此参数遵守 App Store 审核指南或避免触发应用追踪透明度提示。默认值为 `false`。有关 IDFA 收集的更多详情，请参阅[分析集成](analytics-integration#disable-collection-of-advertising-identifiers)部分。

```swift showLineNumbers
let configurationBuilder =
    AdaptyConfiguration
        .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY")
        .with(idfaCollectionDisabled: true)
```

#### 禁用 IP 收集与共享 \{#disable-ip-collection-and-sharing\}

在激活 Adapty 模块时，将 `ipAddressCollectionDisabled` 设置为 `true` 以禁用用户 IP 地址的收集和共享。默认值为 `false`。

当您的应用不需要基于 IP 的功能时，可使用此参数增强用户隐私、遵守地区数据保护法规（如 GDPR 或 CCPA），或减少不必要的数据收集。

```swift showLineNumbers
let configurationBuilder =
    AdaptyConfiguration
        .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY")
        .with(ipAddressCollectionDisabled: true)
```

#### 为 AdaptyUI 中的付费墙配置媒体缓存 \{#set-up-media-cache-configuration-for-adaptyui\}

请注意，AdaptyUI 配置是可选的。您可以在不提供配置的情况下激活 AdaptyUI 模块。但如果您使用配置，则所有参数均为必填项。

```swift showLineNumbers title="Swift"
import AdaptyUI 

// Configure AdaptyUI
        let adaptyUIConfiguration = AdaptyUI.Configuration(
            mediaCacheConfiguration: .init(
                memoryStorageTotalCostLimit: 100 * 1024 * 1024,
                memoryStorageCountLimit: .max,
                diskStorageSizeLimit: 100 * 1024 * 1024
            )
        )

        // Activate AdaptyUI
        AdaptyUI.activate(configuration: adaptyUIConfiguration)
```
参数说明：

| 参数                        | 是否必填 | 描述                                                         |
| :-------------------------- | :------- | :----------------------------------------------------------- |
| memoryStorageTotalCostLimit | 必填     | 存储的总成本限制（字节）。                                   |
| memoryStorageCountLimit     | 必填     | 内存存储的条目数量限制。                                     |
| diskStorageSizeLimit        | 必填     | 磁盘存储的文件大小限制（字节）。0 表示不限制。               |

### 交易完成行为 \{#transaction-finishing-behavior\}

:::info
此功能从 SDK 3.12.0 版本开始提供。
:::

默认情况下，Adapty 在成功验证后会自动完成交易。但如果您需要高级交易验证（例如服务端收据验证、欺诈检测或自定义业务逻辑），可以将 SDK 配置为手动完成交易。

```swift showLineNumbers title="Swift"
let configurationBuilder = AdaptyConfiguration
    .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY")
    .with(transactionsFinishBehavior: .manual) // .auto is the default
```

有关如何完成交易的更多详情，请参阅[指南](ios-transaction-management)。

### 备份恢复时清除数据 \{#clear-data-on-backup-restore\}

当 `clearDataOnBackup` 设置为 `true` 时，SDK 会检测应用从 iCloud 备份恢复的情况，并删除所有本地存储的 SDK 数据，包括已缓存的用户画像信息、产品详情和付费墙。SDK 随后将以全新状态初始化。默认值为 `false`。

:::note
仅删除本地 SDK 缓存。Apple 的交易历史以及 Adapty 服务器上的用户数据不受影响。
:::

```swift showLineNumbers
let configurationBuilder = AdaptyConfiguration
    .builder(withAPIKey: "YOUR_PUBLIC_SDK_KEY")
    .with(clearDataOnBackup: true) // default – false
```

## 故障排除 \{#troubleshooting\}

#### 使用 Tuist 时出现 Swift 6 并发错误 \{#swift-6-concurrency-error-with-tuist\}

使用 [Tuist](https://tuist.dev/) 构建时，您可能会遇到 Swift 6 严格并发编译错误。典型症状包括 `AdaptyUIBuilderLogic` 中的 `@Sendable` 属性不匹配，或类似的跨模块 Sendability 错误。

这是因为 Tuist 从 SPM 包生成 Xcode 项目时不会保留 `swift-tools-version: 6.0` 设置。因此，某些 Adapty 目标（`Adapty`、`AdaptyUI`、`AdaptyUIBuilder`）使用 Swift 5 规则编译，而其他目标使用 Swift 6，从而导致跨模块 `@Sendable` 不匹配。

**解决方案**：升级至 Adapty SDK **3.15.5** 或更高版本，无论混合 Swift 语言版本如何，该版本均可解决此问题。

**临时方案**：如果无法升级，请在您的 Tuist 配置中为所有三个 Adapty 目标显式设置 Swift 6：

```swift showLineNumbers
targetSettings: [
  "Adapty": .init().swiftVersion("6"),
  "AdaptyUI": .init().swiftVersion("6"),
  "AdaptyUIBuilder": .init().swiftVersion("6"),
]
```
