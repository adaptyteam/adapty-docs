Adapty UA 的 Google Ads 集成是双向的：Adapty 从 Google 拉取广告效果数据，并将转化事件回传，让 Google 的智能出价策略能够基于真实的收入数据进行优化。

:::note
Adapty UA 仅支持**应用广告系列**。Performance Max、搜索、展示、购物及其他 Google Ads 广告系列类型不会显示在 Adapty UA 中。
:::
## 工作原理 \{#how-it-works\}

当您连接 Google 账户后，Adapty 会同步您的账户层级、广告系列和广告组。之后：

- **数据指标流入**：Adapty 每 2 小时从 Google 拉取消耗、展示次数、点击量和转化数据，并在您的用户获取分析中呈现。
- **事件流出**：当用户安装您的应用或发生购买行为时，Adapty 会使用您配置的 **App Link ID** 将该事件发送至 Google。Google 将这些事件用于归因和智能出价优化。
:::note
国家级别的数据细分可在广告系列和广告组级别查看。在广告和关键词级别，国家数据通过比例分配方式估算，可能并不精确。
:::

App Link ID 是 Google 在您将 Adapty 添加为转化衡量合作伙伴时为您的应用生成的唯一标识符。您需要在 Adapty 中输入该 ID 以建立连接。
## 第一步：连接您的 Google 账号 \{#step-1-connect-your-google-account\}

1. 在 Adapty UA 中，前往左侧边栏的 **Integrations > Google Ads**，然后点击 **Continue with Google**。

<ZoomImage id="ua-sign-in-google-ads.webp" width="700px" />

2. 使用您的 Google 账号登录并授予所需权限。

完成后，Adapty 将同步您 Google 账号下所有可访问的广告账号，包括管理员账号（MCC）层级结构。
## 步骤 2. 从 Google Ads 获取您的应用关联 ID \{#step-2-get-your-app-link-ids-from-google-ads\}

您需要在 Google Ads 中为每个平台（iOS 和 Android）生成一个关联 ID。Adapty 使用该关联 ID 将转化事件发送到正确的应用。
1. 在 Google Ads 中，点击**工具**图标并选择 **Data manager**。
2. 点击 **+ Connect Product**，选择 **Third-party app analytics**，然后点击 **Continue**。
3. 点击 **Create link ID**。
4. 从提供商列表中选择 **Other provider**，并粘贴 Adapty UA 集成设置页面中的 **Provider ID**。
5. 选择 **iOS** 作为平台，并使用 **Look up your app** 字段查找您的应用。
6. 点击 **Create link ID** 并复制生成的链接 ID — 这就是您的 **iOS App Link ID**。
7. 对 **Android** 重复步骤 3–6，以获取您的 **Android App Link ID**。
:::note
您的链接 ID 状态将显示为 **Unverified**，直到您将其添加到 Adapty 并且转化数据开始流入为止。
:::
## 第 3 步：在 Adapty UA 中创建应用广告活动 \{#step-3-create-an-app-campaign-in-adapty-ua\}

1. 在 Adapty UA 中，前往 **Integrations > Google Ads > App campaigns**。
2. 点击 **Create app campaign**。Adapty 会自动为您当前的应用创建广告活动。

<ZoomImage id="google-ads-create-ad-campaign.webp" width="700px" />

3. 在广告活动设置页面，粘贴您从 Google Ads 复制的 **iOS App Link ID** 和 **Android App Link ID**。

<ZoomImage id="app-link-id.webp" width="700px" />

4. 点击 **Save**。
## 第四步：映射事件 \{#step-4-map-events\}

在 **Events names** 部分，开启您希望发送到 Google 的事件。对于每个已启用的事件，输入或确认相应的 Google 事件名称。

Adapty 通过 App Conversion API 将这些事件发送至 Google —— 包括安装、试用开始、购买、续订等。

点击 **Save**。

<ZoomImage id="ua-google-ads-events.webp" width="700px" />
## 附加配置 \{#additional-configuration\}
### 收入覆盖 \{#revenue-override\}

如果您跟踪试用事件并希望 Google 将收入归因于这些事件，请使用 **Revenue override** 部分。当 **Trial started** 事件启用时，该部分将会显示。

对于每个试用事件目标，设置要上报为收入的订阅价格百分比。例如，设置为 30% 时，Adapty 会将订阅价格的 30% 作为试用事件的转化值发送。

要添加覆盖规则：
1. 在 **Events names** 部分启用 **Trial started**。
2. 在 **Revenue override** 中，点击 **Add override**。
3. 选择目标事件并输入收入百分比（0–100）。
4. 点击 **Save**。

<ZoomImage id="revenue-override.webp" width="500px" />
### 测试集成 \{#test-the-integration\}

在您的广告系列正式投放之前，请点击广告系列设置页面上的 **Send test event**，并选择事件类型。Adapty 将使用您配置的 App Link ID 向 Google 发送一个测试转化事件，并显示结果。
