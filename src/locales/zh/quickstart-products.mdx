在使用 Adapty 的核心功能之前，您需要添加所销售的每个产品，并将其与您支持的每个商店或支付平台关联。此设置使您能够向用户设备交付产品，并在之后的分析中对其进行追踪。

在 Adapty 中，您的应用所销售的任何内容都是一个**产品**。如果同一商品在 App Store、Google Play 或 Stripe 中都有，您可以在 Adapty 中将它们归组为一个产品。只需配置一次，即可从一个地方跨所有平台进行管理。

现在来添加您的第一个产品。

## 添加您的第一个产品 \{#add-your-first-product\}

:::tip
本快速入门介绍了创建产品所需的基本步骤。更多详情请参阅[创建产品](create-product.md)指南。
:::

假设您要添加一个月度订阅产品。

1. 从 Adapty 主菜单进入 [Products](https://app.adapty.io/products)。
2. 点击右上角的 **Create product**。

:::important
**后续步骤取决于您是否已在 App Store 和/或 Google Play 中创建了产品：**
:::

<Tabs groupId="products" queryString>

<TabItem value="no-products" label="尚未在商店中创建产品" default>

:::important
在开始之前，请确保您已配置与 [App Store](initial_ios.md) 和/或 [Google Play](initial-android.md) 的集成。对于 App Store，请确保您已[添加 App Store Connect API 密钥](app-store-connection-configuration#step-6-add-app-store-connect-api-key)，以便 Adapty 可以推送产品。
:::

3. 选择 **Create a new product and push to stores**。

4. 填写产品详情：
    - **Product name**：仅在 Adapty 控制台中对您可见的名称。
    - **Access Level**：决定购买后解锁哪些功能的唯一标识符。如果应用中所有付费用户获得相同的功能访问权限，可以使用默认访问级别：`premium`。对于更复杂的场景，请创建额外的[访问级别](access-level.md)。
    - **Subscription duration**：从列表中选择订阅时长。
        - **Weekly/Monthly/2 Months/3 Months/6 Months/Annual**：订阅时长。
        - **Lifetime**：永久授权适用于永久解锁应用高级功能的产品。
        - **Non-Subscriptions**：对于非订阅且没有时长的产品，使用非订阅类型。可用于解锁附加功能、消耗型商品等。
        - **Consumables**：消耗型商品可以多次购买，在应用使用期间可被消耗。例如游戏内货币和道具。请注意，消耗型商品不会影响访问级别。
    - **Price (USD)**：产品的美元定价。此价格将作为基准，自动计算并设置各国家/地区的价格。您可以在之后于商店中为不同国家和地区自定义价格。请参阅 [App Store](https://developer.apple.com/help/app-store-connect/manage-subscriptions/manage-pricing-for-auto-renewable-subscriptions/) 和 [Google Play](https://support.google.com/googleplay/android-developer/answer/140504?sjid=7303890784165167068-EU) 的指南。

5. 点击 **Save & Continue**，然后切换到 **App Store** 或 **Google Play** 标签页，填写该商店的产品详情。

<Tabs>
<TabItem value="App Store" label="App Store" default>

   - **Product ID**：为产品创建一个永久唯一的 ID。
   - **Product group**：选择您在 App Store Connect 中创建的现有产品组，或点击 **Create new Product Group** 并设置其名称和 ID。Adapty 创建后，您即可从下拉列表中选择。
   - **Screenshot**：上传清晰展示所提供商品或服务的应用内购买截图。该截图仅用于 App Store 审核，不会在 App Store 上展示。截图尺寸和格式要求请参阅[此处](https://developer.apple.com/help/app-store-connect/reference/app-information/screenshot-specifications/)。

:::warning
如果这是该应用的第一个产品，您必须在 App Store Connect 中手动提交审核。之后将不再需要此操作。审核完成后，Adapty 中的产品状态将自动更新。
:::

</TabItem>

<TabItem value="Google Play" label="Google Play" default>

   - **Base Product ID**：为产品创建一个永久唯一的 ID。
   - **Subscription**：选择您在 Google Play Console 中创建的现有订阅组，或点击 **Create new Product Group** 并设置其名称和 ID。Adapty 创建后，您即可从下拉列表中选择。

</TabItem>
</Tabs>

6. 通过从下拉列表中选择 **Free duration** 来配置新用户优惠（免费试用）。在初始设置中，您可以添加免费试用的新用户优惠。主产品获得商店批准后，您可以通过关联商店控制台中的现有 ID 来[添加更多优惠](offers.md)（例如促销优惠、赢回优惠）。

</TabItem>

<TabItem value="products-in-stores" label="商店中已有产品">

3. 选择 **Connect an existing store product**。
4. 填写产品详情：
- **Product name**：仅在 Adapty 控制台中对您可见的名称。
- **Access level ID**：决定购买后解锁哪些功能的唯一标识符。如果应用中所有付费用户获得相同的功能访问权限，可以使用默认访问级别：`premium`。对于更复杂的场景，请创建额外的[访问级别](access-level.md)。
- **Subscription duration**：从列表中选择订阅时长。
  - **Weekly/Monthly/2 Months/3 Months/6 Months/Annual**：订阅时长。
  - **Lifetime**：永久授权适用于永久解锁应用高级功能的产品。
  - **Non-Subscriptions**：对于非订阅且没有时长的产品，使用非订阅类型。可用于解锁附加功能、消耗型商品等。
  - **Consumables**：消耗型商品可以多次购买，在应用使用期间可被消耗。例如游戏内货币和道具。请注意，消耗型商品不会影响访问级别。
- **Price (USD)**：产品的美元定价。如果您的产品已在商店中，此值不会影响其在商店中的实际价格；您可以从列表中选择任意值。

5. 添加商店详情。选择您的商店：

<Tabs>
<TabItem value="App Store" label="App Store" default>

- **App Store Product ID**：用于在设备上访问您产品的唯一标识符。如果找不到，请确认 ID 正确且属于正确的应用。

</TabItem>

<TabItem value="Google Play" label="Google Play" default>

- **Google Play Product ID**：来自 Play Store 的产品标识符。从现有产品 ID 列表中选择。如果找不到，请确认 ID 正确且属于正确的应用。
- **Base plan ID**：在 Play Store 中定义产品基础方案的 ID。
- **Legacy fallback product**：备用产品仅用于使用旧版 Adapty SDK（2.5 及以下版本）的应用。请按以下格式填写：`<subscription_id>:<base_plan_id>`。

<details>
    <summary>点击此处了解在哪里找到 Google Play 产品 ID 和基础方案 ID。</summary>

1. 在您的 [Google Play Console](https://play.google.com/console/developers/android/app) 账户中，前往 **Monetize with Play > Products > Subscriptions**。
2. 打开相应的**订阅**。
3. 您将在 **Subscription details** 部分看到产品 ID，在 **Base plans and offers** 部分的 **ID and duration** 列中看到基础方案 ID。

- **Stripe Product ID**：来自 Stripe 的唯一产品标识符。
- **Stripe Price ID**：来自 Stripe 的与产品关联的价格唯一标识符。

<details>
    <summary>点击此处了解在哪里找到 Stripe 产品 ID 和价格 ID。</summary>

1. 前往 Stripe 中的[产品目录](https://dashboard.stripe.com/products?active=true)。
2. 打开所需产品。
3. 您将看到：
- Stripe 产品 ID（格式类似 `prod_...`）位于右上角。
- Stripe 价格 ID（格式类似 `price_...`）位于 **Pricing** 部分的 **API ID** 列中。

- **Paddle Price ID**：来自 Paddle 的与产品关联的价格唯一标识符。

2. 打开所需产品。
3. 您将看到：
- Paddle 产品 ID（格式类似 `pro_...`）位于 **Additional details** 部分。
- Paddle 价格 ID（格式类似 `pri_...`）位于 **Prices** 部分的 **ID** 列中。

<TabItem value="Custom" label="自定义商店" default>

您可以选择现有的自定义商店，或添加新商店并将产品与之关联。

请注意，Adapty 仅追踪来自 App Store、Google Play 和 Stripe 的交易。对于自定义商店，您需要使用 Adapty 服务端 API 的[设置交易方法](api-adapty/operations/setTransaction.md)来提交交易。

6. 如有需要，您可以为产品[创建优惠](create-offer)。要添加优惠，点击 **Yes, add offers**；否则点击 **No, thanks**。

您的产品将出现在产品列表中。
