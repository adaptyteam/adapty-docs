:::important
如果您拥有自己的身份验证系统，本指南专为您而编写。在这里，您将学习如何在 Adapty 中管理用户画像，以确保其与您现有的身份验证系统保持一致。
:::

如何管理用户的购买行为取决于您应用的身份验证模型：
- 如果您的应用不使用后端身份验证且不存储用户数据，请参阅[关于匿名用户的章节](#anonymous-users)。
- 如果您的应用已有（或将有）后端身份验证，请参阅[关于已识别用户的章节](#identified-users)。

**核心概念**：
- **用户画像** 是 SDK 运行所需的实体，Adapty 会自动创建。
- 用户画像可以是匿名的（**没有 customer user ID**），也可以是已识别的（**有 customer user ID**）。
- 您提供 **customer user ID** 是为了将 Adapty 中的用户画像与您内部的身份验证系统进行关联。

以下是匿名用户和已识别用户的区别：

|                         | 匿名用户                                   | 已识别用户                                                        |
|-------------------------|---------------------------------------------------|-------------------------------------------------------------------------|
| **购买管理** | 应用商店层面的购买恢复                  | 通过 customer user ID 跨设备维护购买历史 |
| **用户画像管理**  | 每次重新安装均创建新用户画像                    | 跨会话和设备使用同一用户画像                            |
| **数据持久性**    | 匿名用户的数据与应用安装绑定 | 已识别用户的数据在应用安装间持久保留                |

## 匿名用户 \{#anonymous-users\}

如果您没有后端身份验证，**您无需在应用代码中处理身份验证**：

1. 当 SDK 在应用首次启动时激活，Adapty **会为用户创建新的用户画像**。
2. 当用户在应用中购买任何内容时，该购买将**与其 Adapty 用户画像及其应用商店账户关联**。
3. 当用户**重新安装**应用或在**新设备**上安装时，Adapty **会在激活时创建新的匿名用户画像**。
4. 如果用户之前在您的应用中进行过购买，默认情况下，其购买记录会在 SDK 激活时自动从 App Store 同步。

因此，对于匿名用户，每次安装都会创建新的用户画像，但这不是问题，因为在 Adapty 分析中，您可以[配置什么被视为新安装](general#4-installs-definition-for-analytics)。

对于匿名用户，您需要按**设备 ID** 统计安装次数。在这种情况下，设备上每次安装应用都计为一次安装，包括重新安装。

## 已识别用户 \{#identified-users\}

您有两种方式在应用中识别用户：

- [**在登录/注册时：**](#during-loginsignup) 如果用户在应用启动后登录，请在他们完成身份验证时使用 customer user ID 调用 `identify()`。

- [**在 SDK 激活时：**](#during-the-sdk-activation) 如果应用启动时您已存储了 customer user ID，请在调用 `activate()` 时一并传入。

:::important
默认情况下，当 Adapty 收到来自某个 Customer User ID 的购买，而该 ID 当前与另一个 Customer User ID 关联时，访问等级将被共享，因此两个用户画像都拥有付费访问权限。您可以配置此设置，将付费访问从一个用户画像转移到另一个，或完全禁用共享。详情请参阅[此文章](general#6-sharing-paid-access-between-user-accounts)。
:::

### 在登录/注册时 \{#during-loginsignup\}

如果您在应用启动后识别用户（例如，在他们登录或注册之后），请使用 `identify` 方法设置其 customer user ID。

- 如果您**之前未使用过该 customer user ID**，Adapty 将自动将其与当前用户画像关联。
- 如果您**之前已使用该 customer user ID 识别过该用户**，Adapty 将切换到与该 customer user ID 关联的用户画像。

:::important
Customer user ID 对每位用户必须唯一。如果您将参数值硬编码，所有用户将被视为同一人。
:::

```dart showLineNumbers
try {
  await Adapty().identify(customerUserId); // Unique for each user
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
}
```

### 在 SDK 激活时 \{#during-the-sdk-activation\}

如果您在激活 SDK 时已知道 customer user ID，可以在 `activate` 方法中直接传入，而无需单独调用 `identify`。

如果您已知道 customer user ID，但仅在激活后才设置，这意味着在激活时 Adapty 会创建一个新的匿名用户画像，只有在您调用 `identify` 后才会切换到现有用户画像。

您可以传入现有的 customer user ID（之前使用过的）或新的 customer user ID。如果传入新的，激活时创建的新用户画像将自动与该 customer user ID 关联。

:::note
默认情况下，创建匿名用户画像不会影响分析看板，因为安装次数是基于设备 ID 统计的。

设备 ID 代表应用在某台设备上从应用商店的单次安装，仅在应用重新安装后才会重新生成。
它不取决于这是首次安装还是重复安装，也不取决于是否使用了现有的 customer user ID。

创建用户画像（在 SDK 激活或退出登录时）、登录或在不重新安装的情况下升级应用，不会产生额外的安装事件。

如果您希望按独立用户而非设备统计安装次数，请前往 **App settings** 并配置 [**Installs definition for analytics**](general#4-installs-definition-for-analytics)。
:::

```dart showLineNumbers"
   try {
       await Adapty().activate(
           configuration: AdaptyConfiguration(apiKey: 'YOUR_API_KEY')
             ..withCustomerUserId(YOUR_CUSTOMER_USER_ID) // Customer user IDs must be unique for each user. If you hardcode the parameter value, all users will be considered as one.
       );
   } catch (e) {
       // handle the error
   }
```

### 退出用户登录 \{#log-users-out\}

如果您有供用户退出登录的按钮，请使用 `logout` 方法。

:::important
退出用户登录会为该用户创建新的匿名用户画像。
:::

```dart showLineNumbers
try {
  await Adapty().logout();
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
  // handle unknown error
}
```

:::info
要将用户重新登录到应用，请使用 `identify` 方法。
:::

### 允许未登录时购买 \{#allow-purchases-without-login\}

如果您的用户在登录前后都可以进行购买，您需要确保他们登录后仍能保留访问权限：

1. 当未登录用户进行购买时，Adapty 会将购买绑定到其匿名用户画像 ID。
2. 当用户登录其账户时，Adapty 切换到其已识别的用户画像。
    - 如果这是一个新的 customer user ID（例如，购买发生在注册之前），Adapty 会将该 customer user ID 分配给当前用户画像，从而保留所有购买历史。
    - 如果这是一个已有的 customer user ID（该 customer user ID 已与某个用户画像关联），您需要在用户画像切换后获取实际的访问等级。您可以在识别后立即调用 [`getProfile`](flutter-check-subscription-status)，或者[监听用户画像更新](flutter-check-subscription-status)以使数据自动同步。
