[Flow Builder](adapty-flow-builder) 将保存与发布分开处理。草稿会将你的工作保存在 Adapty 看板中，而发布则会通过 SDK 将当前版本提供给用户。本文介绍这两个操作及其使用时机。
:::warning
Flow Builder 不会自动保存。如果你关闭页面而未保存，所有未保存的更改都将丢失，并恢复到最近一次保存的版本。
:::
## 将流程保存为草稿 \{#save-a-flow-as-a-draft\}

草稿会将您的进度保存在看板中，但不会影响用户看到的内容。在构建过程中请经常保存，尤其是在离开页面之前。

要保存草稿，请点击流程编辑工具右上角的 **Save draft**，或按 **Cmd/Ctrl + S**。

草稿仅在看板中可见，不会影响用户在应用中看到的内容，即使该流程已分配到某个[版位](placements)也不例外。

## 发布流程 \{#publish-a-flow\}

发布操作会通过 SDK 将当前版本的流程提供给用户使用。发布后，新版本将替换同一流程之前已发布的版本。

要发布流程，请点击 Flow Builder 右上角的 **Publish to Live**。

接下来的操作取决于该流程是否已分配到某个版位：

- **流程已在版位中**：用户在下次请求该版位时即可看到新版本。
- **流程未在版位中**：将流程添加到[版位](create-placement)，即可开始向用户展示。
