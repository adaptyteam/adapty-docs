SDK 返回的每个错误均为 `AdaptyError`。

:::important
如果这些解决方案无法解决您的问题，请在联系支持之前参阅[其他问题](#other-issues)中的步骤，以便我们更高效地为您提供帮助。
:::

| 错误 | 解决方案 |
|----------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| UNKNOWN | 发生了未知或意外错误。 |
| [ITEM_UNAVAILABLE](https://developer.android.com/reference/com/android/billingclient/api/BillingClient.BillingResponseCode#ITEM_UNAVAILABLE()) | 此错误通常出现在测试阶段，可能是因为产品尚未发布到正式环境，或者该用户不属于 Google Play 的测试人员组。 |
| ADAPTY_NOT_INITIALIZED | Adapty SDK 尚未激活。<br/> 最常见的原因是启动页或早期 UI 钩子在 `Adapty.activate` 返回之前就调用了 Adapty 方法。该问题具有偶发性，在模拟器上可能无法复现，因为真机的时序有所不同。请等待 `Adapty.activate` 完成后再调用其他 SDK 方法。完整调用顺序请参见 [Android SDK 调用顺序](android-sdk-call-order)。此外，还需要使用 `Adapty.activate` 方法正确[配置 Adapty SDK](sdk-installation-android#activate-adapty-module-of-adapty-sdk)。 |
| PRODUCT_NOT_FOUND | 请求购买的产品在商店中不可用。 |
| INVALID_JSON | <p>本地备用付费墙的 JSON 格式无效。</p><p>请先修复默认的英文付费墙，然后替换无效的本地付费墙。有关如何修复付费墙，请参阅[使用远程配置自定义付费墙](customize-paywall-with-remote-config)；有关如何替换本地付费墙，请参阅[设置本地备用付费墙](fallback-paywalls)。</p> |
| <p>CURRENT_SUBSCRIPTION_TO_UPDATE</p><p>\_NOT_FOUND_IN_HISTORY</p> | 需要替换的原始订阅在活跃订阅中未找到。 |
| [BILLING_SERVICE_TIMEOUT](https://developer.android.com/google/play/billing/errors#service_timeout_error_code_-3) | 请求在 Google Play 响应前已达到最大超时时间。例如，Play Billing Library 调用所请求的操作执行延迟可能导致此错误。 |
| [FEATURE_NOT_SUPPORTED](https://developer.android.com/reference/com/android/billingclient/api/BillingClient.BillingResponseCode#FEATURE_NOT_SUPPORTED()) | 当前设备的 Play 商店不支持所请求的功能。 |
| [BILLING_SERVICE_DISCONNECTED](https://developer.android.com/google/play/billing/errors#service_disconnected_error_code_-1) | 客户端应用通过 `BillingClient` 与 Google Play 商店服务的连接已断开。 |
| [BILLING_SERVICE_UNAVAILABLE](https://developer.android.com/google/play/billing/errors#service_unavailable_error_code_2) | Google Play Billing 服务当前不可用。大多数情况下，这意味着客户端设备与 Google Play Billing 服务之间的某处存在网络连接问题。 |
| [BILLING_UNAVAILABLE](https://developer.android.com/google/play/billing/errors#billing_unavailable_error_code_3) | <p>购买过程中发生了结算问题，可能原因包括：</p><p>1. 用户设备上缺少 Play 商店应用或版本过旧。</p><p>2. 用户所在国家/地区不受支持。</p><p>3. 用户属于某企业账户，且管理员已禁用购买功能。</p><p>4. Google Play 无法向用户的付款方式扣款（例如信用卡已过期）。</p><p>5. 用户未登录 Play 商店应用。</p> |
| [DEVELOPER_ERROR](https://developer.android.com/google/play/billing/errors#developer_error) | API 使用方式不正确。 |
| [BILLING_ERROR](https://developer.android.com/google/play/billing/errors#error_error_code_6) | Google Play 内部出现问题。 |
| [ITEM_ALREADY_OWNED](https://developer.android.com/reference/com/android/billingclient/api/BillingClient.BillingResponseCode#ITEM_ALREADY_OWNED()) | 该产品已购买。 |
| [ITEM_NOT_OWNED](https://developer.android.com/reference/com/android/billingclient/api/BillingClient.BillingResponseCode#ITEM_NOT_OWNED()) | 用户未拥有该商品，因此无法执行所请求的操作。 |
| [BILLING_NETWORK_ERROR](https://developer.android.com/google/play/billing/errors#network_error_error_code_12) | 设备与 Play 系统之间的网络连接出现问题。 |
| NO_PRODUCT_IDS_FOUND | <p>付费墙中没有任何产品在商店中可用。</p><p>如果遇到此错误，请按以下步骤排查：</p><ol><li>检查所有产品是否已添加到 Adapty 看板。</li><li>确认应用的**包名**与 Google Play Console 中的包名一致。</li><li>核实应用商店中的产品标识符与看板中添加的标识符一致。请注意，标识符不应包含 Bundle ID，除非商店本身已包含。</li><li>确认应用在 Google 税务设置中的付费状态为**有效**，并确保税务信息是最新的、证书有效。</li><li>检查应用是否已绑定银行账户，以便符合变现资格。</li><li>检查产品在您所在地区是否可用。</li><li>确保您的应用处于某个测试轨道中。**内部测试**轨道是最简便的选择，无需审核且对用户不可见。</li></ol> |
| NO_PURCHASES_TO_RESTORE | Google Play 未找到可恢复的购买记录。 |
| AUTHENTICATION_ERROR | 需要使用 `Adapty.activate` 方法正确[配置 Adapty SDK](sdk-installation-android#activate-adapty-module-of-adapty-sdk)。 |
| BAD_REQUEST | 请求无效。<br/> 请确保已完成[与 Google Play 集成](google-play-store-connection-configuration)所需的所有步骤。 |
| SERVER_ERROR | 服务器错误。 |
| REQUEST_FAILED | 发生了无法具体定位的网络问题。 |
| DECODING_FAILED | 无法解析响应。<br/> 请检查代码，确保发送的参数有效。例如，此错误可能表示您使用了无效的 API 密钥。 |
| ANALYTICS_DISABLED | 由于您已[禁用外部分析](analytics-integration#disabling-external-analytics-for-a-specific-customer)，无法处理分析事件。 |
| WRONG_PARAMETER | 某些参数不正确：不能为空的字段为空，或参数类型错误等。 |
## 其他问题 \{#other-issues\}

如果您尚未找到解决方案，可以采取以下后续步骤：

- **将 SDK 升级到最新版本**：我们始终建议升级到最新的 SDK 版本，因为它们更稳定并包含已知问题的修复。
- **联系支持团队或在[支持论坛](https://adapty.featurebase.app/)中获得其他开发者的帮助**。
- **通过 [support@adapty.io](mailto:support@adapty.io) 或在线聊天联系支持团队**：如果您还不打算升级 SDK 或升级后问题仍未解决，请联系我们的支持团队。请注意，如果您[启用详细日志记录](sdk-installation-android#logging)并与团队共享日志，您的问题将得到更快解决。您也可以附上相关代码片段。
