Adapty 用户获取是一套归因解决方案，通过整合广告平台、跟踪链接和您的应用数据，将广告系列与应用安装及订阅收入相关联。它提供统一的营销分析看板，将您所有的获客数据集中在一处。

- 跨所有渠道计算 ROAS（广告支出回报率）
- 在一处查看您的完整应用经济数据
- 获取精准归因数据，助力更优决策
- 随时间分析同期群表现和用户行为

## 为什么选择 Adapty UA？ \{#why-adapty-ua\}

衡量用户获取表现颇具挑战性。数据往往分散在各平台，隐私政策的变化使归因愈加困难，自建解决方案也需要耗费大量时间。

Adapty UA 在单一营销看板中提供内置归因与统一分析。您所有的获客数据指标——从广告支出到安装量再到订阅收入——均可自动整合并实时更新。无需再在多张电子表格之间比对数据，也无需在多个工具之间来回切换。您可以专注于应用增长，而非管理数据系统。

## 工作原理 \{#how-it-works\}

Adapty 用户获取通过整合广告平台、跟踪链接和您的应用数据，将应用安装和订阅收入归因至广告系列。

整体架构如下：
- 广告平台提供系列结构和广告支出数据
- 在 Adapty UA 中生成的跟踪链接，将系列上下文从网页传递至应用安装
- Adapty SDK 从您的应用发送安装和收入事件

归因流程如下：

1. **在 Adapty UA 中生成跟踪链接并添加至广告系列。** 链接包含系列参数，例如平台、广告系列、广告组和广告素材。

2. **用户点击广告并从应用商店安装应用。** 用户通过跟踪链接跳转，并从 App Store 或 Google Play 安装应用。

3. **应用向 Adapty 发送安装事件。** 首次启动时，Adapty SDK 发送安装事件，Adapty 提取与该安装关联的系列参数。

4. **安装被归因至某个广告系列。** Adapty 使用跟踪链接中的系列参数，将该安装与生成它的广告系列关联起来。

5. **广告支出与收入相连接。** Adapty 从受支持的广告平台（目前包括 Meta Ads 和 TikTok for Business）拉取广告支出数据，并将订阅和购买事件关联至已归因的安装记录。

最终，Adapty 在统一分析看板中提供系列级别的数据指标，包括安装量、收入、LTV 和 ROAS。您可以分析同期群、追踪长期表现，并在无需手动整合不同来源数据的情况下，做出基于数据的优化决策。

:::tip
跟踪链接还可以包含自定义参数，让您的应用在处理安装事件时能够实现[延迟深度链接](ua-deferred-data)并响应系列数据。
:::
