**市场细分**是一组过滤条件，用于将具有共同属性的用户分组。使用市场细分可以更有效地为付费墙和 A/B 测试定向。

创建市场细分后，您可以[在版位和 A/B 测试中将其用作**目标受众**](audience)，以控制用户看到哪个付费墙（单个或多个）。示例：

- 向非订阅用户显示标准付费墙，并向曾取消订阅或试用的用户提供折扣。
- 向来自不同国家/地区的用户显示不同的付费墙。
- 根据 Apple Search Ads 归因数据定向用户。
- 确保使用旧版应用的用户继续看到现有付费墙，而新版本用户则看到更新后的付费墙。
- [在分析中](controls-filters-grouping-compare-proceeds#filtering-and-grouping)，按市场细分进行筛选，查看特定用户群体的表现。按市场细分分组，可在**所有用户**范围内比较表现或贡献度。

## 创建 \{#creation\}

要创建市场细分，请输入名称并选择定义其过滤条件的属性。当您选择多个属性时，用户必须满足所有条件。Adapty 在属性之间应用 AND 逻辑。

## 可用属性 \{#available-attributes\}

:::note
虽然许多用户属性会自动设置（如**国家/地区**或**计算所得总收入（美元）**），但**年龄**、**应用用户 ID**、**归因**数据、**性别**和**自定义属性**不会自动定义。如果您想将这些属性用于市场细分，必须[设置用户属性](setting-user-attributes)或[传递归因数据](attribution-integration)。
:::

:::tip
对于基于日期的属性，您可以使用以下方式进行筛选：

- **固定日期**：从日历中选择特定日期（例如，向在黑色星期五至网络星期一期间安装应用的用户显示特别优惠）
- **相对范围**：设置动态时间窗口，如"最近 7 天"或"最近 3 个月"（例如，重新触达 30 天以上未活跃的用户，或定向近期安装用户）

相对范围会自动更新，非常适合持续性活动。固定日期最适合有时间限制的促销活动。
:::

| 属性 | 筛选依据 |
|---------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **年龄** | 用户的年龄。请注意，年龄在 Adapty 首次收到该数据时计算，之后不会更新。 |
| **应用用户 ID** | 用户在您应用中的标识符（[customer_user_id](profiles-crm#user-attributes)）。您可以按其存在或缺失进行筛选，例如，仅向未登录的用户显示付费墙。 |
| **应用版本（当前）** | 安装在用户设备上的当前应用版本，Adapty 最近从该设备接收到事件数据。创建市场细分时，点击**应用版本**旁边的铅笔图标，添加新版本即可立即使用。<br/> 使用**版本 > X.X** 条件，可以衡量所有高于或低于特定版本的应用对转化的影响，无需逐一列出每个版本。<br/><br/> **格式：** 版本字符串必须遵循 [SemVer](https://semver.org/) 格式。任何部分中的前导零均无效——`26.03.4` 不会匹配，而 `26.3.4` 会匹配。无效版本将被静默排除在市场细分之外。 |
| **应用版本（安装时）** | 用户设备上安装的应用版本，即 Adapty 首次收到事件数据时的版本。<br/><br/> **格式：** 版本字符串必须遵循 [SemVer](https://semver.org/) 格式。任何部分中的前导零均无效——`26.3.04` 不会匹配，而 `26.3.4` 会匹配。无效版本将被静默排除在市场细分之外。 |
| **归因：广告组** | 归因广告组。 |
| **归因：广告集** | 归因广告集。 |
| **归因：推广活动** | 营销活动的名称。 |
| **归因：素材** | 归因素材关键词。 |
| **归因：渠道** | 营销渠道的名称。 |
| **归因：来源** | 归因的来源。 |
| **归因：状态** | 归因状态。可能的值：<ul><li> **自然流量** – 用户在没有任何付费营销影响的情况下安装了应用（例如，在 App Store/Google Play 中直接搜索、口碑传播或自然社交媒体触达）。</li><li> **非自然流量** – 用户通过付费营销渠道获取（例如，广告、网红活动、推荐计划）。</li><li> **未知** – 该用户没有可用的归因数据。</li></ul> |
| **计算所得订阅状态** | 用户的[当前订阅状态](profiles-crm#subscription-state)，指示订阅是否处于活跃或已取消状态，或是否存在未解决的账单问题。 |
| **计算所得总收入（美元）** | 该用户产生的总收入。 |
| **国家/地区** | 客户的国家/地区，由其最近的 IP 地址确定。 |
| **商店账户所在国家/地区** | 与用户的 iOS 或 Android 商店账户关联的国家/地区。请注意，Adapty 仅为运行 iOS 13 或更高版本的 iOS 设备收集商店所在国家/地区。 |
| **创建日期** | 用户画像的创建日期（即应用首次安装在用户设备上的时间）。 |
| **设备** | 基于元数据的设备类型。例如，"Samsung Galaxy"或"iPhone 13"。 |
| **性别** | 用户的性别。请注意，该值需要您自行设置。 |
| **安装日期** | 用户安装应用的日期。 |
| **语言** | 用户设备的语言。 |
| **最后使用时间** | 用户最近一次打开应用的日期。 |
| **操作系统** | 用户设备的操作系统版本。 |
| **付费访问等级** | 授予用户的访问等级。 |
| **平台** | 用户的设备平台。可能的值：`iOS`、`macOS`、`iPadOS`、`visionOS`、`Android`。<br/> 如果用户从多个平台（例如 iOS 和 Android）访问您的应用，则市场细分成员资格会使用该特定设备的最新数据针对每个平台分别评估。即使是同一用户画像，也可以实现平台特定的定向。 |
| **订阅到期日期** | 订阅的到期日期，或其存在/缺失状态。永久授权购买显示为 `none`，如果用户有用户画像但从未有过试用、订阅或永久授权购买，则为空。 |
| **订阅产品** | 客户当前活跃订阅的最新产品 ID。 |
| **[自定义属性](profiles-crm#custom-attributes)** | 根据您的应用或业务独有的属性，定义您自己的属性以创建高度定向的市场细分。 |

## 自定义属性 \{#custom-attributes\}

定义自定义属性，基于您的应用或业务独有的属性构建更精准的市场细分。

:::note
- 您可以在移动端 SDK 或 Adapty 看板中设置自定义属性。有关 SDK 设置，请按照[此处](setting-user-attributes#custom-user-attributes)的说明操作。
- 在市场细分中使用自定义属性后，若更改该属性，可能会导致该用户在[分析](controls-filters-grouping-compare-proceeds#filtering-and-grouping)中与该市场细分不同步。数据将反映之前的值。
:::

### 如何配置自定义属性 \{#how-to-configure-a-custom-attribute\}

在 Adapty 看板中，从属性下拉菜单中选择 **Create custom attributes**。

| 字段 | 描述 |
| ------ |--------------------------------------------------------------------------------------------------------------------------------------|
| **Name** | 自定义属性的标签，仅在 Adapty 看板中使用。 |
| **Key** | 属性的唯一标识符。必须与 SDK 中使用的键匹配。 |
| **Type** | 选择以下类型：<ul><li>String：需要预定义的可能值列表。</li><li>Number：仅接受数字值。</li></ul> |
| **Values** | 如果选择 `String`，请输入可能值的列表。如果选择 `Number`，该属性将只接受数字输入。数字属性支持小数值，并可与比较运算符一起使用。 |

填写完必填字段后，您可以在市场细分、[A/B 测试](ab-tests)等中使用自定义属性。

每个用户画像最多可以有 30 个自定义属性。

## 复制市场细分 \{#duplicate-segments\}

如果您需要一个与现有市场细分类似的细分，请直接复制它，而不是从头开始构建。这为同时运行多个活动或具有重叠用户群体的 A/B 测试的团队节省了时间。

复制市场细分会创建一个包含所有过滤条件和描述的副本。新市场细分的名称会添加"(copy)"后缀，以便与原始细分区分。新市场细分与原始细分相互独立，对其中一个的更改不会影响另一个。

在 Adapty 看板中复制市场细分：

1. 在 Adapty 主菜单中打开 **Profiles & Segments** 部分，并切换到 [**Segments**](https://app.adapty.io/segments) 标签页。
2. 点击市场细分旁边的 **3 点**按钮，然后选择 **Duplicate**。

3. 打开新市场细分，根据需要调整其过滤条件。
