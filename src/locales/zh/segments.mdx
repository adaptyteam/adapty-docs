**市场细分**是一组筛选条件，用于将具有共同属性的用户归为一组。通过市场细分，可以更有效地对付费墙和 A/B 测试进行定向投放。

创建市场细分后，你可以[将其作为版位和 A/B 测试中的**目标受众**](audience)，以控制用户看到哪个付费墙（单个或多个）。示例：
- 向非订阅用户展示标准付费墙，向曾经取消订阅或试用的用户提供折扣。
- 向不同国家的用户展示不同的付费墙。
- 根据 Apple Search Ads 归因数据定向触达用户。
- 确保使用旧版应用的用户继续看到现有付费墙，而新版本用户则看到更新后的付费墙。
- [在数据分析中](controls-filters-grouping-compare-proceeds#filtering-and-grouping)，按市场细分筛选，查看特定用户群体的表现；按市场细分分组，比较 **All users** 中各组的表现或贡献。

## 创建 \{#creation\}

要创建一个市场细分，请输入名称并选择用于定义过滤条件的属性。当你选择多个属性时，用户必须满足所有条件。Adapty 在属性之间使用 AND 逻辑。

## 可用属性 \{#available-attributes\}

:::note
许多用户属性会自动设置（例如**国家**或**计算所得总收入（美元）**），但**年龄**、**App 用户 ID**、**归因**数据、**性别**和**自定义属性**不会自动定义。如需将这些属性用于市场细分，必须[设置用户属性](setting-user-attributes)或[传入归因数据](attribution-integration)。
:::

:::tip
对于基于日期的属性，您可以使用以下方式进行筛选：
- **固定日期**：从日历中选择具体日期（例如，向在"黑色星期五"到"网络星期一"期间安装的用户展示特别优惠）
- **相对范围**：设置动态时间窗口，如"最近 7 天"或"最近 3 个月"（例如，重新触达 30 天以上未活跃的用户，或定向最近安装的用户）

相对范围会自动更新，非常适合持续性活动。固定日期则更适合有时间限制的促销活动。
:::
| 属性 | 筛选条件 |
|---------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Age** | 用户年龄。请注意，年龄在 Adapty 首次获取时计算，之后不会更新。 |
| **App User ID** | 用户在您应用中的标识符（[customer_user_id](profiles-crm#user-attributes)）。您可以按其是否存在进行筛选，例如，仅向未登录的用户显示付费墙。 |
| **App version (current)** | 用户设备上当前安装的应用版本，以 Adapty 最后接收到事件数据时为准。创建市场细分时，点击 **App version** 旁的铅笔图标，添加新版本后即可立即使用。<br/> 使用 **version > X.X** 条件，可以在不逐一列举版本的情况下，衡量所有高于或低于某一特定版本的应用转化影响。<br/><br/> **格式：** 版本字符串必须遵循 [SemVer](https://semver.org/) 格式。任意部分中的前导零均无效——`26.03.4` 不会匹配，而 `26.3.4` 则可以。无效版本将被静默排除在市场细分之外。 |
| **App version (on install)** | Adapty 首次接收到事件数据时，用户设备上安装的应用版本。<br/><br/> **格式：** 版本字符串必须遵循 [SemVer](https://semver.org/) 格式。任意部分中的前导零均无效——`26.3.04` 不会匹配，而 `26.3.4` 则可以。无效版本将被静默排除在市场细分之外。 |
| **Attribution: Ad Group** | 归因广告组。 |
| **Attribution: Ad Set** | 归因广告集。 |
| **Attribution: Campaign** | 营销活动名称。 |
| **Attribution: Creative** | 归因素材关键词。 |
| **Attribution: Channel** | 营销渠道名称。 |
| **Attribution: Source** | 归因来源。 |
| **Attribution: Status** | 归因状态。可选值：<ul><li> **Organic** – 用户在没有付费营销影响的情况下安装了应用（例如，在 App Store/Google Play 中直接搜索、口碑传播或自然社交媒体流量）。</li><li> **Non-organic** – 用户通过付费营销渠道获取（例如，广告、网红推广、推荐计划）。</li><li> **Unknown** – 该用户没有可用的归因数据。</li></ul> |
| **Calculated subscription state** | 用户的[当前订阅状态](profiles-crm#subscription-state)，表示订阅是否有效、已取消，或存在尚未解决的账单问题。 |
| **Calculated total revenue USD** | 该用户产生的总收入。 |
| **Country** | 根据用户最近的 IP 地址确定的所在国家/地区。 |
| **Country from store account** | 与用户 iOS 或 Android 商店账户关联的国家/地区。请注意，Adapty 仅针对运行 iOS 13 及更高版本的设备收集商店所在国家/地区信息。 |
| **Creation date** | 用户画像的创建日期（即应用首次安装到用户设备上的时间）。 |
| **Device** | 基于元数据的设备类型，例如"Samsung Galaxy"或"iPhone 13"。 |
| **Gender** | 用户性别。请注意，该值由您自行设置。 |
| **Installation date** | 用户安装应用的日期。 |
| **Language** | 用户设备的语言。 |
| **Last seen** | 用户最近一次打开应用的日期。 |
| **OS** | 用户设备的操作系统版本。 |
| **Paid access level** | 授予用户的访问等级。 |
| **Platform** | 用户的设备平台。可选值：`iOS`、`macOS`、`iPadOS`、`visionOS`、`Android`。<br/> 如果用户从多个平台访问您的应用（例如 iOS 和 Android），则会针对每个平台使用该设备的最新数据单独评估市场细分成员资格。这样即使是同一用户画像，也可以实现基于平台的精准定向。 |
| **Subscription expiration date** | 订阅的到期日期，或其是否存在。永久授权商品显示为 `none`；如果用户有用户画像但从未进行过试用、订阅或永久授权购买，则该字段为空。 |
| **Subscription product** | 用户当前有效订阅的最新产品 ID。 |
| **[Custom attributes](profiles-crm#custom-attributes)** | 自定义属性，基于您的应用或业务特有的属性创建高度精准的市场细分。 |
## 自定义属性 \{#custom-attributes\}

通过定义自定义属性，你可以根据应用或业务的独特属性构建更精准的市场细分。

:::note
- 你可以在移动端 SDK 或 Adapty 看板中设置自定义属性。如需通过 SDK 进行设置，请参阅[此处](setting-user-attributes#custom-user-attributes)的说明。
- 在某个市场细分中使用自定义属性后，若修改该属性，可能导致用户在[分析](controls-filters-grouping-compare-proceeds#filtering-and-grouping)中与该市场细分失去同步。数据将反映修改前的值。
:::
### 如何配置自定义属性 \{#how-to-configure-a-custom-attribute\}

在 Adapty 看板中，从属性下拉菜单中选择 **Create custom attributes**。

| 字段 | 描述 |
| ------ |--------------------------------------------------------------------------------------------------------------------------------------|
| **Name** | 自定义属性的标签，仅在 Adapty 看板中使用。 |
| **Key** | 属性的唯一标识符，必须与 SDK 中使用的键名保持一致。 |
| **Type** | 可选择：<ul><li>String（字符串）：需要预定义可选值列表。</li><li>Number（数字）：仅接受数值。</li></ul> |
| **Values** | 若选择 `String`，请输入可选值列表；若选择 `Number`，该属性仅接受数值输入。数值型属性支持小数，并可与比较运算符配合使用。 |
填写完必填字段后，您可以在市场细分、[A/B 测试](ab-tests)等功能中使用自定义属性。

每个用户画像最多可以设置 30 个自定义属性。

## 复制市场细分 \{#duplicate-segments\}

如果你需要一个与现有市场细分类似的细分，直接复制它，而不必从头构建。对于同时运行多个活动或 A/B 测试且用户群有重叠的团队来说，这样可以节省大量时间。

复制市场细分会生成一个包含所有筛选条件和描述的副本。新市场细分的名称会附加"(copy)"字样，方便你与原版区分。新市场细分与原版相互独立，修改其中一个不会影响另一个。

要在 Adapty 看板中复制市场细分：
1. 在 Adapty 主菜单中打开 **Profiles & Segments** 部分，切换到 [**Segments**](https://app.adapty.io/segments) 标签页。
2. 点击市场细分旁边的 **3-dot** 按钮，选择 **Duplicate**。

## 删除市场细分 \{#delete-segments\}

当某个市场细分不再需要时，你可以将其永久删除。

如果该市场细分当前正被以下任一对象用作目标受众，Adapty 将阻止删除操作：

- **版位**：至少有一个未删除的版位将该市场细分用作其目标受众。
- **A/B 测试（进行中或已完成）**：至少有一个未删除的 A/B 测试将该市场细分用作其目标受众。
对于市场细分的删除，Adapty 将**上线中**和**已完成**的 A/B 测试均视为活跃状态。已完成的测试仍会使用该目标受众向匹配用户展示测试结束后的付费墙或用户引导，且该测试的历史数据图表也归属于此市场细分。只有当 A/B 测试本身被删除后，该市场细分才会被释放。

:::warning
市场细分的删除操作不可撤销，删除后无法恢复。
:::

要在 Adapty 看板中删除市场细分：
1. 在 Adapty 主菜单中进入 **Profiles & Segments**，切换到 [**Segments**](https://app.adapty.io/segments) 标签页。
2. 点击该市场细分旁边的 **3-dot** 按钮，选择 **Delete**。
3. 在确认字段中输入市场细分名称，然后点击 **Delete forever**。

:::info
如果该市场细分正在使用中，对话框会列出引用它的版位和 A/B 测试。
要解除删除限制，请从列表中逐一打开对应的版位或 A/B 测试，将该市场细分从其目标受众中移除，或者直接删除该版位或 A/B 测试。当没有任何内容引用该市场细分时，即可将其删除。
:::
