欢迎使用！我们致力于让应用内购买变得轻松无忧 🚀

我们构建了 Adapty iOS SDK，让您无需为应用内购买烦恼，专注于最擅长的事——打造出色的应用。以下是我们为您处理的事项：

- 开箱即用地处理购买、收据验证和订阅管理
- 无需更新应用即可创建和测试付费墙
- 零配置获取详细的购买分析数据——包含同期群、LTV、流失率和漏斗分析
- 跨应用会话和设备始终保持用户订阅状态最新
- 仅需一行代码即可将应用与营销归因和分析服务集成

:::note
在深入代码之前，您需要将 Adapty 与 App Store Connect 集成并在看板中配置产品。请先查阅我们的[快速入门指南](quickstart)完成所有配置。
:::

## 核心概念 \{#main-concepts\}

在深入代码之前，让我们先了解一下 Adapty 运作的核心概念。

Adapty 方法的优雅之处在于，只有版位是硬编码在应用中的。其他所有内容——产品、付费墙设计、定价和优惠——都可以通过 Adapty 控制台灵活管理，无需更新应用：

1. [**产品**](product) - 应用中可供购买的任何内容，包括订阅、消耗型商品或永久授权。

2. [**付费墙**](paywalls) - 从 Adapty 获取产品并充分发挥其功能的唯一方式。我们这样设计是为了更方便地追踪不同产品组合对变现数据图表的影响。Adapty 中的付费墙既是您产品的特定集合，也包含与之配套的视觉配置。

3. [**版位**](placements) - 用户旅程中您希望展示付费墙的战略节点。可以将版位理解为变现策略中的"地点"和"时机"。常见的版位包括：
   - `main` - 您的主要付费墙位置
   - `onboarding` - 在用户引导流程中展示
   - `settings` - 可从应用设置中访问

   首次集成时，从 `main` 或 `onboarding` 等基础版位开始，然后再[考虑应用中其他用户可能准备好购买的位置](choose-meaningful-placements)。

4. [**用户画像**](profiles-crm) - 当用户购买产品后，其用户画像将被赋予一个**访问等级**，您可以通过该等级来控制用户对付费功能的访问权限。
