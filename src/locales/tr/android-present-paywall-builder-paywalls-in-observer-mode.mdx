Flow Builder veya Paywall Builder kullanarak bir flow veya paywall oluşturduysanız, bunu kullanıcıya göstermek için mobil uygulama kodunuzda render etmeniz konusunda endişelenmenize gerek yoktur. Böyle bir flow veya paywall, içinde ne gösterileceğini ve nasıl gösterileceğini zaten içerir.

:::warning
Bu bölüm yalnızca [Observer mode](observer-vs-full-mode) için geçerlidir. Observer mode'da çalışmıyorsanız bunun yerine [Android - Flow ve paywall'ları gösterme](android-present-paywalls) konusuna bakın.
:::

<SDKv4>

<details>
   <summary>Flow'ları göstermeye başlamadan önce (Genişletmek için tıklayın)</summary>
1. Adapty'nin [Google Play ile](initial-android) başlangıç entegrasyonunu yapın.
   2. Adapty SDK'yı yükleyip yapılandırın. `observerMode` parametresini `true` olarak ayarladığınızdan emin olun. Framework'e özgü talimatlarımıza [Android için](sdk-installation-android) bakın.
   3. Adapty Kontrol Paneli'nde [ürünler oluşturun](create-product).
   4. [Builder'larda flow'lar veya paywall'lar yapılandırın](create-paywall) ve ürünleri bunlara atayın.
   5. Adapty Kontrol Paneli'nde [placement'lar oluşturun ve flow'larınızı ya da paywall'larınızı bunlara atayın](create-placement).
   6. Mobil uygulama kodunuzda [flow'ları ve yapılandırmalarını alın](android-get-pb-paywalls).
</details>

1. `AdaptyUiObserverModeHandler`'ı uygulayın.

`onPurchaseInitiated` olayı, kullanıcının bir satın alma işlemi başlattığını size bildirir. Bu callback'e yanıt olarak özel satın alma flow'unuzu tetikleyebilirsiniz:
<Tabs groupId="current-os" queryString>
<TabItem value="kotlin" label="Kotlin" default>
   ```kotlin showLineNumbers
   val observerModeHandler =
    AdaptyUiObserverModeHandler { product, flow, flowView, onStartPurchase, onFinishPurchase ->
       onStartPurchase()
       yourBillingClient.makePurchase(
           product,
           onSuccess = { purchase ->
               onFinishPurchase()
               //handle success
           },
           onError = {
               onFinishPurchase()
               //handle error
           },
           onCancel = {
               onFinishPurchase()
               //handle cancel
           }
       )
   }
   ```
</TabItem>
<TabItem value="java" label="Java" default>
   ```java showLineNumbers
   AdaptyUiObserverModeHandler observerModeHandler = (product, flow, flowView, onStartPurchase, onFinishPurchase) -> {
       onStartPurchase.invoke();
       yourBillingClient.makePurchase(
           product,
           purchase -> {
               onFinishPurchase.invoke();
               //handle success
           },
           error -> {
               onFinishPurchase.invoke();
               //handle error
           },
           () -> { //cancellation
               onFinishPurchase.invoke();
               //handle cancel
           }
       );
   };
   ```
</TabItem>
</Tabs>
Observer mode'da geri yüklemeleri yönetmek için `getRestoreHandler()` metodunu geçersiz kılın. Varsayılan olarak `null` döndürür; bu durumda Adapty'nin yerleşik `Adapty.restorePurchases()` flow'u kullanılır. Kendi geri yükleme implementasyonunuzu sağlamak için:
<Tabs groupId="current-os" queryString>
<TabItem value="kotlin" label="Kotlin" default>
   ```kotlin showLineNumbers
   val observerModeHandler = object : AdaptyUiObserverModeHandler {
       // onPurchaseInitiated implementation (see above)

       override fun getRestoreHandler() =
           AdaptyUiObserverModeHandler.RestoreHandler { onStartRestore, onFinishRestore ->
               onStartRestore()
               yourBillingClient.restorePurchases(
                   onSuccess = { restoredPurchases ->
                       onFinishRestore()
                       //handle successful restore
                   },
                   onError = {
                       onFinishRestore()
                       //handle error
                   }
               )
           }
   }
   ```
</TabItem>
<TabItem value="java" label="Java" default>
   ```java showLineNumbers
   AdaptyUiObserverModeHandler observerModeHandler = new AdaptyUiObserverModeHandler() {
       // onPurchaseInitiated implementation (see above)

       @Override
       public RestoreHandler getRestoreHandler() {
           return (onStartRestore, onFinishRestore) -> {
               onStartRestore.invoke();
               yourBillingClient.restorePurchases(
                   restoredPurchases -> {
                       onFinishRestore.invoke();
                       //handle successful restore
                   },
                   error -> {
                       onFinishRestore.invoke();
                       //handle error
                   }
               );
           };
       }
   };
   ```
</TabItem>
</Tabs>
AdaptyUI'yı satın alma veya geri yükleme işlemi hakkında bilgilendirmek için aşağıdaki callback'leri çağırmayı unutmayın. Bu, yükleyiciyi göstermek gibi doğru flow davranışı için gereklidir:
| Callback           | Açıklama                                                                                         |
   | :----------------- |:--------------------------------------------------------------------------------------------------|
   | onStartPurchase()  | Satın alma işleminin başladığını AdaptyUI'ya bildirmek için çağrılması gereken callback.          |
   | onFinishPurchase() | Satın alma işleminin tamamlandığını AdaptyUI'ya bildirmek için çağrılması gereken callback.       |
   | onStartRestore()   | İsteğe bağlı. Geri yükleme işleminin başladığını AdaptyUI'ya bildirmek için çağrılabilecek callback. |
   | onFinishRestore()  | İsteğe bağlı. Geri yükleme işleminin tamamlandığını AdaptyUI'ya bildirmek için çağrılabilecek callback. |

| **observerModeHandler** | Observer mode için zorunlu | Bir önceki adımda uyguladığınız `AdaptyUiObserverModeHandler`. |

               onFinishPurchase.invoke();
               //handle cancel
           }
       );
   };
