:::warning
**Onboardinglar SDK v4'te kullanımdan kaldırılmıştır ve gelecekteki bir sürümde kaldırılacaktır.** Artık hata düzeltmesi veya iyileştirme almamaktadır. Bunun yerine [flow'ları](kmp-get-pb-paywalls) kullanın: WebView içinde çalışan onboardingların aksine, flow'lar cihazda native olarak render edilir; bu sayede daha akıcı animasyonlar, tutarlı bir native görünüm, daha hızlı yükleme süreleri ve WebView bağımlılığı ortadan kalkar. Başlamak için [Flow'ları ve paywallları alma](kmp-get-pb-paywalls) ve [Flow'ları ve paywallları gösterme](kmp-present-paywalls) konularına bakın.
:::

Builder ile bir onboarding oluşturduysanız, Kotlin Multiplatform uygulamanızda onu kullanıcıya göstermek için ekstra kod yazmanıza gerek yok. Böyle bir onboarding, hem neyin gösterileceğini hem de nasıl gösterileceğini kendi içinde barındırır.

Başlamadan önce şunları tamamladığınızdan emin olun:

1. [Adapty Kotlin Multiplatform SDK](sdk-installation-kotlin-multiplatform) 3.16.1 veya daha yeni bir sürümünü yüklemiş olun.
2. Bir [onboarding oluşturmuş](create-onboarding) olun.
3. Onboarding'i bir [placement](placements)'a eklemiş olun.

Adapty Kotlin Multiplatform SDK, onboarding'leri sunmak için iki yöntem sağlar:

- **Compose Multiplatform ile**
- **Compose Multiplatform olmadan**

## Compose Multiplatform ile \{#with-compose-multiplatform\}

Bir onboarding göstermek için `createOnboardingView` metoduyla oluşturulan `view` üzerinde `view.present()` metodunu kullanın. Her `view` yalnızca bir kez kullanılabilir. Onboarding'i tekrar göstermeniz gerekiyorsa yeni bir `view` örneği oluşturmak için `createOnboardingView` metodunu bir kez daha çağırın.

:::warning
Aynı `view`'ı yeniden oluşturmadan tekrar kullanmak hataya yol açabilir.
:::

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    AdaptyUI.createOnboardingView(onboarding = onboarding).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```

### iOS sunum stilini yapılandırma \{#configure-ios-presentation-style\}

`present()` metoduna `iosPresentationStyle` parametresini geçerek onboarding'in iOS'ta nasıl sunulacağını yapılandırın. Parametre `AdaptyUIIOSPresentationStyle.FULLSCREEN` (varsayılan) veya `AdaptyUIIOSPresentationStyle.PAGESHEET` değerlerini kabul eder.

```kotlin showLineNumbers

viewModelScope.launch {
    val view = AdaptyUI.createOnboardingView(onboarding = onboarding).getOrNull()
    view?.present(iosPresentationStyle = AdaptyUIIOSPresentationStyle.PAGESHEET)
}
```

### Onboarding'lerdeki linklerin nasıl açılacağını özelleştirme \{#customize-how-links-open-in-onboardings\}

Varsayılan olarak, onboarding'lerdeki linkler uygulama içi tarayıcıda açılır. Bu, web sayfalarını uygulamanızın içinde görüntüleyerek kullanıcıların uygulamalar arasında geçiş yapmadan içerikleri incelemesini sağlar ve kesintisiz bir kullanıcı deneyimi sunar.

Linklerin harici tarayıcıda açılmasını tercih ederseniz, `externalUrlsPresentation` parametresini `AdaptyWebPresentation.EXTERNAL_BROWSER` olarak ayarlayarak bu davranışı özelleştirebilirsiniz:

```kotlin showLineNumbers

viewModelScope.launch {
    AdaptyUI.createOnboardingView(
        onboarding = onboarding,
        externalUrlsPresentation = AdaptyWebPresentation.EXTERNAL_BROWSER // default – IN_APP_BROWSER
    ).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```

## Compose Multiplatform Olmadan \{#without-compose-multiplatform\}

:::note
`createNativeOnboardingView`, `io.adapty:adapty-kmp` çekirdek modülünün bir parçasıdır. Projeniz Compose Multiplatform kullanmıyorsa `io.adapty:adapty-kmp-ui` bağımlılığına ihtiyaç duymazsınız.
:::

Compose Multiplatform olmadan bir onboarding eklemek için `createNativeOnboardingView` çağrısını yapın. Bu çağrı, layout'unuza eklediğiniz bir `AdaptyNativeOnboardingView` döndürür:

<Tabs>
<TabItem value="android" label="Android">
```kotlin showLineNumbers title="Kotlin Multiplatform (Android)"

val nativeView = AdaptyUI.createNativeOnboardingView(
    context = context,
    viewModelStoreOwner = activity,
    onboarding = onboarding,
    observer = myOnboardingObserver,
)

// Embed in your Compose layout:
AndroidView(
    factory = { nativeView.view },
    modifier = Modifier.fillMaxSize()
)
```
</TabItem>
<TabItem value="ios" label="iOS">
KMP arayüz varsayılan metotları Swift'te `@required` hale geldiğinden, `AdaptyUIOnboardingsEventsObserver`'ı doğrudan Swift üzerinden uygulayamazsınız. Önce `iosMain` içinde bir open base class tanımlayın:

```kotlin showLineNumbers title="iosMain (Kotlin)"
open class BaseOnboardingObserver : AdaptyUIOnboardingsEventsObserver
```

Ardından Swift'te yalnızca ihtiyacınız olanları geçersiz kılarak alt sınıf oluşturun:

```swift showLineNumbers title="Swift"
class MyOnboardingObserver: BaseOnboardingObserver {
    override func onboardingViewOnCloseAction(
        view: AdaptyUIOnboardingView,
        meta: AdaptyUIOnboardingMeta,
        actionId: String
    ) {
        // remove nativeView from your view hierarchy
    }
}

let nativeView = AdaptyUI.shared.createNativeOnboardingView(
    onboarding: onboarding,
    observer: MyOnboardingObserver()
)
// nativeView.viewController is a UIViewController.
// Add it to your SwiftUI view or UIKit hierarchy.
```
</TabItem>
</Tabs>

### View'u imha edin \{#dispose-the-view\}

View'u layout'unuzdan kaldırırken `dispose()`'u çağırın. Bu işlem, olay dinleyicisinin kaydını siler ve dahili kaynakları serbest bırakır.

```kotlin showLineNumbers title="Kotlin Multiplatform"
nativeView.dispose()
```
