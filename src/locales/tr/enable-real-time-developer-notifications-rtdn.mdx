Gerçek zamanlı geliştirici bildirimlerini (RTDN) kurmak, veri doğruluğu açısından kritik öneme sahiptir; bu sayede Play Store'dan iade bilgileri ve diğer olaylar dahil güncellemeleri anında alabilirsiniz.

## Bildirimleri Etkinleştirme \{#enable-notifications\}

1. **Google Cloud Pub/Sub**'ın etkin olduğundan emin olun. [Bu bağlantıyı](https://console.cloud.google.com/flows/enableapi?apiid=pubsub) açın ve uygulama projenizi seçin. **Google Cloud Pub/Sub**'ı henüz etkinleştirmediyseniz burada etkinleştirmeniz gerekir.

    margin: '0 auto' /* center alignment */

2. Adapty üst menüsünden [**App settings > Android SDK**](https://app.adapty.io/settings/android-sdk) sayfasına gidin ve **Google Play RTDN topic name** başlığının yanındaki **Enable Pub/Sub API** alanının içeriğini kopyalayın.

<Zoom>
  <img src="/assets/shared/img/a72ff2d-copy_topic.webp"

<p> </p>

:::note
**Enable Pub/Sub API** alanının içeriği yanlış bir formattaysa (doğru format `projects/...` ile başlar), yardım için [Enable Pub/Sub API alanındaki yanlış formatı düzeltme](enable-real-time-developer-notifications-rtdn#fixing-incorrect-format-in-enable-pubsub-api-field) bölümüne bakın.
:::

3. [Google Play Console](https://play.google.com/console/)'u açın, uygulamanızı seçin ve **Monetize with Play** -> **Monetization setup** bölümüne gidin. **Google Play Billing** bölümünde **Enable real-time notifications** onay kutusunu işaretleyin.

4. Adapty **App Settings**'de kopyaladığınız **Enable Pub/Sub API** alanının içeriğini **Topic name** alanına yapıştırın.
5. Google Play Console'da **Save changes**'e tıklayın.

Gerçek zamanlı geliştirici bildirimlerine başarıyla abone olup olmadığınızı kontrol etmek için:

1. Google Play Console ayarlarındaki değişiklikleri kaydedin.
2. Google Play Console'daki **Topic name** alanının altında **Send test notification**'a tıklayın.

## Enable Pub/Sub API Alanındaki Yanlış Formatı Düzeltme \{#fixing-incorrect-format-in-enable-pubsub-api-field\}

**Enable Pub/Sub API** alanının içeriği yanlış bir formattaysa (doğru format `projects/...` ile başlar), sorunu gidermek için şu adımları izleyin:

### 1. API Etkinleştirme ve İzinleri Doğrulama \{#1-verify-api-enablement-and-permissions\}

Gerekli tüm API'lerin etkinleştirildiğinden ve izinlerin servis hesabına doğru şekilde verildiğinden emin olun. Bu adımları daha önce tamamlamış olsanız bile, hiçbir alt adımın atlanmadığından emin olmak için tekrar gözden geçirmeniz önemlidir. Aşağıdaki bölümlerdeki adımları tekrarlayın:

1. [Google Play Console'da Geliştirici API'lerini Etkinleştirme](enabling-of-devepoler-api)
2. [Google Cloud Console'da Servis Hesabı Oluşturma](create-service-account)
3. [Google Play Console'da Servis Hesabına İzin Verme](grant-permissions-to-service-account)
4. [Google Play Console'da Servis Hesabı Anahtar Dosyası Oluşturma](create-service-account-key-file)
5. [Google Play Store Entegrasyonunu Yapılandırma](google-play-store-connection-configuration)

### 2. Alan Politikalarını Düzenleme \{#2-adjust-domain-policies\}

**Domain restricted contacts** ve **Domain restricted sharing** politikalarını değiştirin:

1. [Google Cloud Console](https://console.cloud.google.com/)'u açın ve uygulamanızı yönetmek için servis hesabını oluşturduğunuz projeyi seçin.
2. **Quick Access** bölümünde **IAM & Admin**'i seçin.

   <Zoom>
     <img src="/assets/shared/img/google-cloud-IAM-and-Admin.webp"
     style={{
       border: '1px solid #727272', /* border width and color */
       width: '700px', /* image width */
       display: 'block', /* for alignment */
       margin: '0 auto' /* center alignment */
     }}
   />
   </Zoom>

3. Sol panelde **Organization Policies**'i seçin.
4. **Domain restricted contacts** politikasını bulun.

   <Zoom>
     <img src="/assets/shared/img/google-cloud-policy-action.webp"
     style={{
       border: '1px solid #727272', /* border width and color */
       width: '700px', /* image width */
       display: 'block', /* for alignment */
       margin: '0 auto' /* center alignment */
     }}
   />
   </Zoom>

5. **Actions** sütunundaki üç nokta düğmesine tıklayın ve **Edit policy**'yi seçin.
6. Politika düzenleme penceresinde:
   1. **Policy source** altında **Override parent's policy** radyo düğmesini seçin.
   2. **Policy enforcement** altında **Replace** radyo düğmesini seçin.
   3. **Rules** altında **ADD A RULE** düğmesine tıklayın.

      <Zoom>
        <img src="/assets/shared/img/google-cloud-edit-policy.webp"
        style={{
          border: '1px solid #727272', /* border width and color */
          width: '700px', /* image width */
          display: 'block', /* for alignment */
          margin: '0 auto' /* center alignment */
        }}
      />
      </Zoom>

   4. **New rule** -> **Policy values** altında **Allow All**'u seçin.

      <Zoom>
        <img src="/assets/shared/img/google-cloud-allow-all-policy.webp"
        style={{
          border: '1px solid #727272', /* border width and color */
          width: '700px', /* image width */
          display: 'block', /* for alignment */
          margin: '0 auto' /* center alignment */
        }}
      />
      </Zoom>

   5. **SET POLICY**'ye tıklayın.
7. **Domain restricted sharing** politikası için 4-6. adımları tekrarlayın.

Son olarak, **Google Play RTDN topic name** başlığının yanındaki **Enable Pub/Sub API** alanının içeriğini yeniden oluşturun. Alan artık doğru formatta olacaktır.

Gerçek Zamanlı Geliştirici Bildirimlerini (RTDN) başarıyla etkinleştirdikten sonra güncellenen politikalar için **Policy source**'u **Inherit parent's policy** olarak geri aldığınızdan emin olun.
