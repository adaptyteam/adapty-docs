Bu hızlı başlangıç kılavuzu, bir token para birimini uçtan uca kurar: bir abonelik kullanıcılara ayda 1000 token verir ve uygulamanızdaki ücretli işlemler token harcar. Adapty her bakiyeyi takip eder, bu sayede backend'iniz yalnızca bunları okur ve harcar.

1. `TOKENS` para birimi oluşturmak için [Sanal para birimi oluşturma](create-virtual-currency) kılavuzunu izleyin. Bunu Pro aboneliğinize bağlayın ve **Credit per cycle** değerini 1000 olarak ayarlayın. Ekstra token satmak için tek seferlik token paketi ürünlerini de bağlayın.

   <ZoomImage id="new-currency-combined.webp" width="700px" alt="New virtual currency creation screens" />

2. Oluşturma işlemi başlatılmadan önce kullanıcının bakiyesini okumak için [Sanal para birimi bakiyelerini listele](api-adapty/operations/listVirtualCurrencyBalances) çağrısını yapın:

```bash title="Read balances"
   curl https://api.adapty.io/api/v2/server-side-api/vc/balances/ \
     -H "Authorization: Api-Key {your secret key}" \
     -H "adapty-customer-user-id: user-42"
   ```

   Yanıt, kullanıcının sahip olduğu her para birimini listeler; örneğin 1000 `TOKENS`:

```json title="Response"
   {
     "data": [
       { "code": "TOKENS", "name": "Tokens", "balance": 1000, "held": 0, "available": 1000 }
     ]
   }
   ```
3. Kullanıcı bir üretim işlemi başlattığında, [Sanal para birimi işlemi oluştur](api-adapty/operations/createVirtualCurrencyTransaction) çağrısını negatif bir `amount` değeriyle yaparak token'ları harcayın. Bu örnekte, bir görsel üretimi 100 token'a mal olmaktadır:

   ```bash title="Spend tokens"
   curl -X POST https://api.adapty.io/api/v2/server-side-api/vc/transactions/ \
     -H "Authorization: Api-Key {your secret key}" \
     -H "adapty-customer-user-id: user-42" \
     -H "Content-Type: application/json" \
     -d '{"items": [{"currency_code": "TOKENS", "amount": -100}]}'
   ```

Bir işlem atomik yapıdadır ve güncellenmiş bakiyeyi döndürür. Kullanıcı maliyeti karşılayamazsa istek `insufficient_balance` döndürür ve hiçbir şey değişmez. İstekleri güvenli şekilde yeniden denemek için `Idempotency-Key` başlığını ekleyin.
4. Geri kazanma promosyonu çalıştırmak için aynı endpoint'e pozitif bir `amount` ile token'lar verin. Bu şekilde verdiğiniz kredilerin son kullanma tarihi yoktur.
5. Ekonomiyi istediğiniz zaman denetleyebilmek için kullanıcının [profilindeki](virtual-currency-balance) veya [işlem geçmişindeki](api-adapty/operations/listVirtualCurrencyTransactions) her değişikliği inceleyin.
