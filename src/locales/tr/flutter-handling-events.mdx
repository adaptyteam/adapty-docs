Flow ve Paywall Builder ile oluşturulan paywall'lar, satın alma yapmak ve geri yüklemek için ekstra kod gerektirmez. Ancak uygulamanızın yanıt verebileceği bazı olaylar üretirler. Bu olaylar; buton tıklamaları (kapatma düğmeleri, URL'ler, ürün seçimleri vb.) ve flow veya paywall üzerinde gerçekleştirilen satın alma işlemlerine ilişkin bildirimleri kapsar. Bu olaylara nasıl yanıt vereceğinizi aşağıda öğrenebilirsiniz.

Mobil uygulamanızdaki flow veya paywall ekranında gerçekleşen süreçleri kontrol etmek veya izlemek için `AdaptyUIFlowsEventsObserver` metodlarını uygulayın ve herhangi bir ekran gösterilmeden önce observer'ı ayarlayın:
```dart showLineNumbers title="Flutter"
AdaptyUI().setFlowsEventsObserver(this);
```

Üç observer metodu **zorunludur** — sınıfınız bunlar olmadan derlenmez: `flowViewDidFinishPurchase`, `flowViewDidFinishRestore` ve `flowViewDidReceiveError`. Diğer tüm metodlar isteğe bağlıdır. Daha önce ayarlanmış bir observer'ı kaldırmak için `setFlowsEventsObserver`'a `null` geçin.

### Kullanıcı tarafından oluşturulan olaylar \{#user-generated-events\}

#### Görünüm belirdi \{#view-appeared\}

Bu metot, flow veya paywall görünümü ekranda gösterildiğinde çağrılır.

:::note
iOS'ta, kullanıcı bir paywall içindeki [web paywall butonuna](web-paywall#step-2a-add-a-web-purchase-button) dokunduğunda ve bir web paywall uygulama içi tarayıcıda açıldığında da çağrılır.
:::

```dart showLineNumbers title="Flutter"
void flowViewDidAppear(AdaptyUIFlowView view) {
}
```

#### Görünüm kayboldu \{#view-disappeared\}

Bu metot, flow veya paywall görünümü ekrandan kaldırıldığında çağrılır.
:::note
iOS'ta ayrıca, bir paywall'dan açılan [web paywall](web-paywall#step-2a-add-a-web-purchase-button)'ı uygulama içi tarayıcıda kapandığında da tetiklenir.
:::

```dart showLineNumbers title="Flutter"
void flowViewDidDisappear(AdaptyUIFlowView view) {
}
```

#### Ürün seçimi \{#product-selection\}

Bir ürün satın alma için seçildiğinde (kullanıcı ya da sistem tarafından), bu metot çağrılır:

```dart showLineNumbers title="Flutter"
void flowViewDidSelectProduct(AdaptyUIFlowView view, String productId) {
}
```

#### Satın alma başlatıldı \{#started-purchase\}

Bir kullanıcı satın alma sürecini başlatırsa bu metot çağrılır:

```dart showLineNumbers title="Flutter"
void flowViewDidStartPurchase(AdaptyUIFlowView view, AdaptyPaywallProduct product) {
}
```

<Details>
<summary>Etkinlik örneği (Genişletmek için tıklayın)</summary>
```dart
void flowViewDidStartPurchase(AdaptyUIFlowView view, AdaptyPaywallProduct product) {
  // product — AdaptyPaywallProduct:
  product.vendorProductId;        // 'premium_monthly'
  product.localizedTitle;         // 'Premium Monthly'
  product.localizedDescription;   // 'Premium subscription for 1 month'
  product.price.amount;           // 9.99            (double)
  product.price.currencyCode;     // 'USD'
  product.price.localizedString;  // '$9.99'
}
```

#### Tamamlanan satın alma \{#finished-purchase\}
Bu yöntem **zorunludur**. Bir satın alma başarıyla tamamlandığında, kullanıcı satın almayı iptal ettiğinde veya satın alma işlemi beklemede göründüğünde çağrılır:
```dart showLineNumbers title="Flutter"
void flowViewDidFinishPurchase(AdaptyUIFlowView view, 
                               AdaptyPaywallProduct product, 
                               AdaptyPurchaseResult purchaseResult) {
    switch (purchaseResult) {
      case AdaptyPurchaseResultSuccess(profile: final profile):
        // successful purchase
        break;
      case AdaptyPurchaseResultPending():
        // purchase is pending
        break;
      case AdaptyPurchaseResultUserCancelled():
        // user cancelled the purchase
        break;
      default:
        break;
    }
}
```

<Details>
<summary>Olay örnekleri (Genişletmek için tıklayın)</summary>
```dart
void flowViewDidFinishPurchase(AdaptyUIFlowView view,
                               AdaptyPaywallProduct product,
                               AdaptyPurchaseResult purchaseResult) {
  // product — AdaptyPaywallProduct:
  product.vendorProductId; // 'premium_monthly'

  switch (purchaseResult) {
    case AdaptyPurchaseResultSuccess(profile: final profile):
      // profile — AdaptyProfile:
      profile.accessLevels['premium']?.isActive;  // true
      profile.accessLevels['premium']?.expiresAt; // DateTime(2027, 2, 15, 10, 30)
      break;
    case AdaptyPurchaseResultPending():
      // no additional data
      break;
    case AdaptyPurchaseResultUserCancelled():
      // no additional data
      break;
  }
}
```
</Details>
:::info
v3'ten farklı olarak, bu metodun varsayılan bir davranışı yoktur — başarılı bir satın alma sonrasında görünüm artık otomatik olarak kapatılmaz. Bundan sonra ne olacağına siz karar verin: flow'a devam edin veya `view.dismiss()` çağrısı yapın. Bir ekranı kapatma hakkında ayrıntılar için [Buton eylemlerine yanıt verme](flutter-handle-paywall-actions) bölümüne bakın.
:::

#### Web ödeme navigasyonu tamamlandı \{#finished-web-payment-navigation\}

Bu metod, belirli bir ürün için bir [web paywall](web-paywall) açma girişiminin ardından çağrılır. Hem başarılı hem de başarısız navigasyon girişimleri dahildir:
```dart showLineNumbers title="Flutter"
void flowViewDidFinishWebPaymentNavigation(AdaptyUIFlowView view, 
                                           AdaptyPaywallProduct? product, 
                                           AdaptyError? error) {
}
```

**Parametreler:**
| Parametre   | Açıklama                                                                                              |
|:------------|:------------------------------------------------------------------------------------------------------|
| **product** | Web paywallın açıldığı `AdaptyPaywallProduct`. `null` olabilir.                                       |
| **error**   | Web paywall navigasyonu başarısız olduysa `AdaptyError` nesnesi; navigasyon başarılıysa `null`. |

#### Başarısız satın alma \{#failed-purchase\}
Bu yöntem, bir satın alma başarısız olduğunda (örneğin, ödeme sorunları veya ağ hataları nedeniyle) çağrılır. Kullanıcının başlattığı iptaller veya bekleyen işlemler için **tetiklenmez** — bunlar `flowViewDidFinishPurchase` tarafından ele alınır:

```dart showLineNumbers title="Flutter"
void flowViewDidFailPurchase(AdaptyUIFlowView view, 
                             AdaptyPaywallProduct product, 
                             AdaptyError error) {
}
```

#### Başarısız geri yükleme \{#failed-restore\}

Bir satın alma geri yüklemesi başarısız olursa şu metod çağrılır:

```dart showLineNumbers title="Flutter"

}
```
### Veri çekme ve render etme \{#data-fetching-and-rendering\}

#### Ürün yükleme hataları \{#product-loading-errors\}

Başlatma sırasında ürün dizisini geçmezseniz, AdaptyUI gerekli nesneleri sunucudan kendisi alır. Bu işlem başarısız olursa, AdaptyUI hatayı şu metodu çağırarak bildirir:

```dart showLineNumbers title="Flutter"
void flowViewDidFailLoadingProducts(AdaptyUIFlowView view, AdaptyError error) {
}
```

#### Görünüm hataları \{#view-errors\}
Bu yöntem **zorunludur**. v3'teki `paywallViewDidFailRendering` yönteminin yerini alır: arayüz oluşturma sırasında meydana gelen hatalar ve diğer görünüm hataları bu yöntem çağrılarak raporlanır. Bir kez uyguladığınızda, kapatma işlemi size kalmıştır — bu tür hatalarda görünümü kapatmanızı öneririz; hiçbir gözlemci ayarlanmadığında SDK'nın yerleşik varsayılanı da bunu yapar:

```dart showLineNumbers title="Flutter"
void flowViewDidReceiveError(AdaptyUIFlowView view, AdaptyError error) {
  // log the error and dismiss the broken view
  view.dismiss();
}
```
Normal bir durumda render hataları oluşmamalıdır; eğer biriyle karşılaşırsanız lütfen bize bildirin.
### Analitik olayları \{#analytics-events\}

İsteğe bağlı `flowViewDidReceiveAnalyticEvent` metodu, bir flow'dan gelen özel analitik olaylar için ayrılmıştır. Flow'lar henüz bu olayları kodunuza iletmediğinden, bu metodu uygulamanıza gerek yoktur.
### Observer mode'da satın almaları yönetme \{#handle-purchases-in-observer-mode\}

SDK'yı [Observer mode](implement-observer-mode-flutter)'da etkinleştirdiyseniz ve Adapty tarafından oluşturulan bir flow veya paywall sunuyorsanız, SDK satın alma işlemlerini sizin adınıza gerçekleştirmez. Kullanıcı satın alma veya geri yükleme düğmesine dokunduğunda SDK, `AdaptyUIObserverModeResolver`'ınızı çağırır. Tam kurulum için bkz. [Observer mode'da flow sunma](flutter-present-flows-in-observer-mode).
### Sistem isteklerini yönetme \{#handle-system-requests\}

`AdaptyUISystemRequestsHandler` (`AdaptyUI().setSystemRequestsHandler(...)` aracılığıyla kaydedilir), bir flow'dan gelen sistem istekleri için ayrılmıştır: işletim sistemi izin istemleri (push bildirimleri veya kamera erişimi gibi) ve App Store inceleme istekleri. Flow'lar henüz bu istekleri tetiklemiyor, dolayısıyla bir handler kaydetmeniz gerekmiyor.
Eğer birini kayıt ederseniz, `handlePermission`'ın sınıfın zorunlu metodu olduğunu unutmayın — izni kendi kodunuzla isteyin, ardından `AdaptyUIPermissionResult.granted()` veya `AdaptyUIPermissionResult.denied()` döndürün; `handleAppReviewRequest` ise isteğe bağlıdır.

Kullanıcının gerekli `accessLevel`'a sahip olması durumunda ekranı kapatmanızı öneririz. Bunu nasıl kontrol edeceğinizi öğrenmek için [Abonelik durumu](flutter-listen-subscription-changes) konusuna, paywall ekranını nasıl kapatacağınızı öğrenmek için [Düğme eylemlerine yanıt verme](flutter-handle-paywall-actions) konusuna bakın.

#### Render hataları \{#rendering-errors\}
Arayüz oluşturma sırasında bir hata oluşursa, bu metot çağrılarak hata bildirilir. Varsayılan olarak (v3.15.2'den itibaren), oluşturma hatası meydana geldiğinde paywall otomatik olarak kapatılır; ancak gerekirse bu davranışı değiştirebilirsiniz.

```dart showLineNumbers title="Flutter"
void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
  // Default behavior: view.dismiss()
  // Override with custom logic if needed, for example:
  // - Log the error
  // - Show an error message to the user
}
```

<summary>Etkinlik örneği (Genişletmek için tıklayın)</summary>

void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {

  error.detail;  // platform-specific underlying error, or null
