Adapty Flow Builder veya Paywall Builder kullanarak flow veya paywall oluşturuyorsanız butonları doğru şekilde ayarlamak kritik öneme sahiptir:

1. [Builder'da bir buton ekleyin](paywall-buttons) ve bu butona önceden tanımlanmış bir eylem atayın ya da özel bir eylem ID'si oluşturun.
2. Atadığınız her eylemi işlemek için uygulamanıza kod yazın.

Bu kılavuz, özel ve önceden tanımlanmış eylemleri kodunuzda nasıl ele alacağınızı gösterir.

:::warning
**Yalnızca satın almalar, geri yüklemeler, flow/paywall kapatmaları ve bağlantı açmaları otomatik olarak işlenir.** Özel eylemler gibi diğer tüm buton eylemleri, uygulama kodunda doğru şekilde ele alınmasını gerektirir.
:::

## AdaptyUIFlowsEventsObserver'ı Kurma \{#set-up-the-adaptyuiflowseventssobserver\}

Flow eylemlerini yönetmek için `AdaptyUIFlowsEventsObserver` arayüzünü uygulamanız ve `AdaptyUI.setFlowsEventsObserver()` ile kurmanız gerekir. Bu işlem, uygulamanızın yaşam döngüsünün erken aşamasında, genellikle ana aktivitede veya uygulama başlatma sürecinde yapılmalıdır.

```kotlin

// In your app initialization
AdaptyUI.setFlowsEventsObserver(MyAdaptyUIFlowsEventsObserver())
```

Tüm buton aksiyonları, `flowViewDidPerformAction(view, action)` callback'ine `AdaptyUIAction` sealed class olarak gelir: `CloseAction`, `AndroidSystemBackAction`, `OpenUrlAction` veya `CustomAction`.

:::warning
`flowViewDidPerformAction` override edildiğinde, yalnızca ilgilendiğiniz aksiyonun değil, **tüm** aksiyonların varsayılan davranışı değişir. Aşağıdaki örneklerde gösterildiği gibi, `CloseAction` (flow'u kapat) ve `OpenUrlAction` (URL'i aç) için varsayılan dalları değiştirmeyi amaçlamıyorsanız bunları koruyun.
:::

## Flow ve paywall'ları kapatma \{#close-flows-and-paywalls\}

Flow veya paywall'ınıza kapatma düğmesi eklemek için:

1. Builder'da bir düğme ekleyin ve **Close** aksiyonunu atayın.
2. Uygulama kodunuzda, flow'u kapatan `close` aksiyonu için bir handler uygulayın.

:::info
Kotlin Multiplatform SDK'sında `CloseAction`, varsayılan olarak flow veya paywall'ı kapatmayı tetikler. Ancak gerekirse bu davranışı kodunuzda geçersiz kılabilirsiniz. Örneğin, bir flow'un kapatılması başka bir flow'un açılmasını tetikleyebilir.
:::

```kotlin

class MyAdaptyUIFlowsEventsObserver : AdaptyUIFlowsEventsObserver {
    override fun flowViewDidPerformAction(view: AdaptyUIFlowView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.CloseAction ->
                mainUiScope.launch { view.dismiss() } // default behavior
            is AdaptyUIAction.OpenUrlAction ->
                AdaptyUI.openWebUrl(action.url, action.openIn) // default behavior
            else -> Unit
        }
    }
}

// Set up the observer
AdaptyUI.setFlowsEventsObserver(MyAdaptyUIFlowsEventsObserver())
```

[`createNativeFlowView`](kmp-present-paywalls#without-compose-multiplatform) kullanıyorsanız, `view.dismiss()` çağrısının hiçbir etkisi yoktur — görünüm KMP yığını üzerinden sunulmaz, doğrudan layout'unuza gömülür. Bunun yerine görünümü layout'unuzdan kaldırın ve üzerinde `dispose()` çağırın.

## Android sistem geri tuşunu yönetme \{#handle-the-android-system-back-button\}

Android sistem geri tuşuna basmak (veya geri hareketini kullanmak) `AdaptyUIAction.AndroidSystemBackAction` olayını tetikler. Varsayılan olarak bu eylem yok sayılır — flow açık kalır ve kullanıcı, builder'da tanımladığınız **Close** butonu veya `on_device_back` eylemi gibi bir yoldan çıkış yapar. Sistem geri tuşunun flow'u kapatmasını istiyorsanız, eylemi kendiniz ele alın:

```kotlin
class MyAdaptyUIFlowsEventsObserver : AdaptyUIFlowsEventsObserver {
    override fun flowViewDidPerformAction(view: AdaptyUIFlowView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.CloseAction ->
                mainUiScope.launch { view.dismiss() } // default behavior
            is AdaptyUIAction.AndroidSystemBackAction ->
                mainUiScope.launch { view.dismiss() } // not handled by default
            is AdaptyUIAction.OpenUrlAction ->
                AdaptyUI.openWebUrl(action.url, action.openIn) // default behavior
            else -> Unit
        }
    }
}
```

## Flow ve paywall'lardan URL açma \{#open-urls-from-flows-and-paywalls\}

:::tip
Bir grup bağlantı eklemek istiyorsanız (örneğin, kullanım koşulları ve satın alma geri yükleme), builder'da bir **Link** öğesi ekleyin ve bunu **Open URL** eylemiyle düğmelerle aynı şekilde ele alın.
:::

Flow veya paywall'ınıza bir bağlantı açan düğme eklemek için (örneğin, **Terms of use** veya **Privacy policy**), builder'da bir düğme ekleyin, **Open URL** eylemini atayın ve açmak istediğiniz URL'yi girin.

Varsayılan olarak, SDK alınan URL'yi yerel olarak açar — `action.openIn` değerine bağlı olarak harici veya uygulama içi tarayıcıda — bu nedenle herhangi bir kod gerekmez. Yalnızca önce bir onay iletişim kutusu göstermek gibi özel bir mantık istiyorsanız işleyiciyi geçersiz kılın:

```kotlin

class MyAdaptyUIFlowsEventsObserver(
    private val uriHandler: UriHandler
) : AdaptyUIFlowsEventsObserver {
    override fun flowViewDidPerformAction(view: AdaptyUIFlowView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.OpenUrlAction -> {
                // Show confirmation dialog before opening URL
                mainUiScope.launch {
                    val selectedAction = view.showDialog(
                        title = "Open URL?",
                        content = action.url,
                        primaryActionTitle = "Cancel",
                        secondaryActionTitle = "Open"
                    ).getOrNull()

                    when (selectedAction) {
                        AdaptyUIDialogActionType.PRIMARY -> {
                            // User cancelled
                        }
                        AdaptyUIDialogActionType.SECONDARY -> {
                            // User confirmed - open URL
                            uriHandler.openUri(action.url)
                        }
                        else -> Unit
                    }
                }
            }
            else -> Unit
        }
    }
}

// Set up the observer with UriHandler
AdaptyUI.setFlowsEventsObserver(MyAdaptyUIFlowsEventsObserver(uriHandler))
```

## Uygulamaya giriş yapma \{#log-into-the-app\}

Kullanıcıların uygulamanıza giriş yapmasını sağlayan bir buton eklemek için:

1. Builder'da bir buton ekleyin ve bu butona "login" ID'sine sahip bir **Custom** (Özel) eylem atayın.
2. Uygulama kodunuzda, kullanıcıyı tanımlayan özel eylem için bir işleyici (handler) uygulayın.

```kotlin

class MyAdaptyUIFlowsEventsObserver : AdaptyUIFlowsEventsObserver {
    override fun flowViewDidPerformAction(view: AdaptyUIFlowView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.CustomAction -> {
                if (action.action == "login") {
                    // Handle login action - navigate to login screen
                    // This depends on your app's navigation system
                    // For example, in Compose Multiplatform:
                    // navController.navigate("login")
                }
            }
            else -> Unit
        }
    }
}
```

## Özel eylemleri işleme \{#handle-custom-actions\}

Herhangi bir eylemi işleyen bir buton eklemek için:

1. Builder'da bir buton ekleyin, **Custom** eylemini atayın ve bir kimlik (ID) verin.
2. Uygulama kodunuzda, oluşturduğunuz eylem kimliği için bir işleyici (handler) tanımlayın.

Örneğin, başka bir abonelik teklifi veya tek seferlik satın alma seçeneğiniz varsa, başka bir flow ya da paywall görüntüleyecek bir buton ekleyebilirsiniz:

```kotlin

class MyAdaptyUIFlowsEventsObserver : AdaptyUIFlowsEventsObserver {
    override fun flowViewDidPerformAction(view: AdaptyUIFlowView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.CustomAction -> {
                when (action.action) {
                    "openNewFlow" -> {
                        // Display another flow or paywall
                    }
                }
            }
            else -> Unit
        }
    }
}

// Set up the observer
AdaptyUI.setFlowsEventsObserver(MyAdaptyUIFlowsEventsObserver())
```

## Paywall'lardan URL açın \{#open-urls-from-paywalls\}

:::tip
Bir grup bağlantı (ör. kullanım koşulları ve satın alma geri yükleme) eklemek istiyorsanız paywall builder'a bir **Link** öğesi ekleyin ve bunu **Open URL** aksiyonuyla düğmeleri yönettiğiniz gibi işleyin.
:::

Paywallınızdan bir bağlantı açan bir düğme eklemek için (ör. **Terms of use** veya **Privacy policy**):

1. Paywall builder'da bir düğme ekleyin, **Open URL** aksiyonunu atayın ve açmak istediğiniz URL'yi girin.
2. Uygulama kodunuzda, alınan URL'yi tarayıcıda açan `openUrl` aksiyonu için bir işleyici uygulayın.

:::info
Kotlin Multiplatform SDK'da `OpenUrlAction`, açılması gereken URL'yi sağlar. URL açmayı yönetmek için özel mantık uygulayabilirsiniz; örneğin bir onay iletişim kutusu gösterebilir veya uygulamanızın tercih ettiği URL yönetim yöntemini kullanabilirsiniz.
:::

```kotlin

class MyAdaptyUIPaywallsEventsObserver(
    private val uriHandler: UriHandler
) : AdaptyUIPaywallsEventsObserver {
    override fun paywallViewDidPerformAction(view: AdaptyUIPaywallView, action: AdaptyUIAction) {
        when (action) {
            is AdaptyUIAction.OpenUrlAction -> {
                // Show confirmation dialog before opening URL
                mainUiScope.launch {
                    val selectedAction = view.showDialog(
                        title = "Open URL?",
                        content = action.url,
                        primaryActionTitle = "Cancel",
                        secondaryActionTitle = "Open"
                    ).getOrNull()

                    when (selectedAction) {
                        AdaptyUIDialogActionType.PRIMARY -> {
                            // User cancelled
                        }
                        AdaptyUIDialogActionType.SECONDARY -> {
                            // User confirmed - open URL
                            uriHandler.openUri(action.url)
                        }
                        else -> Unit
                    }
                }
            }
        }
    }
}

// Set up the observer with UriHandler
AdaptyUI.setPaywallsEventsObserver(MyAdaptyUIPaywallsEventsObserver(uriHandler))
```
