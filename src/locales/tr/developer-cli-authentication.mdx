Bir yapay zeka asistanı mı kullanıyorsunuz? LLM'lerin CLI ile çalışmasına yardımcı olmak için bir [Adapty CLI becerisi](https://github.com/adaptyteam/adapty-cli/tree/main/skills/adapty-cli) mevcuttur.
:::

CLI, Adapty API'sini çağırmak için kimlik doğrulama gerektirir.

## Giriş yapma \{#log-in\}

Giriş yapmak için:

1. Terminalinizde şunu çalıştırın:
   ```bash
   adapty auth login
   ```
2. CLI, `XXXX-XXXX` biçiminde bir doğrulama kodu yazdırır ve tarayıcınızda Adapty Kontrol Paneli'ni açar.
3. Yetkilendirme sayfasında, kodun terminal çıktınızla eşleştiğini onaylayın.
4. **Authorize** düğmesine tıklayın. Tarayıcıda "CLI authorized! You can close this tab." mesajı görünür.
5. Terminale geri döndüğünüzde, CLI kimlik doğrulamasının başarılı olduğunu onaylar.

```bash
adapty auth login
```

## Kimlik doğrulamayı yönetme \{#manage-authentication\}

### Kimlik doğrulama durumunu kontrol etme \{#check-authentication-status\}

Mevcut kimlik doğrulama durumunuzu görmek için şunu çalıştırın:

```bash
adapty auth status
```

Kimlik doğrulaması yapıldığında, çıktıda e-posta adresiniz, maskelenmiş bir token öneki ve yerel yapılandırma dosyasının yolu gösterilir:

```
Email: you@example.com
Token: abcd1234****
Config: ~/.config/adapty/config.json
```

Kimlik doğrulaması yapılmadığında:

```
Not authenticated. Run `adapty auth login`.
```

### Tokenınızı doğrulama \{#verify-your-token\}

Tokenınızın geçerli olduğunu onaylamak ve hesap bilgilerinizi görmek için şunu çalıştırın:

```bash
adapty auth whoami
```

`adapty auth status` komutunun aksine, bu komut tokenı doğrulamak için sunucuya canlı bir istek gönderir.

### Çıkış yapma \{#log-out\}

Yerel olarak depolanan kimlik bilgilerinizi temizlemek için şunu çalıştırın:

```bash
adapty auth logout
```

Bu komut `~/.config/adapty/config.json` dosyasını temizler. Token, süresi dolana kadar sunucu tarafında geçerliliğini korur — hemen geçersiz kılmak istiyorsanız bunun yerine `adapty auth revoke` komutunu kullanın.

### Tokenınızı iptal etme \{#revoke-your-token\}

Tokeni sunucuda geçersiz kılmak ve yerel olarak temizlemek için şunu çalıştırın:

```bash
adapty auth revoke
```

Bir tokeni tamamen geçersiz kılmak istediğinizde bunu kullanın — örneğin kimlik bilgilerinizin ele geçirilmiş olabileceği durumlarda. İptal ettikten sonra yeniden kimlik doğrulaması yapmak için `adapty auth login` komutunu çalıştırın.

## Token hataları \{#token-errors\}

Bir token iptal edilirse veya geçersiz hale gelirse, CLI komutları 401 hatası döndürür. Yeniden kimlik doğrulaması yapmak için şunu çalıştırın:

```bash
adapty auth login
```
