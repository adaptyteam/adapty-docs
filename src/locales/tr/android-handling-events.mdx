:::important
Bu kılavuz, satın alma, yenileme, ürün seçimi ve flow render işlemleri için event yönetimini kapsar. Ayrıca düğme işlemlerini de (flow'u kapatma, bağlantı açma vb.) uygulamanız gerekir. Ayrıntılar için [düğme eylemlerini yönetme kılavuzumuza](android-handle-paywall-actions) bakın.

[Flow Builder](adapty-flow-builder) veya [Paywall Builder](adapty-paywall-builder) ile yapılandırılmış flow'lar ve paywall'lar, satın alma ve geri yükleme işlemleri için ekstra kod gerektirmez. Ancak bu ekranlar, uygulamanızın yanıt verebileceği bazı olaylar üretir. Bu olaylar arasında buton tıklamaları (kapat butonları, URL'ler, ürün seçimleri vb.) ve satın alma işlemlerine ilişkin bildirimler yer alır. Bu olaylara nasıl yanıt vereceğinizi aşağıda öğrenebilirsiniz.

<SampleApp />

Satın alma ekranında gerçekleşen süreçleri kontrol etmek veya izlemek istiyorsanız `AdaptyFlowEventListener` metodlarını uygulayın.
Bazı durumlarda varsayılan davranışı korumak istiyorsanız, `AdaptyFlowDefaultEventListener` sınıfını genişletip yalnızca değiştirmek istediğiniz metodları override edebilirsiniz.

Aşağıda `AdaptyFlowDefaultEventListener` içindeki varsayılan değerler yer almaktadır.
### Kullanıcı tarafından oluşturulan olaylar \{#user-generated-events\}

#### Ürün seçimi \{#product-selection\}

Bir ürün satın alma için seçildiğinde (kullanıcı veya sistem tarafından), bu method çağrılır:

```kotlin showLineNumbers title="Kotlin"

    product: AdaptyPaywallProduct,
    context: Context,
```

#### Başarılı, iptal edilmiş veya beklemedeki satın alma \{#successful-canceled-or-pending-purchase\}

Satın alma başarılı olursa şu metot çağrılır:
```kotlin showLineNumbers title="Kotlin"
public override fun onPurchaseFinished(
    purchaseResult: AdaptyPurchaseResult,
    product: AdaptyPaywallProduct,
    context: Context,
) {
    if (purchaseResult !is AdaptyPurchaseResult.UserCanceled)
        context.getActivityOrNull()?.onBackPressed()
}
```

#### Başarısız satın alma \{#failed-purchase\}
Bir satın alma işlemi hata nedeniyle başarısız olursa bu metot çağrılır. Bu durum; Google Play Billing hatalarını (ödeme kısıtlamaları, geçersiz ürünler, ağ hataları), işlem doğrulama hatalarını ve sistem hatalarını kapsar. ```kotlin showLineNumbers title="Kotlin"

```kotlin showLineNumbers title="Kotlin"
    error: AdaptyError,
    product: AdaptyPaywallProduct,
    context: Context,
```

#### Abonelik yükseltme \{#upgrade-subscription\}
Bir kullanıcı aktif bir aboneliği varken yeni bir abonelik satın almaya çalıştığında, bu metodu geçersiz kılarak yeni satın alma işleminin nasıl ele alınacağını kontrol edebilirsiniz. İki seçeneğiniz vardır:
1. **Mevcut aboneliği** yeni olanıyla değiştirin:
```kotlin showLineNumbers title="Kotlin"
public override fun onAwaitingPurchaseParams(
    product: AdaptyPaywallProduct,
    context: Context,
    onPurchaseParamsReceived: AdaptyFlowEventListener.PurchaseParamsCallback,
): AdaptyFlowEventListener.PurchaseParamsCallback.IveBeenInvoked {
    onPurchaseParamsReceived(
        AdaptyPurchaseParameters.Builder()
            .withSubscriptionUpdateParams(AdaptySubscriptionUpdateParameters(...))
            .build()
    )
    return AdaptyFlowEventListener.PurchaseParamsCallback.IveBeenInvoked
}
```
2. **Her iki aboneliği de koru** (yenisini ayrı olarak ekle):
```kotlin showLineNumbers title="Kotlin"
public override fun onAwaitingPurchaseParams(
    product: AdaptyPaywallProduct,
    context: Context,
    onPurchaseParamsReceived: AdaptyFlowEventListener.PurchaseParamsCallback,
): AdaptyFlowEventListener.PurchaseParamsCallback.IveBeenInvoked {
    onPurchaseParamsReceived(AdaptyPurchaseParameters.Empty)
    return AdaptyFlowEventListener.PurchaseParamsCallback.IveBeenInvoked
}
```
:::note
Bu yöntemi geçersiz kılmazsanız, varsayılan davranış her iki aboneliği de aktif tutmaktır (`AdaptyPurchaseParameters.Empty` kullanımıyla eşdeğer).
:::

Gerekirse ek satın alma parametreleri de ayarlayabilirsiniz:
```kotlin
AdaptyPurchaseParameters.Builder()
    .withSubscriptionUpdateParams(AdaptySubscriptionUpdateParameters(...)) // isteğe bağlı - mevcut aboneliği değiştirmek için
    .withOfferPersonalized(true) // isteğe bağlı - kişiselleştirilmiş fiyatlandırma kullanılıyorsa
    .build()
```

### Gezinti \{#navigation\}

#### Sistem geri düğmesi \{#system-back-button\}

Varsayılan olarak, bir flow sistem geri düğmesiyle veya geri hareketiyle kapatılamaz — kullanıcı, flow'dan yalnızca sizin tanımladığınız bir yol aracılığıyla çıkar; örneğin bir **Close** düğmesi veya builder'daki bir `on_device_back` eylemi. Sistem geri düğmesinin flow'u kapatmasını istiyorsanız, `onBackPressed` metodunu geçersiz kılın ve ana activity'nizin ya da fragment'ınızın bu basışı işlemesine izin vermek için `false` döndürün:
```kotlin showLineNumbers title="Kotlin"
public override fun onBackPressed(context: Context): Boolean {
    return false // let the host handle the back press (e.g. finish the activity or pop the fragment)
}
```

Bu callback, yalnızca mevcut ekran için herhangi bir `on_device_back` eylemi yapılandırılmamışsa çağrılır — yapılandırılmış bir eylem öncelik alır ve dahili olarak işlenir. Geri tuşuna basma işlemini tüketmek için `true`, ana bileşenin kendi geri işleme mantığının çalışmasına izin vermek için `false` döndürün.
### Ayrılmış olaylar \{#reserved-events\}

`AdaptyFlowEventListener`, flow'ların henüz kullanmadığı bazı işlevler için birkaç callback tanımlar. Bunları uygulamanıza gerek yok — `AdaptyFlowDefaultEventListener` zaten varsayılan olarak boş (no-op) uygulamalar sağlar.
| Method | Description |
|:-------|:------------|
| **onAnalyticEvent** | Flow'lardan gelen özel analitik etkinlikler için ayrılmıştır. Flow'lar henüz bu etkinlikleri kodunuza iletmediğinden, bunu uygulamanıza gerek yoktur. |
| **onShowAppRate** | Flow'lardan gelen uygulama değerlendirme istekleri için ayrılmıştır. Flow'lar henüz uygulama değerlendirme isteklerini tetiklemediğinden, bunu uygulamanıza gerek yoktur. |
| **onShowRequestPermission** | Flow'lardan gelen sistem izin istekleri (push bildirimleri veya kamera erişimi gibi) için ayrılmıştır. Flow'lar henüz izin isteklerini tetiklemediğinden, bunu uygulamanıza gerek yoktur. |

#### Abonelik yükseltme \{#upgrade-subscription\}
<Tabs groupId="current-os" queryString>
<TabItem value="new" label="SDK version 3.10.0 or later" default>

Bir kullanıcı, aktif bir aboneliği varken yeni bir abonelik satın almaya çalıştığında, bu metodu geçersiz kılarak yeni satın alma işleminin nasıl ele alınacağını kontrol edebilirsiniz. İki seçeneğiniz var:
1. **Mevcut aboneliği** yenisiyle değiştirin:
```kotlin showLineNumbers title="Kotlin"
public override fun onAwaitingPurchaseParams(
    product: AdaptyPaywallProduct,
    context: Context,
    onPurchaseParamsReceived: AdaptyUiEventListener.PurchaseParamsCallback,
): AdaptyUiEventListener.PurchaseParamsCallback.IveBeenInvoked {
    onPurchaseParamsReceived(
        AdaptyPurchaseParameters.Builder()
            .withSubscriptionUpdateParams(AdaptySubscriptionUpdateParameters(...))
            .build()
    )
    return AdaptyUiEventListener.PurchaseParamsCallback.IveBeenInvoked
}
```
2. **Her iki aboneliği de koru** (yenisini ayrı olarak ekle):
```kotlin showLineNumbers title="Kotlin"
public override fun onAwaitingPurchaseParams(
    product: AdaptyPaywallProduct,
    context: Context,
    onPurchaseParamsReceived: AdaptyUiEventListener.PurchaseParamsCallback,
): AdaptyUiEventListener.PurchaseParamsCallback.IveBeenInvoked {
    onPurchaseParamsReceived(AdaptyPurchaseParameters.Empty)
    return AdaptyUiEventListener.PurchaseParamsCallback.IveBeenInvoked
}
```
:::note
Bu metodu override etmezseniz, varsayılan davranış her iki aboneliği de aktif tutmaktır (`AdaptyPurchaseParameters.Empty` kullanımına eşdeğer).
:::

Gerekirse ek satın alma parametreleri de belirleyebilirsiniz:
```kotlin
AdaptyPurchaseParameters.Builder()
    .withSubscriptionUpdateParams(AdaptySubscriptionUpdateParameters(...)) // isteğe bağlı - mevcut aboneliği değiştirmek için
    .withOfferPersonalized(true) // isteğe bağlı - kişiselleştirilmiş fiyatlandırma kullanılıyorsa
    .build()
```

</TabItem>
<TabItem value="old" label="SDK version earlier than 3.10.0" default>
Yeni bir abonelik satın alınırken mevcut bir abonelik hâlâ aktifse, geçerli aboneliği yenisiyle değiştirmek için bu metodu override edin. Aktif abonelik etkin kalmaya devam etmeli ve yeni abonelik ayrı olarak eklenmeliyse `onSubscriptionUpdateParamsReceived(null)` çağırın:

```kotlin showLineNumbers title="Kotlin"
public override fun onAwaitingSubscriptionUpdateParams(
    product: AdaptyPaywallProduct,
    context: Context,
    onSubscriptionUpdateParamsReceived: SubscriptionUpdateParamsCallback,
) {
    onSubscriptionUpdateParamsReceived(AdaptySubscriptionUpdateParameters(...))
}
```
</TabItem>
</Tabs>

### Veri çekme ve render etme \{#data-fetching-and-rendering\}

#### Ürün yükleme hataları \{#product-loading-errors\}

Ürünleri başlatma sırasında geçirmezseniz, AdaptyUI gerekli nesneleri sunucudan kendisi alacaktır. Bu işlem başarısız olursa, AdaptyUI hatayı şu metodu çağırarak bildirir:

```kotlin showLineNumbers title="Kotlin"
public override fun onLoadingProductsFailure(
    error: AdaptyError,
    context: Context,
): Boolean = false
```

<Details>
<summary>Örnek olay (Genişletmek için tıklayın)</summary>
```javascript
{
  "error": {
    "code": "products_loading_failed",
    "message": "Failed to load products from the server",
    "details": {
      "underlyingError": "Network timeout"
    }
  }
}
```
</Details>

`true` döndürürseniz, AdaptyUI isteği 2 saniye sonra tekrar eder.

#### Render hataları \{#rendering-errors\}

```kotlin showLineNumbers title="Kotlin"
public override fun onRenderingError(
    error: AdaptyError,
    context: Context,
```
