Adapty, abonelik olaylarını kullanarak [Pushwoosh](https://www.pushwoosh.com/) profil etiketlerini günceller. Bu sayede aşağıda açıklanan kısa ve kolay entegrasyon ayarlarını yaptıktan sonra push bildirimleriyle hedefli iletişim kurabilirsiniz.

## Pushwoosh entegrasyonu nasıl kurulur \{#how-to-set-up-pushwoosh-integration\}

Pushwoosh'u entegre etmek için [**Integrations** -> **Pushwoosh**](https://app.adapty.io/integrations/pushwoosh) sayfasına gidin, geçişi kapalıdan açık konuma getirin ve alanları doldurun.

Öncelikle Pushwoosh ve Adapty profilleriniz arasındaki bağlantıyı kurmak için kimlik bilgilerini girin. Pushwoosh uygulama kimliği ve kimlik doğrulama token'ı gereklidir.

## Olaylar ve etiketler \{#events-and-tags\}

Kimlik bilgilerinin altında, Adapty'den Pushwoosh'a gönderebileceğiniz üç grup olay yer almaktadır. İhtiyacınız olanları açmanız yeterli. Pushwoosh'a gönderim için olay adlarını da değiştirebilirsiniz. Adapty'nin sunduğu olayların tam listesine [buradan](events) ulaşabilirsiniz.

Adapty, sunucudan sunucuya entegrasyon aracılığıyla abonelik olaylarını Pushwoosh'a gönderir ve tüm abonelik olaylarını Pushwoosh Kontrol Panelinizde görüntülemenizi sağlar.

:::note
Özel etiketler

Adapty ile Pushwoosh entegrasyonunda kendi özel etiketlerinizi de kullanabilirsiniz. Hangi etiketin ihtiyacınıza en uygun olduğunu belirlemek için aşağıdaki etiket listesine başvurabilirsiniz.
:::

| Etiket | Tür | Değer |
|---|----|-----|
| `adapty_customer_user_id` | String | Pushwoosh tarafında bulunan kullanıcının benzersiz tanımlayıcısının değerini içerir. |
| `adapty_profile_id` | String | Adapty [kontrol panelinde](profiles-crm) bulunabilen kullanıcının Adapty Kullanıcı Profili kimliğinin benzersiz tanımlayıcı değerini içerir. |
| `environment` | String | <p>Kullanıcının sandbox mı yoksa production ortamında mı çalıştığını belirtir.</p><p></p><p>Değerler `Sandbox` veya `Production` şeklindedir.</p> |
| `store` | String | <p>Satın alma işleminde kullanılan Store adını içerir.</p><p></p><p>Olası değerler:</p><p>`app_store` veya `play_store`.</p> |
| `vendor_product_id` | String | <p>Apple/Google mağazasındaki Ürün Kimliğinin değerini içerir.</p><p></p><p>Örn. org.locals.12345</p> |
| `subscription_expires_at` | String | <p>En son aboneliğin sona erme tarihini içerir.</p><p></p><p>Değer formatı:</p><p>yıl-ay günTsaat:dakika:saniye</p><p>Örn. 2023-02-10T17:22:03.000000+0000</p> |
| `last_event_type` | String | Entegrasyon için etkinleştirdiğiniz standart [Adapty olayları](events) listesinden alınan son olayın türünü belirtir. |
| `purchase_date` | String | <p>Son işlemin (orijinal satın alma veya yenileme) tarihini içerir.</p><p></p><p>Değer formatı:</p><p>yıl-ay günTsaat:dakika:saniye</p><p>Örn. 2023-02-10T17:22:03.000000+0000</p> |
| `original_purchase_date` | String | <p>İşleme göre ilk satın alma tarihini içerir.</p><p></p><p>Değer formatı:</p><p>yıl-ay günTsaat:dakika:saniye</p><p>Örn. 2023-02-10T17:22:03.000000+0000</p> |
| `active_subscription` | String | Herhangi bir satın alma/yenileme olayında `true`, abonelik sona erdiğinde `false` olarak ayarlanır. |
| `period_type` | String | <p>Satın alma veya yenileme için en son dönem türünü belirtir.</p><p></p><p>Olası değerler:</p><p>Deneme dönemi için `trial`, geri kalanı için `normal`.</p> |

Tüm float değerler int'e yuvarlanır. String değerler aynı kalır.

Önceden tanımlanmış etiket listesine ek olarak, [özel nitelikler](segments#custom-attributes) etiketler aracılığıyla da gönderilebilir. Bu, etikete eklenebilecek veri türünde daha fazla esneklik sağlar ve bir ürün veya hizmete ilişkin belirli bilgilerin takibi için faydalı olabilir. Tüm özel kullanıcı nitelikleri, kullanıcı [entegrasyon sayfasındaki](https://app.adapty.io/integrations/pushwoosh) **Send user custom attributes** onay kutusunu işaretlerse otomatik olarak Pushwoosh'a gönderilir.

## SDK yapılandırması \{#sdk-configuration\}

Adapty'yi Pushwoosh ile ilişkilendirmek için `HWID` değerini bize göndermeniz gerekir:

<TabItem value="swift" label="iOS (Swift)" default>
```swift showLineNumbers
do {
    try await Adapty.setIntegrationIdentifier(
        key: "pushwoosh_hwid", 
        value: Pushwoosh.sharedInstance().getHWID()
    )
} catch {
    // handle the error
}
```
