Adapty Attribution'ın özel S3 uyumlu depolama entegrasyonu, kullanıcı edinimi kampanya verilerinizi kendi S3 uyumlu depolama çözümünüzde güvenli bir şekilde saklamanıza olanak tanır. Kampanya performans verilerinizi, attribution verilerinizi ve kullanıcı edinimi olaylarını özel S3 bucket'ınıza .csv dosyaları olarak kaydedebileceksiniz.
Bu entegrasyonu kurmak için S3 uyumlu depolama konsolunuzda ve Adapty Attribution kontrol panelinde birkaç basit adımı izlemeniz gerekecek.

:::note
Adapty Attribution verilerinizi her **24 saatte bir** saat 04:00 UTC'de gönderir.
Her dosya, UTC saatine göre önceki takvim günü boyunca oluşturulan tüm etkinliklerin verilerini içerir. Örneğin, 8 Mart'ta UTC 04:00'te otomatik olarak dışa aktarılan veriler, 7 Mart'ta UTC 00:00:00 ile 23:59:59 arasında oluşturulan tüm etkinlikleri kapsar.
:::
## Özel S3 entegrasyonunu kurma \{#set-up-custom-s3-integration\}

Veri almaya başlamak için Adapty Attribution'da entegrasyonu yapılandırın:

1. [**Integrations** -> **Custom S3**](https://app.adapty.io/ua/integrations/custom-s3) bölümüne gidin.
2. **Export install events to custom S3** geçiş düğmesini açın.
3. Özel S3 depolama alanınızla Adapty Attribution profilleri arasındaki bağlantıyı kurmak için gerekli alanları doldurun.
| Alan                                    | Açıklama                                                                                                                                                                                                                                            |
|:----------------------------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Access Key ID**                       | S3 uyumlu depolama hizmetinize bir kullanıcı veya uygulamanın erişimini doğrulamak için kullanılan benzersiz tanımlayıcı. Bu kimliği depolama sağlayıcınızın konsolunda bulabilirsiniz.                                                              |
| **Secret Access Key**                   | S3 uyumlu depolama hizmetinize erişimi doğrulamak için Access Key ID ile birlikte kullanılan özel anahtar. Bu anahtarı depolama sağlayıcınızın konsolunda bulabilirsiniz.                                                                            |
| **S3 Bucket Name**                      | Depolama ortamınızdaki belirli bir S3 bucket'ını tanımlayan, küresel olarak benzersiz ad. S3 bucket'ları, kullanıcıların dosyalar ve görseller gibi veri nesnelerini bulutta depolamasına ve almasına olanak tanıyan basit bir depolama hizmetidir. |
| **Region** (İsteğe bağlı)               | Bölgenizi Yönetim Konsolu'ndan alabilirsiniz.                                                                                                                                                                                                       |
| **Folder Inside the Bucket** (İsteğe bağlı) | Seçili S3 bucket'ının içinde oluşturmak istediğiniz klasörün adı. S3'ün klasörleri, nesne anahtar önekleri kullanarak simüle ettiğini unutmayın; bu önekler aslında klasör adlarıdır.                                                             |
| **Custom Endpoint URL**                 | S3 uyumlu depolama hizmetinizin endpoint URL'si. Bu, depolama sağlayıcınız tarafından sağlanmalıdır (ör. MinIO, DigitalOcean Spaces, Wasabi vb.).                                                                                                   |

:::note
S3 bucket adı alanında iç içe dizinler de belirtebilirsiniz, ör. `adapty-ua-events/com.sample-app`
:::

## Manuel veri dışa aktarımı \{#manual-data-export\}

Otomatik etkinlik verisi dışa aktarımına ek olarak, Adapty Attribution; özel S3 depolama alanınıza manuel dosya dışa aktarımı da sunar. Bu özellik sayesinde kullanıcı edinim verilerini dilediğiniz tarihe göre seçerek S3 bucket'ınıza manuel olarak aktarabilirsiniz. Böylece hangi verileri ne zaman dışa aktaracağınız konusunda tam kontrol sizde olur.
## Tablo yapısı \{#table-structure\}

Özel S3 entegrasyonunda Adapty Attribution, kurulum olaylarına ait geçmişe dönük verileri depolamak için bir tablo sağlar. Bu tablo; kullanıcı profili, gelir ve kazanç bilgileri ile kaynak mağaza gibi çeşitli veri noktalarını içerir.

:::warning
Bu yapının zamanla büyüyebileceğini unutmayın; bizim tarafımızdan veya birlikte çalıştığımız üçüncü taraflar tarafından yeni veriler eklenebilir. Bu yapıyı işleyen kodunuzun yeterince dayanıklı olduğundan ve yapının tamamına değil, belirli alanlara dayandığından emin olun.
:::

Olaylara ait tablo yapısı aşağıdaki gibidir:
| Sütun                    | Açıklama                                  |
|--------------------------|-------------------------------------------|
| `adapty_profile_id`      | Benzersiz Adapty profil tanımlayıcısı     |
| `install_id`             | Benzersiz kurulum tanımlayıcısı           |
| `created_at`             | Kayıt oluşturma zaman damgası (ISO 8601)  |
| `installed_at`           | Uygulama kurulum zaman damgası (ISO 8601) |
| `store`                  | Uygulama mağazası (`ios`, `android`)      |
| `country`                | Kullanıcının ülke kodu (ISO 3166-1 alpha-2) |
| `ip_address`             | İstemci IP adresi                         |
| `idfa`                   | iOS Reklamverenler için Tanımlayıcı       |

| `app_set_id`             | Android Uygulama Set Kimliği              |
| `bundle_id`              | Uygulama bundle tanımlayıcısı (ör. `com.example.app`) |
| `device_brand`           | Cihaz markası (ör. `Apple`, `Samsung`)    |
| `device_model`           | Cihaz modeli (ör. `iPhone15,2`)           |
| `os_version`             | Ana işletim sistemi sürümü                |
| `app_version`            | Adapty SDK tarafından bildirilen uygulama sürümü |
| `sdk_version`            | Adapty SDK sürümü                         |

| `campaign_id`            | Kampanya tanımlayıcısı                    |
