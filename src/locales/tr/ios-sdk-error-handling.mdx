Adapty SDK, `AdaptyError` adı verilen tüm hata türleri için kendi sarmalayıcısına sahiptir. Temel olarak, SDK tarafından döndürülen her hata bir `AdaptyError`'dur. Bu hatanın iki kullanışlı özelliği vardır: aşağıda açıklanan `originalError` ve `adaptyErrorCode`.
**originalError**, orijinal hatayla çalışmanız gerektiğinde bunu içerir. [SKError](https://developer.apple.com/documentation/storekit/skerror), [NSError](https://developer.apple.com/documentation/foundation/nserror) veya genel bir Swift [Error](https://developer.apple.com/documentation/swift/error) olabilir. Bu özellik isteğe bağlıdır; çünkü bazı hatalar (tutarsız veya eksik veriler gibi) doğrudan SDK tarafından üretilir ve wrapper'ın başlangıçta oluşturulduğu orijinal hataya sahip olmayabilir.

**adaptyErrorCode**, aşağıdaki gibi yaygın sorunları ele almak için kullanılabilir:
- geçersiz kimlik bilgileri
- ağ hataları
- iptal edilen ödemeler
- faturalama sorunları
- geçersiz makbuz
- ve çok daha fazlası

Hatayı belirli kodlara göre kontrol etmek ve buna uygun tepki vermek oldukça kolaydır.

```swift showLineNumbers title="Swift"
do {
    let info = try await Adapty.makePurchase(product: product)
} catch {
    if error.adaptyErrorCode == .paymentCancelled {
        // purchase was cancelled
        // you can offer discount to your user or remind them later
    }
}
```
:::important
Bu çözümler sorununuzu gidermezse, destek ekibiyle iletişime geçmeden önce atmanız gereken adımlar için [Diğer sorunlar](#other-issues) bölümüne bakın; bu, size daha verimli yardımcı olmamızı sağlar.
:::

## Ağ hataları \{#network-errors\}
| Hata | Kod | Çözüm |
| :------------- | :--- |:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| notActivated | 2002 | Adapty SDK etkinleştirilmemiş. <br/> En sık karşılaşılan durum: bir splash screen veya erken yüklenen bir UI hook, `Adapty.activate` tamamlanmadan önce Adapty metotlarını çağırır. Bu belirti aralıklı olarak ortaya çıkar ve simülatörde yeniden üretilmeyebilir, çünkü gerçek cihaz zamanlaması farklıdır. Diğer SDK çağrılarını planlamadan önce `activate`'in tamamlanma handler'ını veya async sonucunu bekleyin. Tam sıralama için [iOS SDK'da çağrı sırası](ios-sdk-call-order) bölümüne bakın. |
| badRequest | 2003 | Geçersiz istek. <br/> [App Store ile entegrasyon](app-store-connection-configuration) için gereken tüm adımları tamamladığınızdan emin olun. |
| serverError | 2004 | Sunucu hatası. <br/> Bir süre sonra tekrar deneyin. Sorun çözülmezse Adapty destek ekibiyle iletişime geçin. |
| networkFailed | 2005 | Bu hata, kullanıcının cihazında ağ bağlantısı sorunları olduğunu gösterir. <br/> VPN'i devre dışı bırakmayı veya mobil veri ile Wi-Fi arasında geçiş yapmayı deneyin. |
| decodingFailed | 2006 | Bu hata, yanıt çözümlemenin (decoding) başarısız olduğunu gösterir. <br/> Kodunuzu gözden geçirin ve gönderdiğiniz parametrelerin geçerli olduğundan emin olun. Örneğin, bu hata geçersiz bir API anahtarı kullandığınıza işaret edebilir. |
| encodingFailed | 2009 | Bu hata, istek kodlamasının (encoding) başarısız olduğunu gösterir. |

## Diğer sorunlar \{#other-issues\}

Henüz bir çözüm bulamadıysanız, sonraki adımlar şunlar olabilir:
- **SDK'yı en son sürüme yükseltme**: Daha kararlı oldukları ve bilinen sorunlara yönelik düzeltmeler içerdikleri için her zaman en son SDK sürümlerine yükseltmenizi öneririz.
- **Destek ekibiyle iletişime geçin veya diğer geliştiricilerden yardım alın** [destek forumunda](https://adapty.featurebase.app/).
- **Destek ekibiyle [support@adapty.io](mailto:support@adapty.io) adresinden veya sohbet aracılığıyla iletişime geçin**: SDK'yı yükseltmeye hazır değilseniz ya da bu işlem sorununuzu çözmediyse destek ekibimizle iletişime geçin. [Ayrıntılı günlük kaydını etkinleştirirseniz](sdk-installation-ios#logging) ve günlükleri ekiple paylaşırsanız sorununuz daha hızlı çözülecektir. İlgili kod parçacıklarını da ekleyebilirsiniz.
