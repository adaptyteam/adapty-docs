Mobil uygulamanızda paywall'ları görüntülemek, kullanıcılara premium içerik veya hizmetlere erişim sunmanın temel adımlarından biridir. Ancak, paywall'larınızı özelleştirmek için [Paywall Builder](adapty-paywall-builder) kullanıyorsanız, bu paywall'ları yalnızca göstermek satın almaları desteklemek için yeterlidir.
Paywall Builder kullanmıyorsanız, bir satın alma işlemini tamamlamak ve istenen içeriğin kilidini açmak için `.makePurchase()` adlı ayrı bir yöntem kullanmanız gerekir. Bu yöntem, kullanıcıların paywall'larla etkileşime geçmesi ve istedikleri işlemleri gerçekleştirmesi için bir geçit görevi görür.

Kullanıcının satın almaya çalıştığı ürün için paywall'ınızda aktif bir promosyon teklifi varsa, Adapty satın alma sırasında bunu otomatik olarak uygular.

:::warning
Başlangıç teklifinin otomatik olarak uygulanabilmesi için Paywall Builder kullanılarak oluşturulmuş paywall'ları kullanmanız gerektiğini unutmayın.
Diğer durumlarda, [iOS'ta başlangıç teklifi uygunluğunu doğrulamanız](fetch-paywalls-and-products#check-intro-offer-eligibility-on-ios) gerekecektir. Bu adımı atlamak, yayın sırasında uygulamanızın reddedilmesine yol açabilir. Bunun yanı sıra, başlangıç teklifine hak kazanan kullanıcılardan tam ücret alınmasına neden olabilir.
:::

[Başlangıç yapılandırmasını](quickstart) tek bir adımı atlamadan tamamladığınızdan emin olun. Bu olmadan satın almaları doğrulayamayız.
## Satın alma işlemi gerçekleştirme \{#make-purchase\}

:::note
**[Paywall Builder](adapty-paywall-builder) kullanıyor musunuz?** Satın alma işlemleri otomatik olarak gerçekleştirilir; bu adımı atlayabilirsiniz.

**Adım adım rehber mi arıyorsunuz?** Tam bağlamıyla uçtan uca uygulama talimatları için [hızlı başlangıç kılavuzuna](flutter-implement-paywalls-manually) göz atın.
:::
```dart showLineNumbers
try {
  final purchaseResult = await Adapty().makePurchase(product: product);
    switch (purchaseResult) {
      case AdaptyPurchaseResultSuccess(profile: final profile):
        if (profile.accessLevels['premium']?.isActive ?? false) {
          // Grant access to the paid features
        }
        break;
      case AdaptyPurchaseResultPending():
        break;
      case AdaptyPurchaseResultUserCancelled():
        break;
      default:
        break;
    }
} on AdaptyError catch (adaptyError) {
    // Handle the error
} catch (e) {
    // Handle the error
}
```

İstek parametreleri:

| Parametre   | Zorunluluk | Açıklama                                                                                         |
| :---------- | :------- | :-------------------------------------------------------------------------------------------------- |
| **Product** | zorunlu | Paywall'dan alınan bir [`AdaptyPaywallProduct`](https://pub.dev/documentation/adapty_flutter/latest/adapty_flutter/AdaptyPaywallProduct-class.html) nesnesi. |

Yanıt parametreleri:
| Parametre | Açıklama                                                                                                                                                                                                                                                                                                                                                            |
|---------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Profile** | <p>İstek başarılı olursa yanıt bu nesneyi içerir. Bir [AdaptyProfile](https://pub.dev/documentation/adapty_flutter/latest/adapty_flutter/AdaptyProfile-class.html) nesnesi, kullanıcının uygulama içindeki access level'ları, abonelikleri ve tek seferlik satın almaları hakkında kapsamlı bilgi sağlar.</p><p>Kullanıcının uygulamaya gerekli erişime sahip olup olmadığını belirlemek için access level durumunu kontrol edin.</p> |
:::warning
**Not:** Apple'ın StoreKit sürümü v2.0'ın altında ve Adapty SDK sürümü v2.9.0'ın altında kullanıyorsanız, bunun yerine [Apple App Store paylaşılan sırrı](app-store-connection-configuration#step-5-enter-app-store-shared-secret) sağlamanız gerekir. Bu yöntem şu anda Apple tarafından kullanımdan kaldırılmıştır.
:::
## Satın alma sırasında aboneliği değiştirme \{#change-subscription-when-making-a-purchase\}

Bir kullanıcı mevcut aboneliğini yenilemek yerine yeni bir abonelik seçtiğinde, bu işlemin nasıl gerçekleşeceği uygulama mağazasına göre değişir:

- App Store'da abonelik, abonelik grubu içinde otomatik olarak güncellenir. Kullanıcı bir gruptan abonelik satın alırken başka bir gruptan aktif aboneliği varsa, her iki abonelik de aynı anda aktif olur.
- Google Play'de abonelik otomatik olarak güncellenmez. Geçişi aşağıda açıklandığı şekilde mobil uygulama kodunuzda yönetmeniz gerekir.
Android'de aboneliği başka bir abonelikle değiştirmek için `.makePurchase()` metodunu ek parametre ile çağırın:
```dart showLineNumbers
try {
  final subscriptionUpdateParams = AdaptyAndroidSubscriptionUpdateParameters(
    'OLD_PRODUCT_ID',
    AdaptyAndroidSubscriptionUpdateReplacementMode.immediateWithTimeProration,
  );

  final result = await Adapty().makePurchase(
    product: product,
    parameters: AdaptyPurchaseParameters(
      subscriptionUpdateParams: subscriptionUpdateParams,
    ),
  );
  
  // başarılı çapraz yükseltme
} on AdaptyError catch (adaptyError) {
  // Hatayı işle
} catch (e) {
  // Hatayı işle
}
```
Ek istek parametresi:
| Parametre                    | Zorunluluk | Açıklama                                                                                             |
| :--------------------------- | :------- |:--------------------------------------------------------------------------------------------------------|
| **parameters** | zorunlu | `subscriptionUpdateParams` alanı bir [`AdaptyAndroidSubscriptionUpdateParameters`](https://pub.dev/documentation/adapty_flutter/latest/adapty_flutter/AdaptyAndroidSubscriptionUpdateParameters-class.html) nesnesine ayarlanmış `AdaptyPurchaseParameters` nesnesi. |
Google Developer belgelerinde abonelikler ve değiştirme modları hakkında daha fazla bilgi edinebilirsiniz:
- [Değiştirme modları hakkında](https://developer.android.com/google/play/billing/subscriptions#replacement-modes)
- [Google'ın değiştirme modları için önerileri](https://developer.android.com/google/play/billing/subscriptions#replacement-recommendations)
- Değiştirme modu [`CHARGE_PRORATED_PRICE`](https://developer.android.com/reference/com/android/billingclient/api/BillingFlowParams.SubscriptionUpdateParams.ReplacementMode#CHARGE_PRORATED_PRICE()). Not: bu yöntem yalnızca abonelik yükseltmeleri için kullanılabilir. Düşürme işlemleri desteklenmez.
- Değiştirme modu [`DEFERRED`](https://developer.android.com/reference/com/android/billingclient/api/BillingFlowParams.SubscriptionUpdateParams.ReplacementMode#DEFERRED()). Not: Gerçek abonelik değişikliği yalnızca mevcut abonelik faturalandırma dönemi sona erdiğinde gerçekleşir.
## iOS'ta teklif kodlarını kullanma \{#redeem-offer-codes-in-ios\}

<OfferCodesInfo />

Uygulamanızda kod kullanma sayfasını görüntülemek için:

```dart showLineNumbers
try {
  await Adapty().presentCodeRedemptionSheet();
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
  // handle the error
}
```

:::danger
Gözlemlerimize göre, bazı uygulamalarda Teklif Kodu Kullanma sayfası güvenilir şekilde çalışmayabilir. Kullanıcıyı doğrudan App Store'a yönlendirmenizi öneririz.
Bunu yapmak için aşağıdaki formattaki URL'yi açmanız gerekir:
`https://apps.apple.com/redeem?ctx=offercodes&id={apple_app_id}&code={code}`
:::
### Ön ödemeli planları yönetme (Android) \{#manage-prepaid-plans-android\}

Uygulama kullanıcılarınız [ön ödemeli plan](https://developer.android.com/google/play/billing/subscriptions#prepaid-plans) satın alabiliyorsa (örneğin, birkaç aylık yenilenemez bir abonelik), ön ödemeli planlar için [beklemedeki işlemleri](https://developer.android.com/google/play/billing/subscriptions#pending) etkinleştirebilirsiniz.

```dart showLineNumbers title="main.dart"
await Adapty().activate(
  configuration: AdaptyConfiguration(apiKey: 'YOUR_PUBLIC_SDK_KEY')
    ..withGoogleEnablePendingPrepaidPlans(true),
);
```
