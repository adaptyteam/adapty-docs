## Yetkilendirme \{#authorization\}

API isteklerinizi, `Authorization` başlığında gizli API anahtarınızla kimlik doğrulamanız gerekir. Bu anahtarı [App Settings](https://app.adapty.io/settings/general) sayfasında bulabilirsiniz. Format şu şekildedir: `Api-Key {YOUR_SECRET_API_KEY}`, örneğin: `Api-Key secret_live_...`.

:::important
API anahtarları uygulamaya özeldir. Birden fazla uygulamanız varsa her biri için farklı anahtarlar kullandığınızdan emin olun.
:::
## İstek formatı \{#request-format\}

**Başlıklar**

Sunucu tarafı API istekleri, belirli başlıklar ve bir JSON gövdesi gerektirir. İsteklerinizi yapılandırmak için aşağıdaki ayrıntıları kullanın:

| Başlık       | Açıklama                                                     |
| ------------ | ------------------------------------------------------------ |
| Content-Type | (Zorunlu) API'nin isteği işleyebilmesi için `application/json` olarak ayarlayın. |
| Adapty-Tz    | (İsteğe bağlı) Verilerin nasıl gruplandırılacağını ve görüntüleneceğini tanımlamak için saat dilimini ayarlayın. [IANA Saat Dilimi Veritabanı formatını](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones) kullanın (ör. `Europe/Berlin`). |

## Gövde \{#body\}

API, istek için gerekli verileri içeren JSON formatında bir gövde bekler.

## Hız sınırları \{#rate-limits\}

API anahtarı başına saniyede maksimum 2 istek gönderilebilir. Bu sınırı aşmanız durumunda `429 Too Many Requests` hatası döner.
## API anahtarlarını yenileme \{#rotate-api-keys\}

Gizli API anahtarlarını yenilemek için:

1. **Settings → General** sayfasında **Generate new key** butonuna tıklayın, ardından eski anahtarın yanındaki çöp kutusu simgesine tıklayın.
2. Uygulamanızda kullanılan anahtarı güncelleyin.
