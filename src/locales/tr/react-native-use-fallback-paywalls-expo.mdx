:::important
Bu kılavuz **Expo projeleri** için geçerlidir.
**Saf React Native (Expo olmayan)** kullanıyorsanız, bunun yerine [saf React Native yedek paywall kılavuzunu](react-native-use-fallback-paywalls-pure) izleyin.
:::

<FallbackPaywallIntroduction />
Adapty SDK, yedek dosyasını **native** bundle'dan okur; iOS'ta `.app` paketinin içindeki bir kaynak dosyası, Android'de ise `android/app/src/main/assets/` altındaki bir girdi olarak yer alır. Expo projelerinde `npx expo prebuild --clean` komutu her çalıştırıldığında bu dizinleri yeniden oluşturur, bu yüzden dosyaları manuel olarak ekleyemezsiniz. `react-native-adapty` config plugin, dosyayı native bundle'a sizin yerinize bağlar.

:::tip
Tam çalışan bir örnek kurulum [`FocusJournalExpo` örnek uygulamasında](https://github.com/adaptyteam/AdaptySDK-React-Native/tree/master/examples/FocusJournalExpo) mevcuttur.
:::
## Yapılandırma \{#configuration\}

1. Yedek JSON dosyalarını projenizde herhangi bir yere yerleştirin — genellikle diğer asset'lerin yanına:

   ```
   <your-project>/
   └── assets/
       ├── ios_fallback.json
       └── android_fallback.json
   ```

2. `app.json` (veya `app.config.js`) içindeki `react-native-adapty` girdisine `fallbackFile` seçeneğini ekleyin. Her platform anahtarı isteğe bağlıdır — yalnızca ihtiyaç duyduğunuz platformları bağlayın:

   ```json title="app.json"
   {
     "expo": {
       "plugins": [
         [
           "react-native-adapty",
           {
             "fallbackFile": {
               "ios": "./assets/ios_fallback.json",
               "android": "./assets/android_fallback.json"
             }
           }
         ]
       ]
     }
   }
   ```

   :::note
   Adapty her platform için farklı bir yedek JSON dışa aktarır — iOS'ta Apple ürün ID'leri, Android'de Google Play ürün ID'leri. Her platformu kendi dosyasına yönlendirin.
   :::

3. Yerel projeleri yeniden oluşturun:

   ```sh title="Shell"
   npx expo prebuild
   ```

   Eklenti, iOS dosyasını Xcode projesinin paket kaynaklarına ekler ve Android dosyasını `android/app/src/main/assets/` içine kopyalar. Prebuild çıktısı şuna benzer satırlar içerir:

   ```
   [react-native-adapty] Registered ios_fallback.json as iOS bundle resource
   [react-native-adapty] Copied android_fallback.json to android assets/
   ```

4. Dosyayı çalışma zamanında SDK'ya kaydedin:

   ```typescript showLineNumbers title="App.tsx"
   import { adapty } from 'react-native-adapty';

   await adapty.activate('PUBLIC_SDK_KEY');

   await adapty.setFallback({
     ios: { fileName: 'ios_fallback.json' },
     android: { relativeAssetPath: 'android_fallback.json' },
   });
   ```

   `setFallback`'e geçirilen dosya adları, `fallbackFile` altında yapılandırılan dosyaların temel adlarıyla (basename) eşleşmelidir.

:::important
`setFallback`, SDK hedef flow'u, paywall'ı veya onboarding'i getirmeden önce çalışmalıdır.
:::

## Doğrulama \{#verification\}

`npx expo prebuild` çalıştırdıktan sonra her iki platformu da kontrol edin:

- **Android**: `android/app/src/main/assets/` dizininin içeriğini listeleyin. `fallbackFile.android` altında yapılandırılan dosya burada bulunmalı, yalnızca iOS'a ait dosya adı ise burada görünmemelidir.
- **iOS**: `ios/<ProjectName>.xcodeproj/project.pbxproj` dosyasında iOS dosya adını aratın. `PBXFileReference`, `Resources` grubu ve `PBXResourcesBuildPhase` içinde görünmelidir. Yalnızca Android'e ait dosya adı ise `project.pbxproj` içinde yer almamalıdır.
