Bu rehber, Adapty'yi özel paywall'larınıza nasıl entegre edeceğinizi açıklar. Paywall uygulamasının tam kontrolünü elinizde tutarken, Adapty SDK ürünleri getirir, yeni satın almaları gerçekleştirir ve önceki satın almaları geri yükler. Bu rehber Adapty Flutter SDK v4 API'lerini kullanır — v3 kullanıyorsanız, karşılık gelen metod adları için [migration rehberine](migration-to-flutter-sdk-v4) bakın.
:::important
**Bu kılavuz, özel paywall uygulayan geliştiriciler içindir.** Satın alma işlemlerini etkinleştirmenin en kolay yolunu arıyorsanız, [Adapty Paywall Builder](flutter-quickstart-paywalls)'ı kullanın. Paywall Builder ile kodsuz görsel bir editörde paywall'lar oluşturursunuz, Adapty tüm satın alma mantığını otomatik olarak yönetir ve uygulamanızı yeniden yayınlamadan farklı tasarımları test edebilirsiniz.
:::
## Başlamadan Önce \{#before-you-start\}
### Ürünleri ayarlayın \{#set-up-products\}

Uygulama içi satın alımları etkinleştirmek için üç temel kavramı anlamanız gerekir:
- [**Ürünler**](product) – kullanıcıların satın alabileceği her şey (abonelikler, consumable'lar, süresiz erişim)
- [**Paywalllar**](paywalls) – hangi ürünlerin sunulacağını tanımlayan yapılandırmalar. Adapty'de ürünlere yalnızca paywalllar aracılığıyla ulaşılır; bu yapı sayesinde uygulama koduna dokunmadan ürünleri, fiyatları ve teklifleri değiştirebilirsiniz. SDK v4'te bir placement için paywall varyantları bir **flow** nesnesi tarafından taşınır — flow'u çekip ürünlerini sorgularsınız.
- [**Placement'lar**](placements) – uygulamanızda paywallları nerede ve ne zaman göstereceğiniz (örneğin `main`, `onboarding`, `settings`). Kontrol panelinde placement'lar için paywalllar ayarlarsınız, ardından kodunuzda placement ID'siyle bunları talep edersiniz. Bu yapı A/B testleri yürütmeyi ve farklı kullanıcılara farklı paywalllar göstermeyi kolaylaştırır.
Bu kavramları, kendi özel paywall'unuzla çalışsanız bile anladığınızdan emin olun. Bunlar, temelde uygulamanızda sattığınız ürünleri yönetmenin bir yoludur.

Özel paywall'unuzu hayata geçirmek için bir **paywall** oluşturmanız ve bunu bir **placement**'a eklemeniz gerekir. Bu yapı, ürünlerinizi almanızı sağlar. Kontrol panelinde yapmanız gerekenleri anlamak için [buradaki](quickstart) hızlı başlangıç kılavuzunu takip edin.
### Kullanıcıları yönetme \{#manage-users\}

Backend kimlik doğrulaması kullanıp kullanmamak tamamen size bağlıdır.

Ancak Adapty SDK, anonim ve kimliği doğrulanmış kullanıcıları farklı şekilde ele alır. Ayrıntıları öğrenmek ve kullanıcılarla doğru şekilde çalıştığınızdan emin olmak için [kimlik doğrulama hızlı başlangıç kılavuzunu](flutter-quickstart-identify) okuyun.
## 1. Adım. Ürünleri alın \{#step-1-get-products\}

Özel paywallınız için ürünleri almak üzere şunları yapmanız gerekir:

1. [Placement](placements) ID'sini `getFlow` metoduna geçirerek `flow` nesnesini alın.
2. Bu flow için ürünler dizisini `getPaywallProducts` metodunu kullanarak alın.
```dart showLineNumbers

Future<void> loadPaywall() async {
  try {
    final flow = await Adapty().getFlow(placementId: 'YOUR_PLACEMENT_ID');
    final products = await Adapty().getPaywallProducts(flow: flow);
    
    // Use products to build your custom paywall UI
  } on AdaptyError catch (adaptyError) {
    // Handle the error
  } catch (e) {
    // Handle the error
  }
}
```

## Adım 2. Satın almaları kabul etme \{#step-2-accept-purchases\}

Kullanıcı özel paywallınızdaki bir ürüne dokunduğunda, seçilen ürünle birlikte `makePurchase` metodunu çağırın. Bu işlem satın alma akışını yönetir ve güncellenmiş profili döndürür.
```dart showLineNumbers

Future<void> purchaseProduct(AdaptyPaywallProduct product) async {
  try {
    final purchaseResult = await Adapty().makePurchase(product: product);
    
    switch (purchaseResult) {
      case AdaptyPurchaseResultSuccess(profile: final profile):
        // Purchase successful, profile updated
        break;
      case AdaptyPurchaseResultUserCancelled():
        // User canceled the purchase
        break;
      case AdaptyPurchaseResultPending():
        // Purchase is pending (e.g., user will pay offline with cash)
        break;
    }
  } on AdaptyError catch (adaptyError) {
    // Handle the error
  } catch (e) {
    // Handle the error
  }
}
```

## Adım 3. Satın almaları geri yükle \{#step-3-restore-purchases\}

Uygulama mağazaları, abonelik içeren tüm uygulamaların kullanıcılara satın almalarını geri yükleyebilecekleri bir yol sunmasını zorunlu kılar.

Kullanıcı geri yükle butonuna bastığında `restorePurchases` metodunu çağırın. Bu işlem, kullanıcının satın alma geçmişini Adapty ile senkronize eder ve güncel profili döndürür.
```dart showLineNumbers

Future<void> restorePurchases() async {
  try {
    final profile = await Adapty().restorePurchases();
    // Restore successful, profile updated
  } on AdaptyError catch (adaptyError) {
    // Handle the error
  } catch (e) {
    // Handle the error
  }
}
```

## Adım 4. Abonelik durumunu kontrol edin \{#step-4-check-the-subscription-status\}

Bir satın alma veya geri yükleme işleminin ardından, paywall'ı gösterip göstermeyeceğinize ya da ücretli özellikleri açıp açmayacağınıza karar vermek için kullanıcının [access level](access-level) durumunu kontrol edin. `makePurchase` ve `restorePurchases` metodları zaten güncellenmiş profili döndürür; uygulamanın başka bir yerinde mevcut duruma ihtiyaç duyduğunuzda `getProfile` metodunu kullanın:
```dart showLineNumbers

Future<bool> hasPremiumAccess() async {
  try {
    final profile = await Adapty().getProfile();
    return profile.accessLevels['premium']?.isActive ?? false;
  } on AdaptyError catch (adaptyError) {
    // Handle the error
  } catch (e) {
    // Handle the error
  }
  return false;
}
```

Abonelik durumunu kontrol etmenin ve gerçek zamanlı güncellemeleri dinleme dahil izlemenin daha fazla yolu için bkz. [Abonelik durumunu kontrol etme](flutter-check-subscription-status).
## Sonraki adımlar

<SupportForum />

Paywall'ınız uygulamada gösterilmeye hazır. Paywall üzerinden test satın alma işlemini tamamlayabildiğinizden emin olmak için [App Store sandbox](test-purchases-in-sandbox) ortamında veya [Google Play Store](testing-on-android)'da satın almalarınızı test edin. Bunun gerçek bir uygulamada nasıl çalıştığını görmek için örnek uygulamamızdaki [PurchasesObserver](https://github.com/adaptyteam/AdaptySDK-Flutter/blob/master/example/lib/purchase_observer.dart)'a göz atabilirsiniz; bu örnekte hata yönetimi, UI gözlemcileri ve kapsamlı SDK entegrasyonuyla birlikte satın alma işleme akışı gösterilmektedir.
