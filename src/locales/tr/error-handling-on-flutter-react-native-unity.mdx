SDK tarafından döndürülen her hata `AdaptyErrorCode` türündedir. İşte bir örnek:
:::tip
**Hata ayıklamadan önce ayrıntılı günlükleri etkinleştirin.** Çoğu `AdaptyError`, temelinde bir StoreKit, Play Billing, ağ veya backend hatası barındırır. Ayrıntılı günlükler açıkken (`await Adapty().setLogLevel(AdaptyLogLevel.verbose)` — bkz. [Loglama](sdk-installation-flutter#logging)), sarmalanmış hata konsola yazdırılır ve genellikle asıl nedeni ortaya koyar.
:::

:::important
Bu çözümler sorununuzu gidermezse, destek ekibiyle iletişime geçmeden önce atmanız gereken adımlar için [Diğer sorunlar](#other-issues) bölümüne bakın; böylece size daha verimli yardım sunabiliriz.
:::
```dart showLineNumbers
    try {
      final result = await adapty.makePurchase(product: product);
    } on AdaptyError catch (adaptyError) {
      if (adaptyError.code == AdaptyErrorCode.paymentCancelled) {
        // Cancelled
      }
    } catch (e) {
    }
```

## Diğer sorunlar \{#other-issues\}

Henüz bir çözüm bulamadıysanız, sonraki adımlar şunlar olabilir:
- **SDK'yı en son sürüme yükseltme**: En son SDK sürümlerine yükseltmenizi her zaman öneririz; bu sürümler daha kararlıdır ve bilinen sorunlar için düzeltmeler içerir.
- **Destek ekibiyle iletişime geçin veya diğer geliştiricilerden yardım alın** [destek forumunda](https://adapty.featurebase.app/).
- **Destek ekibiyle [support@adapty.io](mailto:support@adapty.io) adresinden veya sohbet yoluyla iletişime geçin**: SDK'yı yükseltmeye hazır değilseniz veya bu işlem sorunu çözmediyse destek ekibimizle iletişime geçin. [Ayrıntılı günlük kaydını etkinleştirip](sdk-installation-flutter#logging) günlükleri ekiple paylaşırsanız sorununuz daha hızlı çözülecektir. İlgili kod parçacıklarını da ekleyebilirsiniz.
