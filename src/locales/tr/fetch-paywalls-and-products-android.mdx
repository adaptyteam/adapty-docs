## Flow bilgilerini getirme \{#fetch-flow-information\}

Adapty'de bir [ürün](product), hem App Store hem de Google Play'den gelen ürünlerin bir kombinasyonudur. Bu platformlar arası ürünler, flow'lara ve paywall'lara entegre edilerek belirli mobil uygulama placement'larında sergilenebilir.

Ürünleri göstermek için `getFlow` metoduyla [placement'larınızdan](placements) birinden bir `AdaptyFlow` almanız gerekir.
:::important
**Ürün ID'lerini hardcode etmeyin.** Hardcode etmeniz gereken tek ID, placement ID'sidir. Flow'lar uzaktan yapılandırıldığından, ürün sayısı ve mevcut teklifler her an değişebilir. Uygulamanız bu değişiklikleri dinamik olarak ele almalıdır; bir flow bugün iki ürün döndürüyorsa ve yarın üç ürün döndürüyorsa, kod değişikliği yapmadan hepsini gösterin.
:::

<Tabs groupId="current-os" queryString>
<TabItem value="kotlin" label="Kotlin" default>
```kotlin showLineNumbers
Adapty.getFlow("YOUR_PLACEMENT_ID") { result ->
    when (result) {
        is AdaptyResult.Success -> {
            val flow = result.value
            // istenen flow
        }
        is AdaptyResult.Error -> {
            val error = result.error
            // hatayı işle
        }
    }
}
```
</TabItem>

```java showLineNumbers
Adapty.getFlow("YOUR_PLACEMENT_ID", result -> {
    if (result instanceof AdaptyResult.Success) {
        AdaptyFlow flow = ((AdaptyResult.Success<AdaptyFlow>) result).getValue();
        // istenen flow
      
    } else if (result instanceof AdaptyResult.Error) {
        AdaptyError error = ((AdaptyResult.Error) result).getError();
        // hatayı işle
      
    }
});
```
</TabItem>

## Ürünleri getir \{#fetch-products\}

Flow'a sahip olduktan sonra, o flow'a karşılık gelen ürün dizisini sorgulayabilirsiniz:

<TabItem value="kotlin" label="Kotlin" default>

```kotlin showLineNumbers
Adapty.getPaywallProducts(flow) { result ->
    when (result) {
        is AdaptyResult.Success -> {
            val products = result.value
            // the requested products
        }
        is AdaptyResult.Error -> {
            val error = result.error
            // handle the error
        }
    }
}
```
</TabItem>

```java showLineNumbers
        // istenen ürünler

        AdaptyError error = ((AdaptyResult.Error) result).getError();
        // hatayı işle
      
    }
```

## Varsayılan kitle flow'u ile flow getirmeyi hızlandırın \{#speed-up-flow-fetching-with-default-audience-flow\}

Genellikle flow'lar neredeyse anında getirilir, bu yüzden bu süreci hızlandırma konusunda endişelenmenize gerek yoktur. Ancak çok sayıda kitle ve placement'ınız olduğunda ve kullanıcılarınızın internet bağlantısı zayıf olduğunda, bir flow'un getirilmesi istediğinizden daha uzun sürebilir. Böyle durumlarda, hiçbir şey göstermemek yerine sorunsuz bir kullanıcı deneyimi sağlamak için varsayılan bir flow görüntülemek isteyebilirsiniz.
Bunu çözmek için, belirtilen placement'ın **All Users** kitlesi için flow'unu getiren `getFlowForDefaultAudience` metodunu kullanabilirsiniz. Ancak şunu anlamak kritik önem taşır: önerilen yaklaşım, yukarıdaki [Flow bilgisini getirme](fetch-paywalls-and-products-android#fetch-flow-information) bölümünde ayrıntılandırıldığı gibi `getFlow` metoduyla flow'u getirmektir.

:::warning
`getFlow` kullanımını neden öneriyoruz

`getFlowForDefaultAudience` metodunun birkaç önemli dezavantajı vardır:
- **Olası geriye dönük uyumluluk sorunları**: Farklı uygulama sürümleri için (mevcut ve gelecek) farklı flow'lar göstermeniz gerekiyorsa, zorluklarla karşılaşabilirsiniz. Ya mevcut (eski) sürümü destekleyen flow'lar tasarlamak ya da mevcut (eski) sürüme sahip kullanıcıların render edilemeyen flow'larla sorun yaşabileceğini kabul etmek zorunda kalırsınız.
- **Hedefleme kaybı**: Tüm kullanıcılar **All Users** kitlesi için tasarlanmış aynı flow'u görecektir; bu da kişiselleştirilmiş hedeflemeyi (ülkelere, marketing attribution'a veya kendi özel niteliklerinize göre hedefleme dahil) kaybetmeniz anlamına gelir.
Bu dezavantajları kabul ederek daha hızlı flow getirme işleminden yararlanmak istiyorsanız, `getFlowForDefaultAudience` metodunu aşağıdaki şekilde kullanın. Aksi takdirde, [yukarıda](fetch-paywalls-and-products-android#fetch-flow-information) açıklanan `getFlow` metodunu kullanmaya devam edin.
:::

## Paywall bilgilerini getirme \{#fetch-paywall-information\}

Adapty'de bir [ürün](product), hem App Store hem de Google Play'den gelen ürünlerin bir kombinasyonunu temsil eder. Bu platformlar arası ürünler, belirli mobil uygulama placement'larında sergilenebilmesi için paywall'lara entegre edilir.

Ürünleri gösterebilmek için `getPaywall` metoduyla [placement](placements)'larınızdan birinden bir [Paywall](paywalls) almanız gerekir.
:::important
**Ürün ID'lerini doğrudan koda yazmayın.** Doğrudan yazmanız gereken tek ID, placement ID'sidir. Paywall'lar uzaktan yapılandırıldığından, ürün sayısı ve mevcut teklifler her an değişebilir. Uygulamanız bu değişiklikleri dinamik olarak ele almalıdır; bir paywall bugün iki ürün döndürüyorsa ve yarın üç ürün döndürüyorsa, kod değişikliği yapmadan hepsini gösterin.
:::

Ürün ID'lerini hardcode etmeyin! Paywall'lar uzaktan yapılandırıldığından, mevcut ürünler, ürün sayısı ve özel teklifler (ücretsiz deneme gibi) zamanla değişebilir. Kodunuzun bu senaryoları ele aldığından emin olun.

Örneğin, başlangıçta 2 ürün alıyorsanız uygulamanız o 2 ürünü göstermelidir. Ancak daha sonra 3 ürün alırsanız, herhangi bir kod değişikliği gerektirmeden uygulamanız 3 ürünü de göstermelidir. Hardcode etmeniz gereken tek şey placement ID'sidir.

| Parametre | Açıklama                                                                                                                                                  |

## Varsayılan kitle paywallı ile paywall getirmeyi hızlandırın \{#speed-up-paywall-fetching-with-default-audience-paywall\}

Genellikle paywalllar neredeyse anında getirilir, bu yüzden bu süreci hızlandırmak için endişelenmenize gerek yoktur. Ancak çok sayıda kitle ve paywalliniz varsa ve kullanıcılarınızın internet bağlantısı zayıfsa, paywall getirme işlemi istediğinizden daha uzun sürebilir. Bu gibi durumlarda, hiç paywall göstermemek yerine sorunsuz bir kullanıcı deneyimi sağlamak için varsayılan bir paywall görüntülemek isteyebilirsiniz.
Bunu çözmek için, belirtilen placement'ın **All Users** kitlesi için paywall'ı çeken `getPaywallForDefaultAudience` metodunu kullanabilirsiniz. Ancak, önerilen yaklaşımın yukarıdaki [Paywall Bilgisini Çekme](fetch-paywalls-and-products-android#fetch-paywall-information) bölümünde açıklandığı gibi `getPaywall` metoduyla paywall'ı çekmek olduğunu anlamanız kritik önem taşır.

:::warning
Neden `getPaywall` kullanmanızı öneriyoruz

`getPaywallForDefaultAudience` metodunun birkaç önemli dezavantajı vardır:
- **Olası geriye dönük uyumluluk sorunları**: Farklı uygulama sürümleri (mevcut ve gelecekteki) için farklı paywall'lar göstermeniz gerekirse zorluklarla karşılaşabilirsiniz. Ya mevcut (eski) sürümü destekleyen paywall'lar tasarlamanız ya da mevcut (eski) sürüme sahip kullanıcıların render edilemeyen paywall'larla sorun yaşamasını kabul etmeniz gerekecektir.
- **Hedefleme kaybı**: Tüm kullanıcılar **All Users** kitlesi için tasarlanmış aynı paywall'ı görür; bu da ülkelere, pazarlama attribution'ına veya kendi özel özelliklerinize dayalı kişiselleştirilmiş hedeflemeyi kaybetmeniz anlamına gelir.
Bu dezavantajları kabul edip daha hızlı paywall yükleme avantajından faydalanmak istiyorsanız, `getPaywallForDefaultAudience` yöntemini aşağıdaki şekilde kullanın. Aksi takdirde, [yukarıda](fetch-paywalls-and-products-android#fetch-paywall-information) açıklanan `getPaywall` yöntemini kullanmaya devam edin.
:::
