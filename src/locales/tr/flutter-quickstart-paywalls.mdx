Uygulama içi satın almaları etkinleştirmek için üç temel kavramı anlamanız gerekir:
- [**Products**](product) – kullanıcıların satın alabileceği her şey (abonelikler, consumable'lar, süresiz erişim)
- [**Paywalls**](paywalls), hangi ürünlerin sunulacağını tanımlayan yapılandırmalardır. Adapty'de ürünlere yalnızca paywall'lar aracılığıyla ulaşılabilir; bu tasarım sayesinde uygulama koduna dokunmadan teklifleri, fiyatlandırmayı ve ürün kombinasyonlarını değiştirebilirsiniz.
- [**Placements**](placements) – uygulamanızda paywall'ları nerede ve ne zaman göstereceğiniz (örneğin `main`, `onboarding`, `settings`). Kontrol panelinde placement'lar için paywall'lar ayarlarsınız, ardından kodunuzda placement ID'siyle bunları talep edersiniz. Bu yapı, A/B testleri çalıştırmayı ve farklı kullanıcılara farklı paywall'lar göstermeyi kolaylaştırır.
Adapty, uygulamanızda satın alma işlemlerini etkinleştirmek için size üç yöntem sunar. Uygulama gereksinimlerinize bağlı olarak birini seçin:
| Uygulama               | Karmaşıklık | Ne zaman kullanılır                                                                                                                                                                                                                                |
|------------------------|-------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Adapty Paywall Builder | ✅ Kolay     | No-code builder'da [eksiksiz, satın almaya hazır bir paywall oluşturursunuz](quickstart-paywalls). Adapty bunu otomatik olarak render eder; karmaşık satın alma akışını, makbuz doğrulamasını ve abonelik yönetimini arka planda sizin için halleder. |
| Manuel oluşturulan paywalllar | 🟡 Orta  | Paywall arayüzünü uygulama kodunuzda kendiniz uygularsınız, ancak ürün tekliflerinde esnekliği korumak için paywall nesnesini yine de Adapty'den alırsınız. [Kılavuza](flutter-quickstart-manual) bakın.                                            |
| Observer mode              | 🔴 Zor    | Kendi satın alma altyapınız zaten mevcut ve bunu kullanmaya devam etmek istiyorsunuz. Observer mode'un Adapty'de bazı kısıtlamaları olduğunu unutmayın. [Makaleye](observer-vs-full-mode) bakın.                                                   |
:::important
**Aşağıdaki adımlar, Adapty paywall builder'da oluşturulmuş bir paywall'un nasıl uygulanacağını göstermektedir.**

Paywall builder'ı kullanmak istemiyorsanız, [elle oluşturulan paywallarda satın alma işlemlerini yönetme kılavuzuna](flutter-making-purchases) bakın.
:::

Adapty paywall builder'da oluşturulmuş bir paywall'u göstermek için uygulama kodunuzda yalnızca şunları yapmanız yeterlidir:
1. **Paywall'ı alın**: Paywall'ı Adapty'den alın.
2. **Paywall'ı gösterin, satın almaları Adapty yönetir**: Uygulamanızda edindiğiniz paywall container'ını gösterin.
3. **Buton aksiyonlarını yönetin**: Paywall'daki kullanıcı etkileşimlerini uygulamanızın yanıtlarıyla ilişkilendirin. Örneğin, kullanıcılar butonlara tıkladığında bağlantıları açın veya paywall'ı kapatın.
## Başlamadan önce \{#before-you-start\}

Başlamadan önce şu adımları tamamlayın:

1. Uygulamanızı Adapty Kontrol Paneli'nde [App Store](initial_ios) ve/veya [Google Play](initial-android)'e bağlayın.
2. Adapty'de [ürünlerinizi oluşturun](create-product).
3. [Bir paywall oluşturun ve ürünleri ekleyin](create-paywall).
4. [Bir placement oluşturun ve paywallınızı ekleyin](create-placement).
5. Uygulama kodunuza [Adapty SDK'yı yükleyin ve etkinleştirin](sdk-installation-flutter).
:::tip
Bu adımları tamamlamanın en hızlı yolu, [hızlı başlangıç kılavuzunu](quickstart) takip etmek veya [Developer CLI](developer-cli-quickstart) kullanarak paywall ve placement oluşturmaktır.
:::
## 1. Paywall'ı alın \{#1-get-the-paywall\}

Paywall'larınız, kontrol panelinde yapılandırılmış placement'larla ilişkilidir. Placement'lar, farklı kitleler için farklı paywall'lar çalıştırmanıza veya [A/B testleri](ab-tests) yapmanıza olanak tanır.

Adapty paywall builder'da oluşturulmuş bir paywall'ı almak için şunları yapmanız gerekir:

1. `getPaywall` metodunu kullanarak [placement](placements) ID'sine göre `paywall` nesnesini alın ve `hasViewConfiguration` özelliğini kullanarak bunun builder'da oluşturulmuş bir paywall olup olmadığını kontrol edin.
2. `createPaywallView` metodunu kullanarak paywall görünümünü oluşturun. Görünüm, paywallı göstermek için gereken UI öğelerini ve stilini içerir.

:::important
Görünüm yapılandırmasını alabilmek için Paywall Builder'daki **Show on device** geçişini açmanız gerekir. Aksi takdirde boş bir görünüm yapılandırması alırsınız ve paywall görüntülenmez.
:::
```dart showLineNumbers

try {
  final paywall = await Adapty().getPaywall(placementId: "YOUR_PLACEMENT_ID", locale: "en");
  // the requested paywall
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
}

try {
  final view = await AdaptyUI().createPaywallView(
        paywall: paywall,
      );
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```

## 2. Paywalli görüntüle \{#display-the-paywall\}

Artık paywall yapılandırmanız olduğuna göre, paywalli görüntülemek için yalnızca birkaç satır eklemeniz yeterli.

Paywalli görüntülemek için `createPaywallView` metoduyla oluşturulan `view` üzerinde `view.present()` metodunu kullanın. Her `view` yalnızca bir kez kullanılabilir. Paywalli tekrar görüntülemeniz gerekiyorsa yeni bir `view` örneği oluşturmak için `createPaywallView` metodunu bir kez daha çağırın.

```dart showLineNumbers title="Flutter"
try {
  await view.present();
} on AdaptyError catch (e) {
  // handle the error
} catch (e) {
  // handle the error
}
```
:::tip
Bir paywall'ın nasıl görüntüleneceğine dair daha fazla ayrıntı için [kılavuzumuza](flutter-present-paywalls) bakın.
:::
## 3. Buton eylemlerini yönetin \{#handle-button-actions\}

Kullanıcılar paywall'daki butonlara tıkladığında Flutter SDK, satın alma ve geri yükleme işlemlerini otomatik olarak gerçekleştirir. Ancak diğer butonların özel veya önceden tanımlanmış ID'leri vardır ve bu eylemlerin kodunuzda ele alınması gerekir.

Paywall ekranındaki süreçleri kontrol etmek veya izlemek için `AdaptyUIPaywallsEventsObserver` metodlarını uygulayın ve herhangi bir ekran gösterilmeden önce observer'ı ayarlayın. Kullanıcı bir eylem gerçekleştirdiğinde `paywallViewDidPerformAction` tetiklenir ve uygulamanızın eylem ID'sine göre yanıt vermesi gerekir.
Örneğin, paywallınızda muhtemelen bir kapat düğmesi ve açılacak URL'ler (örneğin, kullanım koşulları ve gizlilik politikası) bulunmaktadır. Bu nedenle `Close` ve `OpenUrl` ID'lerine sahip aksiyonlara yanıt vermeniz gerekir.

:::tip
Düğme [aksiyonlarını](flutter-handle-paywall-actions) ve [olayları](flutter-handling-events) nasıl ele alacağınıza dair rehberlerimizi okuyun.
:::
Gözlemciyi, bir widget yerine özel ve uzun ömürlü bir nesne olarak uygulayın. Tek bir global gözlemci slotu tüm uygulama genelinde paylaşıldığından, bunu bir `State`'e bağlamak ekranı sızdırır (SDK ona güçlü bir referans tutar) ve bir sonraki ekran kendini kaydettirdiğinde sessizce değiştirilir. `extends` kullanmak ayrıca SDK'nın varsayılan davranışını miras alır; böylece yalnızca önemsediğiniz callback'leri geçersiz kılarsınız.
```dart showLineNumbers title="Flutter"
// A dedicated, long-lived handler for paywall events.
// It does NOT live inside a Widget/State, so it never leaks and is never
// silently replaced when screens are pushed or popped.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // This method is called when user performs an action on the paywall UI
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using url_launcher package
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors
      print('Error launching URL: $e');
    }
  }
}
```

**Herhangi bir paywall gösterilmeden önce,** uygulamanın başlangıcında handler'ı **bir kez** kaydedin:

```dart showLineNumbers title="Flutter"
AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());
```

## Tam örnek \{#full-example\}

Tüm bu adımların uygulamanızda nasıl bir arada kullanılabileceği aşağıda gösterilmiştir.
```dart

void main() {
  // Register a single, long-lived observer once, before any paywall is shown.
  // It is intentionally a plain object (NOT a Widget/State): its lifetime is the
  // whole app, so it never leaks and is never silently replaced when screens are
  // pushed or popped.
  AdaptyUI().setPaywallsEventsObserver(PaywallEventsHandler());

  runApp(MaterialApp(home: PaywallScreen()));
}

/// A dedicated handler for AdaptyUI paywall events.
///
/// It `extends` [AdaptyUIPaywallsEventsObserver] (rather than being implemented
/// by a `State`), which gives you two things for free:
///   * the SDK's sensible defaults (e.g. auto-dismiss on close/finish), so you
///     only override what you actually care about;
///   * a lifecycle that is independent of the widget tree — there is no strong
///     reference back into a `Widget`, so nothing leaks and there is nothing to
///     unregister.
///
/// Every callback receives the [AdaptyUIPaywallView] it relates to, so handling
/// paywall actions never requires a `BuildContext` or widget state.
class PaywallEventsHandler extends AdaptyUIPaywallsEventsObserver {
  // A single, app-wide instance — same idiom as Adapty() and AdaptyUI().
  static final PaywallEventsHandler _instance = PaywallEventsHandler._();
  factory PaywallEventsHandler() => _instance;
  PaywallEventsHandler._();

  // Called when the user performs an action on the paywall UI.
  @override
  void paywallViewDidPerformAction(AdaptyUIPaywallView view, AdaptyUIAction action) {
    switch (action) {
      case const CloseAction():
      case const AndroidSystemBackAction():
        view.dismiss();
        break;
      case OpenUrlAction(url: final url):
        // Open the URL using the url_launcher package.
        _launchUrl(url);
        break;
      default:
        break;
    }
  }

  // Dismiss the paywall once a restore succeeds.
  @override
  void paywallViewDidFinishRestore(AdaptyUIPaywallView view, AdaptyProfile profile) {
    view.dismiss();
  }

  // Handle failures while the paywall is being rendered.
  @override
  void paywallViewDidFailRendering(AdaptyUIPaywallView view, AdaptyError error) {
    print('Paywall rendering failed: $error');
  }

  // Helper method to launch URLs.
  Future<void> _launchUrl(String url) async {
    try {
      final Uri uri = Uri.parse(url);
      if (await canLaunchUrl(uri)) {
        await launchUrl(uri, mode: LaunchMode.externalApplication);
      } else {
        // Handle case where URL cannot be launched.
        print('Could not launch $url');
      }
    } catch (e) {
      // Handle any errors.
      print('Error launching URL: $e');
    }
  }
}

class PaywallScreen extends StatefulWidget {
  const PaywallScreen({super.key});

  @override
  State<PaywallScreen> createState() => _PaywallScreenState();
}

class _PaywallScreenState extends State<PaywallScreen> {
  @override
  void initState() {
    super.initState();
    _showPaywallIfNeeded();
  }

  Future<void> _showPaywallIfNeeded() async {
    try {
      final paywall = await Adapty().getPaywall(
        placementId: 'YOUR_PLACEMENT_ID',
      );

      if (!paywall.hasViewConfiguration) return;

      final view = await AdaptyUI().createPaywallView(paywall: paywall);

      await view.present();
    } catch (_) {
      // Handle any errors (network, SDK issues, etc.)
    }
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Adapty Paywall Example')),
      body: Center(
        // Add a button to re-trigger the paywall for testing purposes.
        child: ElevatedButton(
          onPressed: _showPaywallIfNeeded,
          child: const Text('Show Paywall'),
        ),
      ),
    );
  }
}
```
