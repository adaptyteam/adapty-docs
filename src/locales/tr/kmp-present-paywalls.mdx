<SDKv4>

<MethodPromo method="getFlow" label="Flow ve paywallları göster" />

Bir flow veya paywall oluşturduysanız, bunu kullanıcıya göstermek için mobil uygulama kodunuzda ayrıca render etme konusunda endişelenmenize gerek yoktur. Böyle bir flow veya paywall, içinde neyin gösterileceğini ve nasıl gösterileceğini zaten kapsar.

:::warning

Bu kılavuz, Adapty tarafından render edilen flow'ları ve **yeni Paywall Builder paywall'larını** kapsar. Remote config paywall'ları ve [Observer mode](observer-vs-full-mode) için süreç farklıdır.

- **Remote config paywalls** sunmak için, bkz. [Remote config ile tasarlanmış paywall'ı render etme](present-remote-config-paywalls-kmp).
- **Observer mode**'da flow'ları sunmak için, bkz. [Observer mode'da flow'ları sunma](kmp-present-flows-in-observer-mode).

:::

Aşağıda kullanılan `flow` nesnesini edinmek için bkz. [Flow'ları ve paywall'ları alma](kmp-get-pb-paywalls).

Adapty Kotlin Multiplatform SDK, flow'ları ve paywall'ları sunmak için iki yöntem sunar:

- **Compose Multiplatform ile**
- **Compose Multiplatform olmadan**

## Compose Multiplatform ile \{#with-compose-multiplatform\}

Bir flow veya paywall görüntülemek için [`createFlowView`](kmp-get-pb-paywalls#fetch-the-view-configuration) metoduyla oluşturulan `view` üzerinde `view.present()` metodunu kullanın. Her `view` yalnızca bir kez kullanılabilir. Flow'u tekrar görüntülemeniz gerekiyorsa yeni bir `view` örneği oluşturmak için `createFlowView` metodunu bir kez daha çağırın.

:::warning
Aynı `view`'ı yeniden oluşturmadan tekrar kullanmak hataya yol açabilir.
:::

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    AdaptyUI.createFlowView(flow = flow).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```

### İletişim Kutusu Göster \{#show-dialog\}

Flow veya paywall, Android'de gösterildiğinde yerel uyarı iletişim kutuları yerine bu yöntemi kullanın. Android'de normal uyarılar, flow görünümünün arkasında kalır ve kullanıcılar tarafından görülmez. Bu yöntem, tüm platformlarda flow'un üzerinde iletişim kutusunun düzgün şekilde gösterilmesini sağlar.

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    view.showDialog(
        title = "Close this screen?",
        content = "You will lose access to exclusive offers.",
        primaryActionTitle = "Stay",
        secondaryActionTitle = "Close"
    ).onSuccess { action ->
        if (action == AdaptyUIDialogActionType.SECONDARY) {
            // User confirmed - close the flow
            view.dismiss()
        }
        // If primary - do nothing, user stays
    }.onError { error ->
        // handle the error
    }
}
```

### iOS sunum stilini yapılandırma \{#configure-ios-presentation-style\}

Flow veya paywall'ın iOS'ta nasıl sunulacağını yapılandırmak için `present()` metoduna `iosPresentationStyle` parametresini geçirin. Bu parametre `AdaptyUIIOSPresentationStyle.FULLSCREEN` (varsayılan) veya `AdaptyUIIOSPresentationStyle.PAGESHEET` değerlerini kabul eder.

```kotlin showLineNumbers

viewModelScope.launch {
    val view = AdaptyUI.createFlowView(flow = flow).getOrNull()
    view?.present(iosPresentationStyle = AdaptyUIIOSPresentationStyle.PAGESHEET)
}
```

## Compose Multiplatform Olmadan \{#without-compose-multiplatform\}

:::note
`createNativeFlowView`, `io.adapty:adapty-kmp` çekirdek modülünün bir parçasıdır. Projeniz Compose Multiplatform kullanmıyorsa `io.adapty:adapty-kmp-ui` bağımlılığına ihtiyacınız yoktur.
:::

Compose Multiplatform olmadan bir flow veya paywall yerleştirmek için `createNativeFlowView` metodunu çağırın. Bu metot, layout'unuza eklediğiniz bir `AdaptyNativeFlowView` döndürür:

<Tabs>
<TabItem value="android" label="Android">
```kotlin showLineNumbers title="Kotlin Multiplatform (Android)"

val nativeView = AdaptyUI.createNativeFlowView(
    context = context,
    viewModelStoreOwner = activity,
    flow = flow,
    observer = myFlowObserver,
)

// Embed in your Compose layout:
AndroidView(
    factory = { nativeView.view },
    modifier = Modifier.fillMaxSize()
)
```

Varsayılan olarak, gömülü bir view güvenli alan dolguları uygulamaz — layout'unuzun iç boşlukları kendisinin yönetmesi beklenir. View'ın bu dolguları kendisi uygulamasını istiyorsanız, `createNativeFlowView` fonksiyonuna `androidEnableSafeArea = true` parametresini geçin. Bu parametre yalnızca Android'e özgüdür.
</TabItem>
<TabItem value="ios" label="iOS">
KMP arayüz varsayılan metotları Swift'te `@required` olarak çevrildiğinden, `AdaptyUIFlowsEventsObserver`'ı doğrudan Swift üzerinden uygulayamazsınız. Önce `iosMain` içinde açık bir temel sınıf tanımlayın:

```kotlin showLineNumbers title="iosMain (Kotlin)"
open class BaseFlowObserver : AdaptyUIFlowsEventsObserver
```

Ardından Swift'te yalnızca ihtiyacınız olanı geçersiz kılarak alt sınıf oluşturun:

```swift showLineNumbers title="Swift"
class MyFlowObserver: BaseFlowObserver {
    override func flowViewDidPerformAction(view: AdaptyUIFlowView, action: any AdaptyUIAction) {
        if action is AdaptyUIActionCloseAction {
            // remove nativeView from your view hierarchy
        }
    }
}

let nativeView = AdaptyUI.shared.createNativeFlowView(
    flow: flow,
    observer: MyFlowObserver()
)
// nativeView.viewController is a UIViewController.
// Add it to your SwiftUI view or UIKit hierarchy.
```
</TabItem>
</Tabs>

### Görünümü imha etme \{#dispose-the-view\}

Görünümü düzeninizden kaldırırken `dispose()` metodunu çağırın. Bu işlem, olay dinleyicisinin kaydını siler ve dahili kaynakları serbest bırakır.

```kotlin showLineNumbers title="Kotlin Multiplatform"
nativeView.dispose()
```

## Özel etiketler

Özel etiketler, farklı senaryolar için ayrı flow'lar veya paywall'lar oluşturmaktan kurtarır. Kullanıcı verilerine göre dinamik olarak uyum sağlayan tek bir flow hayal edin. Örneğin, genel bir "Merhaba!" yerine kullanıcıları "Merhaba, Ahmet!" veya "Merhaba, Ayşe!" şeklinde kişisel olarak selamlayabilirsiniz.

Özel etiketleri şu şekillerde kullanabilirsiniz:

- Flow veya paywall'da kullanıcının adını ya da e-posta adresini gösterin.
- Satışları artırmak için haftanın günü bilgisini gösterin (örneğin, "İyi Perşembeler").
- Sattığınız ürünlerle ilgili kişiselleştirilmiş ayrıntılar ekleyin (bir fitness programının adı veya VoIP uygulamasında bir telefon numarası gibi).

Özel etiketler, çeşitli durumlara uyum sağlayan esnek bir flow oluşturmanıza yardımcı olarak uygulamanızın arayüzünü daha kişisel ve ilgi çekici hale getirir.

:::warning
Bazı durumlarda uygulamanız, özellikle kullanıcılar eski bir AdaptyUI SDK sürümü kullanıyorsa, özel etiketi neyle değiştireceğini bilemeyebilir. Bunu önlemek için, bilinmeyen özel etiket içeren satırların yerine geçecek her zaman bir yedek metin ekleyin. Aksi takdirde kullanıcılar etiketleri kod olarak görebilir (`<USERNAME/>`).
:::

Özel etiketleri flow'unuzda veya paywallınızda kullanmak için, flow view oluştururken bunları iletmeniz gerekir:

<Tabs>
<TabItem value="standalone" label="With Compose Multiplatform" default>
```kotlin showLineNumbers title="Kotlin Multiplatform"

    val customTags = mapOf(
        "USERNAME" to "John",
        "DAY_OF_WEEK" to "Thursday"
    )

    AdaptyUI.createFlowView(
        flow = flow,
        customTags = customTags
    ).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```
</TabItem>
<TabItem value="native" label="Without Compose Multiplatform">

```kotlin showLineNumbers title="Kotlin Multiplatform (Android)"
    "USERNAME" to "John",
    "DAY_OF_WEEK" to "Thursday"

    flow = flow,
    observer = myFlowObserver,

```


## Özel zamanlayıcılar \{#custom-timers\}

Zamanlayıcı, süreli özel ve sezonluk teklifleri tanıtmak için harika bir araçtır. Ancak bu zamanlayıcının teklifin geçerlilik süresiyle veya kampanyanın bitiş tarihiyle herhangi bir bağlantısı olmadığını belirtmek gerekir. Yalnızca belirlediğiniz değerden başlayıp sıfıra kadar azalan bağımsız bir geri sayımdır. Zamanlayıcı sıfıra ulaştığında hiçbir şey olmaz; sadece sıfırda kalır.

İstediğiniz mesajı oluşturmak için zamanlayıcının önüne ve arkasına metin ekleyebilirsiniz; örneğin: "Teklif bitiyor: 10:00 sn."

Flow veya paywallınızda özel zamanlayıcılar kullanmak için, flow görünümünü oluştururken bunları aktarın:

<Tabs>
<TabItem value="standalone" label="With Compose Multiplatform" default>
```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    val customTimers = mapOf(
        "CUSTOM_TIMER_NY" to LocalDateTime(2025, 1, 1, 0, 0, 0),
        "CUSTOM_TIMER_SALE" to LocalDateTime(2024, 12, 31, 23, 59, 59)
    )

    AdaptyUI.createFlowView(
        flow = flow,
        customTimers = customTimers
    ).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```
</TabItem>

## Compose Multiplatform ile \{#with-compose-multiplatform\}

:::warning

```kotlin showLineNumbers title="Kotlin Multiplatform"
viewModelScope.launch {

        view.present()
    }.onError { error ->
        // handle the error
    }

```

Bu yöntem, tüm platformlarda paywall'ın üzerinde diyaloğun düzgün şekilde gösterilmesini sağlar.
