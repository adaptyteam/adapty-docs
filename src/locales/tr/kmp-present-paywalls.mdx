<MethodPromo method="getFlow" label="Flow'ları ve paywall'ları görüntüle" />

Bir flow veya paywall oluşturduysanız, bunu kullanıcıya göstermek için mobil uygulama kodunuzda render etme konusunda endişelenmenize gerek yoktur. Böyle bir flow veya paywall, içinde ne gösterilmesi gerektiğini ve nasıl gösterilmesi gerektiğini zaten içerir.

:::warning

Bu rehber, Adapty tarafından render edilen flow'ları ve **yeni Paywall Builder paywall'larını** kapsamaktadır. Remote config paywall'ları ve [Observer mode](observer-vs-full-mode) için süreç farklıdır.

- **Remote config paywalls** sunmak için bkz. [Remote config ile tasarlanmış paywall'u gösterme](present-remote-config-paywalls-kmp).
- **Observer mode**'da flow sunmak için bkz. [Observer mode'da flow sunma](kmp-present-flows-in-observer-mode).

:::

Aşağıda kullanılan `flow` nesnesini almak için bkz. [Flow'ları ve paywall'ları alma](kmp-get-pb-paywalls).

Adapty Kotlin Multiplatform SDK, flow'ları ve paywall'ları sunmak için iki yöntem sunar:

- **Compose Multiplatform ile**
- **Compose Multiplatform olmadan**

## Compose Multiplatform ile \{#with-compose-multiplatform\}

Bir flow veya paywall görüntülemek için, [`createFlowView`](kmp-get-pb-paywalls#fetch-the-view-configuration) metodu tarafından oluşturulan `view` üzerinde `view.present()` metodunu kullanın. Her `view` yalnızca bir kez kullanılabilir. Flow'u tekrar görüntülemeniz gerekirse, yeni bir `view` örneği oluşturmak için `createFlowView` metodunu bir kez daha çağırın.

:::warning
Aynı `view`'ı yeniden oluşturmadan kullanmak hataya neden olabilir.
:::

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    AdaptyUI.createFlowView(flow = flow).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```

### Diyalog göster \{#show-dialog\}

Android'de bir flow veya paywall gösterilirken, yerel uyarı diyalogları yerine bu yöntemi kullanın. Android'de normal uyarılar flow görünümünün arkasında kalır ve kullanıcılar tarafından görülemez. Bu yöntem, tüm platformlarda diyaloğun flow'un üzerinde doğru şekilde gösterilmesini sağlar.

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    view.showDialog(
        title = "Close this screen?",
        content = "You will lose access to exclusive offers.",
        primaryActionTitle = "Stay",
        secondaryActionTitle = "Close"
    ).onSuccess { action ->
        if (action == AdaptyUIDialogActionType.SECONDARY) {
            // User confirmed - close the flow
            view.dismiss()
        }
        // If primary - do nothing, user stays
    }.onError { error ->
        // handle the error
    }
}
```

### iOS sunum stilini yapılandırın \{#configure-ios-presentation-style\}

`present()` metoduna `iosPresentationStyle` parametresini geçirerek flow veya paywall'ın iOS'ta nasıl sunulacağını yapılandırın. Parametre, `AdaptyUIIOSPresentationStyle.FULLSCREEN` (varsayılan) veya `AdaptyUIIOSPresentationStyle.PAGESHEET` değerlerini kabul eder.

```kotlin showLineNumbers

viewModelScope.launch {
    val view = AdaptyUI.createFlowView(flow = flow).getOrNull()
    view?.present(iosPresentationStyle = AdaptyUIIOSPresentationStyle.PAGESHEET)
}
```

## Compose Multiplatform Olmadan \{#without-compose-multiplatform\}

:::note
`createNativeFlowView`, `io.adapty:adapty-kmp` çekirdek modülünün bir parçasıdır. Projeniz Compose Multiplatform kullanmıyorsa `io.adapty:adapty-kmp-ui` bağımlılığına ihtiyaç duymazsınız.
:::

Bir flow veya paywall'ı Compose Multiplatform olmadan yerleştirmek için `createNativeFlowView` fonksiyonunu çağırın. Bu fonksiyon, layout'unuza eklediğiniz bir `AdaptyNativeFlowView` döndürür:

<Tabs>
<TabItem value="android" label="Android">
```kotlin showLineNumbers title="Kotlin Multiplatform (Android)"

val nativeView = AdaptyUI.createNativeFlowView(
    context = context,
    viewModelStoreOwner = activity,
    flow = flow,
    observer = myFlowObserver,
)

// Embed in your Compose layout:
AndroidView(
    factory = { nativeView.view },
    modifier = Modifier.fillMaxSize()
)
```

Varsayılan olarak, gömülü bir view güvenli alan dolgularını uygulamaz — düzeninizin iç boşlukları kendi başına yönetmesi beklenir. View'ın bu dolguları kendisinin uygulamasını istiyorsanız, `createNativeFlowView` fonksiyonuna `androidEnableSafeArea = true` parametresini geçin. Bu parametre yalnızca Android'e özgüdür.
</TabItem>
<TabItem value="ios" label="iOS">
KMP arayüzündeki varsayılan metodlar Swift'te `@required` hale geldiğinden, `AdaptyUIFlowsEventsObserver` arayüzünü doğrudan Swift'ten uygulayamazsınız. Önce `iosMain` içinde açık bir temel sınıf tanımlayın:

```kotlin showLineNumbers title="iosMain (Kotlin)"
open class BaseFlowObserver : AdaptyUIFlowsEventsObserver
```

Ardından Swift'te yalnızca ihtiyacınız olanı geçersiz kılarak alt sınıf oluşturun:

```swift showLineNumbers title="Swift"
class MyFlowObserver: BaseFlowObserver {
    override func flowViewDidPerformAction(view: AdaptyUIFlowView, action: any AdaptyUIAction) {
        if action is AdaptyUIActionCloseAction {
            // remove nativeView from your view hierarchy
        }
    }
}

let nativeView = AdaptyUI.shared.createNativeFlowView(
    flow: flow,
    observer: MyFlowObserver()
)
// nativeView.viewController is a UIViewController.
// Add it to your SwiftUI view or UIKit hierarchy.
```
</TabItem>
</Tabs>

### Görünümü imha etme \{#dispose-the-view\}

Görünümü düzeninizden kaldırırken `dispose()` metodunu çağırın. Bu işlem, olay dinleyicisinin kaydını siler ve dahili kaynakları serbest bırakır.

```kotlin showLineNumbers title="Kotlin Multiplatform"
nativeView.dispose()
```

## Özel etiketler \{#custom-tags\}

Özel etiketler, farklı senaryolar için ayrı flow'lar veya paywall'lar oluşturmanıza gerek kalmadan tek bir flow'u kullanıcı verilerine göre dinamik olarak uyarlamanızı sağlar. Örneğin, genel bir "Merhaba!" yerine kullanıcıları kişisel olarak "Merhaba, Ahmet!" veya "Merhaba, Ayşe!" şeklinde karşılayabilirsiniz.

Özel etiketleri kullanabileceğiniz bazı senaryolar:

- Flow veya paywall üzerinde kullanıcının adını ya da e-posta adresini göstermek.
- Satışları artırmak için haftanın günü gibi bilgileri göstermek (örn. "İyi Perşembeler").
- Sattığınız ürünlerle ilgili kişiselleştirilmiş ayrıntılar eklemek (bir fitness programının adı veya VoIP uygulamasındaki bir telefon numarası gibi).

Özel etiketler, çeşitli durumlara uyum sağlayan esnek bir flow oluşturmanıza yardımcı olur; bu sayede uygulamanızın arayüzü daha kişisel ve ilgi çekici hale gelir.

:::warning
Bazı durumlarda uygulamanız, özel bir etiketi neyle değiştireceğini bilemeyebilir; özellikle kullanıcılar AdaptyUI SDK'nın eski bir sürümünü kullanıyorsa bu sorun yaşanabilir. Bunu önlemek için, bilinmeyen özel etiket içeren satırların yerine geçecek bir yedek metin eklemeyi unutmayın. Aksi takdirde kullanıcılar etiketlerin kod olarak göründüğünü görebilir (`<USERNAME/>`).
:::

Flow ya da paywallınızda özel etiketleri kullanmak için, flow view oluştururken bunları aktarın:

<Tabs>
<TabItem value="standalone" label="With Compose Multiplatform" default>
```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    val customTags = mapOf(
        "USERNAME" to "John",
        "DAY_OF_WEEK" to "Thursday"
    )

    AdaptyUI.createFlowView(
        flow = flow,
        customTags = customTags
    ).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```
</TabItem>
<TabItem value="native" label="Without Compose Multiplatform">
```kotlin showLineNumbers title="Kotlin Multiplatform (Android)"

    "USERNAME" to "John",
    "DAY_OF_WEEK" to "Thursday"
)

    viewModelStoreOwner = activity,
    flow = flow,
    observer = myFlowObserver,
    customTags = customTags,
)
```

```kotlin showLineNumbers title="Kotlin Multiplatform (iOS)"

    "DAY_OF_WEEK" to "Thursday"
```

## Özel zamanlayıcılar \{#custom-timers\}

Zamanlayıcı, süreli özel ve mevsimsel teklifleri öne çıkarmak için harika bir araçtır. Ancak şunu belirtmek gerekir: bu zamanlayıcı, teklifin geçerlilik süresiyle veya kampanyanın bitiş tarihiyle bağlantılı değildir. Yalnızca belirlediğiniz değerden sıfıra doğru azalan bağımsız bir geri sayımdır. Zamanlayıcı sıfıra ulaştığında hiçbir şey olmaz—sadece sıfırda kalır.

İstediğiniz mesajı oluşturmak için zamanlayıcının önüne ve arkasına metin ekleyebilirsiniz; örneğin: "Teklif bitiyor: 10:00 sn."

Flow veya paywallınızda özel zamanlayıcıları kullanmak için, flow görünümünü oluştururken bunları iletin:

<Tabs>
<TabItem value="standalone" label="With Compose Multiplatform" default>
```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    val customTimers = mapOf(
        "CUSTOM_TIMER_NY" to LocalDateTime(2025, 1, 1, 0, 0, 0),
        "CUSTOM_TIMER_SALE" to LocalDateTime(2024, 12, 31, 23, 59, 59)
    )

    AdaptyUI.createFlowView(
        flow = flow,
        customTimers = customTimers
    ).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
}
```
</TabItem>

Paywall Builder kullanarak bir paywall özelleştirdiyseniz, kullanıcıya göstermek için onu mobil uygulama kodunuzda ayrıca render etmenize gerek yoktur. Bu tür bir paywall, hem ne gösterileceğini hem de nasıl gösterileceğini içerir.

:::warning

Bu kılavuz yalnızca **yeni Paywall Builder paywallları** içindir. Paywallları sunma süreci, remote config paywallları ve [Observer mode](observer-vs-full-mode) ile tasarlanan paywalllar için farklıdır.

**Remote config paywalllarını** sunmak için bkz. [Remote config ile tasarlanan paywallı render etme](present-remote-config-paywalls-kmp).

:::

Adapty Kotlin Multiplatform SDK, paywall'ları sunmak için iki farklı yöntem sunar:

- **Compose Multiplatform ile**
- **Compose Multiplatform olmadan**

## Compose Multiplatform ile \{#with-compose-multiplatform\}

Bir paywall görüntülemek için [`createPaywallView`](kmp-get-pb-paywalls#fetch-the-view-configuration-of-paywall-designed-using-paywall-builder) metodu ile oluşturulan `view` üzerinde `view.present()` metodunu kullanın. :::warning

:::

```kotlin showLineNumbers title="Kotlin Multiplatform"

viewModelScope.launch {
    AdaptyUI.createPaywallView(paywall = paywall).onSuccess { view ->
        view.present()
    }.onError { error ->
        // handle the error
    }
```
