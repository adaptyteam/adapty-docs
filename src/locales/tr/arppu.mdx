Ödeme yapan kullanıcı başına ortalama gelir (ARPPU) grafiği, ödeme yapan kullanıcı başına ortalama geliri gösterir. Ödeme yapan müşterilerin oluşturduğu gerçek gelirin, iadeler düşüldükten sonra müşteri sayısına bölünmesiyle hesaplanır. Hangi edinim kanallarının daha yüksek değerli ödeme yapan kullanıcılar getirdiğini görmek için ARPPU'yu attribution'a göre gruplandırın.

## Hesaplama \{#calculation\}

:::warning
Aşağıdaki hesap makinesi [mağaza komisyonu ve vergilendirmeyi](how-adapty-analytics-works#commissions-and-taxes) **dikkate almaz**. Sonucu **brüt gelir** hesaplamalarınızla karşılaştırın.
:::

ARPPU, ödeme yapan kullanıcı başına ortalama geliri gösterir — ödeme yapmayan kullanıcılar paydadan çıkarıldığından genellikle [ARPU](arpu)'dan çok daha yüksektir.

ARPPU, dönemin gelirinin (iadeler düşüldükten sonra) o dönemdeki ödeme yapan kullanıcı sayısına bölünmesiyle hesaplanır.
<CompoundCalculator client:load
    heading="ARPPU"
    formuLatex="\frac{\sum P_i \times Q_i - D}{U_p}"
    variables={[
        {
            nameInTheFormula: "P",
            variableName: "price",
            variableDescription: "Ürün fiyatı",
            variableValue: 10
        },
        {
            nameInTheFormula: "Q",
            variableName: "qty",
            variableDescription: "Satın alınan ürünler",
            variableValue: 1,
            isInteger: true
        },
        {
            nameInTheFormula: "D",
            variableName: "refunds",
            variableDescription: "İade edilen tutar",
            variableValue: 35,
            global: true
        },
        {
            nameInTheFormula: "Up",
            variableName: "users",
            variableDescription: "Ödeme yapan kullanıcılar",
            variableValue: 16,
            global: true,
            isInteger: true
        }
    ]}
    rowFormula="price * qty"
    resultFormula="(_sum - refunds) / users"
    defaultRows={[
        { price: 10, qty: 5 },
        { price: 50, qty: 10 },
        { price: 100, qty: 1 }
    ]}
/>
Satın alması daha sonra iade edilen bir kullanıcı, ödeme yapan kullanıcı paydasında yine de sayılır. Yoğun iadeler, ARPPU'yu beklenenden daha hızlı düşürebilir.
