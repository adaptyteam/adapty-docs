Observer mode'da, Adapty SDK kendi başına mevcut satın alma sisteminiz üzerinden yapılan satın almaları takip edemez. App Store'daki işlemleri uygulamanızdan manuel olarak bildirmeniz gerekir. Analitik hatalarını önlemek için bunu uygulamanızı yayınlamadan **önce** yapılandırmanız son derece önemlidir.

Her işlemi Adapty'nin tanıyabilmesi için `reportTransaction` kullanarak açıkça bildirin.

:::warning
**İşlem bildirmeyi atlama!**
`reportTransaction` çağırmazsanız, Adapty işlemi tanımaz, işlem analizlerde görünmez ve entegrasyonlara gönderilmez.

:::

Adapty paywall'larını kullanıyorsanız, bir işlem bildirirken `variationId`'yi de ekleyin. Bu, satın almayı tetikleyen paywall ile ilişkilendirir ve paywall analizlerinin doğru olmasını sağlar.
```dart showLineNumbers
try {
    // transaction.finish() her çağrıldığında
    await Adapty().reportTransaction(
        "YOUR_TRANSACTION_ID", 
        variationId: "PAYWALL_VARIATION_ID", // isteğe bağlı
    );
} on AdaptyError catch (adaptyError) {
    // hatayı işle
} catch (e) {
    // hatayı işle
}
```

Parametreler:
| Parametre     | Zorunluluk | Açıklama                                                  |
| ------------- | -------- | ------------------------------------------------------------ |
| transactionId | zorunlu | <ul><li> iOS için: İşlemin tanımlayıcısı.</li><li> Android için: Satın almanın `purchase.getOrderId` string tanımlayıcısı; burada purchase, billing library'nin [Purchase](https://developer.android.com/reference/com/android/billingclient/api/Purchase) sınıfının bir örneğidir.</li></ul> |
| variationId   | isteğe bağlı | Varyantın string tanımlayıcısı. Bunu [AdaptyPaywall](https://pub.dev/documentation/adapty_flutter/latest/adapty_flutter/AdaptyPaywall-class.html) nesnesinin `variationId` özelliğini kullanarak alabilirsiniz. |
</TabItem>

Observer modunda, Adapty SDK mevcut satın alma sisteminiz üzerinden gerçekleştirilen satın almaları kendi başına takip edemez. App store'dan işlemleri raporlamanız veya geri yüklemeniz gerekir. Analitik hatalarını önlemek için bunu uygulamanızı yayınlamadan **önce** ayarlamanız kritik önem taşır.

Her işlemi açıkça raporlamak için her iki platformda da `reportTransaction` kullanın; Adapty'nin bunu tanımasını sağlamak için Android'de ek bir adım olarak `restorePurchases` kullanın.

:::warning
**İşlem raporlamayı ve satın alma geri yüklemeyi atlama!**
Bu metotları çağırmazsanız Adapty işlemi tanımaz, işlem analizlerde görünmez ve entegrasyonlara gönderilmez.

:::

Adapty paywall'larını kullanıyorsanız, işlem raporlarken `variationId` değerini ekleyin. Bu, satın almayı tetikleyen paywall ile ilişkilendirir ve paywall analizlerinin doğru olmasını sağlar.
```dart showLineNumbers
// every time when calling transaction.finish()
if (Platform.isAndroid) {
    try {
        await Adapty().restorePurchases();
    } on AdaptyError catch (adaptyError) {
        // handle the error
    } catch (e) {
    }
}

try {
    // every time when calling transaction.finish()
    await Adapty().reportTransaction(
        "YOUR_TRANSACTION_ID", 
        variationId: "PAYWALL_VARIATION_ID", // optional
    );
} on AdaptyError catch (adaptyError) {
    // handle the error
} catch (e) {
    // handle the error
}
```

Parametreler:
| Parametre     | Zorunluluk | Açıklama                                                  |
| ------------- | -------- | ------------------------------------------------------------ |
| transactionId | zorunlu | <ul><li> iOS, StoreKit 1 için: bir [SKPaymentTransaction](https://developer.apple.com/documentation/storekit/skpaymenttransaction) nesnesi.</li><li> iOS, StoreKit 2 için: [Transaction](https://developer.apple.com/documentation/storekit/transaction) nesnesi.</li><li> Android için: String tanımlayıcı (satın almanın purchase.getOrderId değeri; burada purchase, billing library [Purchase](https://developer.android.com/reference/com/android/billingclient/api/Purchase) sınıfının bir örneğidir).</li></ul> |
| variationId   | isteğe bağlı | Varyantın string tanımlayıcısı. Bunu [AdaptyPaywall](https://pub.dev/documentation/adapty_flutter/latest/adapty_flutter/AdaptyPaywall-class.html) nesnesinin `variationId` özelliğini kullanarak alabilirsiniz. |
</TabItem>

**İşlemleri raporlama**

- 3.1.x'e kadar olan sürümler App Store'daki işlemleri otomatik olarak dinler, dolayısıyla manuel raporlama gerekmez.
- 3.2 sürümü Observer Mode'u desteklemez.

**İşlemleri raporlama**
[Mobil Kodda Satın Almaları Geri Yükle](flutter-restore-purchase) sayfasında açıklandığı gibi, Observer Mode'da bir işlemi Adapty'ye bildirmek için `restorePurchases` kullanın.

:::warning
**İşlem bildirmeyi atlama!**
`restorePurchases` çağrısı yapmazsanız, Adapty işlemi tanımaz, analytics'te görünmez ve entegrasyonlara gönderilmez.
:::

**Paywall'ları işlemlerle ilişkilendirme**
Adapty SDK, satın almaları siz işlediğiniz için satın almaların kaynağını belirleyemez. Bu nedenle, Observer mode'da paywall'lar ve/veya A/B testleri kullanmayı planlıyorsanız, uygulama mağazanızdan gelen işlemi mobil uygulama kodunuzda ilgili paywall ile ilişkilendirmeniz gerekir. Uygulamanızı yayınlamadan önce bunu doğru şekilde ayarlamanız önemlidir; aksi takdirde analizlerde hatalara yol açar.
```dart
final transactionId = transaction.transactionIdentifier
final variationId = paywall.variationId

try {
  await Adapty().setVariationId('transactionId', variationId);
} on AdaptyError catch (adaptyError) {
  // handle the error
} catch (e) {
}
```
