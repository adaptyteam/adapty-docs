Uygulama içi satın alımları etkinleştirmek için üç temel kavramı anlamanız gerekir:
- [**Ürünler**](product) – kullanıcıların satın alabileceği her şey (abonelikler, consumable'lar, süresiz erişim)
- [**Flow'lar**](adapty-flow-builder) – ürünleri kullanıcılara sunan ekran dizileri; no-code Flow Builder ile oluşturulur. SDK bunları `getFlow` aracılığıyla alır. Arayüzü kendi kodunuzla oluşturmayı tercih ediyorsanız bunun yerine paywall kullanın — bkz. [Paywall'ları manuel olarak uygulama](android-quickstart-manual).
- [**Placement'lar**](placements) – flow'ların uygulamanızda nerede ve ne zaman gösterileceği (`main`, `onboarding`, `settings` gibi). Flow'ları kontrol panelinde placement'lara bağlar, ardından kodunuzda placement ID'siyle talep edersiniz. Bu sayede A/B testleri çalıştırmak ve farklı kullanıcılara farklı flow'lar göstermek kolaylaşır.
Adapty, uygulamanızda satın almaları etkinleştirmek için size üç yöntem sunar. Uygulama gereksinimlerinize bağlı olarak birini seçin:
| Uygulama | Karmaşıklık | Ne zaman kullanılır |
|---|---|---|
| Adapty Flow Builder | ✅ Kolay | No-code builder'da [satın almaya hazır eksiksiz bir flow oluşturursunuz](quickstart-paywalls). Adapty bunu otomatik olarak render eder; tüm karmaşık satın alma akışını, makbuz doğrulamasını ve abonelik yönetimini arka planda kendisi halleder. |
| Manuel oluşturulan paywall'lar | 🟡 Orta | Paywall arayüzünüzü uygulama kodunuzda kendiniz uygularsınız; ancak ürün tekliflerinde esnekliği korumak için flow nesnesini yine de Adapty'den alırsınız. [Kılavuza](android-quickstart-manual) bakın. |
| Observer modu | 🔴 Zor | Kendi satın alma altyapınız zaten mevcuttur ve bunu kullanmaya devam etmek istersiniz. Observer modunun Adapty'de bazı sınırlamaları olduğunu unutmayın. [Makaleye](observer-vs-full-mode) bakın. |
:::important
**Aşağıdaki adımlar, Adapty Flow Builder'da oluşturulan bir flow'un nasıl hayata geçirileceğini göstermektedir.**

Paywall arayüzünü kendiniz oluşturmayı tercih ediyorsanız, [Paywall'ları manuel olarak uygulama](android-quickstart-manual) bölümüne bakın.
:::

Adapty Flow Builder'da oluşturulan bir flow'u görüntülemek için uygulama kodunuzda yalnızca şunları yapmanız gerekir:

1. **Flow'u alın**: Adapty'den alın.
2. **Görüntüleyin, satın alma işlemlerini Adapty yönetsin**: Görünümü uygulamanızda gösterin.
3. **Buton eylemlerini yönetin**: Kullanıcı etkileşimlerini uygulamanızın bu etkileşimlere verdiği yanıtlarla ilişkilendirin. Örneğin, kullanıcılar butona tıkladığında bağlantıları açın veya flow'u kapatın.
## Başlamadan önce \{#before-you-start\}

Başlamadan önce şu adımları tamamlayın:

1. Adapty Kontrol Paneli'nde [uygulamanızı Google Play'e bağlayın](initial-android).
2. Adapty'de [ürünlerinizi oluşturun](create-product).
3. [Bir flow oluşturun ve buna ürün ekleyin](create-paywall).
4. [Bir placement oluşturun ve flow'unuzu ekleyin](create-placement).
5. Uygulama kodunuza [Adapty SDK'yı yükleyin ve etkinleştirin](sdk-installation-android). Bu kılavuzda Adapty Android SDK v4 (beta) API'leri kullanılmaktadır.
:::tip
Bu adımları tamamlamanın en hızlı yolu [hızlı başlangıç kılavuzunu](quickstart) takip etmek ya da [Developer CLI](developer-cli-quickstart) kullanarak flow'lar ve placement'lar oluşturmaktır.
:::
## 1. Flow'u alın \{#get-the-flow\}

Flow'larınız, kontrol panelinde yapılandırılmış placement'larla ilişkilendirilir. Placement'lar, farklı kitleler için farklı flow'lar çalıştırmanıza veya [A/B testleri](ab-tests) yapmanıza olanak tanır.

Adapty Flow Builder'da oluşturduğunuz bir flow'u almak için şunları yapmanız gerekir:

1. `getFlow` metodunu kullanarak [placement](placements) ID'sine göre `flow` nesnesini alın ve bir görünüm yapılandırmasına sahip olup olmadığını kontrol edin.

2. `getFlowConfiguration` metodunu kullanarak görünüm yapılandırmasını alın. Görünüm yapılandırması, flow'u görüntülemek için gereken UI öğelerini ve stil bilgilerini içerir.
:::important
Görünüm yapılandırmasını almak için Flow Builder'da **Show on device** geçişini açmanız gerekir. Aksi takdirde boş bir görünüm yapılandırması alırsınız ve flow görüntülenmez.
:::

<Tabs groupId="current-os" queryString>

<TabItem value="kotlin" label="Kotlin" default>
```kotlin showLineNumbers
Adapty.getFlow("YOUR_PLACEMENT_ID") { result ->
    if (result is AdaptyResult.Success) {
        val flow = result.value
        
        if (!flow.hasViewConfiguration) {
            return@getFlow
        }
        
        AdaptyUI.getFlowConfiguration(flow) { configResult ->
            if (configResult is AdaptyResult.Success) {
                val flowConfiguration = configResult.value
            }
        }
    }
}
```
</TabItem>
<TabItem value="java" label="Java" default>
```java showLineNumbers

Adapty.getFlow("YOUR_PLACEMENT_ID", result -> {
    if (result instanceof AdaptyResult.Success) {
        AdaptyFlow flow = ((AdaptyResult.Success<AdaptyFlow>) result).getValue();
        
        if (!flow.hasViewConfiguration()) {
            return;
        }
        
        AdaptyUI.getFlowConfiguration(flow, configResult -> {
            if (configResult instanceof AdaptyResult.Success) {
                AdaptyUI.FlowConfiguration flowConfiguration =
                    ((AdaptyResult.Success<AdaptyUI.FlowConfiguration>) configResult).getValue();
                // use loaded configuration
            }
        });
    }
});
```

## 2. Flow'u göster \{#display-the-flow\}

Artık flow yapılandırmanız hazır olduğunda, flow'unuzu görüntülemek için birkaç satır kod eklemeniz yeterlidir.

Görsel flow'u cihaz ekranında gösterebilmek için önce yapılandırmanız gerekir. Bunun için `AdaptyUI.getFlowView()` metodunu çağırın ya da doğrudan `AdaptyFlowView` oluşturun:

## 3. Buton eylemlerini yönetme \{#handle-button-actions\}

Kullanıcılar flow içindeki butonlara tıkladığında, Android SDK satın alma, geri yükleme, flow'u kapatma ve bağlantı açma işlemlerini otomatik olarak gerçekleştirir.

Ancak bazı butonların özel veya önceden tanımlanmış ID'leri vardır ve bu eylemler kodunuzda ele alınmalıdır. Ya da varsayılan davranışlarını geçersiz kılmak isteyebilirsiniz.

Örneğin, aşağıda kapat butonunun varsayılan davranışı gösterilmektedir. Bunu koda eklemeniz gerekmez, ancak gerekirse nasıl yapıldığını buradan görebilirsiniz.
:::tip
Buton [action'ları](android-handle-paywall-actions) ve [event'leri](android-handling-events) nasıl yöneteceğinizi anlatan kılavuzlarımızı okuyun.
:::

<Tabs groupId="current-os" queryString>

<TabItem value="kotlin" label="Kotlin" default>

```kotlin showLineNumbers title="Kotlin"
override fun onActionPerformed(action: AdaptyUI.Action, context: Context) {
    when (action) {
        AdaptyUI.Action.Close -> (context as? Activity)?.onBackPressed() // default behavior
    }
}
```
</TabItem>
<TabItem value="java" label="Java" default>
```java showLineNumbers
@Override
public void onActionPerformed(@NonNull AdaptyUI.Action action, @NonNull Context context) {
    if (action instanceof AdaptyUI.Action.Close) {
        if (context instanceof Activity) {
            ((Activity) context).onBackPressed();
        }
    }
}
```
</TabItem>

## Tam örnek \{#full-example\}

Tüm bu adımları uygulamanızda nasıl bir araya getirebileceğiniz aşağıda gösterilmektedir.

<Tabs groupId="current-os" queryString>

<TabItem value="kotlin" label="Kotlin" default>
```kotlin showLineNumbers title="Kotlin"
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        Adapty.getFlow("YOUR_PLACEMENT_ID") { flowResult ->
            if (flowResult is AdaptyResult.Success) {
                val flow = flowResult.value

                if (!flow.hasViewConfiguration) {
                    // Use custom logic
                    return@getFlow
                }

                AdaptyUI.getFlowConfiguration(flow) { configResult ->
                    if (configResult is AdaptyResult.Success) {
                        val flowConfiguration = configResult.value

                        val flowView = AdaptyUI.getFlowView(
                            this,
                            flowConfiguration,
                            null, // products = null means auto-fetch
                            object : AdaptyFlowDefaultEventListener() {
                                override fun onActionPerformed(action: AdaptyUI.Action, context: Context) {
                                    when (action) {
                                        is AdaptyUI.Action.Close -> {
                                            (context as? Activity)?.onBackPressed()
                                        }
                                    }
                                }
                            }
                        )

                        setContentView(flowView)
                    }
                }
            }
        }
    }
}

```
</TabItem>
